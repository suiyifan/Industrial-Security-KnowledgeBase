# 岗位画像：工控安全分析

## 1. 图片内容整理

### 1.1 JD 标签

- 渗透测试经验
- 计算机 / 信息安全相关专业
- 攻防对抗经验
- 安全产品 / 平台开发经验

### 1.2 JD 职位描述

1. 负责对工控系统软件、协议、设备等开展信息安全研究。包括数据挖掘、软件逆向、协议逆向、脆弱性挖掘、工具开发等。
2. 跟踪业界动态，包括新漏洞、利用工具、攻击手段、防护产品等。
3. 开发安全测试、评估、审计等工作。
4. 撰写安全分析报告、调研报告等。

### 1.3 JD 任职要求

1. 硕士及以上学历，自动化、计算机、网络安全、测控技术、控制工程等专业。
2. 精通至少一种编程语言，如 Python / C 等，具备一定算法设计能力，有良好编程习惯。
3. 熟悉 WinDbg、OD、IDA 等逆向、反汇编工具与手段。
4. 熟悉常见 Web 漏洞、系统漏洞、业务逻辑漏洞利用方法，例如 SQL 注入、WebShell 上传、越权漏洞、远程命令执行漏洞等。
5. 熟悉主流安全产品并了解其基本原理，例如系统漏洞扫描器、Web 漏洞扫描器、基线核查设备、IDS / IPS 等。

### 1.4 关键信息

- 职位类型：网络安全
- 工作地点：杭州西湖区
- 经验与学历：1-3 年，硕士及以上
- 年龄要求：硕士 32 岁及以下，博士条件更宽
- 福利：五险一金、补充医疗保险、全勤奖、交通补助、带薪年假、保底工资
- 公司：浙江腾珑网安科技有限公司

## 2. HR 补充信息整理

HR 表达的岗位重点：

- 公司是工控网络安全创新型企业，服务电力、石油、石化、冶金、水处理等行业。
- 主营方向包括工控网络安全靶场设备、平台类项目、安全运维、网控柜类区域安全项目等。
- 岗位属于项目安全委托类基础设施安全研究方向。
- 主要客户包括能源、电力、石油、国家电网、中煤集团以及区域安全项目。
- 工作内容不是普通的安全运维，也不是单纯挖矿或渗透测试，而是偏工控系统安全研究。
- 重点能力包括：
  - 对工控系统的结构、协议、软件、设备有理解。
  - 能研究产品缺陷和漏洞。
  - 能做 CVE 挖掘。
  - 能围绕工控协议做研究。
  - 能输出报告，并支持面试沟通。
- HR 提到“自动化也能匹配”，说明自动化 / 控制工程背景是加分项。
- HR 暗示面试要聊清楚：
  - 为什么选择工控安全方向。
  - 你对工控系统有什么理解。
  - 你是否能做安全研究、漏洞分析、协议分析。
  - 你有没有项目或作品能证明能力。

## 3. 最终岗位要求总结

这个岗位的本质是：

> 面向工业控制系统的安全研究岗位，需要同时具备网络安全、工控系统、协议分析、逆向分析、漏洞研究、工具开发和报告输出能力。

不是单点能力岗位，而是复合型岗位。能力结构可以拆成七层：

1. 计算机与编程基础：Python / C、Git、Linux、网络、算法与工程习惯。
2. 工业自动化基础：PLC、DCS、SCADA、PID、组态软件、工程站、工业现场架构。
3. 工业网络与协议：TCP/IP、抓包分析、Modbus、S7、OPC UA、IEC 104 等。
4. 安全基础：Web 漏洞、系统漏洞、权限提升、RCE、文件上传、业务逻辑漏洞。
5. 逆向分析：Windows、PE、汇编、IDA、WinDbg、OD、协议逆向、软件逆向。
6. 漏洞研究：漏洞复现、PoC 编写、CVE 分析、工控产品漏洞挖掘。
7. 安全体系与表达：IEC 62443、风险评估、安全测试、审计、报告撰写、面试表达。

## 4. 优先级

### 必须补齐

- Python 安全工具开发
- TCP/IP 与 Wireshark 抓包分析
- 工控系统基本架构
- 至少 2-3 个工业协议的报文结构与安全问题
- Web 漏洞基础
- IDA / WinDbg / OD 的基本逆向流程
- 漏洞分析报告写作

### 面试加分

- 自己写过协议解析 / 探测 / 扫描小工具
- 复现过公开工控 CVE
- 有 GitHub 知识库和实验记录
- 能解释 PLC、DCS、SCADA 在真实工业网络中的位置
- 能把安全漏洞和工业场景后果联系起来

### 可以后置

- 深度二进制漏洞利用
- 高级模糊测试框架
- 大型靶场建设
- 完整安全产品研发
