# 知识库更新工作流

## 每次提问后的更新方式

当你问完一个问题后，按下面规则更新：

1. 如果是概念问题，沉淀到对应 `docs/` 章节。
2. 如果是临时想法、面试反馈、HR 信息，先写入 `memory/Inbox.md`。
3. 如果是实验过程，写入 `labs/`，并在对应 docs 章节链接它。
4. 如果是可复用方法，写入 `templates/` 或 `prompts/`。
5. 如果是项目成果，写入 `docs/12_Research_Projects/`。

## 推荐提问格式

```text
请更新知识库：
问题：
我的理解：
你帮我纠正：
应该归档到：
```

## 每周复盘

每周末更新：

- 本周学了什么
- 做了什么实验
- 哪些概念还不稳
- 下周最重要的 3 件事
- 是否能转化成面试表达

复盘文件建议放在：

```text
memory/Weekly_Review_YYYY-MM-DD.md
```
