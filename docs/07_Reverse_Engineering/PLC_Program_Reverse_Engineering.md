# PLC 程序逆向

## 1. PLC 程序逆向是什么

PLC 程序逆向是从 PLC 工程文件、下载包、字节码、梯形图导出文件、运行时内存或网络传输中恢复控制逻辑。它与普通软件逆向不同，因为目标不是只理解代码执行，而是理解控制对象和工艺意图。

关注问题：

- 程序控制哪些输入、输出和变量。
- 哪些条件会触发启停、联锁、报警、旁路。
- 是否存在硬编码密码、维护后门、危险默认值。
- 下载或上传过程是否能被窃听、篡改或重放。
- 攻击者修改哪一处逻辑会产生真实物理影响。

## 2. PLC 程序的常见表示

IEC 61131-3 常见语言：

- LD：梯形图，适合继电器逻辑。
- FBD：功能块图，适合控制块连接。
- ST：结构化文本，类似 Pascal。
- IL：指令表，部分平台逐渐弃用。
- SFC：顺序功能图，适合步骤流程。

逆向时要能在不同表示之间建立映射。例如一个 PID 控制块在 FBD 中是图形块，在底层可能是函数调用或字节码指令。

## 3. 工程文件分析

PLC 工程文件可能包含：

- 变量表。
- I/O 映射。
- 程序块。
- 功能块。
- 数据块。
- 硬件组态。
- 通信配置。
- 用户和权限配置。
- 注释和符号信息。

工程文件常见风险：

- 明文保存设备 IP、账号、口令。
- 工程文件无加密或弱加密。
- 项目包可被篡改后重新导入。
- 下载包缺少签名。
- 注释泄露工艺逻辑。

## 4. 网络下载流量分析

工程软件下载 PLC 程序时，流量中可能出现：

- 会话建立。
- 设备识别。
- 权限校验。
- 程序块上传/下载。
- 数据块写入。
- 运行/停止命令。
- 校验和确认。

安全关注点：

- 程序块是否明文传输。
- 是否有完整性保护。
- 是否绑定会话和权限。
- 是否能重放旧下载包。
- 是否允许未授权上传读取程序。

## 5. 控制逻辑还原

还原控制逻辑时，不要只看单个变量，要看控制链：

`传感器输入 -> 滤波/换算 -> PID/逻辑判断 -> 联锁/限幅 -> 输出执行器`

重点识别：

- 启停条件。
- 安全联锁。
- 报警阈值。
- 手自动切换。
- 旁路条件。
- 设定值来源。
- 输出限幅。
- 故障处理。

对安全分析而言，旁路、强制、手动模式、远程写入设定值通常是高风险点。

## 6. PLC 字节码与块分析

部分 PLC 平台会把工程编译为专用字节码或块格式。分析思路：

1. 收集同一项目的多个版本。
2. 修改一个变量或一段逻辑，比较文件差异。
3. 找到符号表、块头、长度、校验。
4. 对照工程软件导出结果恢复语义。
5. 在仿真环境中验证修改影响。

差分是 PLC 程序逆向的重要方法。通过 “只改一个点” 可以建立文件字段和工程语义的对应关系。

## 7. 安全风险模式

| 风险 | 表现 | 影响 |
| --- | --- | --- |
| 未授权上传 | 无需认证读取程序 | 工艺逻辑泄露 |
| 未授权下载 | 无需权限写入程序 | 控制逻辑篡改 |
| 下载包可重放 | 无随机数/签名 | 回滚到旧逻辑 |
| 明文传输 | 程序块可抓包读取 | 知识产权和安全泄露 |
| 弱工程文件保护 | 项目可离线篡改 | 供应链或运维风险 |
| 旁路逻辑滥用 | 维护模式未限制 | 安全联锁失效 |

## 8. 面试表达模板

> PLC 程序逆向我会从工程文件、下载流量和运行时行为三个层面看。工程文件里重点看变量表、I/O 映射、程序块、通信配置和权限信息；下载流量里看会话、认证、块传输和校验；控制逻辑上我会按输入、处理、联锁、输出的链路还原工艺影响。安全分析时，我会特别关注未授权上传/下载、程序块明文传输、下载包重放和维护旁路逻辑。

## 9. 实操验证

可用 OpenPLC 或其他仿真环境练习：

- 写一个简单启停逻辑。
- 导出工程文件。
- 修改一个变量名或阈值，比较文件差异。
- 抓取工程软件与运行时通信。
- 画出控制链路。
- 写一页 “如果攻击者修改该逻辑会发生什么”。
