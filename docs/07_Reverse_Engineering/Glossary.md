# 术语表

| 术语 | 解释 |
| --- | --- |
| 逆向工程 | 从二进制、固件、协议、工程文件中恢复设计意图和实现逻辑 |
| 静态分析 | 不运行目标程序，通过反汇编、反编译、字符串和结构分析理解程序 |
| 动态分析 | 运行目标程序，通过调试、断点、抓包、日志观察实际行为 |
| PE | Windows 可执行文件格式，包括 EXE、DLL、SYS 等 |
| ELF | Linux/嵌入式系统常见可执行文件格式 |
| 入口点 | 程序开始执行的位置，也可泛指外部输入进入逻辑的位置 |
| 导入表 | PE 中记录外部 DLL/API 的结构 |
| 导出表 | DLL 对外提供函数的结构 |
| 交叉引用 | 某个函数、字符串、变量被谁引用，或引用了谁 |
| 反汇编 | 将机器码转换为汇编指令 |
| 反编译 | 将机器码转换为近似高级语言伪代码 |
| 控制流 | 程序分支、跳转、循环和调用关系 |
| 数据流 | 数据从输入到变量、函数、输出的传播路径 |
| 状态机 | 程序根据状态执行不同逻辑的模型 |
| 调用约定 | 函数参数、返回值和栈清理由谁负责的规则 |
| 断点 | 调试中让程序在指定位置暂停的机制 |
| 调用栈 | 当前函数调用链 |
| 固件 | 嵌入式设备中运行的软件镜像 |
| binwalk | 常用固件识别和解包工具 |
| 命令码 | 协议中表示操作类型的字段 |
| 功能码 | 工业协议中表示读写、控制等功能的字段 |
| CRC | 循环冗余校验，用于发现传输错误，不等同安全签名 |
| 硬编码密钥 | 直接写在程序或固件中的密钥 |
| 补丁对比 | 对比漏洞版本和修复版本，定位根因和修复点 |
