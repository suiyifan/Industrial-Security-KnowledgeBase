# 项目：工控 CVE 复现与分析

## 1. 项目目标

选择一个适合本地或仿真环境复现的工控相关 CVE，完成从公告阅读、环境搭建、PoC 审计、复现、根因分析到报告输出的闭环。

## 2. 选题标准

优先选择：

- 有公开公告。
- 有明确影响版本。
- 可在本地环境复现。
- 不需要真实生产设备。
- 有补丁或源码可分析。
- 风险类型和 JD 相关，如 Web 漏洞、协议解析、RCE、认证绕过。

## 3. 项目步骤

1. 整理 CVE 信息。
2. 搭建受影响版本。
3. 审计 PoC 安全性。
4. 提取最小触发请求。
5. 记录复现证据。
6. 分析根因。
7. 评估工控影响。
8. 输出修复和缓解建议。

## 4. 产出物

- CVE 信息卡。
- 环境搭建文档。
- PoC 审计记录。
- 最小触发样本。
- 根因分析。
- 工控影响分析。
- Markdown 报告。

## 5. 面试表达

> 我做 CVE 复现时不是只跑 PoC，而是先审计 PoC 安全性，确认环境版本，再提取最小触发请求。复现成功后我会分析根因，比如鉴权缺失、命令拼接或长度校验不足，并进一步评估如果这个组件部署在工控网络中，会对工程站、网关或 HMI 造成什么影响。
