# 项目：OpenPLC + Modbus + PID 实验室

## 1. 项目目标

搭建一个本地工控仿真实验室，用 OpenPLC 或类似环境模拟 PLC，通过 Modbus TCP 进行读写，设计一个简单控制逻辑，观察控制变量、设定值和输出之间的关系。

## 2. 能证明的能力

- PLC 基础。
- Modbus TCP 理解。
- 控制回路理解。
- 抓包分析。
- 安全风险映射。
- 实验报告写作。

## 3. 实验设计

建议模拟：

`设定值 SP -> 控制逻辑/PID -> 输出 MV -> 模拟过程变量 PV`

可以从简单启停逻辑开始，再扩展到 PID 思路。

## 4. 实验步骤

1. 搭建 OpenPLC。
2. 编写简单控制逻辑。
3. 配置 Modbus 地址映射。
4. 用 Python 或 Modbus 客户端读取寄存器。
5. 用 Wireshark 抓取读写流量。
6. 标注功能码、地址、数量和值。
7. 分析如果攻击者修改设定值或强制输出会产生什么影响。

## 5. 产出物

- 网络拓扑图。
- PLC 逻辑截图或代码。
- Modbus 地址表。
- PCAP 文件。
- 报文解析表。
- 风险分析报告。

## 6. 面试表达

> 我搭建过一个 OpenPLC + Modbus 的本地实验环境，用于理解 PLC 控制逻辑和 Modbus 通信。我设计了简单控制变量，抓包分析读写寄存器报文，并把功能码、地址和值对应到控制逻辑。这个项目让我能把协议字段和实际控制影响连接起来，比如写寄存器不只是数据变化，可能对应设定值或输出变化。
