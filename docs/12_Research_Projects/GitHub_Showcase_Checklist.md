# GitHub 展示检查清单

## 项目完整性

- 有清晰 README。
- 有环境说明。
- 有示例输入。
- 有示例输出。
- 有限制条件。
- 有安全声明。
- 有后续计划。

## 技术可信度

- 不是空壳项目。
- 有真实字段表或报告。
- 有代码注释但不过度。
- 有错误处理。
- 有测试样例。
- 有截图或日志证据。

## 求职相关性

- 能对应 JD 要求。
- 能讲出工控场景。
- 能体现协议、漏洞、工具、报告至少一种能力。
- 能讲清楚自己做了什么。

## 风险控制

- 不包含攻击公网目标脚本。
- 不包含默认破坏性 payload。
- 不泄露真实账号、IP、密钥。
- 不夸大项目效果。

## 面试准备

- 准备 30 秒项目摘要。
- 准备 2 分钟技术讲解。
- 准备一个踩坑和修复经历。
- 准备项目后续优化方向。
