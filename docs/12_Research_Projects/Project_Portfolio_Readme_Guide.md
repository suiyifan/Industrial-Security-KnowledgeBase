# 项目 README 写作指南

## 1. README 的目标

README 要让面试官 2 分钟内看懂：

- 你解决什么问题。
- 你怎么做。
- 你产出了什么。
- 你理解哪些安全边界。
- 这个项目和岗位有什么关系。

## 2. 推荐结构

```text
# 项目名称

## 背景
## 目标
## 环境
## 方法
## 核心功能
## 示例输出
## 安全边界
## 结论
## 后续计划
```

## 3. 证据材料

README 中可以链接：

- 抓包截图。
- 字段表。
- 代码文件。
- 报告 Markdown。
- 环境配置。
- 测试样例。

## 4. 常见问题

不要只写 “这是一个扫描器”。要写清：

- 为什么需要它。
- 输入输出是什么。
- 有哪些限制。
- 为什么默认只读。
- 如何避免影响工控设备。

## 5. 面试表达

> 我的项目 README 会按背景、目标、环境、方法、输出和安全边界来写。因为安全工具不能只展示效果，还要说明授权使用、只读限制、速率控制和适用场景。这样别人能复现，也能看到我的工程化意识。
