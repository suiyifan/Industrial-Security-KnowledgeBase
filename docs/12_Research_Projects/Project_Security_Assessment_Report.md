# 项目：安全评估样例报告

## 1. 项目目标

基于仿真拓扑或公开样例，写一份工控安全评估报告，覆盖资产、网络、协议、漏洞、风险路径和整改建议。

## 2. 报告结构

1. 评估范围。
2. 资产清单。
3. 网络拓扑。
4. 通信矩阵。
5. 发现问题。
6. 攻击路径。
7. 工控影响。
8. 整改建议。
9. 优先级计划。

## 3. 可模拟问题

- 工程站和 PLC 未分区。
- HMI 使用弱口令。
- Modbus 明文且允许写操作。
- 远程维护入口访问范围过大。
- 缺少 PLC 下载审计。
- 缺少备份和恢复演练。

## 4. 面试表达

> 我写过一份工控安全评估样例报告，不只是列漏洞，而是从资产、拓扑、通信矩阵和攻击路径分析风险。比如我会说明某个远程维护入口如何到达工程站，再通过工程站访问 PLC，最后映射到可能的下载程序或修改设定值风险，并给出分区分域、访问控制、审计和备份等整改建议。
