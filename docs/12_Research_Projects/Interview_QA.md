# 面试问答

## Q1：你最想展示哪个项目？

我会优先展示 Modbus 流量分析器或 OpenPLC 实验室，因为它们能同时体现工控协议、控制逻辑、抓包分析和安全风险映射。相比单纯 Web 漏洞复现，这类项目更贴近工控安全岗位。

## Q2：项目如何证明你真的理解工控安全？

项目不只展示代码，还会展示控制对象、协议字段、数据流、风险影响和安全边界。比如 Modbus 写寄存器在代码里只是一个功能码，但在工控场景中可能对应设定值修改或输出变化，这种映射能体现理解深度。

## Q3：如果没有真实工控设备怎么办？

可以用 OpenPLC、仿真服务、公开 PCAP、开源项目和本地 CVE 环境构建安全实验。重点是把实验边界说明清楚，不假装自己在真实生产设备上做过。

## Q4：如何讲项目踩坑？

可以讲环境版本不一致、PoC 不安全、协议字段判断错误、抓包和代码不一致、工具异常处理不足等。关键是说明你如何定位问题、修正方法、沉淀经验。

## Q5：项目和岗位要求如何对应？

可以把项目映射到 JD：协议分析、漏洞研究、工具开发、安全评估报告、逆向或 Web 安全。每个项目都准备一段对应岗位能力的说明。
