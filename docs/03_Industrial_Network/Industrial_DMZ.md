# 工业 DMZ 深入理解

## 1. 工业 DMZ 是什么

工业 DMZ 是办公网和控制网之间的缓冲区。它不是普通互联网 DMZ，而是用于隔离企业 IT 与工业控制网络的数据交换区域。

目标：

- 避免办公网直接访问控制网。
- 提供受控的数据交换。
- 集中管理远程运维、补丁、杀毒、文件传输等跨域需求。

## 2. 为什么需要工业 DMZ

很多业务需要跨域：

- 管理层需要看生产报表。
- 办公网需要获取历史数据。
- 控制网需要补丁和杀毒更新。
- 厂商需要远程维护。
- 文件需要从办公网导入生产环境。

如果没有 DMZ，这些需求可能变成办公网直连控制网。

## 3. DMZ 常见资产

| 资产 | 作用 | 风险 |
|---|---|---|
| 跳板机 | 运维入口 | 权限过大、审计不足 |
| 数据中转服务器 | 同步生产数据 | 双向通道风险 |
| 补丁服务器 | 补丁分发 | 补丁源污染 |
| 杀毒更新服务器 | 病毒库更新 | 更新通道被滥用 |
| 文件交换服务器 | 跨域文件传递 | 恶意文件进入控制网 |
| 反向代理 | 发布服务 | 配置错误暴露内部 |
| 堡垒机 | 运维审计 | 账号和会话管理风险 |

## 4. DMZ 通信原则

理想原则：

```text
办公网不直接访问控制网
控制网不直接访问互联网
跨域通信在 DMZ 终止或中转
```

推荐方向：

- 控制网 -> DMZ：上传生产数据。
- 办公网 -> DMZ：读取报表或中转数据。
- DMZ -> 控制网：尽量少，必须严格白名单。

## 5. 常见错误设计

### 错误 1：DMZ 双向全通

如果 DMZ 到办公网、DMZ 到控制网都是全通，DMZ 就失去缓冲价值。

### 错误 2：跳板机直接到 PLC

跳板机通常应访问工程站或运维终端，不应随意直连控制器。

### 错误 3：文件交换无查杀

从办公网带入工程站的文件如果不查杀，可能引入恶意代码。

### 错误 4：远程账号长期有效

厂商维护账号长期开放，是常见高风险入口。

## 6. 评估检查项

- DMZ 是否独立网络区域。
- 办公网是否能绕过 DMZ 访问控制网。
- 控制网是否能直接访问互联网。
- DMZ 中有哪些服务器。
- DMZ 与控制网之间开放哪些端口。
- 是否有双向文件同步。
- 跳板机是否有会话审计。
- 远程运维是否按需开通。
- 是否有恶意文件检测。

## 7. 面试表达

> 工业 DMZ 的作用是作为办公网和控制网之间的缓冲区，承载数据中转、补丁更新、文件交换和远程运维等跨域需求。评估时我会重点看办公网是否绕过 DMZ 直连控制网，DMZ 到控制网是否最小开放，跳板机和文件交换是否有审计和查杀。
