# 分区分域与安全域设计

## 1. 什么是分区分域

分区分域是把系统按功能、重要性和安全需求划分成不同安全区域，并控制区域之间的通信。

白话理解：

> 不同风险等级的资产不要混在一张大网里。该通信的放行，不该通信的阻断。

## 2. Zone 与 Conduit

IEC 62443 中常见两个概念：

- Zone：区域，一组安全需求相似的资产。
- Conduit：管道，区域之间受控通信路径。

示例：

```text
HMI Zone <-> Control Zone
```

中间的通信路径就是 Conduit，应通过防火墙、ACL、协议审计等方式控制。

## 3. 常见安全域

- 办公网域
- 生产管理域
- 工业 DMZ
- 监控操作域
- 控制器域
- 安全系统域
- 远程运维域
- 厂商接入域

## 4. 分区依据

划分安全域时考虑：

- 资产功能
- 生产重要性
- 协议类型
- 是否需要外部访问
- 是否能下发控制命令
- 是否涉及安全联锁
- 是否允许停机维护

## 5. 区域间通信原则

推荐原则：

1. 默认拒绝。
2. 明确业务必要性。
3. 最小端口开放。
4. 明确源和目的。
5. 明确通信方向。
6. 高风险操作单独审批。
7. 有日志和告警。

错误策略：

```text
办公网 -> 控制网: any any allow
```

更合理策略：

```text
指定跳板机 -> 指定工程站: RDP, only during maintenance window
SCADA Server -> PLC: Modbus TCP 502, read/write only if business required
历史采集服务器 -> SCADA Server: database/API, one-way preferred
```

## 6. 安全域设计中的常见问题

- 有分区图，但没有防火墙策略。
- VLAN 划分了，但交换机三层互通无限制。
- 工程师电脑双网卡跨接办公网和控制网。
- 厂商维护 VPN 直接进控制层。
- 历史数据库同时被办公网和控制网访问。
- 工业 DMZ 里部署过多高权限服务。

## 7. 评估时怎么问

- 这个区域的核心功能是什么？
- 这个区域里有哪些高价值资产？
- 哪些区域可以访问它？
- 访问使用什么协议和端口？
- 是否存在跨区域双向通信？
- 是否有日志记录？
- 是否有临时策略长期保留？

## 8. 面试表达

> 分区分域不是简单划 VLAN，而是基于资产功能和安全需求定义区域，再用受控通信管道连接区域。评估时我会关注区域之间是否默认拒绝、通信是否业务必要、端口是否最小开放，以及是否存在办公网直达工程站或控制器的路径。

## 9. 可直接使用的 Zone 定义表

| 字段 | 示例/要求 |
| --- | --- |
| Zone 名称与编号 | Cell-A-Control，避免只写 VLAN 20 |
| 生产功能 | 反应器 A 基本控制与操作监视 |
| 成员 | PLC-A、HMI-A、远程 I/O；例外单列 |
| 关键能力 | 设定、启停、逻辑下载、报警处理 |
| 安全目标 | 周期通信连续，非窗口禁止工程变更 |
| 身份与管理 | 工业域账号；管理仅来自受控工作站 |
| 失联行为 | PLC 自治，HMI Bad Quality，禁止缓存命令 |
| 监测与恢复 | 单元 TAP；可信项目与交换配置恢复 |

划分后做三项反证：同 Zone 内任一资产被攻陷，是否会获得不合理能力；Zone 间每条 Conduit 是否有明确业务；管理、身份、备份和时间是否形成未画出的通道。

## 10. Conduit 评审卡

```text
业务：________________ 责任人：________________
源 Zone/资产/进程：________________________________
目的 Zone/资产/服务：______________________________
发起方向：________ 协议/操作：______________________
身份/证书：________ 正常周期/窗口：__________________
超时/重连/补传：___________________________________
边界控制：________ 监测证据：_______________________
失联后过程状态：___________________________________
恢复与验收：_______________________________________
```

常见否决项包括：以“运维方便”为唯一理由、使用共享管理员、允许整个网段双向访问、未说明故障路径、DMZ 透明转发、同一主机双网卡绕过边界，以及无法说明写操作对应的物理对象。

## 11. 设计验收问题

- 资产是否按控制能力和后果归域，而不是按部门或 IP？
- SIS、BPCS 和管理平台的共同依赖是否单列？
- 每条跨区流能否从业务追到规则、账号、日志和过程状态？
- 主备、维护、启动、停车和恢复流量是否纳入？
- 阻断通道时，控制器、操作员和现场过程进入什么状态？
- 云、蜂窝、无线、虚拟化和供应商连接是否破坏层级假设？
