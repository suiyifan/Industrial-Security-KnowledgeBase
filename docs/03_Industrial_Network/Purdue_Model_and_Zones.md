# Purdue 模型与工控分层

## 1. Purdue 模型是什么

Purdue 模型是一种常用的工业控制系统分层参考模型，用于描述企业 IT、生产管理、监控控制和现场设备之间的关系。

它不是绝对标准，但在工控安全沟通中非常常见。

## 2. 常见层级

| 层级 | 名称 | 常见资产 |
|---|---|---|
| Level 5 | 企业外部 / 互联网 | 云服务、外部供应商 |
| Level 4 | 企业 IT / 办公网 | AD、邮件、OA、ERP、办公终端 |
| Level 3.5 | 工业 DMZ | 跳板机、补丁服务器、数据中转、文件交换 |
| Level 3 | 生产运营管理 | MES、调度、报表、生产数据库 |
| Level 2 | 监控控制 | HMI、SCADA、操作员站、历史库 |
| Level 1 | 基本控制 | PLC、DCS 控制器、RTU、IED |
| Level 0 | 物理过程 | 传感器、执行器、泵、阀、电机 |

## 3. 每一层的安全关注点

### Level 4：企业 IT

关注：

- 钓鱼邮件
- 域控风险
- 办公终端感染
- VPN 账号泄露

工控影响：

> Level 4 常是攻击入口，攻击者可能从办公网横向移动到生产区域。

### Level 3.5：工业 DMZ

关注：

- 数据中转是否单向
- 跳板机审计
- 补丁和杀毒更新路径
- 文件交换审批

工控影响：

> DMZ 是缓冲区，如果设计不当，会变成办公网进入控制网的跳板。

### Level 3：生产管理

关注：

- MES 与 SCADA / 历史库数据交互
- 生产报表访问
- 数据库权限
- 与办公网接口

工控影响：

> 生产管理层通常需要获取控制网数据，但不应随意下发控制命令。

### Level 2：监控控制

关注：

- HMI 权限
- SCADA Server
- 历史数据库
- 操作员站
- 报警和趋势

工控影响：

> Level 2 是人机操作和监控中心，被攻陷后可能影响操作和数据可信度。

### Level 1：基本控制

关注：

- PLC / DCS 控制站
- 工程协议
- 变量读写
- 程序下载
- 控制器状态

工控影响：

> Level 1 直接控制设备，是安全评估中的高价值区域。

### Level 0：物理过程

关注：

- 传感器和执行器
- 现场仪表
- 物理访问
- 信号干扰

工控影响：

> Level 0 的变化就是物理后果，例如阀门开关、电机启停、压力温度变化。

## 4. 分层不是越多越安全

错误理解：

```text
只要画了 Purdue 模型，就安全。
```

正确理解：

```text
分层必须配合访问控制、通信矩阵、日志审计、资产管理和变更管理。
```

如果 Level 4 可以直接 RDP 到 Level 2 工程站，图画得再漂亮也没有意义。

## 5. 面试表达

> Purdue 模型帮助我从层级上理解工控网络。Level 4 是企业 IT，Level 3 是生产管理，Level 2 是监控操作，Level 1 是 PLC/DCS 控制器，Level 0 是现场设备。安全评估时我会重点检查跨层通信是否必要，尤其是办公网到监控层、监控层到控制层、远程运维到工程站这些路径。
