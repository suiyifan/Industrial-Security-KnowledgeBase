# 工控网络安全设备与边界能力

## 1. 为什么要单独理解安全设备

工控安全项目中经常会遇到一堆安全设备：

- 工业防火墙
- 网闸
- 单向传输设备
- 堡垒机
- 工控 IDS
- 日志审计
- 安全隔离与信息交换系统
- VPN 网关
- 数据采集网关

面试时如果只会说“上防火墙”，会显得比较浅。要能说清楚每种设备解决什么问题、不能解决什么问题。

## 2. 工业防火墙

作用：

- 控制不同安全域之间的通信。
- 限制源、目的、端口、协议。
- 部分产品支持工业协议深度解析。

适用：

- 产线边界。
- PLC 前端。
- DMZ 到控制网边界。

局限：

- 策略配置错误会失效。
- 不理解业务容易误拦或放过。
- 不能替代账号权限和工程变更管理。

## 3. 网闸

网闸通常用于高隔离要求场景，实现安全隔离和受控数据交换。

适用：

- 办公网和生产网之间数据交换。
- 生产数据向上层系统发布。
- 文件跨域传输。

关注：

- 是否真的隔离。
- 数据交换方向。
- 文件类型和内容检查。
- 是否有审批和日志。

## 4. 单向传输

单向传输强调数据只能从一个方向流向另一个方向。

常见场景：

```text
控制网 -> DMZ / 办公网
```

用途：

- 上传生产数据。
- 避免上层网络反向访问控制网。

局限：

- 不适合需要双向控制的业务。
- 部署前要确认业务需求。
- 仍需关注数据完整性和接收端安全。

## 5. 堡垒机

堡垒机用于集中管理运维访问。

能力：

- 账号集中管理。
- 访问审批。
- 会话录像。
- 命令审计。
- 文件传输控制。

工控场景关注：

- 是否覆盖工程站、服务器、网络设备。
- 是否禁止绕过堡垒机直连。
- 是否能关联具体人员。
- 是否按维护窗口开通。

## 6. 工控 IDS

工控 IDS 通过被动流量识别资产、协议和异常行为。

关注：

- 新资产上线。
- 异常跨区通信。
- PLC 写操作。
- 工程下载。
- 控制器状态变化。
- 非工程站访问工程协议。

局限：

- 只能看到镜像点流量。
- 告警需要结合业务确认。
- 对加密流量可见性有限。

## 7. 日志审计

日志审计用于收集和分析：

- 服务器日志
- 防火墙日志
- 堡垒机会话日志
- 操作员操作日志
- 工程变更日志
- IDS 告警

工控安全价值：

- 事故追溯。
- 发现异常登录。
- 追踪工程下载。
- 关联报警和网络行为。

## 8. VPN 网关

VPN 提供远程接入能力。

安全关注：

- MFA。
- 账号实名。
- 最小权限。
- 访问范围。
- 日志审计。
- 按需开通。
- 账号到期回收。

## 9. 安全设备常见误区

### 误区 1：买了设备就安全

设备只是能力，策略、运维和审计才决定效果。

### 误区 2：只在最外层部署防火墙

工控网络内部也需要分区边界，尤其是产线之间、监控层到控制层之间。

### 误区 3：IDS 告警越多越好

告警需要可运营。大量误报会导致无人处理。

### 误区 4：堡垒机覆盖了就没有远程风险

如果还有备用 RDP、厂商远控软件或 VPN 绕过堡垒机，风险仍然存在。

## 10. 面试表达

> 我理解工控安全设备要结合场景使用。工业防火墙适合做边界白名单，网闸和单向传输适合高隔离数据交换，堡垒机适合远程运维审计，工控 IDS 适合被动发现资产和异常协议行为。评估时不能只看设备有没有，还要看策略、覆盖范围、日志、告警和是否存在绕过路径。
