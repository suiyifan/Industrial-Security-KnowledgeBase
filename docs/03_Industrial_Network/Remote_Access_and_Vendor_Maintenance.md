# 远程运维与供应商接入

## 1. 原理解释：远程运维为何是复合高风险入口

工业设备分布广、厂商专用性强，故障处理、升级和参数诊断经常需要外部专家。远程运维减少到场时间，却把外部身份、未知终端、互联网路径、文件传输和高权限工程操作连接到控制系统。风险不只来自 VPN 漏洞，还来自授权范围、会话内容、工程语义和维护结束后的残余状态。

一个 VPN 建连成功只证明形成了加密网络通道，不证明用户可信、终端干净、访问目标合理或操作安全。成熟架构要把外部网络接入、身份认证、受控跳转、目标授权、工程操作和现场控制权分层，并让任何一层失效都不直接获得 PLC 工程能力。

## 2. 典型架构与信任终止

```text
供应商人员 / 受管终端
  ↓ VPN / ZTNA + MFA + 设备检查
企业或外部访问区
  ↓ 工业 DMZ 远程访问网关
堡垒机 / 会话代理 / 文件交换
  ↓ 临时最小规则
指定工程站或维护服务器
  ↓ 本地工程权限与现场批准
PLC / DCS / SCADA / 网络设备
```

不建议让供应商 VPN 客户端获得可直接路由到 PLC 的地址。远程会话应在 DMZ 或堡垒终止，再以受控身份连接指定工程站。工程站到控制器的操作仍需项目版本、控制器模式、下载权限和现场许可约束。

架构中的每个信任终止点都应重新认证和授权。若 VPN 用户进入后可使用共享 Windows 管理员横向访问，堡垒只是多一次登录；若堡垒透传供应商密码到所有目标，凭据泄露仍会扩散。

## 3. 远程访问方式与能力边界

### 3.1 VPN

站点到站点 VPN 连接两个网络，远程接入 VPN 连接单个用户或终端。需要限制分配地址、可达网段、加密套件、证书、Split Tunneling、DNS 和空闲/最大时长。站点隧道长期在线时，供应商网络实际上成为受信区域，必须评估对方内部安全和路由发布。

### 3.2 堡垒机与远程桌面网关

堡垒集中认证、目标授权、会话录像和命令审计。录像可支持追溯，但不能自动理解 PLC 逻辑变化；键盘录像也可能遗漏 GUI 工程软件内部操作。应同步收集工程软件审计、项目差异、控制器下载和现场反馈。

### 3.3 ZTNA/应用代理

按用户、设备与应用授予访问可减少网络层全网可达，但工业协议、专用工程工具和旧设备未必兼容。代理可能只覆盖 Web/RDP/SSH，工程站之后的控制器路径仍需治理。产品“零信任”标签不能替代具体数据流验证。

### 3.4 带外与蜂窝维护

4G/5G 路由器、拨号设备、厂商云盒或独立调制解调器可能绕过厂区边界。它们常被临时安装后遗留，使用默认凭据或云端长连接。资产盘点需结合射频、SIM、运营商账单、电源和现场柜内检查。

## 4. 身份、终端与凭据治理

远程用户使用个人实名账号和 MFA，禁止共享“vendor”账号。身份应绑定供应商、合同、工单、目标、角色和有效期；离职、合同结束和紧急访问后及时撤销。服务账号与人员账号分离，API 密钥、证书和本地 PLC 密码纳入保管与轮换。

MFA 降低密码被盗风险，但不能阻止被控制的已登录终端或恶意维护人员。终端应由组织管理，检查补丁、EDR、磁盘加密、禁用不必要网络共享，并限制同时连接互联网和控制会话。供应商自带笔记本若无法满足要求，可使用厂方提供的受管虚拟桌面或现场工程站。

高权限凭据尽量在堡垒或 PAM 中托管，不暴露给供应商终端。临时凭据会话后轮换；本地应急账号使用需告警、复核和重新密封。不能因控制设备不支持集中身份，就在所有 PLC 复用一个长期密码。

## 5. 授权状态机与现场控制权

远程维护应按状态流治理：

```text
Requested → Risk Reviewed → Approved → Scheduled
→ Access Enabled → Identity Verified → Session Active
→ Operation Confirmed → Session Closed → Access Revoked
→ Residual State Checked → Evidence Archived
```

申请记录设备、目的、操作、预计影响、人员、时间和回退。批准前确认生产状态、旁路/联锁、备份和现场联系人。访问启用使用自动到期规则；会话开始由现场人员确认目标和控制权；关键下载或模式切换可要求双人批准。

维护结束不等于关闭 RDP 窗口。退出检查包括 VPN/规则和账号撤销、残余会话终止、文件清理、项目/固件版本核对、控制器模式、强制、旁路、报警抑制、自动重启、设备本地/远程状态以及过程运行确认。

## 6. 文件、剪贴板与工具通道

工程项目、补丁、固件、许可证和诊断工具是远程维护的重要输入。堡垒的剪贴板、磁盘映射、USB 重定向、打印、端口转发和共享目录都可能绕过文件交换流程。默认关闭，确需使用时通过受控通道执行来源验证、签名/哈希、恶意代码检查、审批和归档。

合法签名只能证明签发者和内容未变，不保证版本适用于现场。固件升级要检查硬件型号、引导程序、项目兼容性、冗余顺序和回退能力；工程项目要比较离线与在线版本，禁止未经审查整体覆盖现场调整。

远程诊断工具可能安装驱动、虚拟网卡、抓包组件或后台服务。会话后检查新服务、计划任务、证书、路由和防火墙规则，防止工具留下持久通道。

## 7. 会话审计与证据边界

VPN 日志证明身份何时建立隧道和分配地址；堡垒日志证明连接了哪个目标；录像展示操作画面；工程审计证明项目比较、下载或强制；PLC 日志证明控制器接受操作；过程趋势和反馈证明物理响应。任何单一来源都不足以完成归因。

会话录像要有时间同步、访问控制、完整性和保留策略。GUI 分辨率或网络卡顿可能漏帧；使用脚本、后台传输或二次跳转可能不出现在主画面。关键事件需关联工单、用户、源终端、VPN 会话 ID、堡垒会话、目标登录、文件哈希、工程版本和过程数据。

隐私与商业机密也需考虑。录像和工程包只授予调查与审计角色访问，导出保留审批和哈希，避免审计平台本身成为项目文件泄露点。

## 8. 失联、紧急访问与可用性

远程访问平台故障不应阻止现场安全操作。控制区保持本地工程和应急能力，堡垒/身份服务失联时不自动切换为全网放通。Break-glass 流程定义触发条件、审批、最小目标、短时凭据、全程记录和事后复核。

远程会话中断时，已下发命令、文件或固件可能处于中间状态。应用需定义事务性、断点和恢复：PLC 下载中断是否保留旧程序，固件升级失败是否可回滚，参数批量写入完成到哪一项。会话恢复后不能盲目重试全部操作。

高延迟链路会造成画面陈旧和误操作。远程人员需要看到质量、源时间和命令反馈；高风险动作由现场人员确认真实设备状态。紧急情况下优先稳定过程，而不是强行恢复远程连接。

## 9. 数据流、控制流与状态流

```text
数据流：供应商输入 → VPN/代理 → 堡垒 → 工程站 → 设备与反馈
控制流：工单 → 身份/MFA → 目标授权 → 操作审批 → 现场许可
状态流：Disabled → Enabled → Active → Suspended/Expired
       → Closed → Revoked → Verified
设备流：Local/Run → Maintenance → Program/Stop → Tested → Run/Remote
```

网络访问状态与设备维护状态必须关联。访问已过期但会话未终止、会话关闭但 PLC 仍在 Program、工程站退出但强制仍存在，都是常见控制缺口。

## 10. 安全风险与攻击链

安全风险包括共享供应商账号、无 MFA、长期 VPN、Split Tunnel、全网路由、堡垒直达 PLC、会话录像无工程语义、文件通道绕过、供应商终端感染、厂商云盒影子接入、紧急规则不回收和维护后残余强制。

典型攻击链是：供应商邮箱被钓鱼，攻击者获取 VPN 密码并诱导通过 MFA；进入长期开放的站点隧道后扫描维护网段，利用共享管理员登录工程站；上传被篡改项目，在夜间窗口下载 PLC，并通过 HMI 截图掩盖变化。若设备允许自动运行，逻辑修改可能改变泵阀顺序。

防护分别限制入口、横向与控制能力：实名 MFA 和设备检查；DMZ 会话终止；临时源/目标规则；堡垒隔离与文件审批；工程变更双人复核；控制器下载保护；在线/离线比较；过程一致性监测；SIS 和现场联锁限制后果。

## 11. 工控场景：维护完成后自动重启参数遗留

供应商远程诊断 VFD 间歇欠压，为测试将“来电自动重启”临时启用。会话录像显示测试成功，工单关闭，但退出清单未检查参数差异。数日后停电恢复，维护人员正在泵旁检查，VFD 自动启动。

事件链涉及远程账号、参数写入、审批边界、工程基线、现场作业和能量隔离。网络日志只能证明会话，驱动器参数审计与物理状态才能证明风险。整改包括参数差异自动生成、临时设置到期/回退、维护模式禁止远程启动、锁定挂牌和会话结束的现场复核。

## 12. 评估与实施流程

1. 枚举 VPN、堡垒、云盒、蜂窝、拨号和临时端口等全部入口。
2. 将供应商、账号、终端、合同、工单、目标和权限关联。
3. 还原完整路径与文件/剪贴板/二次跳转通道。
4. 核查申请、审批、启用、监护、关闭和退出状态机。
5. 用工单、日志、录像、工程差异和过程数据抽样复核。
6. 在隔离环境测试超时、会话中断、下载失败和紧急访问。
7. 定期撤销闲置账号、证书、SIM、隧道、规则和工具服务。

## 13. 面试表达

### 60 分答案

> 远程运维应经过 VPN、MFA、工业 DMZ 和堡垒机，使用实名临时账号，只允许在批准时间访问指定目标，并保留会话审计。

### 90 分答案

> 我会把远程运维看作从外部身份到物理控制能力的完整状态机。网络上分层终止 VPN、堡垒和工程会话；身份上绑定人员、终端、工单、目标和有效期；操作上限制文件、剪贴板、二次跳转、工程下载和设备模式；现场上要求控制权、工况和回退确认。审计会关联 VPN、堡垒、文件哈希、工程差异、控制器和过程反馈。结束后撤销规则与凭据，并检查模式、强制、旁路和临时参数，避免会话关闭但风险仍留在现场。

### 常见追问

**问：有 VPN 和堡垒机是否就安全？**

答：不够。还要验证终端、身份、目标范围、文件通道、工程权限、现场批准、残余状态和审计关联。

**问：紧急故障来不及审批怎么办？**

答：预先定义 Break-glass，采用短时最小访问、现场确认、完整记录和强制事后复核，而不是临时全网放通。

## 14. 实操建议

### 实验一：授权状态机

在实验环境建立临时用户、指定目标和自动到期策略，验证未审批、过期、目标外访问和残余会话均被阻断。

### 实验二：文件与会话证据

通过受控堡垒传输无害项目文件，关联工单、用户、会话、文件哈希、目标登录和工程差异，检查录像不能覆盖的证据。

### 实验三：中断与恢复桌面演练

模拟远程下载、参数批量修改和固件升级中断，定义停止条件、现场核实、断点判断、回滚和重新授权步骤。

### 实验四：退出检查

在仿真 PLC/VFD 设置临时强制、模式和参数，结束会话后按清单恢复并交叉验证网络、工程和现场状态。

## 15. 总结与检查清单

- [ ] 能还原全部远程接入与隐蔽旁路；
- [ ] 能设计分层终止而非 VPN 直达控制器；
- [ ] 能治理人员、终端、凭据、文件和工程权限；
- [ ] 能执行完整授权与退出状态机；
- [ ] 能关联网络、会话、工程和过程证据；
- [ ] 能处理失联、紧急访问与中断恢复；
- [ ] 能把远程攻击链映射到物理后果与防护。
