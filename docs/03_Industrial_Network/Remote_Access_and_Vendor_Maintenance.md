# 远程运维与第三方接入

## 1. 为什么远程运维高风险

远程运维是工控网络中最常见、也最危险的入口之一。

原因：

- 外部人员可以进入生产网络。
- 账号可能长期有效。
- 运维工具权限高。
- 维护路径可能绕过工业 DMZ。
- 远程操作可能影响工程站、HMI、PLC。

## 2. 常见远程运维方式

- VPN
- 堡垒机
- RDP
- TeamViewer / AnyDesk 等远控软件
- 专线
- 4G/5G 工业路由器
- 厂商云平台
- 临时拨号或无线接入

## 3. 远程运维风险链路

```text
厂商账号泄露
-> VPN 接入
-> 登录跳板机或工程站
-> 获取工程文件
-> 在线监控 PLC
-> 下载工程或修改变量
```

## 4. 安全设计原则

推荐：

- 按需开通。
- 维护窗口审批。
- 多因素认证。
- 堡垒机统一入口。
- 会话录像。
- 命令审计。
- 账号到期自动回收。
- 禁止直连 PLC。
- 维护后复核工程和日志。

## 5. 评估检查项

- 是否有远程运维清单。
- 哪些厂商有权限。
- 远程入口在哪里。
- 是否经过 DMZ。
- 是否启用 MFA。
- 是否存在共享账号。
- 是否有会话审计。
- 是否允许文件传输。
- 是否能访问工程站。
- 是否能访问控制器。
- 是否有开通和关闭记录。

## 6. 常见问题

- VPN 长期开启。
- 厂商账号多人共用。
- RDP 直接暴露到控制网。
- 远控软件绕过堡垒机。
- 维护完成后账号不回收。
- 远程文件传输不查杀。
- 维护操作无录像。

## 7. 报告表达模板

```text
发现【厂商/运维人员】可通过【远程入口】访问【目标资产】。
该路径绕过/经过【边界设备】。
远程账号具备【权限】。
若账号泄露，攻击者可能【访问工程站/修改配置/下载工程】。
建议按需开通、MFA、堡垒机审计、最小权限和维护后账号回收。
```

## 8. 面试表达

> 我会把远程运维作为工控网络评估重点，因为它往往连接外部人员和内部高价值资产。检查时我会看远程入口是否经过工业 DMZ 和堡垒机，账号是否实名和 MFA，是否允许文件传输，是否能访问工程站或 PLC，以及维护是否有审批、录像和账号回收。
