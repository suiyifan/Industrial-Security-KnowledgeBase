# 03 章阅读路线

你现在可以先不做抓包和实践，按“架构 -> 边界 -> 路径 -> 项目表达”的顺序读。

## 第一轮：建立工业网络全局图

阅读：

1. `Industrial_Network_Overview.md`
2. `Purdue_Model_and_Zones.md`
3. `Typical_Topologies.md`

目标：

- 理解工业网络不是一张扁平网。
- 能说清楚办公网、生产管理网、工业 DMZ、监控层、控制层、现场层。
- 能看懂常见单产线、多站点 SCADA、DCS、工业 DMZ 拓扑。

复习问题：

- Purdue 模型 Level 0 到 Level 4 分别是什么？
- 工业 DMZ 为什么不是普通互联网 DMZ？
- 双网卡工程站为什么危险？

## 第二轮：理解分区和边界

阅读：

1. `Zones_and_Conduits.md`
2. `Industrial_DMZ.md`
3. `Industrial_Firewall_ACL.md`
4. `Network_Security_Devices.md`

目标：

- 理解 Zone 和 Conduit。
- 理解防火墙、网闸、堡垒机、单向传输、数据交换的区别。
- 知道访问控制要基于通信矩阵和最小权限。

复习问题：

- VLAN 为什么不等于安全隔离？
- 什么情况下需要工业 DMZ？
- 防火墙白名单策略至少要写清楚哪些字段？

## 第三轮：理解通信与攻击路径

阅读：

1. `Communication_Matrix.md`
2. `Attack_Path_Modeling.md`
3. `Remote_Access_and_Vendor_Maintenance.md`
4. `Gateway_Protocol_Conversion.md`

目标：

- 能把拓扑图转成通信矩阵。
- 能从通信矩阵中找异常路径。
- 能把远程运维、工业网关、协议转换写成攻击路径。

复习问题：

- 通信矩阵为什么要记录读写权限？
- 厂商 VPN 长期开启有什么风险？
- 工业网关为什么是高价值边界资产？

## 第四轮：理解监测和抓包

阅读：

1. `Passive_Monitoring_and_IDS.md`
2. `Packet_Capture_Methodology.md`
3. `Switch_Router_VLAN.md`
4. `Wireless_Serial_Fieldbus.md`

目标：

- 理解为什么工控网络优先被动监测。
- 知道镜像口、TAP、工业 IDS 的价值。
- 知道抓包要输出通信矩阵，而不只是截图。
- 记住工业网络不只有以太网，还有串口、现场总线、无线和 4G/5G。

复习问题：

- 工业 IDS 重点监测哪些行为？
- 抓包点不同会影响看到哪些流量吗？
- 串口转以太网为什么会扩大攻击面？

## 第五轮：准备项目化表达

阅读：

1. `Network_Assessment_Checklist.md`
2. `Network_Risk_Case_Studies.md`
3. `Project_Style_Interview_Stories.md`
4. `Interview_QA.md`
5. `Glossary.md`

目标：

- 能讲出工业网络评估检查框架。
- 能把网络问题讲成攻击路径。
- 能用项目化语言回答面试官。

面试前背熟五条：

1. 工业网络安全核心是分区、边界、通信矩阵和攻击路径。
2. 工业 DMZ 用于承载跨域数据和运维，不应变成全通跳板。
3. 远程运维要按需开通、MFA、堡垒机、审计、回收账号。
4. 防火墙策略要基于通信矩阵，限制工业协议写操作和工程操作。
5. 抓包分析要沉淀资产、协议、通信方向、写操作和异常跨区通信。

## 第六轮：形成可交付项目包

完成以下最小交付物：

1. 物理、逻辑、业务、安全四视图拓扑；
2. 资产表、通信矩阵和跨域依赖表；
3. 一份多点抓包报告，含时间、丢包和证据边界；
4. 一条六层攻击路径与屏障表；
5. 一个网络变更、停止条件、回退和分层验收方案；
6. 一次事件恢复桌面演练记录。

## 章节验收题

### 架构与分区

- Purdue、Zone、Conduit、VLAN 和子网分别解决什么问题？
- 云边缘、多站点、虚拟化和 SIS 为什么不能机械套层级？
- 如何发现双网卡、管理面和共享身份形成的隐式通道？

### 二三层与可靠性

- ARP 表与 MAC 表如何配合？未知单播在哪个范围泛洪？
- 为什么 VLAN 不等于安全隔离？
- TCP 可靠为什么不等于实时？
- 如何区分链路收敛、应用恢复和数据重新可信？
- PTP 路径非对称如何影响控制和证据？

### 边界与远程

- DMZ 如何终止历史、文件和远程会话？
- 防火墙能否在同一端口内区分读写？不能时如何补偿？
- 远程维护结束后为什么要检查 PLC/VFD 状态？

### 抓包与监测

- “没有抓到响应”有哪些替代解释？
- 如何证明 TCP 重传发生在生产链路而非监测点？
- 被动资产指纹怎样标注置信度和陈旧状态？
- 工业 IDS 告警为什么不应默认自动阻断？

### 接入、攻击与恢复

- 协议网关怎样丢失质量、时间和命令确认语义？
- 远程站重连为什么不能执行缓存控制命令？
- 如何从网络可达逐层证明到物理后果？
- 恢复为什么不能以 ping 通或服务启动为结束？

## 实验路线

```text
虚拟二三层网络
→ VLAN/路由/ACL/NAT 抓包
→ 仿真 PLC/HMI 通信矩阵
→ 链路故障与多层恢复时间线
→ DMZ 代理与历史缓存补传
→ 远程运维授权/退出状态机
→ 工业 IDS 基线与语义告警
→ 六层攻击路径和恢复演练
```

所有写入、欺骗、环路、泛洪、切换和协议异常只在隔离环境完成。生产环境以配置、流量、日志和经批准的低风险验证为主。

## 复习节奏

- 当天：根据章节检查清单复述数据流、控制流和状态流；
- 一周：不看资料重画拓扑和通信矩阵；
- 两周：完成一个抓包或故障实验并写证据报告；
- 一月：用陌生行业场景完成攻击路径、整改和恢复答题；
- 面试前：练习 60 分、90 分和 STAR 三种表达，删除空泛术语。

## 完成判定

能够独立回答“正常数据如何流动、网络和设备怎样维护状态、攻击者能改变什么、监测能看到什么、如何安全验证和恢复”，并给出配置、流量、日志、工程与过程证据，才算完成 03 篇。
