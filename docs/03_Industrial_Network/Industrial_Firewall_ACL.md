# 工业防火墙与访问控制

## 1. 工业防火墙的作用

工业防火墙用于控制不同工业网络区域之间的通信。

它不仅看 IP 和端口，也可能识别工业协议功能码，例如 Modbus 写寄存器、S7 工程操作等。

## 2. 普通防火墙与工业防火墙

| 对比项 | 普通防火墙 | 工业防火墙 |
|---|---|---|
| 主要对象 | IT 协议 | IT + 工业协议 |
| 规则维度 | IP、端口、方向 | IP、端口、协议功能码、设备角色 |
| 场景 | 办公网、互联网边界 | 控制网、产线、PLC 前端 |
| 关注点 | 阻断攻击流量 | 限制异常控制操作 |

## 3. 白名单原则

工控网络更适合白名单：

```text
只允许已知业务通信，其余默认拒绝。
```

原因：

- 工业网络通信相对稳定。
- 资产和通信关系可梳理。
- 异常连接通常更值得关注。

## 4. 规则设计要素

一条规则至少明确：

- 源区域
- 源 IP
- 目的区域
- 目的 IP
- 协议
- 端口
- 方向
- 时间窗口
- 业务说明
- 负责人

更好的规则还包括：

- 工业协议功能限制
- 只读 / 可写限制
- 日志记录
- 告警策略

## 5. 工业协议细粒度控制

以 Modbus 为例：

- 允许读保持寄存器 `0x03`
- 禁止写单寄存器 `0x06`
- 禁止写多寄存器 `0x10`

以工程站维护为例：

- 仅维护窗口允许工程协议。
- 仅指定工程站可访问指定 PLC。
- 下载操作记录日志并告警。

## 6. 常见问题

- 规则过宽：`any any allow`
- 临时规则未关闭。
- 策略没有业务说明。
- 防火墙只部署在外边界，产线内部无隔离。
- 日志不开启。
- 不理解工业协议，无法判断写操作。

## 7. 评估检查项

- 是否有防火墙策略导出。
- 是否存在过宽规则。
- 是否存在长期临时规则。
- 是否有命中统计。
- 是否记录拒绝日志。
- 是否限制工程操作。
- 是否对 PLC 写操作告警。
- 策略变更是否审批。

## 8. 面试表达

> 工业防火墙策略不能只看端口通不通，还要结合业务通信和工业协议功能。比如 SCADA 访问 PLC 的 502 端口可能是正常采集，但如果允许任意主机写寄存器，就有控制风险。我会建议基于通信矩阵做白名单，并对写功能码、工程下载和远程维护操作单独审计。
