# 网络评估检查清单

## 1. 拓扑与资产

- [ ] 是否有最新网络拓扑图。
- [ ] 是否标注安全域。
- [ ] 是否标注边界设备。
- [ ] 是否标注关键资产。
- [ ] 是否标注远程运维入口。
- [ ] 是否存在未知资产。

## 2. 分区分域

- [ ] 办公网与控制网是否隔离。
- [ ] 是否有工业 DMZ。
- [ ] DMZ 是否独立区域。
- [ ] 控制层和监控层是否隔离。
- [ ] SIS / ESD 是否单独隔离。
- [ ] 是否存在双网卡跨接。

## 3. 通信矩阵

- [ ] 是否有通信矩阵。
- [ ] 是否记录源、目的、协议、端口、方向。
- [ ] 是否标注读写权限。
- [ ] 是否确认业务必要性。
- [ ] 是否存在跨域异常通信。
- [ ] 是否存在办公网访问 PLC。

## 4. 边界防护

- [ ] 防火墙策略是否最小化。
- [ ] 是否存在 `any any allow`。
- [ ] 临时策略是否长期保留。
- [ ] 是否记录日志。
- [ ] 是否限制工程协议。
- [ ] 是否监测 PLC 写操作。

## 5. 远程运维

- [ ] 是否有远程运维清单。
- [ ] 是否按需开通。
- [ ] 是否启用 MFA。
- [ ] 是否经过堡垒机。
- [ ] 是否有会话审计。
- [ ] 是否允许文件传输。
- [ ] 账号是否定期回收。

## 6. 交换机与路由

- [ ] VLAN 是否合理。
- [ ] VLAN 间是否有 ACL。
- [ ] 未使用端口是否关闭。
- [ ] 管理口是否限制访问。
- [ ] 默认密码是否修改。
- [ ] SNMP 是否安全配置。

## 7. 工业网关

- [ ] 网关是否跨安全域。
- [ ] 是否允许写操作。
- [ ] 管理界面是否暴露。
- [ ] 默认密码是否修改。
- [ ] 是否连接云平台。
- [ ] 配置是否加密备份。

## 8. 监测与审计

- [ ] 是否部署工业 IDS。
- [ ] 是否有镜像口或 TAP。
- [ ] 是否监测跨区通信。
- [ ] 是否监测工程下载。
- [ ] 是否监测 PLC 状态变化。
- [ ] 是否有日志留存周期。

## 9. 面试表达

> 做工业网络评估时，我会按拓扑资产、分区分域、通信矩阵、边界防护、远程运维、交换路由、工业网关、监测审计几个模块检查。重点不是单纯看设备是否在线，而是判断跨域路径是否合理、通信是否必要、边界是否最小开放、远程入口是否受控。

## 10. 证据记录模板

```text
检查项编号：____________ 资产/区域：________________
检查目标：_________________________________________
资料来源：配置 / 流量 / 日志 / 访谈 / 现场 / 实验
观察时间与版本：___________________________________
事实：_____________________________________________
证据文件与哈希：___________________________________
盲区/未验证假设：__________________________________
控制能力与过程后果：_______________________________
现有屏障及有效性：_________________________________
建议、责任人和期限：_______________________________
验收方式：_________________________________________
```

事实、推断和建议分开书写。没有流量不等于没有资产，规则存在不等于运行有效，端口开放不等于控制命令可执行。每项高风险结论至少给出网络证据和工程/过程证据，无法验证部分明确标注。

## 11. 现场开始与退出清单

### 开始前

- [ ] 授权范围、禁止动作、停止条件和联系人已确认；
- [ ] 工具终端受管、加密、关闭桥接和共享；
- [ ] 接入端口、地址、时间源和流量方向已记录；
- [ ] 生产工况、维护窗口、冗余和安全状态已确认；
- [ ] 配置备份、应急回退和现场监护可用。

### 退出前

- [ ] 临时账号、VPN、规则、路由、镜像和证书已撤销；
- [ ] 测试服务、抓包驱动、文件和计划任务已清理；
- [ ] 交换/防火墙/网关配置与批准版本一致；
- [ ] PLC 模式、强制、旁路、报警和远程控制权正常；
- [ ] 原始证据哈希、保管位置和交接记录完整。

## 12. 风险整改验收清单

- 预期业务是否在稳态、启动、维护和主备切换下正常？
- 不必要源、目标、端口、操作和管理面是否确实被拒绝？
- 防火墙/NAT/代理日志能否关联到真实资产与用户？
- 数据值、质量、源时间、缓存和补传是否正确？
- 链路或平台失联时，控制器和远程站是否安全自治？
- 恢复后旧命令、残余会话、强制和临时配置是否清除？
- 监测规则是否覆盖新架构，且误报/漏报边界已记录？
- 配置、通信矩阵、拓扑和恢复文档是否同步更新？
