# 03 章学习地图

## 1. 本章核心问题

工业网络安全要回答五个问题：

1. 哪些资产在同一个网络里？
2. 哪些资产之间应该通信？
3. 哪些通信是业务必须的？
4. 哪些通信是不合理或高风险的？
5. 攻击者能否从低价值区域走到高价值控制区域？

如果 02 章是理解“设备和控制逻辑”，03 章就是理解“路径和边界”。

## 2. 三条主线

### 主线 1：分层

工业网络不是一张扁平网。通常会按功能分层：

```text
办公网
生产管理网
工业 DMZ
监控层网络
控制层网络
现场设备网络
```

分层的目的不是好看，而是限制攻击路径。

### 主线 2：通信

安全评估要知道：

- 谁访问谁？
- 使用什么协议？
- 端口是什么？
- 方向是什么？
- 频率是否正常？
- 是否需要写操作？

这就是通信矩阵。

### 主线 3：边界

边界是不同安全域之间的控制点：

- 防火墙
- 工业防火墙
- 网闸
- 堡垒机
- VPN
- 路由器 ACL
- 交换机 VLAN
- 工业 IDS 监测点

边界做得好，攻击者即使进入某个区域，也不容易横向移动到控制层。

## 3. 学习时的判断框架

看到一个网络连接，按这个顺序判断：

1. 资产角色：源和目的分别是什么？
2. 网络区域：它们处于哪个安全域？
3. 协议端口：使用什么协议和端口？
4. 通信方向：谁主动发起？
5. 业务必要性：这个通信是否必要？
6. 权限强度：只读、可写、工程下载还是远程桌面？
7. 风险后果：如果被滥用，会影响什么工业动作？
8. 防护措施：是否有白名单、审计、认证、监测？

## 4. 本章产出

本章建议先形成理解，不急着实践。未来可以补：

- 一张工业网络分层图。
- 一张通信矩阵模板。
- 一份远程运维风险清单。
- 一份工业网络评估检查表。
- 一个 Wireshark 工业流量分析报告。

## 5. 面试表达主线

面试时围绕一句话展开：

> 我看工业网络不是只看 IP 和端口，而是看资产角色、网络区域、通信方向、业务必要性和工控影响。比如办公网能直接访问工程站，不只是网络连通问题，而是可能形成从办公网进入控制逻辑修改入口的攻击路径。
