# 03 篇学习地图

## 1. 学习目标：从“能通”走向“可证明、可控制、可恢复”

普通网络学习常以地址、端口和连通性结束；工业网络分析还必须回答通信为何存在、由谁发起、维护什么状态、允许什么控制能力、失败后过程进入什么状态，以及恢复时怎样避免旧命令、错误路由和不一致配置再次生效。

本篇建立六层判断框架：

```text
物理与链路：介质、端口、VLAN、环路、冗余、误码
网络与传输：地址、路由、NAT、组播、TCP/UDP、队列
会话与应用：轮询、订阅、心跳、超时、重连、命令事务
资产与身份：主机、进程、账号、证书、工程工具、维护终端
控制与过程：变量、模式、联锁、执行器、报警、保护层
治理与恢复：审批、基线、证据、故障切换、受控恢复
```

## 2. 四条学习主线

### 2.1 网络转发主线

从源主机开始，解释子网判断、ARP/邻居解析、以太网封装、MAC 学习、802.1Q 标签、三层路由、ACL/NAT、传输层会话和应用报文。重点不是背分层，而是说明每层新增什么标识、设备维护什么表、异常时出现什么报文与状态。

### 2.2 工业业务主线

区分周期 I/O、监控轮询、事件上送、报警、工程下载、远程桌面、历史复制、文件交换和时间同步。不同业务对时延、抖动、丢包、带宽、方向、认证和恢复的要求不同，不能仅凭端口号制定统一策略。

### 2.3 边界与攻击路径主线

把 VLAN、路由器、防火墙、DMZ、代理、网闸、VPN 和堡垒机视为不同控制点。逐步验证攻击者是否获得网络可达、身份凭据、应用会话、工程语义和设备控制权，并检查独立保护是否限制物理后果。

### 2.4 证据与韧性主线

结合配置、转发表、路由表、会话日志、抓包、端点日志、控制器审计和过程趋势还原事件。学习冗余切换、时间同步、备份恢复和变更验收，避免把“设备双机”误认为业务一定连续。

## 4. 每条通信的九问

1. 源和目的分别承担什么控制或生产功能？
2. 谁主动发起，何时建立、保持、超时和重连？
3. 沿途经过哪些 VLAN、路由、NAT、代理和安全设备？
4. 使用什么应用操作，而不只是哪个 TCP/UDP 端口？
5. 是只读、操作写、文件传输、远程桌面还是工程下载？
6. 正常频率、包长、并发、时延和质量状态是什么？
7. 链路失败、重复、乱序或恢复后旧数据如何处理？
8. 攻击者改变该通信后，控制逻辑和物理过程如何响应？
9. 配置、流量、日志和现场反馈分别能证明到哪一步？

## 5. PDF 基线的校正重点

- Ethernet 帧属于数据链路层，物理层传输比特/符号；不能把链路层数据单位直接写成比特。
- ARP 缓存存在于需要进行 IPv4 邻居解析的主机或三层接口，并非只能说“三层设备才有”。
- 交换机对未知单播执行的是在相应 VLAN 内泛洪；广播、多播和已知单播的处理条件不同。
- VLAN 缩小广播域并提供逻辑分段，但不是天然安全边界；跨 VLAN 路由、Trunk、Native VLAN、管理面和错误端口配置都可能破坏预期隔离。
- TCP 提供有序可靠字节流，不保证应用事务成功、固定时延或工业实时性；重传和队头阻塞可能让“可靠”与“及时”发生冲突。
- 抓到 Ethernet/IP/TCP/Modbus 的封装栈仍不足以证明 PLC 动作，需要继续关联事务、应用响应、控制器模式、变量映射和现场反馈。

## 6. 面试表达主线

> 我分析工业网络不会停在 IP、端口和是否连通，而会从业务功能出发，重建端点、转发、边界、应用和过程五层状态。对一条到 PLC 的 TCP 会话，我会确认源身份、路径、会话建立与重传、具体工业操作、控制器响应和现场变量变化；对防护则检查通信矩阵、最小权限、会话审计、冗余收敛、时间一致性和恢复验证。这样才能把网络证据可靠地映射到控制能力和物理后果。
