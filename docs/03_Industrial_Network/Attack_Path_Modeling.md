# 工业网络攻击路径建模

## 1. 攻击路径是什么

攻击路径描述攻击者从入口到目标资产的连续步骤。

工控安全里，攻击路径通常从 IT 侧或远程运维入口开始，最终指向工程站、HMI、PLC、DCS 控制站或关键数据系统。

## 2. 常见入口

- 钓鱼邮件进入办公网。
- VPN 账号泄露。
- 远程桌面暴露。
- 厂商远程运维入口。
- 工业网关 Web 漏洞。
- 文件交换引入恶意文件。
- U 盘进入工程站。

## 3. 常见高价值目标

- 工程站
- HMI / 操作员站
- SCADA Server
- 历史数据库
- PLC / RTU / DCS 控制站
- 工业网关
- SIS / ESD 工程站

## 4. 攻击路径示例：办公网到工程站

```text
钓鱼邮件
-> 办公终端失陷
-> 获取域账号
-> 访问生产管理网共享
-> 发现工程站远程桌面
-> 登录工程站
-> 获取工程文件
-> 修改控制逻辑
```

关键控制点：

- 办公网和生产网隔离。
- 域账号最小权限。
- 远程桌面收敛。
- 工程站登录审计。
- 工程文件完整性校验。

## 5. 攻击路径示例：远程运维到 PLC

```text
厂商 VPN 账号泄露
-> 接入工业 DMZ
-> 登录跳板机
-> RDP 到工程站
-> 在线连接 PLC
-> Force 变量或下载工程
```

关键控制点：

- MFA。
- 维护窗口。
- 会话录像。
- 禁止直连 PLC。
- Force 和下载告警。

## 6. 攻击路径示例：工业网关到控制层

```text
网关 Web 漏洞
-> 获取网关权限
-> 读取协议转换配置
-> 获取 PLC 地址和点表
-> 通过网关写变量
```

关键控制点：

- 网关管理面隔离。
- 默认密码修改。
- 配置加密。
- 写操作限制。
- 协议审计。

## 7. 攻击路径建模方法

1. 列出入口。
2. 列出中间跳板。
3. 列出高价值目标。
4. 画出可达路径。
5. 标注权限和协议。
6. 标注控制措施。
7. 找最短路径和最危险路径。
8. 给出阻断建议。

## 8. 面试表达

> 我会用攻击路径而不是单点漏洞来分析工控网络风险。比如远程 VPN 账号泄露本身是账号风险，但如果它能通过跳板机访问工程站，再连接 PLC 做工程下载或 Force 变量，就形成了从外部入口到控制层的完整攻击路径。整改也要按路径逐段阻断。
