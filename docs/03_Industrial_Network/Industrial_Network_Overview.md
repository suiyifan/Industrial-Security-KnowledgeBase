# 工业网络总体架构

## 1. 工业网络是什么

工业网络连接工业控制系统中的各类资产，包括工程站、操作员站、SCADA Server、历史数据库、PLC、DCS 控制站、RTU、工业网关、传感器和执行器。

普通 IT 网络主要服务办公、业务和数据交换；工业网络还承载实时监控、控制命令、报警、趋势和工程维护。

## 2. 工业网络的核心目标

工业网络设计要优先考虑：

- 稳定性
- 实时性
- 可用性
- 安全隔离
- 可维护性
- 可追溯性

安全不是简单“断网”，而是在保障生产连续性的前提下限制不必要通信。

## 3. 典型工业网络中的资产

| 网络位置 | 常见资产 | 作用 |
|---|---|---|
| 办公网 | 办公电脑、邮件、OA、ERP | 日常办公和管理 |
| 生产管理网 | MES、报表、调度、数据分析 | 生产管理与统计 |
| 工业 DMZ | 数据转发、补丁、杀毒、跳板、文件交换 | 办公网与控制网缓冲区 |
| 监控层 | SCADA Server、HMI、操作员站、历史库 | 监控、操作、数据记录 |
| 控制层 | PLC、DCS 控制站、RTU、工业网关 | 控制逻辑和数据采集 |
| 现场层 | 传感器、执行器、仪表、变频器 | 物理过程采集与动作 |

## 4. 工业网络和普通企业网络的区别

| 对比项 | 普通 IT 网络 | 工业网络 |
|---|---|---|
| 首要目标 | 数据安全、业务连续 | 生产连续、过程安全 |
| 资产生命周期 | 更新较快 | 生命周期长，十年以上常见 |
| 协议 | HTTP、SMB、数据库协议 | Modbus、S7、OPC UA、IEC 104 等 |
| 变更频率 | 相对频繁 | 变更谨慎，需要窗口 |
| 扫描容忍度 | 较高 | 低，主动扫描可能影响设备 |
| 安全后果 | 数据泄露、业务中断 | 停产、设备损坏、安全事故 |

## 5. 工业网络中的常见错误

- 办公网和控制网直连。
- 工程站双网卡跨接两个区域。
- 防火墙策略过宽，使用 `any any allow`。
- 远程桌面直接暴露到控制网。
- 厂商 VPN 长期开启。
- PLC 与上位机之间没有访问控制。
- 交换机没有 VLAN 或端口管理。
- 不知道哪些设备之间应该通信。
- 没有流量监测和日志审计。

## 6. 工控安全视角

工业网络安全关注的不只是“网络通不通”，而是：

- 攻击者能不能从办公网进入生产网？
- 能不能从生产网进入监控层？
- 能不能从 HMI 访问 PLC？
- 能不能通过工程站下载逻辑？
- 能不能通过远程运维绕过边界？
- 能不能通过历史库或数据接口反向进入控制网？

## 7. 面试表达

> 我理解工业网络要按资产功能和安全域来分析。办公网、生产管理网、工业 DMZ、监控层、控制层和现场层的安全要求不同。评估时我会重点看跨域通信是否必要、是否经过边界控制、是否存在远程运维入口，以及攻击者是否能从低安全区域横向移动到工程站、HMI 或 PLC。
