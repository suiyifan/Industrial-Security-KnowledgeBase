# 被动监测、流量镜像与工业 IDS

## 1. 为什么工控网络偏好被动监测

工控环境对主动扫描敏感。某些老旧设备可能因为异常流量导致通信异常、CPU 占用升高或服务异常。

因此工控安全评估常优先使用被动监测：

- 不主动发包。
- 通过镜像口观察流量。
- 识别资产和协议。
- 发现异常通信。

## 2. 流量镜像

流量镜像也叫 SPAN / Port Mirroring。

作用：

```text
把某个交换机端口或 VLAN 的流量复制到监测端口。
```

工业 IDS、Wireshark、流量分析设备通常接在镜像口。

## 3. TAP

TAP 是网络分光设备，用于复制链路流量。

相比交换机镜像：

- 更稳定。
- 不受交换机配置影响。
- 常用于关键链路。

## 4. 工业 IDS 关注什么

工业 IDS 通常关注：

- 资产识别
- 协议识别
- 异常通信
- 新资产上线
- PLC 写操作
- 工程下载
- 控制器状态变化
- 非法功能码
- 异常扫描
- 远程登录行为

## 5. 基线学习

工业网络通信相对稳定，适合建立基线：

- 哪些资产正常通信。
- 正常协议是什么。
- 正常频率是多少。
- 哪些操作很少发生。

偏离基线的行为值得关注：

- 新主机访问 PLC。
- 办公网主机访问控制层。
- 非工程站发起工程协议。
- 夜间出现写寄存器。
- PLC 出现停止 / 下载操作。

## 6. 被动监测局限

- 只能看到经过镜像点的流量。
- 加密流量无法直接解析内容。
- 资产角色需要人工确认。
- 误报需要结合业务。
- 无法发现离线配置问题。

## 7. 部署位置

常见监测点：

- 办公网到工业 DMZ 边界。
- DMZ 到控制网边界。
- SCADA Server 到 PLC 链路。
- 工程站到控制器链路。
- 远程运维入口。
- 关键产线核心交换机。

## 8. 面试表达

> 工控环境不适合一上来主动扫描，所以我会优先考虑被动监测。通过交换机镜像或 TAP 获取流量，建立资产、协议和通信基线。工业 IDS 的价值不仅是报漏洞，还包括发现新资产、异常跨区通信、PLC 写操作、工程下载和控制器状态变化。
