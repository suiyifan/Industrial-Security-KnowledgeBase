# 被动监测、资产识别与工业 IDS

## 1. 原理解释：为什么工业环境偏好被动监测

主动扫描通过发送探测包识别服务和漏洞，但老旧 PLC、串口网关和实时设备可能对异常连接、并发或畸形报文容忍度低。被动监测从 TAP、SPAN 或日志观察已有通信，不向生产资产主动发包，通常更适合持续发现资产、建立通信基线和识别高风险工业操作。

被动不等于零风险，也不等于完整。传感器接入错误、镜像过载、双网卡、管理平台权限和 PCAP 存储都会引入风险；未通信的备用设备、单向链路另一侧、加密载荷和非镜像 VLAN 可能不可见。资产结论必须带来源、首次/最近观察、置信度和盲区。

## 2. 工程架构与监测位置

```text
TAP / SPAN / 虚拟交换镜像
  ↓
被动传感器：解析、元数据、协议状态、告警
  ↓ 单向或受控上送
管理与分析平台：资产、基线、规则、调查
  ↓
SIEM / SOAR / 工单 / 事件响应
```

传感器通常部署在企业—DMZ、DMZ—控制区、SCADA/控制器单元、远程站 WAN 和关键 SIS/BPCS 边界。位置由检测目标决定：边界传感器适合跨区路径，单元传感器适合 PLC/HMI 语义，核心镜像可能因交换转发看不到同交换机其他端口间的单播。

传感器数据接口应尽量只接收流量，不配置生产 IP 或转发；管理接口进入独立管理 Zone。若同一设备用双网卡横跨控制与企业网络，就可能形成旁路。上送使用最小目标、认证加密和缓存限额，平台失联不能影响生产转发。

## 3. 被动资产识别

资产可从 MAC OUI、IP、VLAN、DHCP、ARP、LLDP、主机名、协议字段、设备发现、TLS 证书和通信角色推断。工业协议可能暴露厂商、产品、固件、槽位、站号或项目名，但字段可能缺失、被代理改写或可伪造。

建议资产记录包含：

| 字段 | 说明 |
| --- | --- |
| 观察身份 | MAC、IP、VLAN、序列号、证书等 |
| 工程身份 | 位号、设备名、生产单元、控制功能 |
| 角色 | PLC、HMI、工程站、历史、网关等 |
| 软件/固件 | 值、来源、观察时间、置信度 |
| 通信关系 | 对端、协议、方向、频率、操作 |
| 关键性 | 失效后果、保护层、恢复责任 |
| 证据状态 | 首次、最近、是否陈旧、来源传感器 |

同一资产更换网卡、主备切换或 NAT 后可能出现多个网络身份；同一 IP 也可能被不同设备在不同时间使用。资产归并需要时间、拓扑、序列号和工程资料，不能仅按 IP 去重。

## 4. 通信基线不是静态白名单

基线描述特定工况下正常通信的参与者、方向、周期、包长、会话、应用操作和时间窗口。工业网络流量较稳定，偏差有检测价值，但启动、停车、维护、主备切换、报警洪泛和历史补传都会改变模式。

基线应按生产状态分层：

- 稳态运行：周期 I/O、HMI 轮询、历史采集；
- 启停与批次切换：设备发现、配方、状态迁移；
- 维护窗口：工程软件、上传下载、诊断、备份；
- 故障与恢复：重连、切换、缓存补传、报警突发；
- 备用状态：低频心跳、健康检查和同步。

首次出现的新通信不一定恶意，但必须解释。学习模式不能无限自动吸收偏差，否则攻击者的慢速行为会变成“正常”。基线变更应关联工单、工程版本和责任人确认。

## 5. 检测层次

### 5.1 网络与资产规则

检测新 MAC/IP、地址冲突、异常 ARP、VLAN 变化、新跨区会话、扫描模式、管理协议和互联网连接。这类规则覆盖广，但不能说明工业操作语义。

### 5.2 协议合规与状态机

检查非法长度、异常功能、事务错配、连接状态跳跃、序号异常、未完成握手和违反协议状态机的报文。协议解析器必须处理分片、TCP 重组、厂商扩展和加密边界，避免把解析失败等同攻击。

### 5.3 工业操作语义

关注程序下载、固件升级、模式切换、强制、写设定、启停、报警抑制、配方修改和 SIS 旁路。规则应限定授权源、目标、对象、时间窗口和变化范围。同一写功能可能是正常 HMI 操作，也可能是异常工程行为。

### 5.4 行为与过程一致性

检测周期、流量、会话和对象关系偏差，并把命令与反馈、SP/PV/MV、阀位/电流、物料平衡和保护状态关联。例如 PLC 返回写成功但阀位无变化，可能是逻辑覆盖、设备故障或欺骗；网络 IDS 需要与过程数据平台协同。

### 5.5 威胁情报与漏洞匹配

资产版本可与漏洞情报匹配，但被动指纹不一定准确，CVE 存在也不证明可利用或有物理后果。需确认型号、版本、配置、路径、功能和保护层，区分“疑似受影响”与“验证受影响”。

## 6. 告警质量、置信度与优先级

高质量告警至少说明：源/目的资产与 Zone、协议操作、对象或变量、基线偏差、时间、证据包、可能控制能力、当前工况和建议验证。只有“PLC 异常流量”无法支持处置。

优先级不应只用网络严重度。可综合：

```text
置信度 × 控制能力 × 资产关键性 × 工况暴露 × 保护降级
```

同一工程下载在批准维护窗口可能是低风险审计事件，在满负荷生产且来自未知终端时应高优先。SIS 旁路期间、设备手动模式或现场检修会动态改变风险。

误报可能来自主备切换、固件差异、协议扩展、时钟偏差和镜像重复；漏报可能来自加密、非镜像路径、丢包、协议不支持或攻击使用合法通道。每条规则记录数据依赖和盲区。

## 7. 加密流量与元数据

TLS、VPN、OPC UA 加密和厂商隧道会隐藏应用对象。监测仍可观察端点、证书、握手、版本、SNI（若存在）、连接频率、流量方向、大小和时序，但不能宣称识别内部写命令。解密会增加密钥、隐私、性能和架构风险，不应为了可见性随意终止控制加密。

可把网络元数据与端点、应用和控制器审计关联：工程站进程启动、项目打开、证书使用、控制器下载和过程变化共同提高置信度。检测架构应承认不同传感器的证据边界。

## 8. 响应流程与生产安全

工业 IDS 通常旁路告警，不直接阻断。告警后先确认资产、工况、数据质量和安全状态，再选择响应。立即关闭交换端口、阻断控制会话或断电可能触发输出保持、控制失联或不安全停车。

响应状态流：

```text
Detected → Triaged → Asset/Process Confirmed → Risk Stabilized
→ Evidence Preserved → Contained → Recovered → Validated → Tuned
```

稳定风险可能是现场切换本地控制、暂停高风险操作、确认 SIS 可用或停止远程维护，而不是立即隔离 PLC。网络、安全、自动化、工艺和设备人员共同批准影响生产的措施。

## 9. 平台健康与自身安全

监测系统必须监控抓包丢失、接口状态、时间同步、解析队列、磁盘、规则版本、传感器心跳和上送延迟。传感器显示在线但丢包 50%，告警“正常”没有意义。平台维护窗口和盲区应通知运营人员。

平台汇聚完整拓扑、漏洞、账号和 PCAP，是高价值目标。采用管理面隔离、MFA、角色权限、审计、备份、签名更新和安全导出。传感器规则或协议解析更新先在测试流量验证，避免资源耗尽或解析崩溃。

第三方云分析需评估数据出域、敏感载荷、延迟、断链缓存和供应商访问。控制区不应依赖云平台在线才能维持基本监测或生产。

## 10. 数据流、控制流与状态流

```text
数据流：镜像帧 → 传感器解析 → 元数据/告警 → 平台 → 工单与调查
控制流：资产模型 → 基线 → 检测规则 → 分级 → 响应审批
状态流：Unknown → Observed → Identified → Verified → Changed/Retired
告警流：New → Assigned → Investigating → Confirmed/Benign → Closed → Tuned
```

资产、规则和告警都有生命周期。退役设备若仍发流量可能是资产遗漏、地址复用或未授权接入；告警关闭必须记录依据，不能用静默规则掩盖未解决异常。

## 11. 安全风险与攻击链

监测风险包括镜像范围不完整、传感器双网卡旁路、默认管理凭据、平台连接企业域、规则更新供应链、PCAP 明文泄露、告警疲劳、基线自动污染和响应脚本未经生产验证。

典型攻击链是：攻击者取得监测平台管理员权限，读取资产与漏洞清单选择 PLC；修改规则抑制工程下载告警，并利用传感器管理接口作为进入控制区的跳板；随后从被允许的工程站下载逻辑。若平台仍显示“资产健康”，响应被延迟。

防护包括传感器数据面单向、管理面独立、最小权限与 MFA、规则签名和双人发布、配置/告警审计外送、平台行为监测、离线备份和控制器/过程侧独立检测。监测平台不能成为唯一真相来源。

## 12. 工控场景：合法维护被误判并导致错误隔离

维护窗口内工程师对冗余 PLC 执行固件升级，IDS 报告“未授权程序下载”。值班人员未核对工单和主备状态，立即阻断工程站端口，升级中断使备控制器无法重新加入；随后主控制器故障，系统失去冗余并停车。

根因包括规则缺少维护上下文、告警未关联资产角色、响应预案未评估工程事务中断。整改应把工单、用户、目标、版本和时间窗口送入检测上下文；高风险告警仍记录但由自动化负责人确认；响应手册明确固件/下载中断的安全步骤和回退。

## 13. 面试表达

### 60 分答案

> 工业 IDS 通过 TAP 或镜像口被动识别资产、协议和异常操作，适合不主动打扰生产设备。部署要覆盖关键边界，并建立正常通信基线。

### 90 分答案

> 我会先定义监测点能看到什么，验证镜像方向、丢包和时钟，再把被动身份与工程资产表归并。检测分为网络、协议状态机、工业操作和过程一致性四层，基线按稳态、启停、维护和故障场景管理。告警优先级结合控制能力、工况和保护降级，响应前确认过程安全，避免自动阻断关键会话。平台自身采用单向数据面、独立管理、规则审计和健康监控，并明确加密和未通信资产盲区。

### 常见追问

**问：被动监测能否发现全部资产？**

答：不能。备用、离线、非镜像、单向另一侧和不通信资产可能不可见，需要与配置、现场盘点和受控验证结合。

**问：工业 IDS 是否应该自动阻断？**

答：通常先告警和确认。自动响应只适用于经过风险评估和故障测试的场景，避免中断控制导致更大风险。

## 14. 实操建议

### 实验一：资产置信度

用仿真 PLC/HMI 流量从 MAC、IP、协议和角色推断资产，再与已知清单比较，记录每个字段的证据与置信度。

### 实验二：多工况基线

采集稳态、启动、维护和链路恢复流量，比较周期、会话和工业操作，设计不会把合法切换吸收为攻击或反之的规则。

### 实验三：语义与过程关联

在隔离环境执行读取、设定值修改和模式变化，将 IDS 事件与控制器日志、SP/PV/MV 和反馈关联，评估告警能证明到哪一步。

### 实验四：响应桌面演练

模拟工程下载告警，分别设置正常维护、未知终端和 SIS 旁路工况，设计确认、稳定、隔离、恢复与规则调优流程。

## 15. 总结与检查清单

- [ ] 能选择监测点并验证可见性与丢包；
- [ ] 能合并网络身份与工程资产身份；
- [ ] 能按工况建立和治理通信基线；
- [ ] 能设计网络、协议、操作与过程检测；
- [ ] 能解释加密、备用设备和解析器盲区；
- [ ] 能在不危及生产的前提下响应告警；
- [ ] 能保护监测平台、规则、PCAP 与管理面。
