# 工业以太网交换、VLAN 与二层安全

## 1. 原理解释：二层网络解决什么问题

Ethernet 让同一链路层广播域中的设备通过 MAC 地址交换帧。终端根据 IP 与掩码判断目标是否同网段：同网段时解析目标 MAC，跨网段时解析默认网关 MAC。交换机依据源 MAC 学习入口端口，并依据目的 MAC 决定转发，而不是理解 PLC 变量或控制命令。

应校正几个常见简化：物理层传输比特或符号，Ethernet 帧属于数据链路层；交换机对未知单播是在所属 VLAN 内向除入口外的合适端口泛洪，并非无条件向“所有端口”发送；ARP 缓存存在于需要进行 IPv4 邻居解析的主机或三层接口，不是只有某类设备才有；VLAN 能划分广播域，但不能独立证明安全隔离。

## 2. Ethernet 帧与转发约束

典型 Ethernet II 帧包含目的 MAC、源 MAC、EtherType、负载和 FCS；在线路上还有前导码与帧间间隔。802.1Q 标签插入 VLAN 标识和优先级信息。最小帧、最大传输单元和链路聚合等约束会影响工业报文传输，超过路径 MTU 的 IP 包可能分片或被丢弃。

交换机收到帧后通常执行：校验帧、学习源 MAC 与 VLAN/端口关系、查找目的 MAC，再做已知单播转发、未知单播泛洪、广播泛洪或组播处理。MAC 地址表具有老化时间；设备移动、链路切换或表项溢出会改变转发行为。镜像口看到的流量取决于交换芯片和配置，不能假设天然看到整个 VLAN。

```text
接收帧
  ↓ 校验 FCS / 长度 / 端口状态
学习 (VLAN, Source MAC) → 入端口
  ↓
查询 (VLAN, Destination MAC)
  ├─ 已知单播：转发到指定端口
  ├─ 未知单播：在该 VLAN 内受控泛洪
  ├─ 广播：在该 VLAN 内泛洪
  └─ 组播：按静态项、IGMP Snooping 或泛洪处理
```

## 3. ARP：IP 到链路地址的邻居解析

主机要向同网段目标或默认网关发送 IPv4 包，需要先获得下一跳 MAC。ARP Request 通常以二层广播发送，ARP Reply 通常单播返回；免费 ARP 可用于地址声明、冲突检测或主备切换。ARP 本身没有强认证，主机可能接受未经请求或错误的映射，因此存在欺骗和缓存污染风险。

ARP 表描述 IP 与 MAC 的邻居关系，MAC/CAM 表描述 VLAN 中 MAC 与交换端口的关系。排障时必须同时查看：正确 ARP 但错误 MAC 表可能把帧送错端口；错误 ARP 即使交换转发正常也会把 IP 流量交给攻击者；三层交换机既维护邻居表也维护二层转发表。

ARP 欺骗要形成稳定中间人通常还需双向污染、IP 转发以及处理双方流量。若攻击者只把网关映射到自己却不转发，结果更像拒绝服务。检测证据包括同一 IP 的 MAC 变化、异常 ARP Reply、免费 ARP 频率、交换端口移动、网关邻居冲突和通信时延变化。

## 4. VLAN 与 802.1Q 工程实现

VLAN 将一个物理交换基础设施划分为多个逻辑二层广播域。Access 端口通常接收无标签帧并归入 PVID，发送终端流量时去标签；Trunk 允许多个 VLAN，通常携带 802.1Q 标签；不同厂商对 Hybrid、General、Native/Untagged 的术语和行为不同，必须以实际配置与抓包验证。

不同 VLAN 默认不能在二层直接通信，但可通过路由器、三层交换机 SVI、防火墙、主机双网卡、虚拟交换或错误 Trunk 实现跨域。因此“划了 VLAN 就安全”不成立。安全域还需路由控制、ACL/防火墙、管理面隔离、端口准入、身份治理和监测支持。

VLAN 设计应记录 VLAN ID、名称、用途、子网、网关、成员端口、Tagged/Untagged 行为、允许的 Trunk、管理责任和对应 Zone。避免全网允许所有 VLAN、使用默认 VLAN 承载管理、Native VLAN 不一致、闲置端口启用以及工程笔记本端口自动加入控制 VLAN。

## 5. 广播域、冲突域与工业流量

现代全双工交换端口通常把碰撞域限制在单条点到点链路，传统共享介质或半双工环境才会出现 CSMA/CD 碰撞。广播域由 VLAN 等二层边界限定。ARP、某些设备发现、冗余和工业实时协议可能依赖广播或组播，不能简单把所有非单播视为异常。

广播风暴会占用链路、交换芯片和终端 CPU。低性能 PLC、远程 I/O 或保护设备可能因处理大量无关帧而通信超时。需要配置广播/未知单播/组播抑制，但阈值过低也会阻断合法设备发现或故障切换。策略应基于正常峰值、启动/恢复场景和设备容量验证。

## 6. 环路、生成树与工业环网

二层帧没有类似 IP TTL 的逐跳生存限制，物理环路若无控制会造成广播帧无限复制、MAC 表抖动和网络瘫痪。STP/RSTP/MSTP 通过选举根桥和阻塞冗余端口建立无环转发树；厂商工业环网可能通过专用管理帧实现更快收敛。

冗余不是“接两根线”即可。必须确认协议兼容、根桥/环管理者位置、链路故障检测、收敛时间、MAC 刷新、上层 TCP/应用超时和控制器失联策略。错误连接两个独立环、同时启用不兼容协议或把边缘端口误设为网络端口，都可能引入环路。

安全功能如 BPDU Guard、Root Guard、Loop Guard、边缘端口和风暴抑制需要按拓扑部署。若攻击者或误接设备发送更优 BPDU，可能改变根桥和流量路径；但未经验证直接启用防护也可能关闭关键上联。因此变更要有拓扑、预期状态和回退方案。

## 7. 组播、IGMP Snooping 与实时业务

部分工业协议使用组播分发周期 I/O、发现或冗余状态。若交换机不了解组播成员，组播可能像广播一样泛洪。IGMP Snooping 依据主机成员报告限制组播到需要的端口，通常还需要 Querier 维持成员状态。

常见故障包括没有 Querier 导致成员表老化、快速离开误删共享端口、组播路由边界错误、冗余切换后订阅未恢复，以及监测镜像漏掉只转发到特定端口的组播。排障应关联 IGMP 报文、组播 MAC、成员表、周期序号、I/O 连接状态和控制器诊断。

## 8. QoS、队列与拥塞

802.1Q PCP、DSCP 和交换队列可使关键控制流量优先于视频、备份或工程下载。QoS 不能创造带宽，且错误分类会让非关键流量占据高优先级，或使报警、时间同步被延迟。端到端路径上的所有设备必须对标记和队列有一致策略。

工业网络常见微突发：多个设备在相同时刻上报报警、重连补传或启动发现。平均利用率很低仍可能出现队列丢包。证据应包括端口丢弃、队列计数、接口错误、流量时间分布、TCP 重传和应用超时，而不是只看带宽百分比。

## 9. 数据流、控制流与状态流

```text
数据流：应用报文 → IP 包 → Ethernet 帧 → VLAN/端口 → 目的设备
控制流：端口配置 → VLAN 成员 → STP/环网 → MAC/组播表 → QoS 决策
状态流：Down → Blocking/Discarding → Learning → Forwarding
        邻居：Incomplete → Reachable → Stale/Expired → Re-resolved
```

链路 Up 只说明物理连接成立；端口 Forwarding 才可能转发；MAC 学习正确不等于 ARP 正确；ARP 正确不等于应用在线；应用在线不等于控制过程正常。故障时间线需要区分物理恢复、二层收敛、邻居恢复、传输重连、工业会话重建和过程数据重新可信。

## 10. 安全风险与攻击链

二层安全风险包括未授权设备接入、ARP 欺骗、MAC 泛洪、VLAN 跳跃、恶意或错误 BPDU、广播/组播风暴、Trunk 配置过宽、交换机管理面暴露、镜像口滥用和配置备份泄露。

典型攻击链可以是：攻击者接入未关闭的控制柜端口，通过 DHCP/静态地址进入 HMI VLAN；发送伪造 ARP 将工程站到 PLC 的流量引向攻击主机并转发，观察变量地址和工程会话；随后利用已知协议写入设定值，同时选择性丢弃报警响应。物理后果仍取决于控制器模式、变量映射、许可和保护层。

防护包括关闭闲置端口、固定或认证接入、限制每端口 MAC、DHCP Snooping 与动态 ARP 检查（设备兼容时）、静态关键邻居、Trunk 白名单、管理面独立、配置审计、环路保护、被动监测和物理机柜管理。不能在未验证的生产网批量启用端口安全，因为老旧设备、冗余 MAC 或设备替换可能被误阻断。

## 11. 工控场景：维护接线引发环路与控制超时

检修人员为同时访问两台交换机，把笔记本的 USB 网卡和扩展坞网卡分别接入同一控制 VLAN，并启用了系统桥接。广播帧经笔记本形成环路，交换机 MAC 表不断移动，PLC 与远程 I/O 周期通信超时。设备按预设进入保持输出，部分泵继续运行而新启停命令无法到达。

排障证据包括端口广播速率、MAC Flapping 日志、STP/环网状态、接口错误、PLC I/O 质量、应用重连和过程趋势。恢复不能只拔线：应确认表项稳定、周期通信恢复、数据质量有效、保持输出已解除，并按工艺顺序恢复操作。整改还应控制临时接入、启用边缘端口保护并培训维护人员不得桥接控制网络。

## 12. 面试表达

### 60 分答案

> 交换机根据 MAC 地址表在 VLAN 内转发帧，ARP 用于把 IPv4 下一跳地址解析为 MAC。VLAN 划分广播域，但跨 VLAN 仍可通过路由或其他路径通信，所以还需 ACL、防火墙和端口安全。

### 90 分答案

> 我会同时检查 ARP 邻居表和交换机 MAC 表，区分已知单播、未知单播、广播与组播。VLAN 是二层分段实现，不等于安全域；还要审阅 Access/Trunk、Native VLAN、SVI 路由、管理面和多宿主资产。工业环境尤其关注环路收敛、IGMP Snooping、QoS 队列和风暴抑制，因为二层异常可能造成远程 I/O 超时和输出保持。证据会关联端口状态、表项变化、抓包、控制器质量和过程响应。

### 常见追问

**问：ARP 欺骗为什么能形成中间人？**

答：攻击者让通信双方把对方或网关 IP 映射到攻击者 MAC，并继续双向转发。是否成功还取决于缓存接受规则、端口隔离、静态绑定和网络拓扑。

**问：VLAN 能防止攻击吗？**

答：它能限制二层广播与直接转发范围，但不能阻止经路由、防火墙错误规则、Trunk、多宿主主机或管理面形成的跨域路径。

## 13. 实操建议

### 实验一：ARP 与 MAC 表关联

在隔离交换环境让两台主机通信，抓取 ARP Request/Reply，记录主机邻居表和交换机 MAC 表。移动端口或等待老化后观察重新学习，解释两张表的不同职责。

### 实验二：VLAN 标签与跨域路由

配置两个 Access VLAN 和受控三层网关，在终端侧与 Trunk 侧抓包，比较标签。先验证二层不互通，再通过一条最小 ACL 放通测试服务，证明 VLAN 与安全策略的边界。

### 实验三：环路与组播桌面验证

优先使用仿真器检查 STP 状态、根桥选择、链路故障收敛和 IGMP 成员变化。若使用实验交换机，设置停止条件和独立管理路径，禁止在生产控制网上制造环路或泛洪。

### 实验四：二层攻击链证据

仅在隔离环境模拟 ARP 映射异常，比较端点邻居变化、交换日志、抓包路径和应用时延。实验目标是建立检测证据，不向真实 PLC 注入或篡改控制报文。
