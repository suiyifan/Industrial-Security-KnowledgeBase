# 工业网络抓包分析方法

## 1. 抓包前先确认范围

抓包也需要授权和边界确认。

要确认：

- 抓哪条链路？
- 是否包含敏感生产数据？
- 是否允许保存 PCAP？
- PCAP 存放在哪里？
- 是否需要脱敏？
- 抓包时间窗口多长？
- 是否会影响交换机性能？

## 2. 抓包点选择

不同抓包点看到的东西不同。

| 抓包点 | 能看到 |
|---|---|
| HMI 旁路 | HMI 与 SCADA / PLC 通信 |
| SCADA Server 上联 | 监控系统核心流量 |
| PLC 前端交换机 | 控制器通信 |
| DMZ 边界 | 跨域数据交换 |
| 远程运维入口 | 外部维护行为 |

## 3. 分析步骤

1. 统计 IP 会话。
2. 标注资产角色。
3. 识别协议和端口。
4. 查找工业协议。
5. 区分读操作和写操作。
6. 识别远程登录、文件共享、数据库连接。
7. 找异常源地址。
8. 结合业务确认是否正常。

## 4. Wireshark 常用过滤器

按 IP：

```text
ip.addr == 192.168.1.10
```

按端口：

```text
tcp.port == 502
tcp.port == 102
tcp.port == 4840
tcp.port == 2404
```

按协议：

```text
modbus
s7comm
opcua
```

按 TCP 连接：

```text
tcp.flags.syn == 1
```

## 5. 重点观察

### 5.1 新资产

突然出现的新 IP 访问控制器，需要确认。

### 5.2 写操作

工业协议写操作通常比读操作风险更高。

例如 Modbus：

- `0x03` 读保持寄存器
- `0x06` 写单寄存器
- `0x10` 写多寄存器

### 5.3 工程操作

工程下载、控制器停止、程序上传等操作应重点关注。

### 5.4 远程桌面

RDP、VNC、SSH、Telnet 等远程访问要结合来源和时间判断。

### 5.5 文件共享

SMB、FTP、TFTP 可能涉及工程文件、日志、备份传输。

## 6. 分析输出

抓包分析报告建议包含：

- 抓包时间
- 抓包位置
- 涉及资产
- 协议统计
- 通信矩阵
- 异常通信
- 工业协议写操作
- 远程访问行为
- 风险判断
- 建议

## 7. 面试表达

> 工业网络抓包我会先确认抓包点和授权范围，再统计通信会话，标注资产角色，识别协议端口，重点看工业协议写操作、工程操作、远程登录和跨区通信。抓包结果最终要沉淀成通信矩阵和异常路径，而不是只停留在数据包截图。
