# 典型工业网络拓扑

## 1. 单产线拓扑

```text
办公网
  |
工业防火墙
  |
监控交换机
  |-- 工程站
  |-- 操作员站 / HMI
  |-- SCADA Server
  |-- 历史数据库
  |
控制交换机
  |-- PLC 1
  |-- PLC 2
  |-- 工业网关
  |
现场设备
```

安全关注：

- 办公网是否能直接访问监控交换机。
- 工程站是否能访问所有 PLC。
- SCADA Server 与 PLC 是否需要写权限。
- 历史库是否同时连办公网和控制网。

## 2. 多站点 SCADA 拓扑

```text
调度中心 SCADA
  |
广域网 / 专线 / VPN
  |
远程站点 RTU / PLC
  |
泵站 / 阀室 / 变电站 / 水厂
```

安全关注：

- 远程链路是否加密。
- 站点身份是否认证。
- 调度中心是否能下发遥控。
- 远程站点是否有本地保护。
- 通信中断后系统进入什么状态。

## 3. DCS 流程工业拓扑

```text
工程师站
操作员站
历史站
应用服务器
  |
DCS 控制网络
  |
控制站 / I/O 站
  |
现场仪表 / 执行机构
```

安全关注：

- 工程师站权限。
- 控制策略变更流程。
- 操作员站账号。
- 控制网络冗余。
- 历史站与上层系统接口。

## 4. 工业 DMZ 拓扑

```text
办公网
  |
防火墙 A
  |
工业 DMZ
  |-- 跳板机
  |-- 文件交换
  |-- 补丁服务器
  |-- 数据中转
  |
防火墙 B
  |
控制网
```

安全关注：

- A、B 两侧策略是否独立。
- DMZ 是否能主动访问控制器。
- 文件交换是否有查杀和审批。
- 跳板机是否审计。

## 5. 常见危险拓扑

### 双网卡工程站

```text
办公网 ---- 工程站 ---- 控制网
```

风险：

- 工程站变成绕过边界的桥。
- 办公网攻击可借工程站进入控制网。

### 历史数据库双归属

```text
办公网 ---- 历史库 ---- 控制网
```

风险：

- 办公网可通过历史库影响控制网。
- 数据库账号泄露后可能横向移动。

### 厂商 VPN 直连工程站

```text
互联网 -> VPN -> 工程站 -> PLC
```

风险：

- 外部账号泄露后直接进入高价值资产。

## 6. 面试表达

> 我看拓扑时会重点找跨域路径。比如双网卡工程站、双归属历史库、厂商 VPN 直连工程站，都是常见高风险结构。它们的问题不是简单联网，而是可能绕过工业 DMZ 和防火墙，把办公网或互联网风险带到控制层。

## 6. 拓扑阅读标注规范

一张可审计拓扑至少标注：资产角色与工程名称、接口与 VLAN/子网、物理和逻辑链路、默认网关、路由/NAT、主备、管理面、远程入口、协议发起方向、监测点、时间源和故障域。不要只画设备图标和连线。

建议同时维护四张视图：

1. 物理视图：机柜、端口、光路、电源和共路由；
2. 逻辑视图：Zone、VLAN、子网、VRF、路由和 NAT；
3. 业务视图：周期 I/O、HMI、工程、历史、文件和时间；
4. 安全视图：边界、身份、管理、监测、攻击路径和恢复。

## 7. 拓扑故障推演模板

```text
故障对象：链路 / 交换机 / 网关 / 防火墙 / 时间源 / 服务器
首先可见：________________________________________
二层/三层变化：___________________________________
会话与应用变化：__________________________________
控制器与设备状态：________________________________
过程与保护响应：__________________________________
自动恢复条件：____________________________________
人工确认与回退：__________________________________
```

对每张图至少推演单链路断开、主交换失效、防火墙切换、DMZ 失联、远程 WAN 中断和时间源错误。冗余图若无法说明共因故障、角色仲裁和业务恢复时间，就还不是工程可用拓扑。

## 8. 危险拓扑辨识

- 历史库或工程站双网卡直接连接不同 Zone；
- 企业到控制区端到端路由，DMZ 只作中转；
- 生产与安全系统共享网络管理、虚拟化或备份；
- 远程 VPN/蜂窝云盒直达 PLC 或串口服务器；
- 两条冗余光路同桥架、双机同电源或同管理平台；
- SPAN 传感器同时具备生产转发和企业管理；
- Trunk 允许全部 VLAN，默认 VLAN 承载管理；
- 无线、临时交换机和维护接口未出现在正式图纸。

发现危险拓扑后，继续验证实际路由、主机转发、规则、账号和流量，避免只凭图纸定性。
