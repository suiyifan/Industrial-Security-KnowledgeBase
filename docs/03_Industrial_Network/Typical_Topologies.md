# 典型工业网络拓扑

## 1. 单产线拓扑

```text
办公网
  |
工业防火墙
  |
监控交换机
  |-- 工程站
  |-- 操作员站 / HMI
  |-- SCADA Server
  |-- 历史数据库
  |
控制交换机
  |-- PLC 1
  |-- PLC 2
  |-- 工业网关
  |
现场设备
```

安全关注：

- 办公网是否能直接访问监控交换机。
- 工程站是否能访问所有 PLC。
- SCADA Server 与 PLC 是否需要写权限。
- 历史库是否同时连办公网和控制网。

## 2. 多站点 SCADA 拓扑

```text
调度中心 SCADA
  |
广域网 / 专线 / VPN
  |
远程站点 RTU / PLC
  |
泵站 / 阀室 / 变电站 / 水厂
```

安全关注：

- 远程链路是否加密。
- 站点身份是否认证。
- 调度中心是否能下发遥控。
- 远程站点是否有本地保护。
- 通信中断后系统进入什么状态。

## 3. DCS 流程工业拓扑

```text
工程师站
操作员站
历史站
应用服务器
  |
DCS 控制网络
  |
控制站 / I/O 站
  |
现场仪表 / 执行机构
```

安全关注：

- 工程师站权限。
- 控制策略变更流程。
- 操作员站账号。
- 控制网络冗余。
- 历史站与上层系统接口。

## 4. 工业 DMZ 拓扑

```text
办公网
  |
防火墙 A
  |
工业 DMZ
  |-- 跳板机
  |-- 文件交换
  |-- 补丁服务器
  |-- 数据中转
  |
防火墙 B
  |
控制网
```

安全关注：

- A、B 两侧策略是否独立。
- DMZ 是否能主动访问控制器。
- 文件交换是否有查杀和审批。
- 跳板机是否审计。

## 5. 常见危险拓扑

### 双网卡工程站

```text
办公网 ---- 工程站 ---- 控制网
```

风险：

- 工程站变成绕过边界的桥。
- 办公网攻击可借工程站进入控制网。

### 历史数据库双归属

```text
办公网 ---- 历史库 ---- 控制网
```

风险：

- 办公网可通过历史库影响控制网。
- 数据库账号泄露后可能横向移动。

### 厂商 VPN 直连工程站

```text
互联网 -> VPN -> 工程站 -> PLC
```

风险：

- 外部账号泄露后直接进入高价值资产。

## 6. 面试表达

> 我看拓扑时会重点找跨域路径。比如双网卡工程站、双归属历史库、厂商 VPN 直连工程站，都是常见高风险结构。它们的问题不是简单联网，而是可能绕过工业 DMZ 和防火墙，把办公网或互联网风险带到控制层。
