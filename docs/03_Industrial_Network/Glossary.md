# 03 章术语表

## Purdue Model

工业控制系统分层参考模型，用于描述企业 IT、生产管理、监控控制、基本控制和现场过程。

## Zone

安全区域，一组安全需求相似的资产。

## Conduit

区域之间的受控通信路径。

## Industrial DMZ

工业隔离区，位于办公网和控制网之间，用于数据中转、远程运维、补丁和文件交换。

## VLAN

虚拟局域网，用于在交换机上进行逻辑网络划分。

## ACL

访问控制列表，用于限制不同地址、端口或协议的通信。

## Firewall

防火墙，用于控制网络边界通信。

## Industrial Firewall

工业防火墙，可结合工业协议功能进行更细粒度控制。

## Communication Matrix

通信矩阵，记录资产之间的通信关系、协议、端口、方向和业务说明。

## Jump Server

跳板机，用于集中运维访问。

## Bastion Host

堡垒机，用于账号管理、访问控制和会话审计。

## VPN

虚拟专用网络，常用于远程接入。

## SPAN

交换机端口镜像，把某些流量复制到监测端口。

## TAP

网络分光设备，用于复制链路流量。

## IDS

入侵检测系统，用于发现异常行为和攻击迹象。

## Industrial IDS

工业入侵检测系统，关注工业协议、控制操作、工程下载、PLC 状态变化等。

## Protocol Gateway

协议网关，用于不同工业协议或网络之间转换。

## Serial Server

串口服务器，用于将串口设备接入以太网。

## Remote Maintenance

远程运维，厂商或维护人员通过远程通道进入生产环境进行维护。

## Attack Path

攻击路径，攻击者从入口到目标资产的连续步骤。

## MAC Address Table / CAM Table

交换机维护的 `(VLAN, MAC) → 端口` 转发表，通过源 MAC 学习。它与主机的 ARP/邻居表职责不同；MAC Flapping 可能来自环路、主备、移动或欺骗。

## ARP

IPv4 邻居解析协议，将下一跳 IP 解析为 MAC。Request 通常广播，Reply 通常单播。缺少强认证，可能被污染；分析需同时查看邻居表与交换机 MAC 表。

## SPAN / Port Mirroring

交换机复制指定端口/VLAN 流量到监测端口。可能因方向、容量、芯片实现而漏包或丢标签，“未看到”不能直接证明“未发生”。

## Network TAP

链路中复制流量的专用设备。通常比 SPAN 稳定，但需评估光功率、输出带宽、供电和安装变更风险。

## Longest Prefix Match

路由器选择最具体目的前缀的规则。默认路由只在没有更具体路由时使用；排障必须检查返回路径。

## NAT

改变地址/端口的转换机制，用于地址冲突或边界连接。它不提供身份认证，会改变抓包与日志五元组，并可能破坏含嵌入地址的协议。

## RTO / RPO

RTO 是允许的恢复时间，RPO 是允许的数据/状态损失。工业系统还应定义最大丢包、抖动、安全状态和恢复条件。

## Convergence

故障后拓扑、路由和转发表达到稳定状态的过程。网络收敛不等于 TCP、工业会话、数据质量和过程全部恢复。

## NTP / PTP / Holdover

NTP 用于通用时间同步，PTP 面向更高精度同步；Holdover 指失去参考后依靠本地时钟维持。时间显示分辨率不等于同步准确度。

## Stateful Firewall

维护会话状态并识别返回流的防火墙。允许会话只证明网络路径，不证明应用命令安全或物理动作发生。

## Industrial DPI

解析工业协议功能、对象和读写语义的深度检测。能力受协议版本、加密、分片和厂商扩展限制，不能替代控制器授权。

## Data Diode / Unidirectional Gateway

限制数据方向的物理或逻辑设备。仍需处理两侧代理、数据完整性、缓存、补传和文件内容，不能理解为“有网闸即绝对安全”。

## Baseline

特定工况下资产、通信、周期、应用操作和状态的正常参考。应区分稳态、启停、维护和故障恢复，不能无限自动学习。

## Data Quality / Freshness

Quality 表示值是否可信，Freshness 表示值是否足够新。链路在线或数值未变不代表测量仍有效。

## Protocol Gateway

在介质、地址、类型、质量、时间和命令语义之间转换的设备。北向成功不一定代表南向设备执行。

## Idempotency

同一操作重复执行与执行一次效果相同。远程站重连和缓存补传需要幂等或事务 ID，防止重复命令。

## Break-glass Access

紧急情况下使用的预定义最小访问流程，包含触发条件、短时凭据、记录、事后复核和轮换，不等于临时全网放通。

## Split Brain

两个冗余节点同时认为自己是主节点，可能造成重复转发、写入或控制。需要仲裁、隔离和状态同步。

## Evidence Confidence

对结论证据强度的分级，可区分理论、配置、行为观察、隔离实验、现场确认和后果证实。风险报告应公开未验证假设。

## Process-aware Detection

将网络命令与 SP/PV/MV、模式、设备反馈、物料能量和保护状态关联的检测方法，比单纯端口或阈值更接近真实工业后果。
