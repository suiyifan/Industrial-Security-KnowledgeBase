# 工业网络风险案例化表达

## 案例 1：办公网直达工程站

### 场景

办公网主机可以直接 RDP 到控制网工程站。

### 风险路径

```text
办公终端失陷
-> RDP 访问工程站
-> 获取工程文件
-> 连接 PLC
-> 下载工程或 Force 变量
```

### 工控影响

- 工程站高价值权限暴露。
- 控制逻辑完整性受威胁。
- 攻击者可进一步影响 PLC 控制行为。

### 建议

- 办公网禁止直达工程站。
- 通过工业 DMZ 和堡垒机访问。
- MFA、审批、会话审计。
- 维护窗口后关闭策略。

## 案例 2：防火墙 any any allow

### 场景

工业 DMZ 到控制网防火墙存在全通策略。

### 风险路径

```text
DMZ 主机被攻陷
-> 任意访问控制网资产
-> 扫描 HMI / PLC / SCADA
-> 横向移动
```

### 工控影响

- DMZ 失去缓冲作用。
- 攻击者可扩大控制网暴露面。

### 建议

- 基于通信矩阵重构白名单。
- 删除过宽策略。
- 开启日志。
- 高风险协议单独审计。

## 案例 3：厂商 VPN 长期开启

### 场景

厂商 VPN 账号长期有效，可访问跳板机和工程站。

### 风险路径

```text
厂商账号泄露
-> VPN 登录
-> 访问跳板机
-> RDP 工程站
-> 修改工程或查看点表
```

### 工控影响

- 外部账号成为控制网入口。
- 难以追踪具体操作人。

### 建议

- 按需开通。
- MFA。
- 账号实名。
- 维护后回收。
- 会话录像。

## 案例 4：PLC 网络无访问控制

### 场景

控制层内任意主机都可访问 PLC 502 端口。

### 风险路径

```text
HMI 或普通主机失陷
-> 扫描控制网
-> 发现 PLC 502
-> 读取或写入寄存器
```

### 工控影响

- 过程数据泄露。
- 可写变量可能影响设备状态。
- 攻击范围不受限制。

### 建议

- PLC 前增加工业防火墙或 ACL。
- 只允许 SCADA / 工程站必要访问。
- 监测 Modbus 写功能码。

## 案例 5：双网卡历史库

### 场景

历史数据库同时连接办公网和控制网。

### 风险路径

```text
办公网数据库访问
-> 历史库被攻陷
-> 控制网侧横向移动
-> 访问 SCADA / HMI
```

### 工控影响

- 历史数据被篡改。
- 事故追溯失真。
- 历史库成为跨域跳板。

### 建议

- 通过 DMZ 做数据发布。
- 控制网侧只允许必要采集。
- 办公网访问使用只读副本。
- 加强数据库权限和日志。

## 面试总结

> 工业网络风险要按攻击路径讲。比如办公网直达工程站、防火墙全通、厂商 VPN 长期开启、PLC 无访问控制、双网卡历史库，它们的共同问题都是打破分区边界，让攻击者有机会从低安全区域进入高价值控制资产。

## 案例 6：VLAN 已隔离但三层管理面全通

### 场景

控制器、HMI 和办公终端位于不同 VLAN，但核心交换机 SVI 之间默认路由，办公管理员可 SSH 到全部工业交换机。

### 风险路径

办公管理员凭据失陷 → 网络设备管理面 → 修改 Trunk/ACL/SPAN → 观察或重定向工程流量 → 到达控制区。

### 工控影响

攻击者可能造成环路、流量黑洞、监测盲区或工程路径暴露。VLAN 限制广播域，却未形成真实安全边界。

### 建议与验收

管理面进入独立 Zone，仅受控管理工作站经堡垒访问；分离企业与工业凭据，启用 MFA 和配置审计。验收要验证办公源被拒绝、备用管理路径和故障恢复，而非只检查 VLAN ID。

## 案例 7：时间源漂移破坏事件归因

### 场景

控制器跟随本地 NTP，防火墙跟随企业 NTP，二者相差 90 秒。一次远程写入后，网络日志显示命令似乎发生在报警之后。

### 风险路径

错误/受篡改时间源 → 多系统时钟偏差 → 攻击与过程事件顺序颠倒 → 调查和响应判断错误。

### 工控影响

SOE、报警、批次、证书和历史数据都可能受影响。显示到毫秒不能证明同步到毫秒。

### 建议与验收

建立分层可信时间源、多源偏差监测和 holdover 方案；事件报告保存时钟状态和误差区间。通过已知事件在抓包、控制器、历史和防火墙中校正时间线。

## 案例 8：远程站重连执行旧命令

### 场景

中心 SCADA 在蜂窝链路中断期间多次下发开阀，网关将请求排队；重连后依次发送，现场阀门在不合适工况动作。

### 风险路径

链路失联 → 中心状态陈旧 → 命令无有效期/幂等 → 网关缓存 → 重连批量执行。

### 工控影响

可能造成管线压力波动、溢流或设备启停顺序错误。问题不一定是攻击，也属于可被攻击者利用的状态设计缺陷。

### 建议与验收

控制命令设置事务 ID、有效期和目标状态；现场 RTU 自治，重连先同步模式和设备状态，再重新授权远程控制。实验验证历史数据可补传而过期命令被丢弃。

## 案例分析统一模板

```text
业务与工况 → 现状证据 → 入口/故障 → 网络与身份路径
→ 获得的工业能力 → 控制/状态变化 → 过程与保护响应
→ 可信度/未验证假设 → 临时缓解 → 根因整改 → 验收与恢复
```

报告避免“存在漏洞，可能导致停产”式跳跃。明确攻击者还需要什么、哪道屏障会阻断、什么证据能证明物理后果。
