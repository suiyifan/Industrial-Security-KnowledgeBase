# 工业网络风险案例化表达

## 案例 1：办公网直达工程站

### 场景

办公网主机可以直接 RDP 到控制网工程站。

### 风险路径

```text
办公终端失陷
-> RDP 访问工程站
-> 获取工程文件
-> 连接 PLC
-> 下载工程或 Force 变量
```

### 工控影响

- 工程站高价值权限暴露。
- 控制逻辑完整性受威胁。
- 攻击者可进一步影响 PLC 控制行为。

### 建议

- 办公网禁止直达工程站。
- 通过工业 DMZ 和堡垒机访问。
- MFA、审批、会话审计。
- 维护窗口后关闭策略。

## 案例 2：防火墙 any any allow

### 场景

工业 DMZ 到控制网防火墙存在全通策略。

### 风险路径

```text
DMZ 主机被攻陷
-> 任意访问控制网资产
-> 扫描 HMI / PLC / SCADA
-> 横向移动
```

### 工控影响

- DMZ 失去缓冲作用。
- 攻击者可扩大控制网暴露面。

### 建议

- 基于通信矩阵重构白名单。
- 删除过宽策略。
- 开启日志。
- 高风险协议单独审计。

## 案例 3：厂商 VPN 长期开启

### 场景

厂商 VPN 账号长期有效，可访问跳板机和工程站。

### 风险路径

```text
厂商账号泄露
-> VPN 登录
-> 访问跳板机
-> RDP 工程站
-> 修改工程或查看点表
```

### 工控影响

- 外部账号成为控制网入口。
- 难以追踪具体操作人。

### 建议

- 按需开通。
- MFA。
- 账号实名。
- 维护后回收。
- 会话录像。

## 案例 4：PLC 网络无访问控制

### 场景

控制层内任意主机都可访问 PLC 502 端口。

### 风险路径

```text
HMI 或普通主机失陷
-> 扫描控制网
-> 发现 PLC 502
-> 读取或写入寄存器
```

### 工控影响

- 过程数据泄露。
- 可写变量可能影响设备状态。
- 攻击范围不受限制。

### 建议

- PLC 前增加工业防火墙或 ACL。
- 只允许 SCADA / 工程站必要访问。
- 监测 Modbus 写功能码。

## 案例 5：双网卡历史库

### 场景

历史数据库同时连接办公网和控制网。

### 风险路径

```text
办公网数据库访问
-> 历史库被攻陷
-> 控制网侧横向移动
-> 访问 SCADA / HMI
```

### 工控影响

- 历史数据被篡改。
- 事故追溯失真。
- 历史库成为跨域跳板。

### 建议

- 通过 DMZ 做数据发布。
- 控制网侧只允许必要采集。
- 办公网访问使用只读副本。
- 加强数据库权限和日志。

## 面试总结

> 工业网络风险要按攻击路径讲。比如办公网直达工程站、防火墙全通、厂商 VPN 长期开启、PLC 无访问控制、双网卡历史库，它们的共同问题都是打破分区边界，让攻击者有机会从低安全区域进入高价值控制资产。
