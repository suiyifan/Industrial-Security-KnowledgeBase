# 工业网关、协议转换与边界设备

## 1. 工业网关是什么

工业网关用于连接不同协议、不同网络或不同系统。

常见作用：

- 协议转换
- 数据采集
- 边缘计算
- 远程运维
- 云平台接入
- 串口转以太网
- 站点数据汇聚

## 2. 常见协议转换

- Modbus RTU <-> Modbus TCP
- Serial <-> Ethernet
- PLC 私有协议 <-> OPC UA
- OPC DA <-> OPC UA
- IEC 104 <-> 其他电力协议
- 工业协议 <-> MQTT / HTTP API

## 3. 安全风险

工业网关常常跨边界，风险高：

- 一侧连接 PLC，一侧连接上层网络或云平台。
- Web 管理界面可能有漏洞。
- 默认密码常见。
- 配置文件可能包含 PLC 地址和点表。
- 协议转换可能把低安全协议暴露到高层网络。
- 远程运维功能可能绕过边界。

## 4. 数据采集网关

数据采集网关通常从 PLC / RTU 读取数据，再转发给 SCADA、MES、云平台或数据库。

评估要问：

- 只读还是可写？
- 是否能下发控制命令？
- 采集周期是多少？
- 数据上传到哪里？
- 是否加密？
- 是否有本地缓存？
- 配置谁能修改？

## 5. 串口服务器

串口服务器把 RS-232 / RS-485 设备接入以太网。

风险：

- 传统串口设备本来没有网络暴露。
- 接入以太网后攻击面扩大。
- 缺少认证和加密。
- 可能暴露 Modbus RTU 等协议。

## 6. 云接入网关

云接入网关把现场数据上传到云平台。

安全关注：

- 是否经过审批。
- 上传哪些数据。
- 是否有远程控制能力。
- 云端账号权限。
- 证书和密钥存放。
- 断网后的行为。

## 7. 评估检查项

- 网关型号和固件版本。
- 管理界面是否暴露。
- 默认账号是否修改。
- 是否启用不必要服务。
- 协议转换方向。
- 是否允许写操作。
- 配置备份是否加密。
- 是否有云连接。
- 是否记录操作日志。

## 8. 面试表达

> 工业网关是工控网络里的关键边界资产，因为它经常连接不同协议和不同安全域。评估时我会关注它是否只是采集数据，还是能下发控制命令；是否存在默认密码、Web 漏洞、配置泄露；以及协议转换是否把原本只在现场侧的协议暴露到上层网络。
