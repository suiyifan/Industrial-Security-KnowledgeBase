# 无线、串口与现场总线风险

## 1. 为什么不能只看以太网

工业网络不只有 TCP/IP。很多现场仍然存在：

- RS-232
- RS-485
- Modbus RTU
- Profibus
- CAN
- HART
- 工业无线
- 4G/5G 路由器
- Wi-Fi 维护网络

这些链路可能不在传统扫描范围内，但同样影响控制系统。

## 2. 串口通信

串口通信常用于老设备、仪表、RTU、变频器。

特点：

- 简单
- 稳定
- 老旧
- 缺少认证
- 常通过网关接入以太网

风险：

- 串口转以太网后暴露范围扩大。
- Modbus RTU 缺少认证。
- 报文可被重放或伪造。

## 3. 现场总线

现场总线连接控制器和现场设备。

常见：

- Profibus
- CANopen
- DeviceNet
- Foundation Fieldbus
- HART

安全关注：

- 是否通过网关连接上层网络。
- 是否有物理接入风险。
- 是否有诊断接口暴露。
- 是否能影响现场设备参数。

## 4. 工业无线

工业无线包括：

- Wi-Fi
- 4G/5G
- ZigBee
- WirelessHART
- 私有无线

风险：

- 弱密码。
- 非授权 AP。
- 维护 Wi-Fi 长期开启。
- 无线网桥暴露。
- SIM 卡和路由器管理风险。

## 5. 4G/5G 工业路由器

常见于远程泵站、阀室、光伏、风电、环保监测。

安全关注：

- 是否有公网 IP。
- 是否开启远程管理。
- 默认密码是否修改。
- 是否接入控制器。
- 是否有 VPN。
- SIM 卡和平台账号是否管理。

## 6. 评估问题

- 是否存在无线网络？
- 是否有隐藏或临时 AP？
- 串口设备是否通过网关接入以太网？
- 现场总线是否有诊断口？
- 远程站点是否通过 4G/5G 回传？
- 无线和蜂窝设备谁负责管理？

## 7. 面试表达

> 工业网络评估不能只看以太网。很多现场设备通过串口、现场总线、无线或 4G/5G 工业路由器接入。尤其是串口转以太网和远程站点蜂窝路由器，可能把原本局部的控制链路暴露到更大网络范围，所以也要纳入资产和边界评估。
