# 无线、串口与现场总线接入

## 1. 原理解释：工业通信不只存在于以太网

现场设备可能位于旋转机械、移动车辆、长距离管线或危险区域，铺设以太网成本高或不可行；大量存量仪表只支持串口和现场总线。无线、RS-485、CAN、PROFIBUS、HART 等技术解决不同距离、实时性、供电和抗干扰约束，是控制链的重要组成。

这些链路经网关接入 IP 网络后，安全团队容易只看到北向 TCP 会话，忽略南向共享总线、射频覆盖和现场自治。真实分析必须追踪介质状态、总线仲裁、节点地址、应用事务、网关缓存和设备控制状态。

## 2. 串口与物理层工程

RS-232 多用于短距离点对点，电平相对地；RS-485 使用差分信号，支持较长距离和多点总线。工程参数包括波特率、数据位、奇偶校验、停止位、流控、二线/四线、终端电阻、偏置、隔离、屏蔽和接地。

RS-485 推荐总线型而非任意星型，支线过长、终端错误、地电位差和电磁干扰会导致反射、误码和间歇超时。串口“能收数据”不代表参数和布线正确，需结合示波/诊断、错误计数、重试和不同工况验证。

半双工链路需方向控制，主从协议依赖请求、静默间隔、响应与超时。一个从站持续占线或网关重试过密，会阻塞整条总线。串口没有交换机隔离，每个节点通常能看到共享介质上的通信，物理接入风险高。

## 3. 现场总线的共同模型

不同总线细节在 04 篇协议章节展开，这里建立网络工程共性：

- 节点地址或设备标识；
- 总线主站、令牌、优先级或仲裁机制；
- 周期数据与非周期诊断；
- 连接、运行、故障和安全状态；
- 总线长度、速率、终端和拓扑约束；
- 网关、控制器和工程工具的接入方式。

CAN 使用基于标识符优先级的位仲裁，高优先级帧可持续占用总线；PROFIBUS DP 主站周期访问从站；HART 在 4–20 mA 上叠加数字通信，模拟量仍可承载过程值。不能用“现场总线都不安全”概括，应明确攻击者能否接入、注入、改变地址/参数，以及设备失联策略。

## 4. 工业 Wi-Fi

Wi-Fi 用 AP 和客户端提供局域无线接入，适合移动 HMI、AGV、仓储终端和难布线区域。设计关注频段、信道、发射功率、天线、漫游、覆盖、干扰、容量、QoS 和认证。信号强度高不等于链路稳定，噪声、同频竞争、重传和漫游延迟都会影响周期通信。

安全优先使用 WPA2/3-Enterprise、802.1X/EAP 和独立证书或账号，避免共享 PSK 长期不变。无线客户端进入专用 Zone，经防火墙访问指定应用；AP 管理面和控制器管理面与业务分离。WPS、开放管理、默认凭据、弱加密和企业/控制 SSID 混用均属风险。

漫游时客户端在 AP 间切换，认证、密钥与三层地址可能变化。对 AGV 或移动操作终端，应测试移动路径中的切换时延、丢包、应用重连、旧命令和本地安全状态，而不只做静态覆盖热力图。

## 5. 专用工业无线与低功耗网络

WirelessHART、ISA100.11a 等低功耗网状网络适合过程仪表，节点通过网关和网络管理器维护路由、时隙、密钥和设备加入。网状自愈提高覆盖，但路由变化、干扰和电池状态会改变延迟与数据新鲜度。

设备加入通常需要 Join Key 或安装码。共享或泄露的加入凭据可能允许未授权节点接入；网关和安全管理器成为高价值资产。轮换密钥需考虑离线节点和维护流程，不能因更换电池而使用全厂通用默认密钥。

LoRaWAN、专用电台或厂商遥测链路适合低速远距离数据，不适合默认承载高频闭环。链路预算、占空比、确认重传和下行容量会限制控制能力。安全评估要区分遥测只读与远程控制。

## 6. 蜂窝 4G/5G 与公网/专网接入

蜂窝路由器用于远程泵站、风机、管线和移动设备。连接可能采用公网 APN、专用 APN、运营商 VPN、IPsec 或云平台反向隧道。专用 APN 不是端到端认证，仍需控制 SIM、设备身份、路由、云账号和管理接口。

常见风险包括公网管理口、默认密码、长期云连接、SMS 控制、SIM 被转移、DNS/NTP 来自公网、固件不更新和蜂窝链路绕过厂区防火墙。流量清单应包含运营商、APN、SIM/IMEI、分配地址、VPN、云域名、远程管理和失联策略。

蜂窝网络具有时延、抖动、地址变化和覆盖波动。远程站必须自治：链路断开继续安全控制，本地缓存遥测，控制命令有有效期和确认；重连时先同步状态，再接受新命令，禁止执行缓存的过期启停。

## 7. 射频工程、干扰与可用性

无线可用性取决于链路预算、路径损耗、Fresnel 区、天线极化、遮挡、多径、噪声和同频占用。RSSI 只是接收功率，需结合 SNR、重试、调制速率、丢包和漫游判断。工业现场的电机、变频器、焊机和金属结构会改变射频环境。

恶意干扰与普通干扰症状相似。判断需结合频谱、多个 AP/站点、时间模式、设备错误和现场活动。无线 IDS 可检测未知 AP、伪造 SSID、去认证和异常关联，但无法单独证明射频干扰来源。

冗余无线链路若共用频段、天线位置、电源或运营商，存在共因故障。关键控制优先保留有线或本地保护，不以单一无线链路承担唯一安全功能。

## 8. 远程站、自治与命令状态

远程 RTU/PLC 需要在中心失联时保持本地控制、报警缓存和安全状态。链路恢复后，中心看到的状态可能陈旧，现场模式也可能已改变。建议使用源时间、质量、序号和命令有效期。

自治策略应明确哪些设定可在本地修改、哪些动作必须中心与现场共同授权，以及失联持续到何时需要降级或派员。电池、燃料、存储和本地日志容量也会限制自治时间。恢复远程控制前，操作员应核对设备位置、累计量、报警、旁路和正在执行的本地计划，避免中心假设与现场事实发生冲突，并留存交接记录。

```text
链路状态：Online → Degraded → Offline → Reconnecting → Synchronizing → Online
控制权：Remote → Communication Lost → Local Autonomous → Resync → Remote Authorized
命令：Created → Sent → Received → Validated → Executed → Feedback / Expired
```

重连先交换设备状态、时间和控制权，再处理遥测补传；历史数据可补传，实时控制命令通常过期丢弃。若中心在失联期间重复下发开阀，重连后全部执行会造成危险。

## 9. 数据流、控制流与状态流

```text
数据流：现场设备 → 总线/无线 → 网关/RTU → WAN → SCADA
控制流：操作命令 → 权限/有效期 → WAN → RTU 本地逻辑 → 执行器
状态流：介质质量 → 链路关联 → 协议会话 → 数据质量 → 设备/过程状态
```

无线关联成功不等于 SCADA 会话恢复；串口端口打开不等于从站响应；从站响应不等于工程量正确；命令确认不等于执行器动作。多层证据仍需关联。

## 10. 管理面与现场接入

AP、无线控制器、串口服务器、蜂窝路由器、现场总线主站和网关都有配置接口。限制管理源，使用安全协议、独立凭据、MFA（支持时）、签名固件和配置备份。禁用不需要的 Telnet、HTTP、发现和云管理。

现场控制柜端口、天线馈线、调试接口和串口接线是物理边界。机柜上锁、端口封堵、维护接入审批和巡检可以补偿旧协议缺少认证。设备替换后检查地址、密钥、波特率、路由、云绑定和失联策略。

## 11. 安全风险与攻击链

安全风险包括共享无线密钥、未知 AP、开放串口服务器、蜂窝旁路、默认网关凭据、总线物理接入、未授权节点加入、射频干扰、重放、旧命令重连执行、管理云账号泄露和密钥无法轮换。

典型攻击链是：攻击者获得厂商云平台账号，登录远程站蜂窝路由器并修改端口转发，使串口服务器暴露；通过 Raw TCP 枚举从站和读取状态，随后在中心链路短时中断时排队写命令。重连后网关把命令发送到泵控制器，造成非计划启动。

防护包括云 MFA 和最小租户权限、专用 APN/VPN、入站默认拒绝、串口客户端白名单、命令有效期和本地许可、配置变更告警、现场模式控制、独立设备保护和过程反馈监测。

## 12. 工控场景：AGV 漫游抖动导致状态错位

AGV 在两个 AP 间漫游时丢包，调度系统未及时收到“到位”状态，却重发移动任务。AGV 本地控制器已进入下一工位，恢复后旧任务被再次接受，导致路线冲突并触发安全激光停车。网络平均信号正常，问题只出现在移动路径和漫游窗口。

调查需关联 AP 漫游、重传、应用任务 ID、AGV 本地状态、位置和安全事件。整改包括快速漫游与覆盖优化、任务幂等/序号、旧命令过期、本地状态校验和恢复握手。不能只通过提高 AP 发射功率解决。

## 13. 工程评估流程

1. 盘点所有无线、蜂窝、串口、总线和现场调试接口。
2. 记录介质、拓扑、参数、地址、密钥、管理与云依赖。
3. 还原周期、仲裁、超时、重试、缓存和重连状态机。
4. 测量覆盖、SNR、重传、总线错误和链路容量。
5. 核查本地自治、命令有效期、控制权和安全状态。
6. 审阅现场物理保护、凭据、固件和配置备份。
7. 在隔离/授权窗口验证失联与恢复，不干扰生产射频或总线。

## 14. 面试表达

### 60 分答案

> 工业现场除以太网外还有 RS-485、现场总线、Wi-Fi、专用无线和 4G/5G。评估要检查物理拓扑、通信参数、认证加密、网关和失联策略。

### 90 分答案

> 我会从介质和状态机分析非以太网接入。串口检查电气、终端、半双工仲裁和帧时序；现场总线检查主站/优先级、周期和故障状态；无线检查链路预算、SNR、漫游、认证与干扰；蜂窝检查 APN、VPN、SIM、云账号和旁路。远程站重点是本地自治、源时间、质量、命令有效期和重连同步，避免链路恢复后执行旧命令。最终用网关两侧流量、设备诊断和过程反馈验证。

### 常见追问

**问：专用 APN 是否等于安全隔离？**

答：不是。它提供运营商侧逻辑接入，还需端到端 VPN、设备身份、路由白名单、云账号和管理面保护。

**问：无线信号强为什么仍会丢包？**

答：还可能有噪声、同频竞争、多径、漫游和队列问题，应结合 SNR、重传和时延分析。

## 15. 实操建议

### 实验一：RS-485 时序

在隔离环境用虚拟/实验串口改变波特率、校验和超时，观察帧错误、重试与总线占用，不连接生产设备。

### 实验二：无线漫游

用测试终端在两个实验 AP 间移动，记录 RSSI、SNR、重传、漫游和应用状态，比较链路与业务恢复时间。

### 实验三：远程站重连

模拟 WAN 中断，生成遥测与命令；验证历史补传、命令过期、状态同步和远程控制权重新授权。

### 实验四：攻击路径桌面推演

以云账号—蜂窝路由器—串口服务器—设备为路径，列出前提、证据、阻断点和过程后果。

## 16. 总结与检查清单

- [ ] 能解释串口电气、拓扑、参数与半双工状态；
- [ ] 能抽象现场总线仲裁、周期和故障机制；
- [ ] 能评估 Wi-Fi 覆盖、漫游、认证和干扰；
- [ ] 能审阅蜂窝、APN、VPN、SIM 与云管理；
- [ ] 能设计远程站自治、命令有效期和重连同步；
- [ ] 能保护管理面、现场接口、凭据和配置；
- [ ] 能将链路异常映射到控制与物理后果。
