# 文件上传、WebShell 与工程文件风险

文件上传漏洞是应用没有正确限制文件类型、内容、路径或执行权限，导致攻击者可以上传可执行脚本或危险文件。

工控平台常见上传入口包括报表附件、设备固件、工程备份、日志导入、图片或组态资源、文件交换区。

```text
攻击者上传文件 -> Web 服务器保存 -> 文件被解析或执行 -> 获得服务器权限 -> 读取配置 / 横向移动
```

工控风险：

- WebShell 获取服务器权限。
- 读取工程文件和配置。
- 通过文件交换进入工程站。
- 替换固件或配置包。

> 面试表达：工控场景下文件上传漏洞要关注上传内容是否能进入控制环境。文件交换服务器如果没有查杀和审批，恶意文件可能被带入工程站。
