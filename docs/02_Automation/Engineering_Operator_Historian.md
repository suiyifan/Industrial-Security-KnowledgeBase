# 工程站、操作员站与历史数据库

## 1. 为什么这三类资产重要

在工控安全中，工程站、操作员站、历史数据库经常比普通办公电脑更关键。

因为它们分别对应：

- 工程站：能改系统。
- 操作员站：能操作系统。
- 历史数据库：能记录系统。

## 2. 工程站

工程站用于工程配置和维护。

常见功能：

- 编写 PLC 程序。
- 配置 DCS 控制策略。
- 配置 HMI / SCADA 画面。
- 下载工程到控制器。
- 上传现场工程。
- 在线监控变量。
- 诊断设备状态。

安全价值：

> 工程站是高价值目标，因为拿到工程站权限，可能意味着攻击者能理解并修改控制逻辑。

## 3. 工程文件

工程文件可能包含：

- 控制逻辑
- 点表
- 设备地址
- 通信配置
- 账号或连接信息
- 画面配置
- 报警阈值
- 脚本

安全关注：

- 工程文件是否加密。
- 是否有版本管理。
- 是否有访问控制。
- 是否有离线备份。
- 是否能验证工程完整性。

## 4. 操作员站

操作员站用于日常监控和操作。

常见功能：

- 查看实时画面。
- 启停设备。
- 调整设定值。
- 确认报警。
- 查看趋势。
- 切换自动 / 手动模式。

安全关注：

- 操作权限是否分级。
- 是否存在共用账号。
- 是否记录操作日志。
- 误操作是否有二次确认。
- 关键操作是否需要审批。

## 5. 历史数据库

历史数据库用于存储过程数据。

常见数据：

- 温度、压力、液位、流量等趋势。
- 报警记录。
- 操作记录。
- 事件记录。
- 生产报表数据。

安全价值：

- 支持事故追溯。
- 支持异常分析。
- 支持合规审计。
- 支持攻击影响判断。

安全风险：

- 数据被删除。
- 数据被篡改。
- 采集被中断。
- 攻击者通过历史库横向移动。
- 历史接口暴露给办公网。

## 6. 三者之间的关系

```text
工程站 -> 配置 / 下载 -> PLC / DCS / SCADA
操作员站 -> 监控 / 操作 -> 生产过程
历史数据库 <- 采集 / 存储 <- 实时数据与事件
```

安全分析时可以这样判断：

- 攻陷工程站：偏控制逻辑和工程完整性风险。
- 攻陷操作员站：偏误操作、错误显示和命令下发风险。
- 攻陷历史数据库：偏追溯失真、数据完整性和横向移动风险。

## 7. 面试表达

可以这样说：

> 工程站、操作员站和历史数据库分别对应配置、操作和记录三个关键环节。工控安全评估时，我会把工程站视为高价值资产，因为它可能用于修改控制逻辑和下载工程；操作员站关系到实时操作；历史数据库关系到趋势、报警和事故追溯的可信度。
