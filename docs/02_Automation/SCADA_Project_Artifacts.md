# SCADA 工程资料审阅表

本专题列出 SCADA/HMI 项目常见资料及审阅重点，用于项目访谈、离线分析和版本比较。

## 1. 资料清单

- 系统架构、站点、通信链路和前置服务器；
- 驱动、通道、设备和协议连接配置；
- 标签数据库、数据类型、单位、质量和刷新周期；
- HMI 画面、模板、弹窗、动画和导航；
- 按钮、命令、选择—执行和反馈逻辑；
- 报警限、优先级、迟滞、延时、搁置和抑制；
- 趋势、历史点、采样、压缩和保留策略；
- 脚本、配方、报表、数据库查询和 API；
- 用户、角色、区域、电子签名和服务账号；
- 主备、灾备、缓存、补传和失联恢复；
- 工程版本、发布包、许可证和备份。

## 2. 标签审阅

抽样追踪：

```text
PLC/RTU 变量 → 协议对象 → 驱动地址
 → SCADA 标签 → HMI/报警/历史/报表
```

核对类型、字节序、缩放、单位、可写性、质量、源时间和失联替代值。标签名相同不代表映射正确。

## 3. 画面与命令审阅

- 对象显示的是命令、反馈还是计算状态；
- 坏质量、手动、旁路、强制和维护是否可见；
- 高风险按钮是否显示目标、当前状态和后果；
- 权限是在界面隐藏还是后端拒绝；
- 命令是否有确认、序列号、超时和结果；
- 弹窗和模板更新是否影响多台设备；
- 脚本是否能批量写标签或调用系统功能。

## 4. 报警与历史审阅

检查报警的设计依据、优先级、状态抑制、搁置期限和操作指导。检查历史采样、死区、压缩、插值、缓存与补传。事故导出需保留质量、源时间和查询方式。

## 5. 权限与接口审阅

- 默认、共享和自动登录账号；
- 操作、班长、工程和管理员角色；
- Web、移动、API、数据库与报表接口；
- 服务账号权限与凭据保管；
- 远程会话、会话锁定和审计；
- 主备、灾备权限是否一致。

## 6. 失联与恢复审阅

确认现场自治、HMI 坏质量、命令队列、事件缓存、累计量、重连去重和旧命令取消。恢复测试应验证数据、命令、报警、历史和用户会话，不只看服务器进程启动。

## 7. 版本比较输出

每次比较至少列出：

```text
对象类型：标签/画面/报警/脚本/权限/驱动
旧版本与新版本：
改变内容与原因：
受影响设备和过程：
写能力或显示语义变化：
测试证据：
回退方法：
```

## 8. 安全视角

高风险变化包括标签交换、只读变可写、质量丢失、报警禁用、脚本新增外部调用、服务账号提权和重连后旧命令执行。结论需与 PLC 逻辑和现场反馈交叉验证。

## 9. 面试速答

> 审阅 SCADA 工程时，我会从驱动和标签开始，追到画面、命令、报警、历史与权限。重点确认值、质量和时间是否一起传递，高风险命令是否有服务端授权与反馈，失联后是否存在陈旧值和旧命令。版本比较还要评估模板、脚本和服务账号对多个站点的传播影响。
