# SCADA 项目资料与配置深挖

SCADA / HMI 项目里有大量安全线索。它不只是“画面软件”，而是变量、权限、脚本、报警、趋势和通信驱动的集合。

## 1. SCADA 项目常见资料

| 资料 | 内容 | 安全价值 |
|---|---|---|
| 画面文件 | 工艺流程、按钮、数值显示 | 理解操作入口 |
| 变量库 | 点名、地址、类型、读写属性 | 判断关键变量 |
| 通信驱动配置 | PLC 地址、协议、轮询周期 | 理解通信路径 |
| 用户权限配置 | 角色、账号、操作权限 | 判断越权风险 |
| 报警配置 | 报警条件、级别、确认方式 | 判断报警风险 |
| 趋势配置 | 采集变量、周期、存储位置 | 判断追溯能力 |
| 脚本 | 自动动作、按钮逻辑、数据处理 | 查找危险执行 |
| 报表配置 | 生产统计、数据源 | 判断数据可信 |

## 2. 画面分析怎么看

HMI 画面通常暴露工艺结构。

看画面时关注：

- 哪些设备可操作？
- 哪些按钮是启动、停止、复位？
- 哪些参数可输入？
- 哪些报警长期存在？
- 是否有管理员或维护入口？
- 是否有隐藏画面或快捷键？

## 3. 按钮背后的动作

一个按钮可能不只是写一个变量。

可能动作：

- 写 PLC 变量。
- 执行脚本。
- 调用系统命令。
- 切换画面。
- 写数据库。
- 发送控制指令。

安全分析要追踪：

```text
按钮 -> 脚本 / 变量 -> 通信驱动 -> PLC 地址 -> 现场动作
```

## 4. 用户权限配置

常见角色：

- Viewer：只读查看
- Operator：操作员
- Engineer：工程师
- Administrator：管理员
- Maintenance：维护人员

风险：

- 所有人共用 operator。
- 操作员具有工程师权限。
- 管理员密码长期不变。
- 离职账号未删除。
- 关键操作无二次确认。

## 5. 通信驱动配置

通信驱动连接 SCADA 和 PLC / RTU / DCS。

重点看：

- 目标 IP
- 协议类型
- 端口
- 轮询周期
- 读写权限
- 超时设置
- 重连策略

安全意义：

- 可以还原 SCADA 与控制器的通信矩阵。
- 可以发现不该存在的连接。
- 可以判断哪些变量被频繁读写。

## 6. 报警配置细节

报警配置通常包括：

- 报警变量
- 高高限 / 高限 / 低限 / 低低限
- 报警级别
- 是否需要确认
- 是否可以屏蔽
- 报警声音
- 报警记录位置

安全风险：

- 阈值被调得过高。
- 高级别报警被降级。
- 报警被长期屏蔽。
- 报警记录被删除。

## 7. 趋势与历史配置

趋势配置关注：

- 哪些变量被记录。
- 采样周期。
- 保存周期。
- 存储路径。
- 是否压缩。
- 是否同步到历史数据库。

如果关键变量没有趋势记录，事故后很难追溯。

## 8. 脚本安全

组态软件可能支持 VBScript、JavaScript、C 脚本或厂商脚本。

关注：

- 是否调用系统命令。
- 是否读写文件。
- 是否访问数据库。
- 是否拼接 SQL。
- 是否包含硬编码账号密码。
- 是否能被普通用户修改。

## 9. 面试可讲版本

> 我分析 SCADA/HMI 时不会只看界面，而会追踪画面元素背后的变量、脚本、权限和通信驱动。一个启动按钮可能对应写 PLC 变量或执行脚本。评估时我会检查变量读写权限、用户角色、报警屏蔽、趋势记录和通信驱动配置，判断它是否可能影响现场操作和数据可信度。
