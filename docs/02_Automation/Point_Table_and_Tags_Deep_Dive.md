# 点表、变量与工程资料深挖

点表是把“协议地址”翻译成“工业含义”的关键资料。没有点表，看到 `40001` 只是一个寄存器；有了点表，才知道它可能是液位、温度、阀门开度或启停命令。

## 1. 点表通常包含什么

常见字段：

| 字段 | 含义 |
|---|---|
| Tag Name | 变量名 / 点名 |
| Description | 中文描述 |
| Type | DI、DO、AI、AO、内部变量 |
| Address | PLC 地址或协议地址 |
| Data Type | Bool、Int、Float、Double、String |
| Scale | 工程量转换比例 |
| Unit | 单位 |
| Range | 量程 |
| Alarm | 报警阈值 |
| Access | 只读 / 可写 |
| Source | 数据来源设备 |
| Destination | 使用该变量的画面或逻辑 |

## 2. 点名命名规律

常见点名可能包含：

```text
设备区域_设备编号_变量类型
```

示例：

```text
PUMP01_RUN
PUMP01_FAULT
TANK01_LEVEL
VALVE02_OPEN_CMD
REACTOR_TEMP_PV
REACTOR_TEMP_SP
```

常见后缀：

| 后缀 | 含义 |
|---|---|
| `_PV` | 过程值 |
| `_SP` | 设定值 |
| `_CMD` | 命令 |
| `_FB` | 反馈 |
| `_RUN` | 运行状态 |
| `_STOP` | 停止命令 |
| `_FAULT` | 故障 |
| `_ALM` | 报警 |
| `_HH` | 高高限 |
| `_LL` | 低低限 |

## 3. 只读点和可写点

安全分析里要特别区分：

### 只读点

通常是状态或测量值：

- 温度 PV
- 压力 PV
- 电机运行反馈
- 阀门开到位反馈

风险：

- 被读取可能泄露生产状态。
- 被伪造可能误导上位系统。

### 可写点

通常是命令或设定值：

- 电机启动命令
- 阀门开度设定
- PID 设定值
- 模式切换
- 报警屏蔽

风险：

- 可直接影响设备或控制目标。

## 4. 地址映射示例

一个简化点表：

| Tag | Type | Address | Description | Access |
|---|---|---|---|---|
| TANK01_LEVEL_PV | AI | 40001 | 1 号水箱液位 | Read |
| TANK01_LEVEL_HH | AI | 40002 | 液位高高限 | Read/Write |
| PUMP01_START_CMD | DO | 00001 | 1 号泵启动命令 | Write |
| PUMP01_RUN_FB | DI | 10001 | 1 号泵运行反馈 | Read |
| VALVE01_OPEN_SP | AO | 40010 | 1 号阀开度设定 | Read/Write |

如果漏洞允许写 `40010`，风险不是“写寄存器”，而是“可能改变阀门开度设定”。

## 5. 点表与 HMI 画面

HMI 画面上的每个数字、按钮、颜色变化，背后通常都绑定变量。

例如：

- 水箱液位数字绑定 `TANK01_LEVEL_PV`
- 泵启动按钮写 `PUMP01_START_CMD`
- 泵图标绿色状态绑定 `PUMP01_RUN_FB`
- 报警闪烁绑定 `TANK01_LEVEL_HH_ALM`

安全分析时要问：

- 哪些画面元素能写变量？
- 写变量是否需要登录？
- 是否有二次确认？
- HMI 脚本是否会执行多个联动操作？

## 6. 点表与协议流量

抓包看到：

```text
Function Code: Write Single Register
Address: 40010
Value: 80
```

结合点表才能解释：

```text
该操作将 VALVE01_OPEN_SP 从原值改为 80%，可能导致阀门开度变化。
```

这就是协议分析和自动化知识结合的价值。

## 7. 点表泄露的风险

点表泄露后，攻击者可以：

- 理解设备命名和工艺流程。
- 区分关键点位与普通点位。
- 定位可写变量。
- 构造更精准的攻击。
- 降低协议逆向难度。

报告表达：

> 点表属于敏感工程资料，一旦泄露，攻击者可将协议地址映射到真实设备和工艺变量，显著提高针对性攻击能力。

## 8. 面试可讲版本

> 点表是我理解工控系统的重要资料。协议里看到的地址本身没有工业含义，必须结合点表才能知道它对应液位、压力、泵启动还是阀门开度。安全分析时，我会优先区分只读点和可写点，再判断可写点是否涉及设备启停、设定值、报警屏蔽或模式切换。
