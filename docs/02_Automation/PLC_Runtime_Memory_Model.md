# PLC Runtime、扫描周期与内存模型深挖

## 1. 原理解释：PLC 为什么采用扫描周期

PLC 不像普通上位机程序那样主要依赖事件驱动，而是采用周期扫描。

典型流程：

```text
输入采样
  -> 冻结输入映像区
  -> 执行用户程序
  -> 修改输出映像区
  -> 输出刷新
  -> 通信 / 诊断
  -> 下一轮扫描
```

这样设计的核心目标是确定性。

## 2. 为什么冻结输入

如果没有输入映像区，程序可能出现：

```text
程序前半段看到 I0.0 = 1
程序后半段看到 I0.0 = 0
```

同一扫描周期内逻辑会不一致。

输入映像区相当于在扫描开始时给现场世界“拍照”。

## 3. 为什么输出统一刷新

程序执行过程中可能多次写同一个输出。

如果每次写都立即作用到现场，继电器、阀门、电机可能在一次扫描中抖动。

输出映像区保证现场输出在扫描末尾统一刷新。

## 4. 工程实现：梯形图背后的执行模型

梯形图是工程师友好的表达，不是 CPU 真正执行的形态。

梯形图：

```text
I0.0 ---- I0.1 ---- ( Q0.0 )
```

等价于：

```c
Q0_0 = I0_0 && I0_1;
```

再底层可以理解为：

```text
LOAD I0.0
LOAD I0.1
AND
STORE Q0.0
```

## 5. PLC 内存区域

| 区域 | 本质 | 生命周期 / 用途 |
|---|---|---|
| I 区 | 输入过程映像 | 一次扫描的现场快照 |
| Q 区 | 输出过程映像 | 一次扫描后统一输出 |
| M 区 | 共享逻辑缓存 | 中间状态、运行标志 |
| DB 区 | 业务状态 / 对象实例 | PID、设定值、配方、模式 |
| L 区 | 局部变量 | 块执行期间临时存在 |
| T/C | 定时器 / 计数器 | 时间和次数状态 |
| Retain | 掉电保持数据 | 重启后仍保留 |

## 6. DB 为什么重要

DB 区通常保存更有业务含义的数据：

- PID 积分
- 上次误差
- SP 设定值
- 配方
- 模式
- 报警阈值
- 设备状态

HMI 修改 SP 或报警限，本质上经常是在写 DB。

## 7. OB、FB、FC 的本质

| 类型 | 含义 | 安全关注 |
|---|---|---|
| OB | Runtime 调度任务 | 改 OB 可能改变任务入口 |
| OB1 | 循环任务 | 主逻辑 |
| OB35 | 周期中断任务 | PID 等固定周期任务 |
| OB100 | 启动初始化 | 可用于启动持久化 |
| FC | 无状态函数 | 执行后局部变量释放 |
| FB | 有状态功能块 | 需要实例 DB 保存状态 |

PID 不宜随便放在周期不稳定的 OB1 中，因为积分计算依赖固定 `Δt`。

## 8. 数据流：HMI 修改设定值

```text
操作员在 HMI 输入 SP
  -> HMI 写通信驱动
  -> 工业协议写 PLC 变量
  -> DB 中 SP 被修改
  -> PID 下个周期读取新 SP
  -> 输出变化
  -> 阀门 / 变频器动作
  -> 物理过程变化
```

## 9. 安全视角

攻击点：

- 修改 I 区或输入映像：伪造现场输入。
- 修改 Q 区或输出：影响现场动作。
- 修改 M 区：改变逻辑状态。
- 修改 DB：改变设定值、配方、PID 状态。
- 修改 Retain DB：造成重启后仍存在的持久化影响。
- 修改 OB100：在启动时执行恶意初始化。
- 修改 OB35：影响周期控制任务。

## 10. 工业案例：Retain DB 持久化影响

如果攻击者把某个 Retain DB 中的 PID 参数或报警阈值改掉，即使 PLC 重启，参数仍可能保留。

这类攻击比单次写寄存器更危险，因为它具有持久性。

## 11. 面试表达

### 60 分答案

> PLC 通过扫描周期读取输入、执行程序、刷新输出，保证控制逻辑确定性。它有 I、Q、M、DB、定时器、计数器等内存区域，不同区域保存不同生命周期的数据。

### 90 分答案

> 我理解 PLC 的安全风险要结合 Runtime 和内存模型看。I/Q 区连接现场输入输出，M 区保存中间逻辑状态，DB 区保存设定值、配方、PID 状态等业务数据。攻击 DB 或 Retain DB 可能比攻击普通输出更隐蔽，因为它可能长期改变控制行为。OB35、OB100 这类任务也很关键，前者影响周期控制，后者可能影响启动初始化。

### 常见追问

问：为什么 PID 更适合固定周期任务？

答：

> 因为积分项与时间步长有关，如果执行周期不稳定，积分计算会不准确，可能导致控制效果变差甚至振荡。
