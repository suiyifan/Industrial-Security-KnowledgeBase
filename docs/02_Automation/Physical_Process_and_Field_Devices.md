# 工业现场与物理过程

## 1. 为什么要理解物理过程

工控安全的特殊性在于：网络攻击可能影响物理世界。

例如：

- 改一个温度设定值，可能导致产品质量异常。
- 让泵频繁启停，可能导致设备损坏。
- 篡改液位数据，可能导致溢流或空转。
- 屏蔽报警，可能导致操作员错过危险状态。

所以学习工控安全，必须知道现场设备和物理量代表什么。

## 2. 常见过程变量

过程变量是工业现场被测量或控制的参数。

| 变量 | 英文 | 例子 | 安全影响 |
|---|---|---|---|
| 温度 | Temperature | 炉温、反应釜温度 | 超温可能引发质量问题或危险 |
| 压力 | Pressure | 管道压力、罐体压力 | 超压可能导致泄漏、爆裂 |
| 液位 | Level | 水箱、储罐液位 | 过高溢流，过低空转 |
| 流量 | Flow | 水、气、油流量 | 影响配比、冷却、输送 |
| 转速 | Speed | 电机、风机转速 | 影响设备负载和工艺稳定 |
| 位置 | Position | 阀门开度、机械臂位置 | 错位可能导致碰撞或误动作 |
| 电压/电流 | Voltage/Current | 电机、电网参数 | 异常可能导致停机或损坏 |

## 3. 传感器

传感器负责把物理世界的数据转换成控制系统能读取的信号。

常见传感器：

- 温度传感器
- 压力变送器
- 液位计
- 流量计
- 接近开关
- 编码器
- 电流互感器

安全关注：

- 数据是否可信。
- 是否可能被干扰。
- 是否存在校准错误。
- 是否可能被网络侧数据伪造替代。

## 4. 执行器

执行器负责把控制命令转化成物理动作。

常见执行器：

- 电机
- 水泵
- 阀门
- 变频器
- 继电器
- 气缸 / 液压缸
- 加热器

安全关注：

- 是否能被未授权启停。
- 是否能被异常频繁控制。
- 是否能被改到危险状态。
- 是否有手动、自动、远程、本地模式切换。

## 5. 数字量与模拟量

### 5.1 数字量

数字量只有有限状态，常见是 0 / 1。

例子：

- 开 / 关
- 启动 / 停止
- 到位 / 未到位
- 报警 / 正常

### 5.2 模拟量

模拟量是连续变化的数值。

例子：

- 温度 86.5 摄氏度
- 压力 1.2 MPa
- 液位 65%
- 流量 120 m3/h

安全分析时要区分：写一个数字量可能是启停设备；写一个模拟量可能是修改设定值。

## 6. I/O 点

I/O 是 Input / Output，表示输入输出点。

常见分类：

- DI：Digital Input，数字量输入
- DO：Digital Output，数字量输出
- AI：Analog Input，模拟量输入
- AO：Analog Output，模拟量输出

例子：

```text
DI：急停按钮状态
DO：电机启动命令
AI：压力传感器数值
AO：阀门开度给定
```

## 7. 点表

点表是工业系统里非常关键的资料，它记录变量名称、地址、类型、含义、量程、单位等。

示例：

| 点名 | 类型 | 地址 | 含义 | 单位 |
|---|---|---|---|---|
| TANK_LEVEL | AI | 40001 | 水箱液位 | % |
| PUMP_START | DO | 00001 | 水泵启动 | - |
| VALVE_OPEN | AO | 40010 | 阀门开度 | % |

安全意义：

- 有点表，攻击者更容易理解变量含义。
- 没有点表，协议分析难度更高。
- 报告中分析风险时，点表能帮助说明“这个地址到底控制什么”。

## 8. 面试表达

可以这样说：

> 我理解工业现场由传感器、执行器和被控对象组成。安全分析时不能把寄存器或变量当成普通数据，因为某些变量可能对应电机启停、阀门开度、压力设定值或报警状态。判断漏洞影响时，需要结合点表和工艺过程理解真实后果。
