# 物理过程、传感器与执行器

工控安全最终保护的不是 IP 地址，而是承载物料和能量的工业过程。传感器把物理世界投影为控制器能够处理的数据，执行器再把控制器输出转化为动作。只看网络侧变量而不了解测量原理、执行器能力和过程惯性，很容易高估攻击效果、漏掉危险后果，或把普通仪表故障误判为网络攻击。

## 1. 原理解释：过程变量为什么是安全分析起点

过程变量描述生产系统当前状态，常见类别包括：

| 变量 | 常见单位 | 典型危险 | 常见测量方式 |
| --- | --- | --- | --- |
| 温度 T | ℃、K | 过热、反应失控、材料性能下降 | 热电偶、热电阻、红外 |
| 压力 P | Pa、kPa、MPa、bar | 超压、真空失稳、泄漏 | 压阻、电容、差压 |
| 流量 F | m³/h、kg/h、L/min | 断流、过量投加、配比错误 | 差压、电磁、涡街、质量流量计 |
| 液位 L | mm、m、% | 溢流、抽空、泵汽蚀 | 差压、雷达、超声、浮子 |
| 速度/转速 S | m/s、r/min | 超速、堵转、机械损伤 | 编码器、测速传感器 |
| 位置/位移 Z | mm、°、% | 越限、碰撞、阀位错误 | 限位开关、编码器、位置反馈 |
| 成分/质量 A | pH、浓度、电导率、含氧量 | 产品不合格、腐蚀、环境超标 | 电化学、光学、分析仪表 |

变量不是孤立数值。安全判断还需要知道正常范围、报警限、联锁限、变化速率、测量精度、刷新周期、滞后和单位。例如压力从 1.0 升至 1.2 MPa 是否危险，取决于容器设计压力、升高速率、泄放能力和联锁设定，而不是单看 20% 的变化。

## 2. 工程实现：从传感元件到控制器变量

完整测量链通常包含：

```text
被测过程
  ↓
一次元件 / 取压取样装置
  ↓
传感元件
  ↓
变送器：放大、补偿、线性化
  ↓
4–20 mA / 0–10 V / 脉冲 / 数字总线
  ↓
隔离器、安全栅、端子、I/O 模块
  ↓
原始码 → 量程转换 → 工程量 → 质量码
  ↓
控制、报警、趋势、历史与上层应用
```

### 2.1 一次元件与取样条件

测量准确性不仅取决于变送器。差压流量测量依赖孔板和导压管；pH 分析依赖取样和探头维护；温度套管安装深度影响响应速度；雷达液位可能受泡沫和罐内构件影响。网络数据正常并不能排除一次元件堵塞、取样管冻结或安装方向错误。

### 2.2 变送器

变送器把传感信号转换为标准输出，并进行零点、量程、阻尼、线性化和温度补偿。智能变送器还可能支持远程参数化和诊断。改变量程上限不会立即制造异常通信，却会让相同物理输入对应不同工程量，是典型的语义层攻击面。

### 2.3 信号调理与隔离

隔离器用于切断地环路并抑制干扰；安全栅限制进入危险区域的能量；浪涌保护器抵御瞬态过压。错误接线、供电故障或隔离器量程不一致，可能使多个上层系统同时获得错误数据。

## 3. 传感器的关键特性

- **量程**：可测范围，超量程后可能饱和或损坏；
- **精度**：测量结果接近真实值的程度；
- **重复性**：相同条件下多次测量的一致程度；
- **分辨率**：能够识别的最小变化；
- **响应时间**：输入变化到输出达到规定比例所需时间；
- **漂移**：相同输入下输出随时间缓慢变化；
- **迟滞**：上升与下降方向到达同一输入时输出不同；
- **阻尼**：平滑波动但同时引入延迟；
- **失效输出**：故障时输出低端、高端、保持或特定诊断值。

攻击检测需要把这些物理特性纳入模型。完全没有噪声、无延迟、长期精确保持的过程值，反而可能是冻结、回放或模拟数据。

## 4. 执行器如何改变物理过程

### 4.1 调节阀

调节阀由阀体、执行机构、定位器和位置反馈组成。控制器输出通常表示目标开度，但目标开度不等于真实阀位，真实阀位也不等于实际流量：阀门可能卡涩、气源不足、机械脱落，管路压差也会影响流量。

阀门常见失效位置：

- **FC，故障关**：失气或失电时关闭，常用于切断危险进料；
- **FO，故障开**：故障时打开，可能用于冷却水或泄放；
- **FL，故障保持**：保持最后位置，适用于某些连续过程。

“安全位置”由过程危害分析决定，不存在所有场景通用答案。冷却阀通常倾向故障开，但在可能导致热冲击或污染的过程里仍需综合判断。

### 4.2 开关阀与电磁阀

开关阀通常只有开/关命令，并通过开到位、关到位限位反馈状态。命令与两个反馈组合可以诊断：命令开但关到位仍在，可能是动作失败；开到位和关到位同时出现，可能是接线、反馈或逻辑异常。

### 4.3 泵、电机与风机

电机通常通过接触器、软启动器或变频器驱动。PLC 输出启动命令，MCC 或 VFD 执行动作，再返回运行、故障、就地/远程和速度反馈。设备启动还受液位、阀位、润滑、冷却和下游许可限制。

### 4.4 加热器、断路器与机械机构

加热器改变热量输入，断路器改变电能通路，气缸和伺服机构改变机械位置。这些对象常具有较高能量和快速动作能力，安全分析必须关注命令持续时间、互锁、机械极限和失效后的剩余能量。

## 5. 数据流、控制流、状态流与能量流

以离心泵为例：

```text
数据流：吸入口液位、出口压力、运行反馈、故障码 → PLC/HMI
控制流：自动模式 + 启动请求 + 许可条件 → 启动输出
状态流：Stopped → Starting → Running → Stopping/Faulted
能量流：电源 → 电机转矩 → 叶轮 → 流体压力与流量
```

远程写入启动位只改变控制请求。真正启动还可能依赖远程模式、液位许可、阀门到位、MCC 就绪和故障复位。反过来，若攻击者直接强制物理输出或修改许可逻辑，就可能绕过状态机设计。

## 6. 常见故障模式与证据

### 6.1 传感器故障

- **开路/短路**：模拟量进入低端或高端诊断区；
- **卡死**：数值长时间不变，但相关变量继续变化；
- **漂移**：缓慢偏离，可与冗余测点、物料平衡或校验记录比较；
- **噪声**：高频波动，可能来自接地、电磁干扰或机械振动；
- **量程错误**：变化趋势看似合理，但绝对值比例错误；
- **方向反转**：过程上升时显示下降，常见于差压方向、接线或缩放错误。

### 6.2 执行器故障

- 命令变化但位置反馈不变；
- 反馈变化但过程变量没有预期响应；
- 阀门卡涩产生锯齿或死区；
- 电机收到命令但保护跳闸；
- VFD 频率指令与实际频率、转速不一致；
- 就地模式使远程命令无效。

### 6.3 共同原因故障

多个测点同时异常，不一定是协同攻击。它们可能共享供电、I/O 模块、接地、通信网关、空气源或取样系统。资产关系图应记录这些共因依赖。

## 7. 安全视角：可被操纵的工程语义

本节讨论的安全风险不是“仪表联网所以可能被攻击”这种泛化结论，而是测量和执行语义被改变后，正常控制与人员决策如何偏离真实过程。

攻击面不仅是传感器通信接口，还包括：

- 变送器零点、量程、阻尼、故障方向和设备地址；
- I/O 通道类型、滤波、缩放、线性化和诊断使能；
- PLC 中的工程量换算、替代值、坏质量处理和仿真开关；
- HMI 标签绑定、单位、小数位、报警限和显示颜色；
- 执行器行程、正反作用、速度限制、就地/远程和失效位置；
- 维护终端、手持配置器、资产管理系统和校验记录。

### 7.1 反馈欺骗

伪造 PV 可诱使正常控制算法产生危险 MV。例如实际液位上升但反馈保持偏低，控制器会继续进料。隐蔽操纵往往同时考虑过程模型，使伪造值维持“合理变化”，因此需要物料平衡、冗余测点和执行响应进行交叉验证。

### 7.2 执行欺骗

HMI 和 PLC 显示阀门已关闭，但现场阀门因定位器参数、机械故障或反馈伪造仍开启。单独验证输出位不能证明物理隔离已经建立。

### 7.3 诊断抑制

把坏质量替换为最后有效值、关闭断线诊断或延长滤波时间，能让异常更晚暴露。可用性设计中的“平滑显示”如果没有明显质量标识，会削弱操作员态势感知。

典型攻击链可表示为：取得仪表或资产管理权限 → 修改量程、阻尼或故障方向 → PLC 接收到格式正常但语义错误的 PV → 控制算法与 HMI 基于错误值动作 → 执行器改变物料或能量 → 独立测量或保护决定最终后果。每一步都需要独立证据。

## 8. 工控场景：储罐液位测量被缩放

某储罐真实量程为 0—10 m，PLC 原始输入被错误配置为 0—20 m。真实液位 8 m 被显示为 16 m，可能造成误停车；若反向错误使 8 m 显示为 4 m，则控制器可能继续进料并逼近溢流。

完整分析应核对：

1. 变送器量程和输出电流；
2. I/O 模块原始码和诊断状态；
3. PLC 缩放上下限与单位；
4. HMI 标签缩放和报警限；
5. 就地液位计、物料进出流量和历史趋势；
6. 高高液位独立开关或 SIS 是否仍有效。

风险不在“参数被修改”这一抽象事实，而在错误测量如何驱动控制与操作决策，以及独立保护是否能阻止后果。

## 9. 防护与工程治理

- 对仪表参数、I/O 组态和执行器配置建立基线与变更审批；
- 关键测点使用不同原理或独立通道进行合理性校验；
- 在 HMI 明确展示坏质量、替代值、仿真、旁路和维护状态；
- 对命令、反馈和过程响应做一致性监测，而非只监测单点越限；
- 保护现场配置接口和资产管理系统，限制远程写参数；
- 定期执行回路检查、校准、阀门行程测试和保护功能验证；
- 恢复时同时验证数字参数、现场接线、仪表输出和执行器实际位置。

## 10. 面试表达

### 60 分答案

> 传感器和变送器把温度、压力、流量、液位等物理量转换成标准信号，经 I/O 进入控制器；控制器再通过阀门、电机等执行器改变过程。安全分析要关注量程、断线、漂移、卡死和执行器反馈。

### 90 分答案

> 我会把现场设备放进完整测量链和执行链分析。过程值通常要经过一次元件、变送器、隔离与接线、I/O 原始码、PLC 缩放、通信标签和 HMI 展示，多次语义转换后才成为操作员看到的数值。控制输出也要经过逻辑、I/O、定位器或驱动器才形成真实动作。因此我不会用单一 PLC 值证明现场状态，而会交叉检查质量码、原始码、就地表计、位置反馈、相关过程变量和物料能量平衡，再判断是故障、配置错误还是恶意操纵。

### 常见追问

**问：为什么 4 mA 而不是 0 mA 表示量程下限？**

答：活零点可以给两线制仪表供电，并帮助区分正常零值与断线；但具体诊断阈值和故障方向仍取决于设备及组态。

**问：阀门显示 50% 就代表流量是 50% 吗？**

答：不代表。显示值可能是控制指令、定位器反馈或估算位置；阀门流量还受阀特性、压差、介质和机械状态影响。

**问：怎样识别传感器欺骗？**

答：使用独立测点、物料能量平衡、命令—反馈—响应关系、变化速率、质量码和现场检查交叉验证，不能只设一个静态阈值。

## 11. 实操建议

### 实验一：模拟量缩放

在离线脚本或 PLC 仿真器中把 4—20 mA 映射为 0—10 m，分别输入 3.6、4、12、20、21 mA，记录工程量和诊断状态。再故意改变上限，观察 HMI 数值与报警如何变化。

### 实验二：命令—反馈—响应矩阵

为一台泵或阀门建立表格：命令、位置/运行反馈、过程响应、合理结论。至少覆盖拒动、反馈错误、就地模式和过程堵塞四种异常。

### 实验三：多源验证

在水箱仿真中冻结液位 PV，同时保留进出流量变化。用物料平衡判断液位趋势，比较 PLC 值、计算值和历史趋势，记录发现异常所需时间。

## 12. 总结与检查清单

- [ ] 能解释测量链中每次信号与语义转换；
- [ ] 能区分控制指令、执行器反馈和真实过程响应；
- [ ] 能列出传感器与执行器的主要故障模式及证据；
- [ ] 能说明失效位置为何必须由具体过程危害决定；
- [ ] 能用至少三个独立证据判断故障、误配置或恶意操纵；
- [ ] 能把现场参数变更映射到控制动作、保护响应和物理后果。
