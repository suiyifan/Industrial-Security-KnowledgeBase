# 控制逻辑阅读方法

安全人员不一定要像自动化工程师一样写完整 PLC 程序，但要能读懂关键逻辑，尤其是启停、联锁、报警、模式和通信相关逻辑。

## 1. 先找设备控制逻辑

设备控制逻辑通常围绕一个设备展开。

例如泵：

- 启动命令
- 停止命令
- 运行反馈
- 故障反馈
- 自动 / 手动模式
- 就地 / 远程模式
- 联锁条件
- 启动允许
- 停止条件

阅读时先问：

```text
什么条件下允许启动？
什么条件下必须停止？
哪些信号只是显示，哪些信号会真的动作？
```

## 2. 典型泵控制逻辑

简化逻辑：

```text
启动允许 = 无故障 AND 液位不低 AND 远程模式 AND 未急停

如果 自动模式 AND 液位低于启动线 AND 启动允许:
    启动泵

如果 液位高于停止线 OR 故障 OR 急停:
    停止泵
```

安全分析：

- 如果液位 PV 被伪造，自动启停可能错误。
- 如果启动允许被强制为真，可能绕过部分保护。
- 如果停止条件被篡改，可能导致泵无法及时停止。

## 3. 联锁逻辑怎么看

联锁常见形式：

```text
IF pressure > high_high_limit THEN close_inlet_valve
IF motor_fault THEN stop_pump
IF emergency_stop THEN stop_all
```

关注点：

- 联锁触发条件是什么？
- 联锁动作是什么？
- 联锁是否能被旁路？
- 旁路是否有权限和记录？
- 联锁是否依赖网络数据？

如果联锁依赖来自网络的变量，就要关注数据可信度。

## 4. 报警逻辑怎么看

报警一般不会直接控制设备，但会影响人的判断。

常见逻辑：

```text
IF tank_level > high_limit THEN alarm_high = true
IF tank_level > high_high_limit THEN alarm_high_high = true
```

关注：

- 报警阈值是否可改。
- 报警是否可屏蔽。
- 报警确认是否记录。
- 报警是否分级。

攻击者如果屏蔽报警，可能让操作员错过异常。

## 5. 模式切换怎么看

常见模式：

- 手动 / 自动
- 本地 / 远程
- 维护 / 运行
- 调试 / 生产

安全关注：

- 模式切换是否需要权限。
- 模式切换是否有记录。
- 调试模式是否降低保护。
- 远程模式是否允许上位机控制。

## 6. 通信逻辑怎么看

通信逻辑可能处理：

- 从上位机读取命令。
- 向上位机发送状态。
- 与其他 PLC 交换数据。
- 与变频器、仪表、网关通信。

关注：

- 外部写入的数据是否校验。
- 通信中断时进入什么状态。
- 是否有超时处理。
- 是否有默认安全状态。

## 7. 常见危险写点

优先关注这些变量：

- 启动命令
- 停止命令
- 设定值
- 模式切换
- 报警屏蔽
- 联锁旁路
- PID 参数
- 复位命令
- 下载 / 维护标志

## 8. 从安全报告角度描述逻辑风险

模板：

```text
该控制逻辑中，【变量】用于【工业动作】。
当前系统允许【访问主体】通过【路径】修改该变量。
若被未授权修改，可能导致【设备动作】或【工艺后果】。
建议增加权限控制、操作确认、日志审计，并限制写入路径。
```

## 9. 面试可讲版本

> 我阅读控制逻辑时会先找设备启停、联锁、报警、模式切换和通信处理。比如泵控制逻辑里，我会关注启动允许、停止条件、液位反馈、故障反馈和远程模式。如果漏洞能修改启动命令、液位值或联锁旁路变量，就可能影响泵的真实运行状态。
