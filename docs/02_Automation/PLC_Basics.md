# PLC 系统

## 1. PLC 是什么

PLC 是 Programmable Logic Controller，可编程逻辑控制器。

它是工业现场常见的控制器，负责读取输入、执行控制逻辑、输出控制命令。

白话理解：

> PLC 就像工业现场的专用控制大脑。它不负责漂亮界面，主要负责稳定、实时、可靠地控制设备。

## 2. PLC 的典型工作流程

PLC 通常按扫描周期工作：

1. 读取输入。
2. 执行用户程序。
3. 更新输出。
4. 通信和诊断。
5. 进入下一轮扫描。

这叫扫描周期。

## 3. 为什么扫描周期重要

扫描周期决定控制逻辑响应速度。

安全分析中要理解：

- 如果通信异常导致变量延迟，可能影响控制。
- 如果程序被修改，下一轮扫描就可能执行恶意逻辑。
- 如果 PLC 负载异常，扫描周期可能变长，影响实时性。

## 4. PLC 程序语言

IEC 61131-3 定义了常见 PLC 编程语言：

| 语言 | 特点 |
|---|---|
| LD 梯形图 | 像电气继电器图，现场常见 |
| FBD 功能块图 | 用功能块连接表达逻辑 |
| ST 结构化文本 | 类似高级语言 |
| SFC 顺序功能图 | 表示流程步骤 |
| IL 指令表 | 类似低级指令，较少新用 |

安全人员不必一开始精通编程，但要能看懂基本逻辑：输入条件、输出动作、定时器、计数器、联锁条件。

## 5. PLC 常见组成

- CPU 模块
- 电源模块
- DI / DO 模块
- AI / AO 模块
- 通信模块
- 背板 / 总线
- 存储卡

## 6. PLC 变量和地址

PLC 中常见数据对象：

- 输入点
- 输出点
- 内部继电器
- 数据寄存器
- 定时器
- 计数器
- 数据块

不同厂商命名不同。

安全分析时，协议地址和 PLC 变量之间的映射非常关键。攻击者如果知道某个地址对应“启动水泵”，写入风险会显著提高。

## 7. 工程下载

工程站会把控制程序下载到 PLC。

常见操作：

- 上传程序：从 PLC 读取工程。
- 下载程序：把工程写入 PLC。
- 在线监控：实时查看变量。
- 强制变量：临时强制输入或输出状态。
- 修改参数：改变设定值或控制参数。

安全关注：

- 下载操作是否需要认证。
- 工程文件是否有保护。
- 是否允许远程修改逻辑。
- 是否存在默认口令或弱口令。
- 是否记录工程变更日志。

## 8. Force 强制变量

强制变量是非常敏感的功能。

例子：

```text
强制 PUMP_START = 1
```

可能导致水泵启动，即使正常逻辑不满足启动条件。

安全分析要重点关注：

- 谁可以强制变量？
- 强制状态是否有明显提示？
- 是否会被联锁覆盖？
- 是否会保留到重启后？

## 9. PLC 通信

常见 PLC 通信协议：

- Modbus TCP
- Siemens S7comm
- EtherNet/IP
- Profinet
- OPC UA
- 厂商私有协议

PLC 安全风险经常来自通信接口：

- 未授权读写。
- 明文传输。
- 缺少身份认证。
- 工程服务暴露。
- 诊断接口可被滥用。

## 10. PLC 常见安全问题

- 默认口令或弱口令。
- 工程服务暴露在不可信网络。
- 控制逻辑未加密或未签名。
- 允许未授权上传 / 下载程序。
- 协议缺少认证和加密。
- Web 管理界面漏洞。
- 固件漏洞。
- 物理接口暴露。

## 11. PLC 漏洞影响如何描述

不要只写：

```text
攻击者可以写寄存器。
```

要进一步写：

```text
如果该寄存器对应设备启停、阀门开度或控制设定值，攻击者可能影响现场设备状态，造成生产异常、设备损坏或安全风险。
```

## 12. 面试表达

可以这样说：

> PLC 是现场控制的核心设备，通常通过扫描周期读取输入、执行逻辑、更新输出。安全分析时，我会重点关注工程下载、变量强制、通信协议、认证机制和控制逻辑保护。如果漏洞允许未授权读写 PLC 变量，就要进一步结合点表判断它是否会影响设备启停、设定值或联锁条件。
