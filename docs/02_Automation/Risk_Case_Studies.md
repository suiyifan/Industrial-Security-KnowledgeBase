# 工控风险案例化表达

这一节用于把抽象知识变成“像做过项目”的表达。面试时不要只讲漏洞名，要讲资产、路径、动作、后果、建议。

## 案例 1：HMI 共用账号

### 场景

某水处理站 HMI 使用统一操作员账号，多个班组共用，密码长期不变。

### 风险链路

```text
共用账号泄露
-> 登录 HMI
-> 查看泵站画面和变量状态
-> 下发泵启动 / 停止命令
-> 影响液位和供水压力
```

### 工控影响

- 无法追踪具体操作人。
- 异常启停可能影响供水稳定。
- 攻击者可利用合法界面执行操作。

### 整改建议

- 账号实名化。
- 操作员、工程师、管理员分权。
- 关键操作二次确认。
- 记录操作日志。

## 案例 2：PLC 未授权读写

### 场景

PLC 的 Modbus TCP 服务暴露在控制网络中，未做访问控制。

### 风险链路

```text
进入控制网络
-> 发现 TCP 502
-> 读取寄存器
-> 结合点表识别关键变量
-> 写设定值或控制命令
```

### 工控影响

- 读取过程数据导致生产状态泄露。
- 写可控变量可能改变设备状态。
- 如果写入设定值，可能改变控制目标。

### 整改建议

- 限制访问源。
- 工业防火墙白名单。
- 监测功能码写操作。
- 对关键变量写入增加上位权限控制。

## 案例 3：工程站感染恶意软件

### 场景

工程站长期连接办公网，并允许 U 盘拷贝工程文件。

### 风险链路

```text
办公网恶意文件
-> 工程站感染
-> 工程文件被窃取
-> 控制逻辑和点表泄露
-> 攻击者构造针对性修改
```

### 工控影响

- 工程资料泄露。
- 控制逻辑可能被篡改。
- 攻击者更容易理解生产流程。

### 整改建议

- 工程站最小化联网。
- U 盘管控。
- 工程文件加密和备份。
- 工程变更审计。

## 案例 4：报警长期屏蔽

### 场景

某 HMI 画面中多个高优先级报警长期处于屏蔽状态。

### 风险链路

```text
报警屏蔽
-> 异常过程状态不提示
-> 操作员延迟响应
-> 工艺参数继续偏离
```

### 工控影响

- 异常不能及时发现。
- 事故追溯困难。
- 攻击者可能利用屏蔽状态掩盖动作。

### 整改建议

- 报警屏蔽审批。
- 屏蔽状态明显展示。
- 定期复核长期屏蔽报警。
- 记录屏蔽、恢复和确认日志。

## 案例 5：远程运维入口暴露

### 场景

厂商通过 VPN 和远程桌面维护工程站，账号长期有效。

### 风险链路

```text
远程账号泄露
-> VPN 接入
-> RDP 登录工程站
-> 上传 / 下载工程
-> 修改控制逻辑
```

### 工控影响

- 外部攻击者获得控制网络入口。
- 工程站成为跳板。
- 控制逻辑完整性受影响。

### 整改建议

- 远程访问按需开通。
- MFA。
- 堡垒机审计。
- 维护窗口审批。
- 维护后账号回收。

## 面试总结表达

> 我会把工控风险按“资产角色、攻击路径、工业动作、生产影响、整改建议”来表达。比如 PLC 未授权读写不是简单的端口问题，而是要结合点表判断是否能写设定值、启停命令或联锁旁路变量；HMI 共用账号也不是普通弱口令问题，因为它可能直接关联设备操作和报警确认。
