# 工控风险案例化表达

案例的作用不是制造惊险故事，而是练习从事实到后果的证据链。以下场景均为教学模型，真实项目需根据资产、逻辑、工况和保护层重新验证。

## 2. 工程站共享管理员账号

### 已知事实

多名工程师与承包商共用本地管理员，工程软件保存控制器连接，缺少个人操作审计。

### 风险链

凭据泄露或误用 → 登录工程站 → 读取工程与点表 → 在线修改、强制或下载 → 控制状态变化。物理后果取决于控制器权限、目标逻辑、当前工况和保护。

### 证据与保护

检查登录、远程会话、工程软件审计、控制器下载记录和工程差异。网络隔离和控制器钥匙可能限制能力，但共享账号破坏归责。

### 整改

个人账号、最小权限、跳板与会话审计；紧急账号密封管理；工程操作关联工单。复测需确认旧账号回收且工程功能仍可用。

## 3. PLC 外部变量未授权写入

### 已知事实

测试环境证明任意可达主机能写某变量，但尚未确认变量语义。

### 风险链

先通过点表和交叉引用确认变量。若为启动请求，还需模式、许可和握手；若为直接输出或旁路，影响路径更短。写入成功不等于设备动作。

### 证据与保护

工程文件、在线变量、任务时序、输出反馈和过程趋势。控制器模式、程序覆盖、设备就地状态和 SIS 都可能限制后果。

### 整改

限制可达性和可写标签、启用控制器访问保护、命令握手和语义监测。生产不执行主动写入复测，优先隔离设备验证。

## 4. HMI 标签绑定错误

### 已知事实

检修后两个同类阀门的命令和反馈标签被交换。

### 风险链

操作员点击阀 A → 实际阀 B 动作 → 反馈也映射错误 → 画面显示成功 → 物料路径与操作意图不一致。

### 证据与保护

比较 HMI 标签、PLC 地址、点表、现场阀位和流量。若过程联锁检查真实流量或阀组组合，可能阻止危险路径。

### 整改

组态版本控制、标签引用自动检查、端到端点对点测试。复测必须到真实反馈和过程响应层。

## 5. 报警长期搁置

### 已知事实

高压力报警因抖动被搁置，持续时间超出班次且无补偿措施。

### 风险链

仪表抖动 → 报警搁置 → 真实压力升高时操作员无提示 → BPCS 输出饱和 → 独立高高压联锁成为最后保护。

### 证据与保护

搁置日志、用户、原因、压力/SP/MV 趋势和 SIS SOE。联锁有效可限制后果，但仍可能造成停车和生产损失。

### 整改

修复仪表、合理迟滞/延时、搁置限时与交接、关键报警禁止无限期搁置。复测包括洪泛和真实动态测试。

## 6. 远程 I/O 失联保持最后值

### 已知事实

液位 AI 在通信中断后保持最后高值，控制逻辑没有使用质量码。

### 风险链

失联 → PV 冻结 → 低液位许可持续成立 → 泵继续运行 → 真实液位下降 → 汽蚀或干转。

### 证据与保护

连接状态、质量、原始值、泵电流、入口压力和独立低液位开关。独立保护可停车。

### 整改

质量参与控制、超时策略、独立低低液位和命令—反馈—响应监测。复测断链、重连和恢复。

## 7. VFD 自动重启与维护

### 已知事实

驱动器电源恢复后自动重启，维护人员依赖 HMI Stop 而未做能量隔离。

### 风险链

瞬时掉电 → 电机停转 → 人员接近 → 电源恢复和 VFD 自动启动 → 机械伤害风险。

### 证据与保护

VFD 参数、控制源、事件记录、STO/隔离状态和作业票。软件停止不能替代锁定挂牌。

### 整改

维护物理隔离、自动重启状态告警、维护模式和安全功能验证。复测不得让人员暴露于运动区域。

## 8. DCS 主备状态不同步

### 已知事实

主控制器在线修改 PID 模式和参数，备机同步异常未告警。

### 风险链

主机故障 → 备机接管旧参数/积分状态 → MV 突跳 → 过程超调 → 报警或联锁动作。

### 证据与保护

主备版本、同步诊断、切换 SOE、SP/PV/MV、实际阀位。输出跟踪和独立保护可限制冲击。

### 整改

同步状态监测、在线变更后全节点比较、定期受控切换。复测观察完整动态而非只看切换成功。

## 9. SIS 与 BPCS 共享测点

### 已知事实

正常压力控制和高高压 SIF 使用同一变送器与取压管，文档却把 SIS 视为独立保护。

### 风险链

取压堵塞或测量被操纵 → BPCS 与 SIS 同时看到低值 → 控制继续加热且安全动作不触发 → 真实压力上升。

### 证据与保护

P&ID、回路图、SRS、I/O 和现场取压。机械泄放可能仍提供缓解。

### 整改

重新评估独立性和风险降低信用，必要时采用独立取样与传感。证明测试覆盖共同路径。

## 10. 配方版本被替换

### 已知事实

批次系统允许高权限服务账号导入配方，文件缺少签名和双人审批。

### 风险链

配方被替换 → 投料量、温度或保持时间变化 → PLC 按合法步骤执行错误目标 → 产品质量或反应安全受影响。

### 证据与保护

配方版本、导入日志、批次记录、物料称量、实验室数据和过程趋势。参数边界和 SIS 可能限制安全后果，但不一定保护质量。

### 整改

配方签名、版本审批、参数约束、服务账号最小权限和批次追溯。复测比较下发内容与执行记录。

## 11. 历史服务器资源耗尽

### 已知事实

大量报表查询导致历史接口积压、操作员趋势延迟，但 PLC 控制未中断。

### 风险链

资源耗尽 → 趋势与报警历史延迟 → 操作员缺少动态证据 → 异常处置和事故追溯受损 → 可能扩大生产损失。

### 证据与保护

服务器资源、接口队列、数据缺口、画面延迟和控制器实时状态。不能把“PLC 正常”当作无影响。

### 整改

查询隔离、资源限制、接口缓存监测和降级告警。复测验证高负荷下实时操作链不受影响。

## 12. 案例表达原则

- 先陈述可验证事实，再给风险推理；
- 写清攻击前提和当前工况；
- 区分命令、反馈、状态和真实过程；
- 说明报警、设备保护和 SIS 是否有效；
- 给结论可信等级，不制造确定性；
- 整改包含临时措施、长期方案、回退和复测；
- 不在公开材料泄露真实地址、凭据和可利用细节。

## 13. 面试表达

> 我描述工控风险时不会从漏洞直接跳到物理事故，而会说明资产能力、目标变量、模式许可、控制逻辑、执行器和过程响应，再评估报警、设备保护与 SIS。证据不足时明确是理论可能还是工程可信。整改也会考虑生产窗口、补偿措施、回退和过程功能复测。
