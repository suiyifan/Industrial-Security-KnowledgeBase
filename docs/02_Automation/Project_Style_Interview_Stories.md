# 项目式面试表达素材

项目式表达不等于虚构项目。没有真实生产经验时，应明确说明使用公开资料、仿真器或实验设备完成了哪些工作，哪些结论是工程推演，哪些已经实验验证。可信度比夸张经历更重要。

## 1. 表达结构

使用六段式：

```text
背景与边界
 → 我的任务
 → 分析方法
 → 关键证据
 → 结果与限制
 → 复盘与下一步
```

避免只列工具；要说明工具输出怎样改变了判断。

## 2. 30 秒定位表达

> 我关注自动化与网络安全的交叉方向。我的学习不是只背 PLC、DCS 和 SCADA 定义，而是从物理过程、I/O、控制逻辑、操作监控和 SIS 建立完整控制链，再用工程文件、仿真趋势、抓包和日志验证安全判断。对没有现场验证的结论，我会明确标为工程推演，不夸大成真实生产影响。

## 3. 项目故事：PLC 控制链审阅

### 背景

> 我在隔离仿真环境搭建了一个水箱、泵和阀门控制场景，目标是练习从外部写点判断物理影响。

### 方法

> 我先画 SP、PV、MV 和物料流，再建立 I/O 与标签表，编写包含自动/手动、液位许可、反馈超时、联锁和复位的泵状态机。随后从最终输出反向做交叉引用，检查通信请求、模式、许可、锁存和反馈。

### 发现

> 最初逻辑只在启动沿检查低液位许可，设备运行后液位下降不会跳闸。我通过冻结液位反馈和改变真实水箱模型复现了泵继续运行的状态。

### 改进

> 我把低液位改为运行中持续联锁，增加坏质量和入口压力校验，并为通信命令加入请求—确认和序列号。

### 限制

> 这是仿真验证，不能证明某一真实厂商或现场存在同样问题，但方法可以迁移到工程审阅。

## 4. 项目故事：工业流量语义分析

> 我分析一组模拟 PLC/HMI 流量时，没有止于源目 IP 和功能码，而是把地址映射到标签、数据类型、单位和可写属性。对写请求，我继续检查 PLC 读取位置、任务周期、边沿、许可和反馈。结果发现部分写入会被下一扫描覆盖，但一个请求位会触发锁存，因此“值立刻恢复”不等于无影响。我最终把规则设计为同时关联写对象、控制器模式、命令确认和过程响应。

可展示证据：点表、抓包字段、交叉引用图、时序图和仿真趋势。

## 5. 项目故事：DCS 主备与变更审阅

> 我用桌面推演和仿真数据分析 DCS 在线参数变更。先列出主备必须同步的静态配置和 PID 积分、模式、顺控步位等动态状态，再构造主机参数已改、备机未同步的切换场景。通过 SP/PV/MV、阀位和切换 SOE 说明为什么“切换成功”不代表无扰。我形成了变更后节点比较、同步检查和受控切换验证清单。

## 6. 项目故事：SCADA 失联与旧命令

> 我实现了一个简化的选择—执行命令状态机，模拟站点失联、命令超时和重连。初始版本保留未确认命令，重连后会迟到执行。我加入命令有效期、序列号、状态前置条件和人工重新确认，并分别记录源时间、接收时间和反馈。这个实验让我理解 SCADA 安全不仅是加密，还包括失联状态和命令生命周期。

## 7. 项目故事：报警与事件时间线

> 我构造了报警、操作日志、控制器 SOE 和历史趋势四种数据，故意加入时钟偏差和历史压缩。校正后重建命令—输出—PV—联锁顺序，发现只看历史趋势会把补传时间误当事件时间。最终输出包含时间源、精度、校正方法和不确定区间。

## 8. 项目故事：安全评估方法设计

> 我按真实项目方法设计了一次自动化安全评估：先定义允许和禁止动作，再收集 P&ID、拓扑、点表、工程和变更记录；按物理过程建立资产—功能—通信—保护模型；技术验证从离线审阅和被动证据开始，主动写入转移到隔离实验。每个发现都要求写明控制对象、必要前提、保护层和证据等级，并给出维护窗口、回退和过程复测方案。

## 9. 回答“没有真实项目经验怎么办”

> 我目前不会把实验室练习包装成真实现场项目。我的补足方式是把知识做成可检查的工程产物：控制链图、点表、PLC 状态机、抓包解析、趋势分析、变更清单和恢复演练。这样进入项目后，我至少能听懂工艺、仪表和自动化人员的语言，知道哪些动作不能在生产验证，也能把安全发现写成有前提、有保护层、有证据边界的结论。

## 10. 常见追问

**问：你在项目里具体做了什么？**

答：只讲自己实际完成的建模、编码、抓包、审阅、验证和文档，不把团队或资料中的内容说成个人实施。

**问：结果怎样证明？**

答：给工程版本、实验输入、趋势、抓包、日志、差异和复现步骤，同时说明实验与生产的边界。

**问：最大的失误或复盘是什么？**

答：选择真实改进点，例如早期只看变量值、忽略质量和时序，之后补入命令—反馈—过程响应模型。

## 11. 简历项目写法模板

```text
项目：隔离环境下的 PLC 控制链安全分析
- 建立水箱、泵、阀门与液位闭环模型，整理 I/O、标签、模式和联锁；
- 从协议写点追踪至 PLC 任务、状态机和物理输出，验证瞬态写入与锁存差异；
- 注入反馈冻结、通信中断和重启状态不一致，使用趋势与物料平衡定位异常；
- 输出风险链、证据等级、低扰动验证和恢复检查表。
```

## 12. 表达检查清单

- [ ] 明确真实现场、实验验证和工程推演的边界；
- [ ] 有清楚的问题、个人动作、证据和结果；
- [ ] 工具输出映射到控制行为；
- [ ] 说明物理过程和保护层；
- [ ] 承认限制并给出下一步；
- [ ] 不泄露真实凭据和敏感现场细节。
