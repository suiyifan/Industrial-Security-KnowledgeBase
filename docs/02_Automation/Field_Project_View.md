# 现场项目视角：一次工控安全评估怎么展开

这一节的目标是让你具备“项目现场感”。即使你还没真正进过现场，也要知道专业人员通常怎么拆解工控安全项目。

## 1. 项目开始前先明确边界

工控安全评估第一步不是扫描，而是确认边界。

要问清楚：

- 评估对象是整个厂区，还是某条产线、某个车间、某套 DCS、某个泵站？
- 是否涉及生产环境？
- 是否允许主动扫描？
- 是否允许登录设备？
- 是否允许读取配置？
- 是否允许导出工程文件？
- 是否允许抓包？
- 哪些资产是禁止触碰的？
- 发生异常时联系谁？

面试表达：

> 工控环境不能一上来就扫描。我会先确认授权范围、生产状态、测试窗口和禁止操作清单，尤其是 PLC、DCS、SIS、ESD 这类关键控制或安全系统，要优先采用访谈、配置核查、镜像抓包和离线资料分析。

## 2. 资料收集阶段看什么

常见资料：

| 资料 | 用途 | 安全价值 |
|---|---|---|
| 网络拓扑图 | 理解区域和通信路径 | 判断边界、横向路径 |
| 资产清单 | 确认设备和系统 | 识别高价值资产 |
| 点表 | 理解变量含义 | 判断读写影响 |
| 工程文件 | 理解控制逻辑和组态 | 判断逻辑篡改风险 |
| 账号清单 | 理解权限分配 | 检查弱口令、共用账号 |
| 远程运维记录 | 理解外部入口 | 发现 VPN/RDP/堡垒机风险 |
| 报警清单 | 理解异常条件 | 判断报警屏蔽风险 |
| 历史趋势 | 理解过程变化 | 支持异常分析 |
| 备份记录 | 理解恢复能力 | 判断勒索和破坏影响 |

如果没有资料，要在报告中写清楚“资料缺失导致资产识别、影响判断和恢复能力评估存在不确定性”。

## 3. 资产识别阶段怎么分层

不要把资产只写成 IP 列表。要按功能分层：

```text
企业管理层：办公终端、OA、ERP、邮件
生产管理层：MES、调度系统、报表系统
监控层：SCADA Server、HMI、操作员站、历史库
控制层：PLC、DCS 控制站、RTU、工业网关
现场层：传感器、执行器、变频器、仪表
安全层：SIS、ESD、火气系统、安全栅
```

工控安全报告里，资产的“功能角色”比 IP 更重要。

## 4. 访谈阶段问什么

访谈对象：

- 自动化工程师
- 仪表工程师
- 运维人员
- 网络管理员
- 安全管理员
- 生产班组
- 厂商维护人员

核心问题：

- 哪些系统停机会影响生产？
- 哪些设备不能扫描？
- 工程文件谁能下载？
- 控制逻辑变更是否审批？
- 是否有远程运维？
- 是否存在共用账号？
- 备份多久做一次？
- 最近是否发生过通信中断、误动作、异常报警？

## 5. 现场观察看什么

如果进入现场或控制室，重点观察：

- 操作员站是否锁屏。
- 是否有共享账号贴在屏幕或桌面。
- 工程站是否连接互联网或办公网。
- 是否插着未知 U 盘。
- 是否存在远程桌面窗口。
- 网络柜中是否有未标识网线。
- 安全设备是否旁路。
- HMI 是否有报警被长期屏蔽。
- 是否存在老旧 Windows 系统。

这类细节会让面试官觉得你知道真实环境的复杂性。

## 6. 技术检查怎么安排

安全检查建议从低风险到高风险：

1. 文档审查。
2. 访谈确认。
3. 被动流量分析。
4. 配置核查。
5. 账号权限核查。
6. 低速资产探测。
7. 漏洞验证。
8. 离线工程分析。

不要把高风险动作放在前面。

## 7. 发现问题怎么写

一个好发现不只是“存在弱口令”，而是：

```text
发现：HMI 操作员站存在共用账号，且具备设备启停权限。
资产角色：该 HMI 用于泵站运行监控和远程操作。
影响路径：账号泄露后，攻击者可登录 HMI 并执行启停操作。
工控影响：可能造成泵组异常启停、供水压力波动或生产中断。
整改建议：按角色分配账号，禁用共用账号，关键操作增加二次确认和日志审计。
```

## 8. 项目复盘看什么

项目结束后要沉淀：

- 资产关系图
- 关键通信矩阵
- 高风险资产清单
- 风险发现列表
- 整改建议
- 不能验证的风险与原因
- 后续持续监测建议

## 9. 面试可讲版本

> 如果我参与工控安全评估，我会先确认授权范围和生产边界，再通过拓扑、资产清单、点表、工程文件、账号和远程运维记录理解系统。技术上优先做被动流量和配置核查，再根据窗口做低风险验证。风险描述时我不会只写漏洞名称，而会说明资产角色、攻击路径和可能的生产影响。
