# 现场项目视角：一次自动化安全评估如何展开

现场评估不是把 IT 扫描器带进控制网。项目需要同时管理生产边界、人员协作、工程证据和验证风险。本章给出从项目启动到复测关闭的通用流程，具体动作仍需服从合同授权、厂区制度和现场负责人指令。

## 1. 项目启动与边界确认

启动阶段明确：

- 评估目标：合规、风险、架构、配置、漏洞或事件专项；
- 组织、厂区、装置、系统和资产范围；
- 明确排除的 SIS、保护装置、生产控制和无线频段；
- 允许的时间、地点、工具、协议和测试强度；
- 禁止动作：扫描、写入、登录尝试、下载、重启等；
- 现场联系人、停止权和紧急沟通；
- 数据保密、证据存储与销毁；
- 发现高危问题时的即时通报流程。

授权必须可追溯，范围模糊时默认不执行主动动作。

## 2. 多专业团队与沟通

项目常涉及工艺、生产、仪表、电气、自动化、网络、系统、安全、设备厂商和承包商。不同人员掌握不同事实：

- 工艺人员解释危险边界和正常窗口；
- 操作员知道真实操作与异常处置；
- 自动化人员掌握逻辑、组态和变更；
- 仪表、电气人员掌握现场回路与设备保护；
- IT/网络人员掌握身份、主机和边界；
- 功能安全人员掌握 SIS、旁路和测试。

发现应与责任专业复核，避免只依据安全人员推断。

## 3. 资料收集

优先索取：

- 工艺流程图、P&ID、单线图和控制说明；
- 网络拓扑、区域边界和通信矩阵；
- 资产清单、固件、操作系统和软件版本；
- PLC/DCS/SCADA 工程、点表和因果矩阵；
- 用户、角色、远程运维和承包商流程；
- 变更、补丁、备份、恢复和应急记录；
- 报警、历史、时间同步和日志策略；
- SIS 安全需求、旁路和证明测试摘要。

记录资料版本、日期、来源和适用范围。

## 4. 现场踏勘

按物理过程走查资产，而非只按 IP：

```text
现场仪表/执行器 → I/O/控制柜 → PLC/DCS/SIS
 → 工程与操作站 → 服务器/历史 → 边界与远程维护
```

观察机柜门禁、端口、无线、移动介质、临时接线、控制权、强制旁路、时间显示、备件和环境条件。拍照和接入设备需事先获准。

## 5. 资产与功能建模

为资产补充：

- 过程单元与物理对象；
- 控制、操作、工程、保护或历史角色；
- 关键通信与可写方向；
- 高风险能力；
- 依赖的电源、网络、时间和身份；
- 最大可信后果与恢复顺序。

这样资产清单才能支持风险排序。

## 6. 架构与数据流审阅

检查实际通信是否符合设计：

- 跨区连接、双宿主和临时通道；
- 工程站到控制器的路径；
- 历史数据上送和远程维护下行；
- 控制器间、远程 I/O 和驱动器周期通信；
- 身份、时间、备份和更新服务；
- SIS 与 BPCS 的共享依赖。

拓扑图需结合交换机、路由、防火墙配置和被动流量验证。

## 7. 主机与应用配置审阅

关注：

- 支持状态、补丁与补偿控制；
- 本地/域账号、共享账号和服务账号；
- 远程桌面、文件共享、数据库和 Web 服务；
- 工程软件、驱动、插件和许可证；
- 白名单、防病毒、日志和时间同步；
- 备份、恢复介质和敏感文件；
- 客户端自动登录和会话锁定。

配置建议必须考虑厂商兼容、控制性能和维护窗口。

## 8. 控制工程审阅

对关键回路和设备抽样：

- 点表、I/O、单位、质量和标签映射；
- PLC 任务、模式、强制、保持和重启；
- 启停、许可、联锁、报警和复位；
- DCS 主备、在线变更和报警管理；
- SCADA 命令、缓存和失联策略；
- VFD 参数、命令源和保护；
- SIS 独立性、旁路和测试。

把工程差异与工单、操作记录和现场状态关联。

## 9. 技术验证分级

### 9.1 被动验证

读取现有配置、日志、抓包和工程文件，风险最低。

### 9.2 非侵入查询

使用厂商只读诊断或已批准查询，仍需评估连接资源和协议稳定性。

### 9.3 隔离复现

在同型号设备、仿真器或备份副本验证漏洞和配置影响。

### 9.4 生产最小验证

仅在必要、授权、维护窗口和现场监护下实施。预先定义输入、预期输出、停止条件、回退和过程观察。默认不写控制变量、不切模式、不下载程序。

## 10. 发现形成与现场复核

每个发现应包含：

```text
事实与证据
 → 可利用前提
 → 受影响资产与控制能力
 → 控制/状态/数据流变化
 → 过程与保护影响
 → 可信度和限制
 → 补偿措施与长期整改
 → 实施、回退和复测方法
```

先与资产和工艺负责人复核事实，再确定风险。避免泄露真实凭据、工艺机密和可直接利用细节。

## 11. 恢复能力评估

抽查备份是否：

- 来源可信、版本明确并离线保护；
- 能在可用工具版本中打开和编译；
- 包含硬件、网络、程序、HMI、报警和服务器配置；
- 不依赖丢失的许可证、证书或账号；
- 有设备替换、主备恢复和受控启动步骤；
- 经过实际演练并记录恢复时间；
- 包含现场状态、物料和保护验证。

文件存在不等于可恢复。

## 12. 工控场景：发现工程站存在高危漏洞

项目组不直接在生产工程站执行利用，而是：

1. 确认版本、暴露面和可达路径；
2. 审查账号权限与控制器信任；
3. 在副本或同版本实验环境复现；
4. 确认成功后可获得的工程能力；
5. 与控制逻辑和保护层映射物理风险；
6. 设计网络隔离、应用控制和访问限制作为临时措施；
7. 与厂商制定升级、回退和功能验证计划。

## 13. 项目交付与复测

交付物可包括资产与数据流图、证据清单、风险报告、整改路线、配置基线、验证记录和管理层摘要。复测不只确认端口关闭，还要确认：

- 控制功能和报警未受损；
- 主备和通信仍正常；
- 账号、策略和日志生效；
- 工程版本与现场一致；
- 补偿控制可持续；
- 回退与恢复记录完整。

## 14. 面试表达

### 60 分答案

> 工控评估先确认授权和生产边界，再收集拓扑、资产、工程和运维资料，通过访谈、踏勘、配置审阅和被动流量识别问题，最后结合过程影响提出整改。

### 90 分答案

> 我会先确定装置、系统、允许动作和停止权，再按物理过程建立资产—功能—通信—保护模型。技术验证从离线工程、被动流量和日志开始，主动测试优先移到同型号隔离环境。每个发现都从事实、前提、控制能力、状态变化、过程后果和保护层形成证据链。建议包含补偿措施、维护窗口、回退和过程验收，复测还要证明控制、报警、主备和恢复能力没有被整改破坏。

## 15. 总结与检查清单

- [ ] 授权、范围、禁项、停止权和沟通清晰；
- [ ] 资料版本、来源和证据保管可追溯；
- [ ] 资产映射到过程功能和恢复顺序；
- [ ] 验证强度遵循被动优先和隔离复现；
- [ ] 风险结论有控制与保护证据；
- [ ] 整改与复测包含回退和生产验收。
