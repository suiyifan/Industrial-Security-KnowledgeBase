# 资产访谈与现场调研问题清单

访谈用于理解文档无法完整表达的运行事实，不是逐条审问。每个问题都应追问证据、例外和最近一次实际案例，并与工程配置、日志和现场观察交叉验证。

## 1. 使用方法

对每个问题记录：

```text
受访角色：
回答摘要：
适用系统/装置：
支撑证据：
已知例外：
风险信号：
待验证事项：
责任人与期限：
```

不要在访谈现场记录真实密码，也不要要求操作员演示危险控制动作。

## 2. 工艺与生产

- 过程的安全、质量和产量目标是什么？
- 哪些温度、压力、流量、液位、速度或成分最关键？
- 哪些设备或公用工程失效会导致整线停车？
- 从异常出现到必须处置有多少时间？
- 正常停车、紧急停车和恢复启动有什么区别？
- 当前有哪些临时工况、降级运行或保护旁路？
- 哪些状态只能由现场人员确认？

证据：PFD/P&ID、操作规程、工艺卡、异常处置记录。风险信号：只描述产量、不清楚安全边界；恢复完全依赖个人经验。

## 3. 自动化架构

- PLC、DCS、SCADA、RTU、HMI、工程站、历史和 SIS 分别服务哪些单元？
- 哪个工程数据库或备份是权威版本？
- 控制器、网络、服务器和电源如何冗余？
- 哪些系统跨区或双网连接？
- 上层失联后现场如何自治？
- 最近一次扩容、迁移或厂商升级改变了什么？

证据：架构图、节点清单、主备状态、工程版本。风险信号：图纸与现场明显不一致；无人能说明版本权威。

## 4. PLC 与控制逻辑

- CPU 模式、任务周期、最大扫描和看门狗如何设置？
- 谁能上传、下载、在线修改、强制或切换模式？
- 当前是否存在强制、仿真、旁路或在线差异？
- 保持变量和首次扫描逻辑怎样处理重启？
- 启停、许可、联锁、报警和复位如何设计？
- 通信命令是否有握手、序列号、超时和反馈？
- 控制器故障或远程 I/O 失联后输出如何处理？

证据：工程比较、强制表、任务诊断、变更单、逻辑测试。风险信号：强制由口头交接；恢复 RUN 即认为生产恢复。

## 5. DCS 与操作系统

- 主备控制器是否同步程序与动态状态？
- 在线变更如何传播到控制站、画面、报警和历史？
- 操作权如何按区域和角色分配？
- 手动、自动、串级和远程模式谁可切换？
- 报警洪泛、搁置、抑制和常驻如何治理？
- 服务器或网络切换最近一次测试是什么时候？
- SOE、操作日志和历史时间是否一致？

证据：同步状态、操作审计、报警指标、切换测试。风险信号：冗余多年未测试；工程与运行节点版本不一致。

## 6. SCADA、HMI 与远程站

- 标签、协议地址、单位、质量和时间戳如何管理？
- 高风险遥控是否使用选择—执行、确认和超时？
- 通信中断时显示最后值还是坏质量？
- 断链期间命令、事件和累计量是否缓存？
- 重连后旧命令如何取消、数据如何补传和去重？
- 脚本、配方、Web 客户端和 API 有哪些写能力？
- 现场 RTU/PLC 在中心失联后执行什么策略？

证据：组态导出、命令日志、失联演练、标签表。风险信号：界面隐藏按钮代替后端授权；重连行为从未验证。

## 7. 仪表、I/O 与现场设备

- 关键测点的量程、单位、失效方向和校准周期是什么？
- 是否有独立测点、物料平衡或其他合理性校验？
- I/O 通道断线、超量程和坏质量怎样处理？
- 阀门、电机和 VFD 的命令源、反馈和保护是什么？
- 设备就地/远程和维护状态是否返回 HMI？
- 关键输出故障时保持、清零还是安全值？
- 设备替换后怎样做点对点和回路测试？

证据：仪表表、回路图、I/O 组态、校准与测试记录。风险信号：PLC 值被视为现场唯一事实；输出策略统一清零。

## 8. SIS 与功能安全

- 哪些危险场景由哪些 SIF 保护？
- 安全需求规格、设定和最终元件是什么？
- BPCS 与 SIS 在测点、供电、网络、工程站和阀门上共享什么？
- 当前有哪些旁路、强制或测试状态？
- 旁路如何授权、提示、交接和超时？
- 证明测试是否覆盖完整回路和最终元件？
- 安全程序下载、签名和钥匙由谁管理？

证据：SRS、因果矩阵、旁路记录、测试报告。风险信号：仅因使用安全 PLC 就声称满足 SIL；旁路无期限。

## 9. 身份、远程维护与承包商

- 工程、操作、维护和管理账号是否个人化？
- 哪些共享或默认账号仍存在，原因是什么？
- 远程接入经过哪些网关、审批和会话审计？
- 承包商设备、软件和移动介质如何检查？
- 紧急远程支持如何启用和关闭？
- 离职、转岗和项目结束后权限怎样回收？
- 服务账号和设备内置账号怎样保管和轮换？

证据：账号清单、远程会话、审批、回收记录。风险信号：长期常开远程工具；多人共用工程管理员。

## 10. 变更、补丁与供应链

- 变更如何做过程影响分析、离线测试和回退准备？
- 谁审批 PLC/DCS/SIS、HMI、VFD 和仪表参数变更？
- 工程软件、库、插件、设备包和固件来源如何验证？
- 补丁如何测试厂商兼容性和控制功能？
- 在线修改如何合并回主工程？
- 临时变更、旁路和调试状态怎样关闭？

证据：工单、差异、哈希、测试和验收。风险信号：仅记录“优化程序”，没有对象与版本差异。

## 11. 备份、恢复与应急

- 备份覆盖哪些工程、服务器、固件、证书和许可证？
- 备份是否离线、不可变、加密并定期校验？
- 最近一次实际打开、编译、下载或恢复是什么时候？
- 替换控制器、服务器和工程站分别需要多长时间？
- 恢复顺序如何考虑现场阀位、物料、能量和 SIS？
- 事件期间谁有权停车、隔离和恢复生产？
- 如何保全工程、日志、SOE、趋势和现场证据？

证据：恢复演练、RTO、介质清单、应急记录。风险信号：有备份文件但从未验证；只恢复 IT 系统不核对过程。

## 12. 结束访谈的复核问题

- 哪项风险最担心但当前没有资源解决？
- 哪个系统“绝不能停”，理由和失效策略是什么？
- 最近一次意外停机、错误下载或报警洪泛教训是什么？
- 哪些临时措施已经存在很久？
- 哪份文档或哪位人员能纠正今天的回答？

访谈结束后把事实、推断和意见分开，形成待验证清单，不把个人回答直接写成最终结论。
