# DCS 系统

## 1. DCS 是什么

DCS 是 Distributed Control System，分布式控制系统。

它常见于流程工业，例如石油、化工、电力、冶金、水处理等场景。

白话理解：

> PLC 更像现场设备控制器，DCS 更像面向大型连续生产过程的一整套控制系统。

## 2. DCS 适合什么场景

DCS 常用于连续过程控制：

- 炼油
- 化工反应
- 发电
- 冶金
- 水处理
- 造纸
- 制药

这些场景通常有大量连续变量，例如温度、压力、流量、液位。

## 3. DCS 的典型组成

- 控制站
- 操作员站
- 工程师站
- 历史站
- 服务器
- 控制网络
- I/O 模块
- 现场仪表

## 4. DCS 与 PLC 的区别

| 对比项 | PLC | DCS |
|---|---|---|
| 典型场景 | 设备控制、离散制造 | 流程工业、连续控制 |
| 系统形态 | 控制器为核心，可组合 | 一整套平台 |
| 控制规模 | 小到中大型均可 | 大规模、多回路 |
| 操作界面 | 常配 HMI / SCADA | 操作员站体系更完整 |
| 数据历史 | 可能外接历史库 | 通常内置历史站 |
| 工程管理 | 工程软件 | 集成工程环境 |

现实中 PLC 和 DCS 的边界并非绝对，有些大型 PLC 系统也能完成复杂控制。

## 5. DCS 中的控制回路

DCS 常有大量控制回路。

一个典型回路：

```text
压力传感器 -> 控制站 PID -> 调节阀 -> 管道压力变化 -> 压力传感器
```

安全分析中要关注：

- 回路设定值是否可被修改。
- PID 参数是否可被修改。
- 操作员是否能看到真实过程值。
- 报警是否可信。

## 6. DCS 权限体系

DCS 通常有不同角色：

- 操作员
- 班长
- 工程师
- 管理员
- 维护人员

权限可能涉及：

- 查看画面
- 操作设备
- 修改设定值
- 确认报警
- 修改组态
- 下载控制策略

安全关注：

- 是否共用账号。
- 是否弱口令。
- 是否有操作日志。
- 是否能越权修改关键参数。

## 7. DCS 安全风险

- 工程师站被攻陷导致控制策略被改。
- 操作员站被攻陷导致误操作或错误显示。
- 历史站被篡改导致追溯失真。
- 控制网络被横向移动。
- 组态文件泄露导致攻击者理解工艺。
- 补丁困难导致系统长期带漏洞运行。

## 8. 面试表达

可以这样说：

> DCS 更常见于石油化工、电力、冶金、水处理等连续过程场景，通常包含控制站、操作员站、工程师站和历史站。相比单个 PLC，DCS 更强调多回路、集中监控和工程管理。安全分析时，我会关注工程师权限、控制策略下载、报警和历史数据完整性，以及控制网络横向移动风险。
