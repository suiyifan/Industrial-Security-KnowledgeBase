# SCADA、HMI 与组态工程

SCADA 负责对分散现场进行监视、数据采集和监督控制，HMI 则把过程数据、报警和操作命令呈现给人员。二者不只是“显示界面”：标签绑定、通信驱动、命令确认、质量码、脚本、配方和权限共同决定操作员看到什么、能改变什么，以及通信中断时系统怎样退化。

## 1. 原理解释：监督控制与现场自治

SCADA 常用于输油管线、电网、水务、交通和分布式站点。中央系统需要跨广域网络采集遥测和事件、下发调度命令，但现场 PLC、RTU 或 IED 在中心失联时仍应维持基本控制与保护。

典型功能包括：

- 遥测：模拟量和累计量；
- 遥信：开关、故障、状态和事件；
- 遥控：开关设备、启停机组；
- 遥调：修改设定值或控制参数；
- 报警、趋势、历史、报表和事件顺序；
- 多站点调度、权限和通信质量管理。

SCADA 是监督层，不能把所有关键保护都依赖中央服务器和广域链路。

## 2. 工程实现：典型架构

```text
现场传感器/执行器
   ↕
PLC / RTU / IED / 站控系统
   ↕ 串口、无线、专网、工业以太网
前置通信服务器 / 网关
   ↕
实时数据库 / SCADA 主备服务器
   ↕
HMI 客户端、报警、历史、报表、调度接口
```

大型系统可能包含多级主站、灾备中心和区域控制中心。每一级都需明确数据权威、命令路由和失联策略。

## 3. HMI 画面与对象模型

HMI 画面通常由以下对象构成：

- 数值、仪表、趋势和状态符号；
- 按钮、面板、弹窗和导航；
- 报警列表、事件列表和确认控件；
- 配方、批次和参数表；
- 脚本、动画和条件表达式；
- 用户会话与区域权限。

一个泵符号可能绑定运行反馈、故障、模式、启动命令、许可和联锁等多个标签。颜色应表达明确状态，不能用绿色同时表示“运行”“可用”和“安全”。

## 4. 标签、通信驱动与质量码

HMI 标签可以直接映射控制器变量，也可经过内部计算、缩放、别名或脚本。完整映射为：

```text
现场对象 → PLC 变量 → 协议地址/符号
 → SCADA 驱动 → 实时数据库标签
 → HMI 对象 / 报警 / 历史 / 报表
```

标签应包含数据类型、单位、上下限、访问方向、刷新周期、质量和时间戳。通信中断后系统可能显示最后值、替代值或空值；若没有明显坏质量标识，操作员会把陈旧数据当成实时状态。

## 5. 轮询、报告变化与事件

SCADA 获取数据的方式包括周期轮询、变化上送、事件上送和文件/批量传输。

- 轮询简单可控，但大量站点会增加延迟和带宽；
- 变化上送减少流量，但需要可靠保存与重传；
- 事件通常在现场打时间戳，以保留真实顺序；
- 累计量需处理断链期间增量、回卷和复位。

网络恢复后可能出现积压事件集中上送，不能把接收时间误当发生时间。

## 6. 命令链与选择—执行

高风险遥控不应只是写一个位。常见流程：

```text
操作员选择设备
 → 系统验证身份、区域、模式和设备状态
 → 显示目标与动作供确认
 → 执行命令并分配序列号
 → 现场校验许可后动作
 → 返回确认、结果码和状态反馈
```

选择—执行可降低误操作，但不是身份认证的替代。命令需要超时、去重、反馈和审计，避免链路重试造成重复动作。

## 7. 画面设计与态势感知

有效 HMI 应帮助操作员识别偏差，而非追求装饰：

- 正常状态低饱和度，异常状态突出；
- 显示过程上下文、趋势和关键约束；
- 明确区分命令、反馈、许可、联锁和坏质量；
- 高风险动作显示目标设备、当前状态和预期后果；
- 旁路、强制、手动和维护状态持续可见；
- 导航支持从总览下钻到回路与设备。

错误动画、重叠对象或脚本异常可能造成显示与实际状态不一致。

## 8. 脚本、配方与扩展接口

组态平台常支持脚本、数据库查询、报表、ActiveX/插件、Web 客户端和 API。它们增强功能，也引入通用软件风险：

- 脚本可批量写标签或执行系统命令；
- 配方导入可改变大量参数；
- 数据库连接可能使用高权限账号；
- 老旧插件和运行库扩大主机攻击面；
- Web 发布可能暴露远程操作能力；
- 自定义驱动解析外部数据，可能存在内存或输入验证缺陷。

必须区分“查看客户端”和“可控制客户端”，并最小化服务账号权限。

## 9. 会话、权限与控制权

权限可以按角色、区域、设备、动作和时段限制。重点检查：

- 默认和共享账号；
- 客户端自动登录与会话锁定；
- 班长批准、双人确认和电子签名；
- 远程会话与本地控制权冲突；
- 脚本、报表和服务账号绕过界面权限；
- 灾备切换后权限配置是否一致。

界面按钮灰掉不等于后端写入被拒绝，授权必须在服务或控制器侧再次验证。

## 10. 失联、缓存与恢复

SCADA 与现场失联时需定义：

- 现场是否继续自治控制；
- 中央命令队列是否取消或等待；
- HMI 如何显示坏质量和最后更新时间；
- 事件、历史与累计量怎样缓存；
- 恢复后数据如何补传、去重和排序；
- 旧命令是否可能在重连后执行；
- 操作员如何确认现场与中心状态一致。

最危险的不是明确离线，而是部分标签陈旧、部分标签实时形成虚假一致性。

## 11. 数据流、控制流与状态流

```text
数据流：现场值 → 驱动 → 实时库 → 画面/报警/历史
控制流：会话权限 → HMI 动作 → 命令服务 → 现场许可 → 输出
状态流：通信在线/离线、数据好/坏、命令选择/执行/确认/失败
```

安全调查应保存标签值、质量、源时间、接收时间、命令序列号、用户和现场反馈。

## 12. 安全视角：SCADA/HMI 攻击面

SCADA/HMI 安全风险集中在“中心看到的状态、下发的命令与现场事实不一致”。攻击者可能只操纵其中一条链，也可能同时操纵数据、操作和审计。

### 12.1 画面与标签欺骗

改变标签绑定、单位、动画或质量显示，可隐藏真实过程。即使 PLC 未被修改，操作员决策也会受影响。

### 12.2 命令与权限滥用

共享账号、暴露 API 或服务端授权缺失，使攻击者可绕过界面发送遥控或遥调。

### 12.3 通信与缓存操纵

阻断、延迟、重放或选择性修改数据，会让中心看到不一致状态。重连后的旧命令和积压事件也可能造成混乱。

### 12.4 组态与供应链

恶意脚本、驱动、插件或配方文件可通过工程站传播到多个客户端和服务器。

### 12.5 SCADA 攻击链

```text
取得客户端、服务器、前置机或远程接口权限
 → 修改标签、质量、命令、脚本、权限或队列
 → 数据流/控制流/状态流在中心与现场之间失配
 → 操作员错误决策或现场收到异常命令
 → PLC/RTU 许可与设备动作
 → 过程变化及保护响应
```

证据至少包括源时间、接收时间、质量、用户、命令序列、前置通信日志、现场确认和过程趋势。只看中心服务器日志无法证明远程站执行了命令。

## 13. 前置机、数据集中器与协议转换

前置通信层承担多链路轮询、协议转换、缓存、时间处理和命令路由。审阅重点：

- 站点地址与协议对象映射；
- 主备前置机和链路切换；
- 轮询优先级、超时、重试和限速；
- 值、质量和源时间是否完整转换；
- 累计量、事件和文件传输如何去重；
- 写命令是否限制目标、方向和有效期；
- 配置变化如何发布到全部前置节点。

协议转换若只传值不传质量，会让上层丢失关键语义；重复映射或站号错误可能把命令发往错误站点。

## 14. 广域通信的时间与容量约束

无线、卫星、专线和运营商网络具有不同带宽、抖动、丢包和费用。SCADA 设计需在实时性与容量间折中：

- 关键事件优先于普通遥测；
- 变化上送需处理抖动与死区；
- 轮询周期要覆盖站点规模与链路容量；
- 断链缓存需有容量、溢出和保留策略；
- 重连补传不能挤占实时命令；
- 时间同步丢失时要标注精度下降。

资源耗尽可能来自攻击，也可能来自新增站点、错误轮询或事件风暴，应结合配置变更和链路统计判断。

## 15. 命令安全状态机

高风险命令应记录完整状态：Draft、Selected、Confirmed、Sent、Accepted、Executed、Failed、Expired、Cancelled。每个状态需明确责任系统和时间戳。

命令结果至少区分：通信接收、逻辑接受、输出执行和现场到位。把协议 ACK 当成阀门已动作，会造成错误的闭环确认。

## 16. 工控场景：旧遥控在重连后执行

管线阀站通信中断，操作员下发关闭命令但未收到确认，随后改用电话通知现场保持开启。若主站把未确认命令留在队列，链路恢复后旧关闭命令可能被发送，造成流量中断。

防护需要命令短有效期、序列号、状态前置条件、明确取消、现场许可和重连后人工确认。调查应关联操作日志、命令队列、链路事件和阀位反馈。

## 17. 防护与恢复

- 中央监督与现场自治、联锁和保护合理分工；
- 标签显示值、质量和最后更新时间；
- 高风险命令使用选择—执行、超时、去重和结果确认；
- 服务端和控制器端验证权限与过程条件；
- 组态、画面、脚本、驱动和配方纳入版本控制；
- 主备与灾备系统同步验证权限、队列和配置；
- 定期演练失联、重连、数据补传和命令取消；
- 事件调查区分源时间、接收时间和显示时间。

## 18. 面试表达

### 60 分答案

> SCADA 用于远程监视和数据采集，HMI 为操作员提供画面、报警、趋势和控制。现场 PLC/RTU 负责实时控制，中心下发监督命令。

### 90 分答案

> 我会把 SCADA/HMI 分成现场自治、通信前置、实时数据库、操作界面和历史应用几层。分析时追踪标签从 PLC 变量、协议对象、驱动和实时库到画面的完整映射，同时保留质量码和源时间。对于遥控，我关注选择—执行、序列号、超时、服务端权限、现场许可和反馈确认。广域链路失联时还要检查缓存、旧命令、事件补传和部分数据陈旧，避免中心与现场状态不一致。

### 常见追问

**问：HMI 显示正常为何过程仍可能异常？**

答：可能显示最后值、错误标签、伪造反馈或动画脚本结果，应与 PLC 在线值、质量、现场表计和过程平衡交叉验证。

**问：按钮隐藏是否能阻止未授权操作？**

答：不能，攻击者可能直接调用 API 或写标签，权限必须在后端和控制器侧实施。

## 19. 实操建议

### 实验一：标签映射

建立一个 PLC 变量、SCADA 标签、HMI 数值和历史点，故意制造单位、缩放和质量显示错误并定位。

### 实验二：命令状态机

实现选择、确认、执行、超时和反馈状态，模拟重复命令、失联和重连，验证旧命令不会执行。

### 实验三：坏质量显示

断开仿真 PLC 通信，比较保持最后值但无标识与明确灰化、显示时间戳两种界面对判断的影响。

### 实验四：前置机补传

模拟远程站持续产生事件，断开主站链路后缓存，再恢复通信。检查实时遥测、历史事件和命令的优先级、去重、源时间与接收时间。

### 实验五：四级命令确认

分别记录命令发送、现场逻辑接受、输出变化和设备到位四个时刻。注入许可拒绝和执行器拒动，验证界面不会把通信成功误显示为物理成功。

## 20. 总结与检查清单

- [ ] 能区分 SCADA 监督控制与现场自治；
- [ ] 能追踪标签、质量和时间戳的完整数据链；
- [ ] 能解释选择—执行、确认、超时和防重放；
- [ ] 能审阅画面、脚本、配方、驱动和后端权限；
- [ ] 能分析失联、缓存、旧命令和重连行为；
- [ ] 能用现场反馈验证 HMI 所示状态。
