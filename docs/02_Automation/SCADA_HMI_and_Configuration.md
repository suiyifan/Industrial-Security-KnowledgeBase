# SCADA、HMI 与组态软件

## 1. SCADA 是什么

SCADA 是 Supervisory Control And Data Acquisition，监控与数据采集系统。

它负责从远程或分散设备采集数据，进行监控、告警、趋势展示和调度控制。

常见场景：

- 电力调度
- 水务
- 油气管线
- 城市基础设施
- 大范围分布式站点

## 2. HMI 是什么

HMI 是 Human Machine Interface，人机界面。

它给操作员提供可视化界面：

- 看设备状态
- 看工艺流程
- 看报警
- 修改设定值
- 启停设备
- 切换控制模式

HMI 不只是显示器，它经常能下发控制命令。

## 3. 组态软件是什么

组态软件用于搭建 HMI / SCADA 画面和逻辑。

常见功能：

- 画工艺流程图
- 绑定变量点
- 配置报警
- 配置趋势
- 配置脚本
- 配置权限
- 配置通信驱动

安全关注：

- 组态工程文件是否泄露。
- 脚本是否存在命令执行风险。
- 变量绑定是否暴露敏感点位。
- 权限配置是否过宽。

## 4. SCADA 典型架构

```text
现场 PLC / RTU
    |
通信网络
    |
SCADA Server
    |
HMI Client / 操作员站
    |
历史数据库 / 报表 / 调度系统
```

## 5. HMI 画面中的关键信息

HMI 画面通常包含：

- 工艺流程
- 实时数据
- 设备状态
- 报警状态
- 趋势曲线
- 控制按钮
- 参数输入框

攻击者如果能看到 HMI 画面，就可能理解工艺结构；如果能操作 HMI，就可能直接影响生产。

## 6. 报警

报警用于提示异常状态。

常见报警：

- 高高报警
- 高报警
- 低报警
- 低低报警
- 通信中断
- 设备故障
- 联锁触发

安全关注：

- 报警是否可被屏蔽。
- 报警阈值是否可被修改。
- 报警确认是否有权限控制。
- 是否记录报警历史。

## 7. 趋势

趋势是过程变量随时间变化的曲线。

用途：

- 观察工艺稳定性。
- 发现异常波动。
- 事故追溯。
- 判断攻击是否产生物理影响。

安全关注：

- 趋势数据是否被篡改。
- 历史数据是否缺失。
- 数据采样周期是否异常。

## 8. SCADA / HMI 常见安全问题

- 默认口令。
- Web 访问控制缺陷。
- 工程文件泄露。
- 脚本执行漏洞。
- 通信驱动漏洞。
- 客户端权限过高。
- 老旧 Windows 系统。
- 远程桌面暴露。
- 操作日志缺失。

## 9. 面试表达

可以这样说：

> SCADA 更偏监控和数据采集，HMI 是操作员与设备交互的界面，组态软件用于配置画面、变量、报警、趋势和通信驱动。安全分析时，我会关注 HMI 是否能下发控制命令、组态文件是否泄露、报警和趋势是否可信，以及 SCADA Server 是否成为控制网络中的高价值目标。
