# DCS 运行维护与安全关注点

DCS 的安全分析要更关注连续过程、权限、工程变更、报警管理和历史追溯。

## 1. DCS 日常运行视角

操作员关注：

- 当前工艺参数是否正常。
- 报警是否出现。
- 趋势是否异常。
- 设备是否运行。
- 控制回路是否自动。
- 设定值是否需要调整。

工程师关注：

- 控制策略是否正确。
- 回路参数是否稳定。
- 组态是否需要变更。
- 控制站是否健康。
- 通信是否正常。

安全人员要把两者联系起来。

## 2. 控制回路管理

DCS 中常见大量回路：

```text
FIC: Flow Indicating Controller
PIC: Pressure Indicating Controller
TIC: Temperature Indicating Controller
LIC: Level Indicating Controller
```

命名示例：

```text
TIC-101  101 号温度指示控制回路
PIC-202  202 号压力指示控制回路
```

安全关注：

- 谁能修改 SP？
- 谁能切换手动 / 自动？
- 谁能修改 PID 参数？
- 是否记录修改历史？
- 是否存在批量参数修改入口？

## 3. 报警泛滥

报警泛滥是工业现场常见问题。

含义：

> 短时间内大量报警出现，操作员无法有效判断优先级。

安全影响：

- 真实攻击或异常可能被淹没。
- 操作员忽略重要报警。
- 攻击者可能利用报警噪声掩盖操作。

评估问题：

- 是否有报警分级？
- 是否统计报警频率？
- 是否存在长期未处理报警？
- 是否有报警屏蔽审批？

## 4. 旁路与抑制

DCS / SIS / 报警系统可能存在：

- Bypass：旁路
- Suppress：抑制
- Disable：禁用
- Override：覆盖

这些功能在维护时有用，但安全风险高。

要问：

- 谁可以旁路？
- 旁路是否有时间限制？
- 旁路是否记录？
- 旁路状态是否在画面明显提示？

## 5. 工程变更管理

DCS 变更通常比普通 IT 变更更敏感。

变更应包含：

- 申请
- 审批
- 风险评估
- 备份
- 测试
- 执行窗口
- 回退方案
- 记录归档

安全风险：

- 未审批修改控制策略。
- 无备份直接下载。
- 现场版本与备份不一致。
- 第三方维护后未复核。

## 6. 时间同步

DCS、历史库、报警服务器、操作员站需要时间一致。

安全意义：

- 事故追溯依赖时间线。
- 日志关联依赖时间同步。
- 攻击调查需要准确时间戳。

风险：

- 时间漂移导致日志无法对齐。
- NTP 配置错误。
- 时间服务器被篡改。

## 7. 面试可讲版本

> DCS 安全分析不只看漏洞，还要看运行管理细节，比如控制回路参数、报警泛滥、旁路状态、工程变更、历史趋势和时间同步。连续过程系统对稳定性要求很高，所以任何设定值、PID 参数、报警阈值或工程策略的异常变更都可能带来生产风险。
