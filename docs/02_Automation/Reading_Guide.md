# 02 章阅读路线与复习问题

你目前先不做实践也可以。本章建议按“读懂系统 -> 建立安全映射 -> 准备面试表达”的方式阅读。

## 第一轮：只建立整体理解

阅读：

1. `Industrial_Automation_Overview.md`
2. `Physical_Process_and_Field_Devices.md`
3. `Control_Theory_Basics.md`

目标：

- 理解工业自动化在控制什么。
- 理解传感器、控制器、执行器的关系。
- 理解开环、闭环、PID 的基本思想。

复习问题：

- 为什么工控安全会影响物理世界？
- SP、PV、CV 分别是什么意思？
- 传感器数据被伪造会造成什么问题？
- 执行器被异常控制会造成什么问题？

## 第二轮：理解核心系统组件

阅读：

1. `PLC_Basics.md`
2. `DCS_Basics.md`
3. `SCADA_HMI_and_Configuration.md`
4. `Engineering_Operator_Historian.md`

目标：

- 区分 PLC、DCS、SCADA、HMI。
- 理解工程站、操作员站、历史数据库的安全价值。
- 知道控制逻辑、组态、点表、报警、趋势是什么。

复习问题：

- PLC 的扫描周期是什么？
- 工程站为什么是高价值目标？
- HMI 为什么不只是显示界面？
- 历史数据库被篡改有什么影响？

## 第三轮：理解行业与安全后果

阅读：

1. `Industrial_Scenarios.md`
2. `Safety_SIS_and_Interlock.md`
3. `Automation_Attack_Surface.md`
4. `Automation_for_ICS_Security.md`

目标：

- 能把行业场景和风险后果联系起来。
- 理解报警、联锁、SIS、ESD 的区别。
- 能从资产、功能、协议三个视角看攻击面。
- 能把漏洞影响写成工控风险。

复习问题：

- 电力、石化、水处理场景的安全影响有什么不同？
- 报警和联锁的区别是什么？
- 工控系统攻击面可以从哪三条线分析？
- 远程命令执行发生在工程站和办公终端上，风险有何不同？

## 第四轮：准备面试表达

阅读：

1. `Interview_QA.md`
2. `Glossary.md`

目标：

- 把每个知识点压缩成 30 秒到 1 分钟表达。
- 不夸大，不背书面定义，用自己的话解释。

复习问题：

- 用 1 分钟解释你怎么理解工控系统。
- 用 1 分钟解释 PLC、DCS、SCADA 的区别。
- 用 1 分钟解释工程站为什么关键。
- 用 1 分钟解释你如何把漏洞影响转化为工控风险。

## 读完后的最低掌握标准

不用做实验的情况下，至少要能口头回答：

1. 工控系统由哪些核心组件组成？
2. PLC、DCS、SCADA、HMI 各自负责什么？
3. 传感器、执行器、控制器如何形成闭环？
4. 工程站、操作员站、历史数据库分别有什么安全风险？
5. 同一个漏洞为什么在工控场景中影响更复杂？
6. 如果面试官问“你没有工控项目经验，为什么能转这个方向”，你如何回答？

最后一个问题可以这样组织：

```text
我会先承认自己还在补齐实战经验，但我的切入路径是清晰的：
第一，我先理解自动化系统本身，包括 PLC、DCS、SCADA、HMI、工程站和控制链路；
第二，我用网络和 Python 能力去做协议分析和工具开发；
第三，我用安全基础去理解漏洞利用、权限、Web 和系统风险；
第四，我把每个漏洞都放回工业场景，判断它是否影响控制逻辑、设备状态、报警趋势或安全联锁。
```

## 第五轮：读出项目现场感

阅读：

1. `Field_Project_View.md`
2. `Point_Table_and_Tags_Deep_Dive.md`
3. `PLC_Engineering_Workflow.md`
4. `Control_Logic_Reading.md`
5. `SCADA_Project_Artifacts.md`
6. `DCS_Operations_Deep_Dive.md`
7. `Asset_Interview_Checklist.md`
8. `Risk_Case_Studies.md`
9. `Project_Style_Interview_Stories.md`

目标：

- 知道真实项目不是直接扫漏洞，而是先确认授权边界、生产状态和资料完整性。
- 知道点表、工程文件、组态画面、报警、趋势、历史库分别有什么安全价值。
- 知道如何把“端口开放”“未授权访问”“弱口令”讲成具体控制风险。
- 面试时能表现出你理解项目检查项，而不是只背名词。

复习问题：

- 如果你拿到一张点表，你会优先看哪些字段？
- 如果发现 HMI 有共用账号，你会怎么描述工控影响？
- 如果发现 PLC 允许未授权写寄存器，你怎么判断是否危险？
- 如果工程站可以远程登录，你会继续问哪些问题？
- 如果报警被长期屏蔽，你会怎么写风险和整改建议？

## 第六轮：把表达压成面试答案

面试前重点背熟五条表达主线：

1. 工控系统不是单台设备，而是一条控制链路。
2. 工控风险要结合资产角色判断：工程站、HMI、PLC、历史库的影响不同。
3. 协议地址要结合点表，才能知道真实工业含义。
4. 关键操作包括工程下载、变量写入、Force、报警屏蔽、模式切换。
5. 安全测试必须尊重生产边界，优先低风险方法。
