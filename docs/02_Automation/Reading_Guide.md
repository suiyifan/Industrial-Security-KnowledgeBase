# 阅读、复习与验收指南

本指南用于判断是否真正掌握 02 篇。阅读完成不是验收标准；能够建模、解释、验证、表达和恢复，才表示知识已转化为工程能力。

## 1. 第一轮：建立全局控制链

阅读 2.1—2.3，完成：

- 画出物理过程、传感器、I/O、控制器、执行器和操作监控；
- 标出数据流、控制流、状态流和能量流；
- 为关键变量记录单位、范围、质量和时间；
- 区分命令、反馈和真实过程响应。

验收问题：HMI 显示液位 50% 时，你能列出从现场到画面的所有转换和可能失真点吗？

## 2. 第二轮：理解动态与状态

阅读 2.4—2.6，完成：

- 为一个对象解释增益、时间常数、滞后和积分特性；
- 比较 P、PI、PID 的趋势；
- 画出含异常、超时和恢复的设备状态机；
- 区分许可、联锁、报警、跳闸、旁路和复位；
- 解释连续控制与顺序/批次控制的差异。

验收问题：为什么 PV 仍在正常范围，系统却可能已经处于危险演化过程？

## 3. 第三轮：进入 PLC 工程

阅读 2.7—2.10，完成：

- 画任务、程序、功能块和变量调用图；
- 区分输入映像、输出映像、实例、保持区和通信变量；
- 从高后果输出反向追踪所有写点；
- 模拟一次通信写入被覆盖、锁存或触发边沿；
- 完成一次受控工程比较、下载和回退；
- 验证重启后的程序状态与现场模型一致。

验收问题：协议写入成功但数值下一扫描恢复，为什么仍不能判定无影响？

## 4. 第四轮：理解系统级操作与保护

阅读 2.11—2.16，完成：

- 解释 DCS 工程数据库、主备动态状态和在线变更；
- 追踪 SCADA 标签、命令、质量、缓存和重连；
- 重建报警、SOE、操作和历史的时间线；
- 建立跨 P&ID、I/O、PLC、协议和 HMI 的点表；
- 区分电机控制、动力、保护和反馈回路；
- 为一项 SIF 画出传感器、逻辑、最终元件和共因依赖。

验收问题：为什么 BPCS 与 SIS 使用两台不同控制器，仍不一定构成独立保护？

## 5. 第五轮：迁移到行业和安全项目

阅读 2.17—2.18 及篇级扩展材料，完成：

- 选择一个行业描述物料、能量、关键变量、时间尺度和保护；
- 把一个漏洞沿控制链映射到可信物理后果；
- 为结论标记理论可能、工程可信或实验验证；
- 制定低扰动验证、补偿控制、回退和恢复计划；
- 使用访谈清单补齐未知事实；
- 用 60 秒和 3 分钟两种长度表达项目。

验收问题：你的风险结论中，哪些是证据，哪些是假设，哪些保护可能阻断？

## 6. 必做综合实验：水箱控制系统

### 6.1 环境

使用 OpenPLC、CODESYS、其他仿真 PLC 或纯软件模型，禁止连接生产设备。

### 6.2 功能

- 液位 AI、进出阀、出水泵；
- 手动/自动和就地/远程语义；
- PID 或简单滞环控制；
- 泵状态机、启动许可、低液位联锁和反馈超时；
- HMI 数值、质量、报警和趋势；
- 请求—确认命令与事件记录；
- 重启、保持和恢复逻辑。

### 6.3 异常注入

- PV 冻结、偏置和量程错误；
- 远程 I/O 失联；
- 通信写入和重复命令；
- 输出反馈错误；
- PID 饱和；
- 联锁旁路；
- CPU 重启和保持状态不一致。

### 6.4 交付证据

- 架构和控制链图；
- I/O/点表和状态机；
- 程序与工程版本；
- 实验前提、停止条件和恢复点；
- SP/PV/MV、命令、反馈和报警趋势；
- 每种异常的保护响应和可信后果；
- 恢复步骤与最终状态验证。

## 7. 章节自测矩阵

| 能力 | 基础 | 合格 | 成熟 |
| --- | --- | --- | --- |
| 原理 | 能定义术语 | 能解释设计原因 | 能分析约束与例外 |
| 工程 | 知道设备名称 | 能画实现链路 | 能定位任务、状态和参数 |
| 数据 | 能读单点值 | 能追踪跨层映射 | 能处理质量、时间和一致性 |
| 安全 | 能列攻击点 | 能写控制风险链 | 能限定前提、保护与可信度 |
| 实操 | 能跟步骤操作 | 能复现并记录 | 能设计异常、停止与恢复 |
| 表达 | 能给 60 分答案 | 能给工程案例 | 能应对追问并承认边界 |

## 8. 复习节奏

- 当天：用自己的话复述机制并画图；
- 一周：不看正文重建控制链和关键状态；
- 一月：重新执行一个异常实验并比较证据；
- 跨篇学习时：把网络、协议、漏洞和事件分析重新映射回 02 篇模型。

复习不以背诵页数衡量，以能否重新生成模型和证据衡量。

## 9. 最终验收清单

- [ ] 能从现场物理量追踪到 HMI、报警和历史；
- [ ] 能解释动态过程、PID、顺控和状态恢复；
- [ ] 能分析 PLC 任务、内存、工程和逻辑；
- [ ] 能解释 DCS、SCADA、VFD 和 SIS 的工程边界；
- [ ] 能区分命令成功、设备动作和过程后果；
- [ ] 能用多源证据判断故障、误配或攻击；
- [ ] 能在隔离环境完成异常注入与恢复；
- [ ] 能把发现写成有前提、有保护、有证据的风险；
- [ ] 能说明真实现场、实验验证和工程推演的边界；
- [ ] 能以成熟工控安全工程师的方式回答追问。
