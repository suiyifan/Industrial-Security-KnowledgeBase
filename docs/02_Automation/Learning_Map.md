# 02 章学习地图

## 1. 本章要建立的核心模型

工业自动化可以先用一条控制链路理解：

```text
传感器 -> 控制器 -> 执行器 -> 物理过程 -> 传感器
```

展开后是：

```text
现场设备 -> PLC / RTU / 控制器 -> HMI / SCADA / DCS -> 工程站 / 历史数据库 / 上层系统
```

安全研究要继续追问：

```text
谁能读取数据？
谁能下发命令？
谁能修改控制逻辑？
谁能改变报警和趋势？
谁能影响安全联锁？
```

## 2. 三层递进

### 第一层：理解工业现场

先知道工业现场在做什么：

- 测量温度、压力、液位、流量、转速。
- 控制阀门、电机、泵、加热器、变频器。
- 让生产过程稳定、安全、连续运行。

### 第二层：理解控制系统

再理解系统怎么控制：

- PLC 适合离散控制和设备控制。
- DCS 适合流程工业的大规模连续控制。
- SCADA 适合远程监控、数据采集和调度。
- HMI 给人看状态、点按钮、确认报警。
- 工程站用于配置、编程、下载工程。

### 第三层：理解安全影响

最后从安全视角看：

- 工程站被攻陷，可能导致控制逻辑被篡改。
- HMI 被攻陷，可能导致操作员看到错误状态或下发错误命令。
- PLC 被未授权访问，可能导致寄存器、线圈、变量被读写。
- 历史数据库被篡改，可能影响事故追溯和趋势判断。
- 安全联锁被绕过，可能造成设备损坏或人身安全风险。

## 3. 学习时要避免的误区

### 误区 1：只背名词

不要只背 PLC、DCS、SCADA 的定义。要能说出它们在控制链路中的位置。

### 误区 2：把工控安全当普通 Web 安全

普通 Web 漏洞通常影响数据、权限和业务流程。工控安全还可能影响物理过程，例如阀门开关、电机启停、压力超限。

### 误区 3：过早陷入协议细节

协议很重要，但协议字段背后代表工业操作。比如写寄存器不是抽象的“写数据”，它可能是在改设定值、启停设备或改变控制模式。
