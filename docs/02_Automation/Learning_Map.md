# 02 篇学习地图：从物理过程到安全后果

工业自动化学习最容易陷入两个极端：只背设备名词，或者直接钻进某一厂商软件。本篇采用“过程—控制—系统—安全—证据”五阶段路线，先建立跨厂商模型，再理解具体实现。

## 第一阶段：认识被控制的物理世界

对应章节：2.1—2.3。

核心问题：

- 生产目标是什么，哪些变量必须维持在允许区间？
- 温度、压力、流量、液位、速度和位置如何被感知？
- 阀门、泵、电机、加热器和断路器如何改变过程？
- 现场信号如何经过变送、隔离、接线、I/O 转换成为控制器变量？
- 断线、短路、漂移、卡死、饱和和量程错误分别表现为什么证据？

阶段产出：画出一条完整测量链和执行链，能够把原始码、工程量、质量码、报警限值与现场设备对应起来。

## 第二阶段：理解控制为什么能够稳定运行

对应章节：2.4—2.6。

核心问题：

- 对象增益、惯性、滞后和扰动为什么决定控制策略？
- 反馈如何纠偏，又为什么可能振荡、超调或失稳？
- PID 的 P、I、D 分别改变什么动态特性？
- 控制器饱和、积分累积、模式切换和手自动切换怎样影响输出？
- 连续调节、离散逻辑、状态机、顺控和批次配方有什么区别？

阶段产出：完成一个闭环趋势分析和一个顺序控制状态图，能解释正常迁移、超时、异常恢复与危险跳步。

## 第三阶段：进入控制器和工程系统内部

对应章节：2.7—2.11。

核心问题：

- PLC 的输入采样、逻辑执行、通信服务和输出刷新如何调度？
- 周期任务、事件任务和中断如何影响确定性？
- 变量、数据块、保持区、I/O 映像和通信映射如何组织？
- 工程项目如何完成组态、编译、下载、在线修改、强制和版本回退？
- LD、FBD、ST、SFC 中如何追踪启停、模式、联锁、报警与通信写点？
- DCS 如何管理控制回路、冗余控制器、操作权限和在线工程变更？

阶段产出：用仿真器或实验设备完成一次受控变更，保留变更前后工程版本、变量趋势、控制器状态和恢复证明。

## 第四阶段：理解操作、监视与保护体系

对应章节：2.12—2.16。

核心问题：

- SCADA 如何完成广域遥测、遥信、遥控和事件处理？
- HMI 按钮、面板、脚本和配方最终写入什么对象？
- 报警从产生、确认、抑制、搁置到归档经历哪些状态？
- 趋势、历史数据、事件顺序记录为何可能互相矛盾？
- 点表如何连接仪表位号、PLC 地址、协议对象、HMI 标签和历史点？
- MCC、接触器、保护继电器和 VFD 如何控制并保护电机？
- SIS 为什么要独立于 BPCS，旁路和复位为何属于高风险操作？

阶段产出：完成一份跨层点表和一份保护层分析，能够区分控制动作、操作动作与安全动作。

## 第五阶段：把自动化知识转化为安全判断

对应章节：2.17—2.18 及篇级扩展材料。

分析任何自动化安全问题时，使用以下链路：

```text
入口与权限
  ↓
被改变的工程对象、变量或控制状态
  ↓
控制算法、顺控、联锁或操作画面的行为变化
  ↓
过程变量与设备状态偏移
  ↓
BPCS / 报警 / SIS / 人工操作是否发现并阻断
  ↓
安全、环境、质量、设备和生产连续性后果
  ↓
隔离、回退、复位、校验和恢复生产所需证据
```

阶段产出：选择水处理、石化、电力、冶金或离散制造中的一个场景，完成一份不依赖漏洞编号的过程导向威胁分析。

## 推荐实验环境

- 控制逻辑：CODESYS、OpenPLC 或厂商教学版；
- 过程仿真：Factory I/O、MATLAB/Simulink、Python 简化模型或其他隔离仿真器；
- HMI/SCADA：仿真标签、OPC 测试环境或开源监控平台；
- 网络与证据：Wireshark、控制器诊断缓冲区、工程软件比较功能、系统日志；
- 版本管理：Git 保存代码、点表、状态图、实验步骤和结论，不保存真实生产凭据。

## 整篇验收问题

如果不能清楚回答以下问题，就还没有完成 02 篇：

1. 同一次 HMI 数值变化，如何区分真实过程变化、传感器故障、通信异常和显示欺骗？
2. 为什么 PLC 在线变量正常，不代表现场执行器一定正确动作？
3. 为什么删除报警不等于消除危险，强制变量恢复也不等于过程已经恢复？
4. 同样一次“写寄存器”，写 SP、模式、联锁旁路和直接输出为何后果不同？
5. 为什么攻击者控制工程站后仍需要理解控制逻辑和过程约束才能稳定操纵物理过程？
6. 事故恢复时为什么必须同时核对工程版本、控制器状态、现场位置和历史趋势？
