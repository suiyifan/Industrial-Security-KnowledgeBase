# 控制理论入门：开环、闭环、PID

## 1. 为什么安全人员要懂一点控制理论

你不需要一开始成为控制理论专家，但必须理解控制系统为什么能稳定运行。

因为很多工控安全事件的影响不是“机器被黑”这么简单，而是：

- 控制回路被扰动。
- 设定值被篡改。
- 反馈数据被伪造。
- 执行器动作异常。
- 联锁条件被绕过。

这些都和控制逻辑有关。

## 2. 开环控制

开环控制没有反馈。

例子：

```text
按下按钮 -> 电机运行 10 秒 -> 停止
```

系统不会关心实际结果是否达到目标。

优点：

- 简单。
- 成本低。

缺点：

- 抗干扰能力弱。
- 无法自动纠正偏差。

## 3. 闭环控制

闭环控制有反馈。

例子：水箱液位控制。

```text
目标液位 60%
当前液位 45%
控制器判断液位偏低
启动水泵或打开阀门
液位上升
传感器反馈新液位
```

闭环控制核心是：

```text
设定值 - 实际值 = 偏差
```

控制器根据偏差调整输出。

## 4. 设定值、过程值、控制输出

| 名称 | 英文 | 含义 |
|---|---|---|
| 设定值 | Setpoint / SP | 目标值 |
| 过程值 | Process Variable / PV | 实际测量值 |
| 控制输出 | Control Output / CV | 控制器输出 |

例子：

```text
SP：水箱液位目标 60%
PV：当前液位 45%
CV：阀门开度 80%
```

安全分析中，SP 被篡改可能改变生产目标；PV 被伪造可能误导控制器；CV 被异常修改可能直接影响执行器。

## 5. PID 控制

PID 是工业控制中非常常见的控制算法。

PID 包括：

- P：Proportional，比例
- I：Integral，积分
- D：Derivative，微分

## 6. P：比例

比例控制根据当前偏差调整输出。

偏差越大，动作越大。

白话理解：

> 离目标差得越远，就用越大的力去调。

问题：

- 可能存在稳态误差。
- 参数太大可能振荡。

## 7. I：积分

积分控制关注偏差累积。

白话理解：

> 如果长期偏离目标，就慢慢加大修正力度。

作用：

- 消除长期小偏差。

问题：

- 可能导致超调。
- 响应可能变慢。

## 8. D：微分

微分控制关注变化趋势。

白话理解：

> 如果变化太快，就提前刹车。

作用：

- 抑制快速变化。
- 减少超调。

问题：

- 对噪声敏感。

## 9. 安全视角下的 PID 风险

攻击者如果能修改 PID 参数，可能导致：

- 控制不稳定。
- 过程振荡。
- 响应过慢。
- 超调严重。
- 设备频繁动作。

攻击者如果伪造 PV，可能让控制器以为系统正常，实际已经偏离安全范围。

## 10. 控制模式

常见模式：

- 手动模式：人直接控制输出。
- 自动模式：控制器根据逻辑自动控制。
- 本地模式：现场设备本地控制。
- 远程模式：上位机或远程系统控制。

安全关注：

- 谁能切换模式？
- 切换有没有权限控制？
- 远程模式是否可被滥用？
- 手动模式是否能绕过保护？

## 11. 面试表达

可以这样说：

> 我理解控制系统通常通过设定值、过程值和控制输出形成闭环。PID 控制根据偏差进行调节。安全分析时，如果攻击者能篡改设定值、伪造过程值或修改 PID 参数，就可能造成控制不稳定、设备异常动作或生产过程偏离安全范围。
