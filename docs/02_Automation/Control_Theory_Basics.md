# 控制理论与反馈控制基础

控制理论研究如何让动态系统在扰动、测量误差和设备约束下保持目标状态。对工控安全工程师而言，关键不是推导所有数学公式，而是理解过程为什么会延迟、超调、振荡或失稳，以及攻击者改变 SP、PV、MV 或模式后，物理过程会以什么速度和方向响应。

## 1. 原理解释：控制解决什么问题

工业过程会受到负荷、原料、环境和设备老化等扰动。控制器不断比较目标与实际状态并调整执行器，使过程变量维持在允许范围。

```text
误差 e(t) = SP(t) - PV(t)
控制器根据 e(t) 计算 MV(t)
过程在 MV(t) 和扰动 d(t) 作用下产生新的 PV(t)
```

控制目标通常包括：稳定、快速、准确、鲁棒和经济。但这些目标会冲突：响应过快可能超调或振荡，过度滤波会延迟，追求零稳态误差可能积累过大控制输出。

## 2. 工程实现：开环与闭环

### 2.1 开环控制

开环只按预先规则输出，不利用结果纠偏。例如定时运行加热器、按固定速度输送。结构简单，但无法自动补偿环境或负荷变化。

### 2.2 闭环控制

闭环测量 PV 并反馈给控制器。当 SP 与 PV 有偏差时，控制器调整 MV。闭环可以抑制扰动，但错误反馈也会让控制器“正确地做错事”。

### 2.3 前馈与反馈

前馈根据可测扰动提前补偿，反馈根据最终偏差纠正。锅炉负荷变化可通过蒸汽流量前馈提前增加燃料，再由压力反馈消除剩余误差。攻击前馈信号可能在 PV 尚未异常前改变输出。

## 3. 动态对象的关键特性

### 3.1 增益

对象增益描述 MV 变化最终引起多少 PV 变化。阀门增加 10% 可能使流量增加 5%，也可能因阀特性和压差仅增加 1%。错误估计增益会导致控制过强或过弱。

### 3.2 时间常数

时间常数反映过程响应速度。一阶对象在一个时间常数后达到最终变化的约 63%。流量回路可能以秒响应，大型储罐液位可能以分钟或小时响应。相同攻击持续时间在两类过程中后果完全不同。

### 3.3 纯滞后

输入变化后，输出一段时间没有可见响应。例如长管线运输、分析仪取样和炉温传导。滞后让控制器在看不到结果时继续动作，增加超调风险，也给攻击者制造隐藏窗口。

### 3.4 积分过程

储罐液位是典型积分过程：只要进出流量不平衡，液位就持续变化，不会自然停在新值。一个很小但持续的偏差也可能最终造成溢流或抽空。

### 3.5 非线性与工作点

阀门特性、泵曲线、化学反应和摩擦都可能非线性。同一控制参数在低负荷稳定，在高负荷可能振荡。安全分析不能假设所有状态下响应相同。

## 4. 稳态与瞬态指标

- **稳态误差**：过程稳定后 SP 与 PV 的差；
- **上升时间**：PV 从低比例达到高比例所需时间；
- **超调量**：PV 超过目标的最大程度；
- **调节时间**：进入并保持在允许误差带所需时间；
- **振荡**：PV 或 MV 周期性波动；
- **变化率**：单位时间内变量变化大小；
- **稳定裕度**：系统距离失稳还有多少余量。

报警只看静态上限可能发现得太晚。对于压力、温度等高后果变量，变化率和预测到限时间同样重要。

## 5. 稳定性与闭环失稳

稳定系统受到有限扰动后会回到平衡或新的有限状态；不稳定系统的振荡或偏差会不断增大。失稳可能来自：

- 控制增益过高；
- 测量或通信延迟增加；
- 执行器饱和、死区或反向；
- 多个回路相互耦合；
- 手自动切换不连续；
- 错误反馈、采样周期或控制参数。

网络延迟并不必然导致失稳。要比较延迟相对于对象动态和控制周期的大小。给慢速液位回路增加几十毫秒可能无明显影响，对高速运动控制则可能不可接受。

## 6. 采样、离散控制与时序

PLC 和 DCS 以离散周期执行控制。采样过慢会遗漏快速变化并增加相位滞后；采样过快会放大噪声、增加计算与通信负荷。工程上通常让控制周期明显快于过程主要动态，但具体比例依对象、算法和厂商实现确定。

时序链包括：

```text
传感器响应 + I/O 更新 + 网络传输 + 任务等待
+ 算法执行 + 输出刷新 + 执行器响应 + 过程响应
```

攻击者不必完全阻断通信，只要让关键反馈延迟、乱序或冻结到足以破坏控制判断，就可能影响闭环。

## 7. 控制模式与无扰切换

常见模式包括：

- **手动**：操作员直接给 MV；
- **自动**：控制器根据本地 SP 调节；
- **串级**：上级控制器输出成为下级 SP；
- **远程/监督**：SP 来自优化器、SCADA 或上层系统；
- **跟踪**：控制器内部状态跟随外部输出，为切换做准备。

模式切换时若内部输出与实际输出不一致，会产生突跳。无扰切换通过输出跟踪、积分项初始化或 SP 跟踪减少冲击。模式位、跟踪值和远程 SP 都是高风险变量。

## 8. 数据流、控制流与状态流

以串级温度控制为例：

```text
主温度 SP - 主温度 PV → 主控制器 → 蒸汽流量 SP
蒸汽流量 SP - 流量 PV → 副控制器 → 蒸汽阀 MV
```

数据流包含两个 PV、两个 SP 和一个最终 MV；控制流由主副回路调用关系决定；状态流包括每个回路的手动/自动/串级模式；能量流是蒸汽热量进入过程。攻击中间流量 SP 可能绕过主回路的正常调节意图。

## 9. 安全视角：控制闭环的四类操纵

控制系统的安全风险来自攻击行为与对象动态共同作用。一次短暂写入是否危险，取决于过程增益、滞后、积分特性、执行器限幅以及独立保护的响应时间。

### 9.1 改变 SP

控制算法和反馈都正常，但目标被改变。此类操纵可能表现为合法操作命令，需要结合权限、工艺窗口和变更记录识别。

### 9.2 欺骗 PV

控制器依据错误反馈调整输出。冻结、偏置、比例变换和动态回放的隐蔽性逐步增加。

### 9.3 改变 MV 或执行路径

直接强制输出、切换手动模式、改变阀门方向或干预驱动器，可绕过控制算法。PV 随后可能暴露异常，除非反馈也被操纵。

### 9.4 改变模型与参数

修改控制增益、滤波、采样周期、上下限、前馈系数或模式逻辑，会改变系统动态，却可能不立即越限。

### 9.5 典型攻击链

```text
取得写参数或过程变量的能力
 → 改变 SP、PV、MV、模式或动态参数
 → 闭环误差和控制作用发生变化
 → 执行器在限幅与速率约束下动作
 → 过程按增益、惯性和滞后响应
 → 报警、操作员、设备保护和 SIS 依次介入
```

攻击链中的时间十分关键。如果过程安全时间为数十分钟，缓慢偏置可能长期隐蔽；如果设备在数十毫秒内完成危险动作，传统服务器日志可能来不及形成有效证据。

## 10. 过程耦合与多变量影响

工业过程往往不是单输入单输出。改变一个阀门可能同时影响流量、压力和液位；两个控制器可能争夺同一操纵量。常见耦合包括：

- 上游流量改变下游液位与压力；
- 蒸汽阀同时影响温度和设备压力；
- 多台泵并联运行改变单泵工作点；
- 炉膛燃料与空气必须协调，单独改变一方影响安全与效率；
- 电网有功、无功、电压和频率相互作用。

当一个回路振荡时，不能立刻认定该回路参数错误。异常可能来自另一个回路、上游扰动或共享执行器。安全监测要分析变量间的因果时间关系。

## 11. 模型、残差与合理性校验

简单过程模型可以提供独立判断。例如水箱满足近似物料平衡：

```text
液位变化率 ≈ (进流量 - 出流量) / 截面积
```

用流量估计的液位与传感器 PV 之差称为残差。残差持续增大可能表示测量漂移、流量误差、泄漏或欺骗，但不能单独证明攻击。模型需要考虑噪声、未测扰动和参数变化。

可用合理性规则包括：

- 阀位增加后流量应在合理时间内同向变化；
- 加热输出增加后温度不应瞬间跳变；
- 相邻压力点变化应符合流体传播方向；
- 电机停止后转速应按机械惯性衰减；
- 质量与能量不能无来源地增加或消失。

这些规则把控制理论转化为异常检测证据。

## 12. 工控场景：慢速液位偏置攻击

攻击者给液位 PV 叠加逐渐增大的正偏置，使控制器认为液位偏高并持续减少进料。由于偏置变化缓慢，静态报警不触发；真实液位逐渐下降，最终可能造成出料泵吸空。

可检测证据包括：进出流量物料平衡、就地液位计、泵入口压力、PV 与模型估计残差、偏置变化率和操作记录。保护措施包括独立低低液位开关及泵干转保护。

## 13. 防护与监测

- 对 SP、模式、参数、限幅和控制周期变更建立审计；
- 同时监测 PV、MV、误差、模式和饱和状态；
- 使用变化率、命令—响应关系和物料能量平衡发现合理范围内的欺骗；
- 对远程 SP 和上层优化命令设置工艺约束与速率限制；
- 关键回路保留独立保护，不依赖同一测量和控制逻辑；
- 变更控制参数后观察足够长的动态过程再确认稳定。

## 14. 面试表达

### 60 分答案

> 闭环控制比较设定值和过程值，再调整控制输出。对象具有增益、惯性和滞后，因此输入变化不会立即得到结果。安全分析要关注 SP、PV、MV 和控制模式是否被改变。

### 90 分答案

> 我分析控制风险时会先识别对象动态，包括增益、时间常数、纯滞后、积分特性和工作点，再画出 SP、PV、MV、扰动和保护层。之后判断异常改变了目标、反馈、执行输出还是控制参数，并比较攻击持续时间、采样延迟与过程安全时间。这样才能解释为什么同样的网络延迟对慢速液位回路影响很小，却可能破坏高速运动控制，也能通过物料平衡、变化率和命令—响应关系寻找证据。

### 常见追问

**问：反馈控制为什么可能不稳定？**

答：控制作用过强、延迟过大、执行器非线性或回路耦合会使纠偏到达时已经过量，形成不断放大的振荡。

**问：PV 在正常范围是否代表安全？**

答：不一定。PV 可能被欺骗，变化率可能已异常，MV 可能长期饱和，或独立保护已旁路。需要结合动态和多源证据。

## 15. 实操建议

### 实验一：一阶对象响应

用 Python、Simulink 或其他离线仿真器建立一阶对象，改变增益、时间常数和滞后，比较阶跃响应的上升时间与调节时间。

### 实验二：反馈冻结

在液位模型中冻结 PV，观察控制器输出和真实液位。分别增加变化率报警、物料平衡和独立低液位保护，比较发现与阻断时间。

### 实验三：模式切换

模拟手动切自动时内部输出不跟踪与正确跟踪两种情况，记录 MV 突跳和 PV 超调。所有实验只在隔离模型中完成。

### 实验四：耦合回路

建立两个共享扰动或操纵量的简化回路，改变其中一个控制参数，观察另一个回路趋势。尝试只看单回路与看跨回路数据两种诊断方式，比较结论差异。

实验报告还应记录模型假设、采样周期、初始状态和评价指标，避免只保存一张趋势截图而无法复现结论。

对任何异常现象，都要同时保存 SP、PV、MV、模式和扰动，才能区分控制器问题与对象变化。

## 16. 总结与检查清单

- [ ] 能解释增益、时间常数、纯滞后、积分过程和非线性；
- [ ] 能用稳态误差、超调和调节时间评价响应；
- [ ] 能说明采样、延迟和执行器约束怎样影响稳定性；
- [ ] 能区分 SP、PV、MV、参数和模式操纵；
- [ ] 能结合过程安全时间判断攻击窗口；
- [ ] 能用动态关系和独立测量验证异常。
