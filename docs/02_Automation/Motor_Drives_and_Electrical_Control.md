# 电机、电气控制与变频驱动

泵、风机、压缩机、输送机和搅拌器最终都依赖电机与电气回路。PLC 发出的启动位只是控制链的一部分，真正的能量通断由接触器、断路器、保护继电器、MCC 或变频器完成。工控安全分析必须从数字命令追踪到电气状态、机械运动和过程后果。

## 1. 原理解释：控制回路与动力回路

电机系统通常分为：

- **动力回路**：把电源经断路器、接触器或驱动器送到电机；
- **控制回路**：由按钮、继电器、PLC 和保护触点决定是否允许动力回路接通；
- **保护回路**：检测过载、短路、缺相、接地、过温、堵转和超速并跳闸；
- **反馈回路**：返回运行、故障、就绪、就地/远程、速度和电流。

PLC 输出失效不应自动等于保护失效。关键电气保护通常在驱动器或保护继电器中独立实现。

## 2. 工程实现：直接启动回路

典型直接启动：

```text
启动请求 + 许可 + 无跳闸
 → PLC DO / 中间继电器
 → 接触器线圈
 → 主触点闭合
 → 电机得电
 → 辅助触点返回运行反馈
```

停止通常优先于启动。硬接线急停、热继电器和保护触点可串联在线圈回路中，使 PLC 无法绕过。

接触器辅助触点只证明接触器状态，不必然证明电机转动；还需速度、电流、压力或流量等过程反馈。

## 3. MCC、电气柜与控制权

MCC 集中安装电机馈线、断路器、接触器、软启动器、VFD 和保护。抽屉单元可能具有断开、试验和运行位置。

常见控制位置：

- 就地按钮盒；
- MCC 柜门；
- PLC/HMI 远程；
- VFD 本地面板；
- 维护或试验模式。

就地/远程选择必须明确反馈。HMI 显示远程命令成功，不代表设备不处于就地模式。

## 4. 电机启动方式

### 4.1 直接启动

结构简单，但启动电流和机械冲击大。

### 4.2 星—三角或降压启动

通过切换接法降低启动电流。切换时序和互锁错误可能造成短路或启动失败。

### 4.3 软启动器

通过控制电压斜坡减少冲击，通常用于不需要连续调速的场合。

### 4.4 变频器 VFD

通过改变输出频率和电压调节速度与转矩，支持加减速、限流、制动、PID 和多种通信控制。VFD 是具有本地参数和状态机的控制设备，不是简单 AO 执行器。

## 5. VFD 控制与参数

重要对象包括：

- 启停与正反转命令；
- 频率/速度给定来源；
- 本地、端子、网络控制权；
- 最小/最大频率；
- 加速、减速和 S 曲线时间；
- 电机铭牌与自整定参数；
- 电流、转矩、温度和直流母线状态；
- 故障、警告、复位和自动重启；
- 安全转矩关闭 STO 等安全功能。

改变控制源或自动重启参数，可能让维护人员误判设备不会启动。

## 6. 正反转、联锁与机械约束

正反接触器必须电气和逻辑互锁，防止同时吸合。某些泵和压缩机禁止反转，输送机反转可能造成物料堆积。方向命令需结合设备设计，而非仅看一个布尔位。

频繁启停会增加热负荷和机械磨损。程序常设置最小运行时间、最小停止时间和每小时启动次数。

## 7. 保护继电器与跳闸

保护可包括：

- 短路、过流和过载；
- 缺相、相序、欠压和过压；
- 接地故障；
- 轴承或绕组过温；
- 堵转、欠载和失速；
- 振动、超速或差动保护。

跳闸状态通常锁存，需要确认原因后复位。远程复位若没有检查现场条件，可能造成反复启动或扩大故障。

## 8. 命令、反馈与过程响应

```text
命令：Start、Stop、SpeedRef、Direction、Reset
状态：Ready、Running、AtSpeed、Fault、Local/Remote
电气：接触器、频率、电流、转矩、保护码
过程：流量、压力、速度、位置、振动
```

诊断应沿四层比较。例如 `Running=1` 但电流为零，可能只是逻辑状态；电流存在但流量为零，可能泵空转、阀门关闭或机械脱开。

## 9. 数据流、控制流、状态流和能量流

```text
HMI/自动逻辑 → PLC 命令 → MCC/VFD → 电机
VFD/继电器反馈 → PLC → HMI/报警
状态：Stopped → Starting → Running → Stopping/Faulted
能量：电源 → 电磁转矩 → 机械轴功 → 流体/输送过程
```

安全结论需要走到能量流。网络命令只有在控制权、许可、电气回路和设备条件允许时才形成运动。

## 10. 安全视角：高风险操纵

- 切换本地/远程或命令源；
- 绕过启动许可和最小停机时间；
- 修改最大频率、加减速或转矩限值；
- 反复远程复位跳闸；
- 改变正反转和自动重启；
- 隐藏电流、速度、故障或保护状态；
- 修改 VFD 电机参数导致保护失配；
- 在维护期间远程启动。

STO 等安全功能应按安全回路验证，普通网络停止命令不能替代它。

### 10.1 安全风险必须映射到设备工况

电气控制的安全风险不能只写成“变频器参数可被修改”。同一参数在不同工况下后果不同：空载试车时提高频率可能只触发超速保护；泵出口阀关闭时高速运行可能造成温升与汽化；输送机带料时缩短加速时间会增加机械冲击；风机越过共振转速区间可能导致持续振动。评估至少确认电机负载、设备曲线、工艺通路、控制权、保护定值和人员作业状态。

网络侧看到写入成功，也不代表电机已获得转矩。应继续验证 PLC 是否在远程和自动模式、许可是否满足、VFD 是否接受该命令源、STO 是否释放、接触器是否吸合、电流与速度是否变化，以及流量、压力或机械位置是否产生预期响应。反过来，HMI 显示停止也不代表动力回路已隔离，残余能量、重力、压力和自动重启仍可能造成伤害。

### 10.2 典型攻击链

一条工程可信的攻击链可以写为：攻击者进入维护笔记本，复用驱动器管理凭据连接 MCC 网络，读取参数表识别关键泵，将命令源从硬接线切到网络，放宽最高频率与自动重启条件，再通过 PLC 或驱动器下发启动。若同时抑制过流告警或伪造运行反馈，操作员可能延迟发现；最终后果由泵曲线、阀门位置、管网阻力和独立机械保护决定。

该链路存在多个可阻断点：维护访问审批和跳板审计可阻止入口；参数签名与变更告警可发现配置漂移；硬接线许可、STO 和设备保护可限制动作；电流、速度、阀位与流量的交叉验证可识别反馈欺骗。分析报告应把每一步所需权限、可观测证据、失败条件和物理后果分别列出。

### 10.3 故障、攻击与误操作的辨析

速度异常不必然是攻击。给定频率正常而实际速度下降，可能来自负载过大、皮带打滑、电机故障或测速异常；给定频率被改变，则继续检查操作记录、配方、PLC 逻辑和参数审计。工程人员应建立“命令—驱动输出—电气量—机械量—过程量”证据矩阵，避免仅凭单一网络包或故障码定性。

### 10.4 驱动系统的模式与状态治理

电机控制通常同时存在就地按钮、柜面选择开关、PLC 自动逻辑、HMI 手动命令和驱动器本体面板。系统必须定义控制权优先级，并把当前命令源可靠反馈给操作员。若 HMI 按钮可点击但驱动器处于 Local，命令可能被拒绝；若切换 Remote 后保留旧启动位，又可能在模式转换瞬间启动。因此模式切换应采用边沿确认、命令清零或重新授权，而不是简单复用持续置位的启动信号。

建议把状态拆成可解释变量：`Available` 表示电源、通信和保护允许；`Permissive` 表示工艺启动条件满足；`Commanded` 表示控制系统已请求；`Running` 来自接触器、驱动器或速度反馈；`ProcessProven` 表示流量、压力或运动已建立；`Trip` 表示保护动作。把这些状态压缩成一个绿色“运行”位，会掩盖空转、联轴器断裂、阀门关闭或反馈粘连。

从安全监测角度，重点是非法状态迁移和长时间不一致：没有命令却出现转速、发出启动后电流长期为零、运行反馈成立但过程量无响应、保护跳闸后被连续复位、Local/Remote 在非维护窗口改变。这类规则比只检测某个寄存器写入更接近真实控制行为。

### 10.5 变更与恢复的工程验收

更换 VFD、恢复参数或修改 PLC 电机模块后，应执行方向点动、最小与最大速度、加减速、急停或 STO、故障反馈、通信丢失策略和工艺许可测试。对于泵和风机，还要确认旋转方向、阀门位置、压力流量与振动；对于输送和提升设备，要确认制动、限位、张力及机械位置。

恢复使用的参数备份必须与设备型号、固件、电机铭牌和现场接线匹配。直接向替换设备批量导入旧参数，可能覆盖电机保护、编码器类型或安全功能配置。可信恢复应包含批准版本、差异审阅、离线校验、受控下载、现场功能测试和观察期，而不只是“通信恢复正常”。

## 11. 工控场景：泵自动重启造成维护风险

泵因瞬时欠压停机，VFD 被配置为电源恢复后自动重启。维护人员看到设备停止后接近联轴器，但未执行隔离上锁；电压恢复后驱动器自行启动。

风险涉及参数、状态显示、作业许可和能量隔离。防护包括锁定挂牌、STO/隔离装置、自动重启状态告警和维护模式禁止远程启动。

## 12. 防护与治理

- 对 VFD 参数、命令源、自动重启和保护定值建立基线；
- 维护依赖物理能量隔离，不依赖软件停止；
- 启停、方向、复位和速度上限实施最小权限；
- 同时监测命令、运行反馈、电流、速度和过程响应；
- 电气保护与 PLC 联锁分层设计并定期试验；
- 网络中断时定义驱动器保持、停车或本地控制策略；
- 设备替换后验证电机铭牌、方向、保护和控制源。

## 13. 面试表达

### 60 分答案

> PLC 通过 MCC、接触器、软启动器或 VFD 控制电机，保护继电器负责过流、过载、缺相和过温等保护。命令位不等于电机真实运行，要看反馈和过程响应。

### 90 分答案

> 我会把电机控制分为控制回路、动力回路、保护回路和反馈回路。分析远程启动时，会确认控制权、许可、PLC 输出、中间继电器、接触器或 VFD 状态、电流转速以及最终流量或机械运动。对于 VFD 还要审阅命令源、频率上下限、加减速、自动重启、保护参数和 STO。维护安全必须依靠可验证的能量隔离，不能把 HMI 停止或网络命令当作隔离措施。

### 常见追问

**问：运行反馈为 1 是否证明泵在送液？**

答：不能，只能证明某一级状态。还需电流、转速、阀位、压力和流量验证。

**问：普通 Stop 与 STO 有什么区别？**

答：Stop 是功能控制命令，可能失效或被重启；STO 是按安全设计阻止产生转矩的功能，具体等级和接线需按设备验证。

## 14. 实操建议

### 实验一：电机状态机

用仿真 PLC 建立许可、启停、反馈超时、最小停机时间和跳闸复位，观察命令与状态分离。

### 实验二：VFD 参数审阅

在离线手册或仿真环境整理命令源、速度限值、斜坡、自动重启和保护参数基线，不连接生产驱动器。

### 实验三：过程验证

模拟运行反馈正常但流量为零，建立电气、机械和工艺三类可能原因及证据表。

### 实验四：攻击链与阻断点

在离线模型中构造“命令源切换—上限修改—远程启动—反馈隐藏”序列，逐项记录所需权限、驱动器事件、PLC 状态、电流速度与过程响应。分别加入硬接线许可、参数基线和多变量监测，比较每个防护能阻断或发现哪一步。

### 实验五：维护隔离验证

使用桌面推演区分 HMI Stop、VFD 禁止、STO、断路器断开和锁定挂牌的证据边界。给出电源恢复、远程复位、重力回转和残余压力四种情形，说明仅有软件状态为何不能证明零能量。

## 15. 总结与检查清单

- [ ] 能区分控制、动力、保护和反馈回路；
- [ ] 能解释 MCC、接触器、软启动和 VFD；
- [ ] 能审阅命令源、速度、斜坡、自动重启和保护；
- [ ] 能沿命令—电气状态—机械动作—过程响应验证；
- [ ] 能区分软件停止、设备跳闸和安全能量隔离；
- [ ] 能把参数操纵映射到机械与生产后果。
