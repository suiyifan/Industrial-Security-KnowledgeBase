# 顺序控制、状态机与批次过程

连续控制关注变量如何稳定在目标附近，顺序控制关注设备和工艺步骤能否按合法条件迁移。生产线启停、反应釜投料、CIP 清洗、压缩机启动和批次配方都依赖状态机。攻击者不一定修改模拟量，只要跳过步骤、伪造完成条件或改变配方，就可能造成高后果。

## 1. 原理解释：为什么需要状态与步序

复杂设备不能靠一组互不相关的开关命令安全运行。状态机明确：

- 当前处于什么状态；
- 哪些条件允许进入下一状态；
- 进入、保持和退出状态时执行什么动作；
- 超时、故障和人工中止时怎样恢复；
- 哪些联锁在所有状态下始终有效。

```text
Idle → Preparing → Starting → Running → Completing → Idle
  ↘ Faulted ← 任意异常状态 → Recovering ───────────────┘
```

状态变量是对真实设备行为的抽象。程序处于 `Running` 不等于设备真的运行，仍需反馈和过程响应验证。

## 2. 工程实现：状态机的组成

### 2.1 状态

状态应互斥、可解释并覆盖异常。常见状态包括停止、准备、启动、运行、暂停、完成、故障、复位和维护。

### 2.2 转移条件

转移条件通常由命令、许可、反馈、定时器和过程条件组合。例如泵启动完成可能要求：启动命令存在、电机运行反馈到达、出口压力建立且未超时。

### 2.3 状态动作

动作可分为进入动作、持续动作和退出动作。进入状态时清零计时器，持续状态时保持阀门，退出时释放资源。动作定义不完整会留下“上一步输出残留”。

### 2.4 超时与异常分支

每个等待现场反馈的步骤都应有合理超时和失败去向。超时不能简单跳过，否则会把未完成动作当作完成。

## 3. 许可、联锁、报警和跳闸

- **许可 Permissive**：启动或进入状态前必须满足；
- **联锁 Interlock**：运行中持续约束危险动作；
- **报警 Alarm**：通知人员关注，不一定自动动作；
- **跳闸 Trip**：满足危险条件后自动停止或进入安全状态；
- **旁路 Bypass**：经授权临时忽略特定条件；
- **复位 Reset**：故障原因消除后清除锁存状态，不等于强制恢复运行。

安全审阅要检查这些概念是否被混用。例如把关键低液位保护只做成报警，会把自动保护依赖于人工反应。

## 4. 模式与命令仲裁

典型模式包括就地/远程、手动/自动、维护/生产和单步/连续。多个命令来源必须有明确优先级：

```text
紧急停车 > 安全跳闸 > 就地保护 > 联锁停止
        > 操作员停止 > 自动顺控 > 启动请求
```

实际项目优先级由危害分析确定。模式切换应验证当前状态，避免从维护模式直接恢复到自动后设备突然启动。

## 5. SFC、梯形图与结构化文本实现

- **SFC/GRAFCET**直观表达步、转移和并行分支；
- **梯形图**常用锁存线圈、比较、定时器和步位实现；
- **结构化文本**适合 `CASE` 状态机、配方和复杂计算；
- **功能块**封装设备状态、命令和诊断，便于复用。

阅读时先找状态变量或步位，再找每个状态的动作与转移，不要从单个输出线圈逆向猜全部流程。

## 6. 顺控中的并行、互斥与竞态

多个设备可能并行启动，也可能共享管线、机器人区域或物料源而必须互斥。常见问题：

- 两个分支同时争用同一阀门；
- 一个分支完成后过早释放公共资源；
- 快速输入在两个任务间被重复消费；
- 通信延迟使完成反馈晚于超时；
- 复位同时清除了尚未解决的多个故障；
- 重启后保持步位与现场真实状态不一致。

状态机的安全性依赖原子性、优先级和恢复设计，不只是步骤顺序。

## 7. 批次控制与配方

批次过程按配方生产有限数量产品。可用层级理解：过程单元、单元、设备模块和控制模块。配方通常包含：

- 原料与目标量；
- 步骤顺序与并行关系；
- 温度、压力、搅拌速度等参数；
- 保持时间和完成条件；
- 异常、暂停、恢复与报废规则；
- 批号、操作员和审计记录。

修改配方可能不改 PLC 程序，却能改变投料比例、处理时间和质量。配方服务器、下载路径和审批流程属于关键攻击面。

## 8. 数据流、控制流、状态流和物料流

以反应釜批次为例：

```text
配方参数 → 批次引擎 → 步序状态 → 设备命令
现场反馈 → 完成条件/联锁 → 状态迁移
物料流：投料 → 搅拌 → 加热 → 保温 → 冷却 → 出料
```

状态变化应与物料和能量变化相符。若程序显示“投料完成”，但储罐重量、阀位和流量累积不支持该结论，就需要调查反馈或逻辑。

## 9. 异常恢复与重启一致性

停电、控制器重启或网络中断后，程序必须重新确定现场状态：

- 哪些状态和累计量需要保持；
- 哪些输出必须清除；
- 半成品能否继续处理；
- 阀门和电机是否处于预期位置；
- 是否需要人工确认或退回安全步骤；
- 重复执行某一步是否会二次投料或重复动作。

“从上次步号继续”可能造成危险；“全部从头开始”也可能重复投料。恢复策略必须结合物料实际状态。

## 10. 安全视角：顺控操纵路径

顺控安全风险通常表现为“程序声称的步骤”和“现场真实物料/设备状态”分离。高风险变量包括步号、完成位、配方参数、模式、旁路、累计量和恢复选择。

### 10.1 跳步与伪造完成

强制完成位或修改转移条件，可绕过吹扫、预热、阀门到位和质量检查。

### 10.2 模式与许可操纵

切换维护模式、强制远程或屏蔽许可，可能让正常界面上的启动命令变得危险。

### 10.3 配方篡改

改变投料量、温度、保持时间或顺序，可导致质量事故、危险反应或设备超限。参数仍处于单个变量允许范围，不代表组合安全。

### 10.4 恢复逻辑滥用

反复触发重启、复位或断网，可能让状态机进入未充分测试的恢复路径，造成重复动作或保护失效。

### 10.5 典型攻击链

```text
取得配方、步位或维护命令写权限
 → 跳步、伪造完成、改变参数或切换模式
 → 状态机进入设计外或条件不完整的状态
 → 阀门、电机和物料动作顺序改变
 → 质量、设备或过程安全边界被突破
 → 联锁、SIS 与人工检查决定最终后果
```

验证该攻击链必须比较批次版本、状态事件、设备反馈、流量/重量累计和实验室质量，不能只看当前步号。

## 11. 批次完整性与事务边界

一个批次动作常由多个系统共同完成：MES 下发工单，批次服务器选择配方，PLC 执行步骤，称重或流量累计证明投料，实验室系统给出质量结果。如果这些系统分别更新而没有一致的批次标识，就可能出现：

- 新配方配旧批号；
- PLC 已执行但上层认为失败而重复下发；
- 配方版本改变但审计只记录名称；
- 网络中断后离线结果重复补传；
- 人工补料没有进入自动累计；
- 批次暂停期间过程仍在发生物理变化。

工程上可使用批次 ID、配方版本、步骤序列号、完成确认和不可变审计建立事务边界。安全监测应关注跨系统标识是否一致。

## 12. 暂停、中止与恢复策略

- **Hold/Pause**：暂时停止步序推进，但要定义搅拌、加热和计时是否继续；
- **Abort**：中止批次并进入受控安全状态，可能需要排空或隔离；
- **Restart**：从已验证状态重新开始，不等于简单恢复旧步号；
- **Recovery**：根据现场物料、设备位置和质量决定继续、返工或报废。

暂停状态本身不是静止状态。反应、冷却、沉降和挥发可能继续发生，恢复窗口需要工艺判断。

## 13. 工控场景：CIP 清洗跳过冲洗步骤

清洗流程为排空、预冲洗、碱洗、过渡冲洗、酸洗、终冲洗和生产就绪。攻击者伪造电导率达标或强制步骤完成，使流程提前进入生产就绪。HMI 显示顺控完成，但管路仍残留清洗剂，造成产品污染和腐蚀风险。

调查证据包括步序事件、配方版本、电导率原始趋势、阀门反馈、流量累积、操作确认和实验室质量结果。

## 14. 防护与工程治理

- 对状态机、转移条件、配方和模式仲裁建立版本基线；
- 关键步骤使用独立或多条件完成证明；
- 对跳步、单步、旁路、强制完成和配方下载实施高权限与审计；
- 状态变化记录时间、操作者、来源和原因；
- 设计明确的超时、故障、暂停、中止和恢复路径；
- 重启后执行现场状态核对，不盲信保持变量；
- 对高后果顺控进行仿真、异常注入和恢复演练。

## 15. 面试表达

### 60 分答案

> 顺序控制用状态和转移条件组织设备动作，许可决定能否启动，联锁和跳闸在运行中阻止危险。批次控制还包括配方、步骤参数和批次记录。

### 90 分答案

> 我会从状态、转移条件、状态动作、超时和异常恢复五部分审阅顺控，再核对模式仲裁、许可、联锁、报警和跳闸的优先级。安全上重点关注强制完成、跳步、配方修改、保持步位和重启恢复，因为程序状态可能与真实阀位、物料量和设备状态不一致。验证时要把步序事件与流量累积、重量、阀位反馈和质量数据关联，而不是只看 HMI 显示“步骤完成”。

### 常见追问

**问：报警、联锁和跳闸有什么区别？**

答：报警主要通知人，联锁持续约束动作，跳闸在危险条件下主动改变状态。具体实现以项目因果矩阵和逻辑为准。

**问：为什么控制器重启后不能直接继续原步？**

答：保持的程序状态可能与现场物料和设备位置不一致，继续可能重复投料或跳过必要动作，需要重新验证状态。

## 16. 实操建议

### 实验一：泵状态机

用仿真 PLC 实现 Stopped、Starting、Running、Faulted 和 Resetting，加入启动许可、反馈超时和停止优先级。

### 实验二：异常注入

分别模拟反馈不来、反馈迟到、控制器重启和强制完成，记录状态、输出、报警与恢复结果。

### 实验三：配方差异

建立两版简单配方，用 Git 或表格比较参数与步骤差异，并说明每项变化可能影响的物料和质量。

### 实验四：暂停与恢复

在加热或搅拌步骤中触发暂停，分别设计“物理动作继续”和“全部动作停止”两种语义，观察计时、温度和恢复结果。形成恢复前现场核对清单。

### 实验五：跨系统批次一致性

用两个简单脚本或表格分别模拟上层批次服务器与 PLC。上层发送批次 ID、配方版本和步骤序列号，PLC 返回接收与完成状态。依次注入重复下发、确认丢失、旧配方重放和结果补传，观察没有去重机制时怎样发生重复动作。随后加入唯一批次 ID、单调序列号、有效期和幂等确认，比较状态差异。

实验记录必须包含每次消息、PLC 状态、物料累计和恢复决定。该实验用于理解事务边界，不得连接真实批次或 MES 系统。

完成后再执行一次控制器重启，验证持久化的批次标识、步骤状态与实际累计量是否一致，并记录需要人工确认的恢复条件。

若无法自动证明一致，应保持设备停止并由工艺与操作人员共同决定继续、返工或报废。

## 17. 总结与检查清单

- [ ] 能画出含异常与恢复路径的状态机；
- [ ] 能区分许可、联锁、报警、跳闸、旁路和复位；
- [ ] 能解释模式与多命令来源的优先级；
- [ ] 能识别并行、互斥、超时和重启一致性风险；
- [ ] 能把配方与步序变化映射到物料、质量和安全后果；
- [ ] 能使用现场反馈而非单一步位证明步骤完成。
