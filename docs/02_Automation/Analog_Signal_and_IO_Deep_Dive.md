# 4~20mA、I/O 与真实世界连接

## 1. 原理解释：PLC 为什么需要 I/O

PLC 不能直接“看到”温度、压力、液位，也不能直接“推动”阀门、电机、水泵。

它看到的是电信号，输出的也是电信号。

I/O 模块负责把物理世界编码成数字世界，再把数字世界解码成物理动作。

## 2. 工程实现：4~20mA 信号链路

4~20mA 是工业连续变量中非常常见的模拟信号。

完整链路：

```text
压力 / 温度 / 液位
  -> 变送器
  -> 4~20mA 电流信号
  -> AI 模块
  -> ADC 转换
  -> 工程量 PV
  -> PID 计算
  -> AO 模块
  -> 4~20mA 输出
  -> 阀门定位器
  -> 调节阀动作
  -> 工艺过程变化
```

## 3. 为什么是 4~20mA

4~20mA 的设计不是随意的：

- `4mA` 表示真实零点。
- `0mA` 可以表示断线故障。
- 电流信号适合长距离传输。
- 抗干扰能力比电压信号更强。
- 便于量程映射。

例子：

```text
4mA  -> 0%
20mA -> 100%
12mA -> 50%
```

如果液位量程是 0 到 10 米，12mA 可能代表 5 米。

## 4. I/O 类型

| 类型 | 含义 | 例子 | 安全关注 |
|---|---|---|---|
| DI | 数字量输入 | 急停按钮、限位开关 | 状态是否被伪造 |
| DO | 数字量输出 | 电机启动、继电器输出 | 是否被异常启停 |
| AI | 模拟量输入 | 温度、压力、液位 | PV 是否可信 |
| AO | 模拟量输出 | 阀门开度、变频器频率 | 输出是否被篡改 |

## 5. 数据流：从现场到 HMI

以压力变送器为例：

```text
真实压力
  -> 变送器输出 4~20mA
  -> AI 模块采样
  -> PLC 内部原始值
  -> 工程量换算
  -> 数据块 / 寄存器
  -> SCADA / HMI 读取
  -> 操作员看到压力数值
```

如果 HMI 显示 1.2MPa，背后经历了传感器、模拟信号、AI、工程量转换、通信协议和画面绑定。

## 6. 安全视角：攻击点

可能攻击点：

- 传感器本体
- 变送器参数
- 4~20mA 信号线
- AI / AO 模块
- 工程量转换参数
- PLC 内部变量
- 通信协议
- HMI 画面绑定

### 反馈欺骗

攻击 AI 或 PV 数据，可能让控制器看到假的过程值。

### 执行欺骗

攻击 AO 或执行机构，可能让控制器输出正确但现场动作错误。

### 量程篡改

如果工程量转换比例被改，HMI 显示值和真实物理量会不一致。

## 7. 工业案例：液位量程被篡改

假设原本：

```text
4mA  -> 0m
20mA -> 10m
```

被错误改成：

```text
4mA  -> 0m
20mA -> 20m
```

真实 8m 液位可能被显示成 16m 或导致控制逻辑判断错误，具体取决于转换方式。

安全分析要关注：

- 量程参数谁能改？
- 修改是否有审批？
- HMI 和 PLC 使用的量程是否一致？
- 历史趋势是否能发现突变？

## 8. 面试表达

### 60 分答案

> PLC 通过 I/O 模块连接真实世界。传感器把温度、压力、液位等物理量转换成电信号，AI 模块再转换成 PLC 内部变量；PLC 计算后通过 DO 或 AO 控制电机、阀门等执行器。

### 90 分答案

> 我会把 I/O 看成物理过程和控制程序之间的翻译层。比如 4~20mA 不只是电气信号，它承载了压力、液位、阀门开度等工程量。安全分析时，攻击不一定发生在 PLC 程序，也可能发生在变送器、量程映射、AI/AO 模块或 HMI 画面绑定上。最终判断风险时要看这个信号对应什么物理动作或控制回路。

### 常见追问

问：为什么 0mA 不表示 0%？

答：

> 因为 4mA 表示真实零点，0mA 通常用于表示断线或故障。这样系统能区分“测量值为零”和“信号链路断开”。
