# 控制理论深挖：从闭环稳定到攻击影响

## 1. 原理解释：控制的本质

自动控制不是为了“让设备自己动”，而是为了让工业过程在扰动下保持稳定、安全、连续运行。

工业对象不是静止的。锅炉压力、反应器温度、精馏塔液位、压缩机转速会持续受到扰动：

- 原料波动
- 负荷变化
- 环境温度变化
- 阀门磨损
- 传感器漂移
- 工艺耦合
- 网络和采样延迟

控制系统真正控制的是过程变量 `PV`，不是 PLC 或阀门本身。PLC、DCS、调节阀、变送器只是实现控制的工具。

基本结构：

```text
设定值 SP
  -> 控制器
  -> 执行机构
  -> 被控对象
  -> 传感器
  -> 反馈 PV
  -> 控制器
```

## 2. 工程实现：SP、PV、MV/CV 如何工作

常见变量：

| 名称 | 含义 | 工程例子 |
|---|---|---|
| SP | Setpoint，设定值 | 反应器目标温度 80 摄氏度 |
| PV | Process Variable，过程值 | 当前温度 76.5 摄氏度 |
| MV / CV | Manipulated Variable / Control Output，控制输出 | 蒸汽阀门开度 65% |
| Error | 偏差 | `SP - PV` |

工程中，操作员可能在 HMI 上修改 SP，控制器读取 PV，PID 算法计算输出 MV，再通过 AO 模块或通信协议控制阀门、变频器或执行器。

## 3. 数据流：一次闭环控制如何流动

以水箱液位为例：

```text
液位传感器
  -> 4~20mA 信号
  -> AI 模块
  -> PLC/DCS 内部 PV 变量
  -> PID 计算
  -> AO 输出或通信写入
  -> 阀门 / 水泵
  -> 水箱液位变化
  -> 液位传感器再次反馈
```

这条链路中任何环节被篡改，都会改变控制结果。

## 4. 开环与闭环

开环控制只执行动作，不知道结果。

```text
打开水阀 30 分钟 -> 关闭
```

闭环控制会测量结果并不断修正。

```text
目标液位 60%
当前液位 45%
控制器增大进水
液位上升
传感器反馈新液位
```

反馈的价值在于：控制器不需要知道扰动是什么，只需要知道结果偏离了目标。

## 5. 动态系统：为什么不能只看当前数值

工业对象通常不是“输入立即变输出”的静态对象，而是具有动态特性：

- 惯性：锅炉、反应釜、液位、电机不会瞬间变化。
- 延迟：热传递、物料运输、传感器采样、网络通信都会滞后。
- 非线性：阀门开度和流量不一定线性。
- 耦合：一个变量变化可能影响多个控制回路。

安全分析时，看到一次写寄存器不能只问“写了什么值”，还要问：

- 这个值作用于哪个对象？
- 对象响应多快？
- 是否有滞后？
- 是否会传播到其他回路？
- 最终是收敛、振荡还是失稳？

## 6. 一阶系统、二阶系统与振荡

很多液位、温度、压力过程可以近似为一阶系统。它们通常表现为指数响应：离目标越远变化越快，越接近目标变化越慢。

二阶系统可能出现超调和振荡，例如电机、机械系统、带惯性的转动设备。

PID 参数会影响系统响应：

- `Kp` 增大：响应更快，但可能超调。
- `Ki` 增大：消除稳态误差，但可能积分饱和。
- `Kd` 增大：增加阻尼，但对噪声敏感。

## 7. 安全视角：攻击闭环的四种方式

### 攻击 SP

改变控制目标。

例子：把反应器温度设定值从 80 改到 95。

### 攻击 PV

伪造反馈，让控制器看到假的世界。

例子：真实液位已经高高限，但反馈仍显示 60%。

### 攻击 MV / CV

控制器计算正确，但执行机构收到错误输出。

例子：PID 输出应关小阀门，但阀门实际被打开。

### 攻击控制参数

修改 `Kp/Ki/Kd`、采样周期、滤波参数，使系统从稳定变成振荡或响应迟缓。

## 8. 工业案例：反馈欺骗

假设水箱真实液位已经升到 95%，但攻击者让 PLC / HMI 看到的 PV 始终是 60%。

可能后果：

- 操作员误以为系统正常。
- 控制器继续保持进水。
- 高高液位报警可能不触发。
- 最终导致溢流或设备联锁动作。

这类攻击比直接停机更隐蔽，因为系统“自认为正常”。

## 9. 面试表达

### 60 分答案

> 自动控制的核心是通过传感器、控制器和执行机构形成闭环，让过程变量在扰动下保持稳定。工控安全分析时，我会关注 SP、PV 和控制输出是否可信，因为修改设定值、伪造反馈或改变输出都可能影响真实物理过程。

### 90 分答案

> 我会把控制系统看成一条闭环链路：SP 进入控制器，控制器根据 PV 计算输出，执行机构改变物理过程，再由传感器反馈。攻击不一定要直接控制设备，也可以攻击反馈链路、控制参数或采样周期，让系统在“看似正常”的状态下逐渐偏离。比如修改 PID 参数可能不是立刻改变 PV，而是改变闭环动态特性，让系统产生振荡、超调或响应迟缓。

### 常见追问

问：为什么反馈攻击危险？

答：

> 因为控制器和操作员都依赖反馈判断现场状态。如果反馈被伪造，系统可能在真实异常时仍认为正常，导致错误控制持续存在。

## 10. 实操验证

建议后续用以下路线验证：

1. 用 Scilab Xcos 或 MATLAB/Simulink 建立一阶液位对象。
2. 调整 `Kp/Ki/Kd`，观察上升时间、超调、振荡。
3. 在 OpenPLC 或 CODESYS 中把 SP、PV、Kp、Ki、Kd 映射到寄存器。
4. 用 Python 修改 SP 或 PID 参数。
5. 用 Wireshark 抓取写寄存器流量。
6. 观察 PV 和输出变化曲线。
