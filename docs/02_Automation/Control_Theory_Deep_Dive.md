# PID 控制与工程整定

PID 是过程工业中最常见的反馈控制算法，但安全分析不能停留在“P 看当前、I 看历史、D 看趋势”。还要理解控制方向、输出限幅、积分饱和、滤波、手自动切换、串级关系和执行器约束，因为这些工程细节决定参数修改或反馈操纵是否真正形成物理影响。

## 1. 原理解释：PID 如何修正误差

理想并联形式可写为：

```text
u(t) = Kp × [e(t) + 1/Ti × ∫e(t)dt + Td × de(t)/dt]
e(t) = SP(t) - PV(t)
```

- **比例 P** 根据当前误差立即响应；
- **积分 I** 累积历史误差，消除稳态偏差；
- **微分 D** 根据误差变化趋势提前抑制快速变化。

厂商可能采用增益或比例带、积分时间或重复次数、理想式或并联式等不同定义。同一组数字不能脱离算法形式直接复制到另一平台。

## 2. 工程实现：P、I、D 的真实作用

### 2.1 比例作用

增大 `Kp` 通常使响应更快、偏差更小，但会放大噪声并降低稳定裕度。只有比例控制时，为维持非零输出往往需要保留一定误差，这就是比例偏差。

### 2.2 积分作用

积分会持续累积误差，直到消除稳态偏差。积分过强可能造成慢振荡和超调；执行器饱和时积分仍继续累积，会导致解除饱和后输出长时间无法回归。

### 2.3 微分作用

微分对快速变化敏感，可增加阻尼，但也会放大测量噪声。工程实现通常对微分项滤波，并常对 PV 而非误差求导，以避免 SP 阶跃导致“微分冲击”。液位、流量等回路不一定需要 D。

## 3. 正作用、反作用与控制方向

控制方向由“MV 增加会让 PV 怎样变化”和执行器作用共同决定：

- 加热功率增加使温度上升，温度高于 SP 时应减少输出；
- 冷却阀开度增加使温度下降，温度高于 SP 时应增加输出；
- 出料阀开度增加使液位下降；进料阀开度增加使液位上升。

控制方向配置错误会形成正反馈：偏差越大，控制器越把过程推向错误方向。上线前应通过小幅、受控的输出变化验证过程方向，不应仅依据标签名判断。

## 4. 输出限幅、速率限制与死区

实际 MV 受物理约束：阀门只能在 0—100% 行程内，电机频率、加热功率和阀位变化速度也有限。

- **上下限**保护设备和工艺边界；
- **速率限制**避免输出突变造成水锤、热冲击或机械冲击；
- **死区**减少阀门频繁动作，但会降低小偏差响应；
- **最小开度/最小转速**保证设备在有效工作区；
- **分程控制**让一个输出区间驱动多个执行器。

修改这些约束可能比修改 PID 参数更直接地产生危险。

## 5. 积分饱和与抗饱和

当输出已到上限但误差仍存在，积分项可能继续增长。之后即使误差反向，控制器也要先“消化”积累量，造成长时间饱和。

常见抗饱和方法：

- 输出饱和时停止积分；
- 只允许能使输出离开饱和方向的积分；
- 用实际受限输出反算积分状态；
- 手动模式下跟踪实际输出。

安全监测应关注输出长期贴边、积分状态异常和执行器实际输出与控制器计算输出不一致。

## 6. 手动、自动、串级与无扰切换

### 6.1 手动模式

操作员直接设置 MV，PID 内部仍应跟踪实际输出，为切回自动做准备。手动模式适合检修和异常处置，但也可能绕过正常调节。

### 6.2 自动模式

PID 根据本地 SP 调节。操作员或程序改变 SP，控制器自动响应。

### 6.3 串级模式

主回路输出成为副回路 SP。副回路快速处理局部扰动，主回路维持最终目标。任一回路掉出串级，控制链都会改变。

### 6.4 无扰切换

切换前使 SP、积分状态或内部输出跟踪当前值，避免 MV 跳变。需要审计模式位、跟踪值、远程 SP 和切换原因。

## 7. 前馈、比值、分程与选择控制

- **前馈控制**根据可测扰动提前补偿；
- **比值控制**维持两种流量或原料的比例；
- **分程控制**用同一控制输出驱动加热/冷却或两只阀；
- **高低选择器**从多个控制需求中选择最安全或最约束的输出；
- **约束控制**在设备极限附近接管正常控制目标。

安全人员如果只看最终 PID 块，可能漏掉上游选择器和下游限幅逻辑。一次写入能否生效，要沿整个信号链追踪。

## 8. 整定原则与方法

整定目标是在响应速度、超调、鲁棒性、执行器磨损和扰动抑制之间折中。

### 8.1 经验试调

在安全、稳定工况下逐步增加比例作用，再加入积分，必要时加入微分。每次只改变一类参数并等待足够的过程响应。生产系统不得为“试效果”随意制造大阶跃。

### 8.2 阶跃试验与对象模型

在手动模式对 MV 做受控小阶跃，估计对象增益、时间常数和滞后，再计算初始参数。试验前需确认工艺窗口、联锁、停止条件和操作授权。

### 8.3 继电反馈或临界比例法

通过受控振荡估计临界增益和周期。此类方法可能引起过程波动，不适合高风险或约束紧的生产设备。

### 8.4 自动整定

自动整定仍需要正确的控制方向、测量质量、输出权限和过程稳定条件。自动生成参数不能替代工程确认。

## 9. 参数变更后的验证

参数“能下载”不代表变更安全。至少验证：

1. 算法形式和单位是否一致；
2. 控制方向、输出限幅和速率限制是否正确；
3. 手动/自动/串级切换是否无扰；
4. SP 阶跃和常见扰动下是否稳定；
5. 输出是否频繁动作或长期饱和；
6. 报警、联锁与独立保护是否仍有足够响应时间；
7. 变更记录、旧参数和回退方案是否完整。

## 10. 数据流、控制流与状态流

```text
SP、PV、模式、前馈 → PID 内部 P/I/D 状态
                    → 限幅/速率/选择逻辑
                    → MV → 执行器 → 过程

状态：Manual / Auto / Cascade / Tracking / Bad-PV / Saturated
```

PID 不只是一个输出数值，还具有积分记忆、模式、质量、饱和和跟踪状态。事件调查应保留这些状态，而不能只导出 SP/PV 趋势。

## 11. 安全视角：PID 操纵路径

PID 安全风险不局限于三参数被改。模式、跟踪、限幅、选择器、前馈和下游执行器参数共同决定最终输出，因此应比较完整控制块和信号链。

### 11.1 参数缓慢劣化

逐步提高比例增益或缩短积分时间，可能让回路从稳定变为轻微振荡，再演化为设备磨损或质量波动。单次参数差异不大，但趋势异常。

### 11.2 上下限与模式操纵

放宽温度 SP 上限、解除输出速率限制或切换手动，比直接改 PID 三参数更容易产生确定动作。高风险变量应按控制语义分类审计。

### 11.3 反馈与状态协同欺骗

攻击者冻结 PV 并把回路保持自动，积分会不断推动输出饱和；若同时伪造 MV 显示，操作员可能更晚发现。独立现场反馈和过程平衡是关键证据。

### 11.4 参数恢复不完整

只恢复 `Kp/Ti/Td` 而未恢复限幅、跟踪值、模式和积分状态，过程仍可能异常。恢复需要比较完整控制块配置和运行状态。

### 11.5 典型攻击链

```text
取得工程参数或操作权限
 → 修改 PID 参数、模式、限幅或远程 SP
 → P/I/D 内部状态和计算输出改变
 → 下游选择与执行器约束决定实际 MV
 → 过程按滞后与惯性发生偏移
 → 报警、联锁和 SIS 是否在过程安全时间内动作
```

证据应包含变更身份、前后参数、算法形式、模式、积分状态、计算 MV、实际阀位和过程趋势。仅看到 `Kp` 变化，无法证明已形成特定后果。

## 12. 回路性能异常的诊断顺序

回路振荡或控制差时，按以下顺序排查：

1. PV 是否真实、质量良好且单位量程正确；
2. 执行器是否卡涩、死区、饱和或控制方向错误；
3. 模式、串级、选择器和限幅是否符合预期；
4. 过程负荷或工作点是否显著变化；
5. 任务周期、通信延迟和滤波是否变化；
6. 最后才判断 PID 参数是否需要调整。

常见现象与线索：MV 锯齿可能是阀门卡涩；PV 噪声引起 MV 高频动作可能是微分或滤波问题；MV 长期饱和而 PV 无法到达 SP，可能是设备能力不足而非参数太弱。

## 13. 工控场景：反应器温度回路被改为强积分

攻击者缩短积分时间并放宽蒸汽阀输出上限。进料温度下降形成负偏差后，控制器快速积累并把蒸汽阀推到高位；由于反应器存在较长热滞后，PV 尚未上升时积分继续累积，最终产生明显超调。

完整风险链：

```text
PID 参数与输出限制被改
 → 热滞后期间积分持续累积
 → 蒸汽阀长期高开度
 → 反应器温度超调
 → 高温报警/BPCS 限制/SIS 是否及时动作
 → 产品失控、设备超温或停车
```

调查应比较工程版本、在线参数、模式事件、SP/PV/MV 趋势、阀位反馈、蒸汽流量和保护动作时间。

## 14. 防护与监测

- 对 PID 参数、控制方向、限幅、模式和远程 SP 建立基线；
- 将在线参数变更与工单、操作员身份和工程版本关联；
- 检测长期饱和、振荡增强、模式频繁切换和执行器过度动作；
- 使用独立测量或软测量校验关键 PV；
- 参数变更实行双人复核，并保留一键可用的已验证回退集；
- 对高后果回路使用独立保护层限制最坏结果。

## 15. 面试表达

### 60 分答案

> PID 用比例、积分和微分处理当前误差、累计误差和变化趋势。P 提高响应，I 消除稳态误差，D 增加阻尼。参数过强会造成超调或振荡。

### 90 分答案

> 我分析 PID 不只看三参数，还会确认厂商算法形式、控制方向、采样周期、滤波、输出上下限、速率限制、抗积分饱和以及手自动和串级跟踪。风险判断会把参数变化放回对象增益、时间常数和滞后中，观察 SP、PV、计算输出、实际阀位和过程响应。恢复时也要恢复模式、积分状态、限幅和选择逻辑，不能只把 Kp、Ti、Td 改回去。

### 常见追问

**问：为什么 PID 输出已经到 100%，PV 仍未到 SP？**

答：可能是执行器能力不足、阀门故障、对象负荷过大、控制方向错误、反馈错误或输出在下游被限制，需要沿命令—反馈—过程响应排查。

**问：参数改回原值后为什么还有超调？**

答：积分状态可能已累积，执行器和过程也有惯性；还要检查模式、跟踪、限幅和真实过程状态。

## 16. 实操建议

### 实验一：P、PI、PID 对比

在一阶滞后模型上分别使用 P、PI、PID，记录上升时间、超调、稳态误差和 MV 动作量。

### 实验二：积分饱和

限制 MV 上限并施加不可达到的 SP，比较无抗饱和与反算抗饱和的恢复时间。

### 实验三：无扰切换

在手动模式把 MV 调到非零，再切自动；比较有无输出跟踪时的突跳。仅在离线仿真环境进行。

### 实验四：诊断而非盲目整定

分别注入 PV 噪声、阀门死区和过程增益变化，让三种情况都表现为趋势波动。使用 PV、计算 MV、实际阀位和过程响应判断根因，避免把所有振荡都用降低 `Kp` 处理。

## 17. 总结与检查清单

- [ ] 能解释 PID 不同实现形式为何不能直接复制参数；
- [ ] 能判断控制方向、限幅和积分饱和；
- [ ] 能解释手动、自动、串级和跟踪状态；
- [ ] 能沿选择、前馈和限幅链追踪最终 MV；
- [ ] 能设计参数变更验证与回退；
- [ ] 能把 PID 操纵映射到过程动态、保护响应与物理后果。
