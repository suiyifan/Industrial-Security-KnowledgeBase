# 工业自动化系统全景

工业自动化的目标不是“让设备联网”，而是在人员无法持续、精确、快速操作的条件下，使物理过程按照安全、质量、产量和能耗目标运行。工控安全工程师必须先理解自动化系统正常情况下如何维持生产，才能判断某个网络包、变量写入、工程变更或主机漏洞是否真的会改变控制行为。

## 1. 原理解释：从生产目标理解自动化

一个工业过程通常同时受到多类约束：

- **安全约束**：压力不能超过容器设计上限，炉温不能进入危险区；
- **质量约束**：配比、温度、速度和加工尺寸必须处于工艺窗口；
- **产量约束**：流量、节拍、设备利用率需要达到计划；
- **设备约束**：电机不能频繁启动，泵不能干转，阀门不能超出行程；
- **环境约束**：排放、泄漏和废水指标不得超限；
- **经济约束**：能耗、原料消耗和维护成本需要优化。

控制系统把这些目标转换为测量、计算、动作和保护。安全事件的严重性也应围绕这些约束评估，而不能只套用“机密性、完整性、可用性”三个抽象词。

## 2. 最小控制闭环

最小闭环由四类对象组成：

```text
设定值 SP
   ↓
[控制器] -- 控制输出 MV/CV --> [执行器] --> [物理过程]
   ↑                                            |
   └--------- 过程值 PV ← [传感器/变送器] <----┘
                              ↑
                         扰动进入过程
```

- **物理过程**是被控制对象，例如水箱、锅炉、反应器、输送线或电机；
- **传感器与变送器**把温度、压力、流量、液位、位置等变成标准信号；
- **控制器**根据 SP、PV、模式、联锁和算法计算输出；
- **执行器**把电信号转换为阀位、转速、开关状态、机械位移或热量；
- **扰动**是影响过程但不完全受控制器支配的外部变化，例如进料波动、环境温度或下游负荷。

例如液位控制中，液位变送器提供 PV，操作员设定液位 SP，控制器调节进水阀开度 MV，出水流量变化构成扰动。篡改 SP、伪造 PV、固定 MV 和修改算法都能影响液位，但攻击路径、可见证据和保护响应完全不同。

## 3. 工程实现：自动化系统的功能层级

现场系统并不存在绝对统一的分层，但可以用以下功能层理解职责：

| 功能层 | 典型对象 | 主要职责 | 关键状态 |
| --- | --- | --- | --- |
| 物理过程层 | 管道、储罐、炉窑、输送线、发电机组 | 物料与能量转换 | 温度、压力、流量、液位、速度、位置 |
| 现场感知与执行层 | 传感器、变送器、阀门、电机、VFD、保护继电器 | 测量和执行 | 信号质量、阀位、转速、跳闸、就地/远程 |
| 控制层 | PLC、PAC、DCS 控制器、RTU、IED、SIS 逻辑解算器 | 实时控制、顺控、联锁与保护 | RUN/STOP、任务状态、模式、强制、旁路 |
| 监控操作层 | HMI、操作员站、SCADA 服务器、报警与历史服务器 | 人机操作、态势展示、记录和调度 | 会话、权限、报警、命令、趋势、通信质量 |
| 工程管理层 | 工程站、资产管理站、维护终端、域服务、补丁/备份服务器 | 组态、编程、维护和生命周期治理 | 工程版本、下载权限、备份、证书、审计 |
| 生产协同层 | MES、LIMS、排产、质量与能源管理 | 连接控制过程与经营计划 | 工单、配方、批次、质量结果、生产报表 |

层级只是分析工具，不等于可信边界。工程站可能同时连接管理网和控制网；远程 I/O 可能通过工业以太网接入控制器；历史数据可能向上复制；无线维护终端可能临时进入现场。评估必须以真实通信和权限关系为准。

## 4. PLC、DCS、SCADA 与 SIS 的职责差异

### 4.1 PLC

PLC 擅长确定性的逻辑、顺序和设备控制，常见于生产线、设备机组、泵站和公用工程。它按任务周期读取输入、执行程序、更新输出，也可能处理高速计数、运动控制和网络通信。

### 4.2 DCS

DCS 强调过程工业中的大量连续控制回路、统一工程数据库、操作管理、冗余和在线维护。它不是“更大的 PLC”，其价值在于控制、操作、报警、历史、权限和工程生命周期的系统化集成。

### 4.3 SCADA

SCADA 面向分散地理区域或多站点的监视与数据采集，典型对象包括电网、输油管线、水务和交通。中央系统通过 RTU、PLC、IED 或网关获得遥测和事件，并下发遥控、设定值或调度指令。通信中断时，现场控制往往仍需自治运行。

### 4.4 SIS

SIS 在预先定义的危险条件下执行安全动作，例如紧急停车、切断燃料或泄放压力。SIS 应与基本过程控制系统 BPCS 保持足够独立，避免同一故障同时破坏控制和保护。安全分析不能把 SIS 当作普通备用 PLC。

## 5. 四类流：工控安全分析的共同语言

### 5.1 数据流

描述数值和语义如何传播：

```text
压力 → 变送器电流 → AI 原始码 → 工程量 PV
     → 控制器标签 → 通信对象 → HMI 画面
     → 报警判断 → 历史归档 → 上层报表
```

每次转换都可能引入量程、单位、符号、字节序、时间戳和质量码问题。画面显示“5.0”没有意义，除非知道它是 5 bar、5 MPa、5% 还是失效后保留的最后值。

### 5.2 控制流

描述谁决定程序执行和动作：周期任务调用功能块，模式选择决定自动或手动路径，许可条件允许启动，联锁在危险条件下禁止或停止设备，操作员命令触发状态迁移。

### 5.3 状态流

描述设备和系统的离散状态变化，例如：

```text
Stopped → Starting → Running → Stopping → Stopped
               ↘ Faulted → Resetting ────────────┘
```

远程命令是否危险，取决于当前状态和迁移条件。对已运行泵写入“启动”可能没有效果，而绕过许可后从 Stopped 强制进入 Running 可能造成干转。

### 5.4 能量流

电能、压力能、热能、动能和化学能才是物理后果的直接载体。网络请求本身不会让管道破裂；它通过控制器和执行器改变阀位、转速或加热功率，进而改变过程能量平衡。风险分析必须走完这条映射链。

## 6. 生命周期：运行系统之外的攻击面

自动化系统经历设计、集成、调试、投运、运行、检修、升级和退役。每个阶段有不同风险：

- 设计阶段错误的安全假设会固化进逻辑和网络；
- 集成调试时常出现临时账号、旁路和宽松通信；
- 投运前需要验证量程、方向、联锁、报警和失效位置；
- 运行期强调变更审批、最小操作权限和持续监测；
- 停机检修期间远程维护、移动介质和临时接线增多；
- 升级与迁移可能带来版本不兼容、参数丢失和回退失败；
- 退役设备、工程备份和存储介质可能泄露工艺与凭据。

工控安全评估不能只扫描在线 IP，还要检查工程文件、备份介质、维护流程和恢复能力。

## 7. 正常运行、故障与攻击的区分

同一异常现象可能有多种原因。例如 HMI 液位突然固定：

- 过程确实稳定；
- 变送器卡死；
- 模拟输入模块故障；
- 通信中断后 HMI 保留最后值；
- 标签绑定错误；
- 历史查询时间范围错误；
- 攻击者重放或篡改数据。

判断需要交叉验证现场表计、I/O 诊断、PLC 在线值、通信质量、报警、趋势、控制输出和实际物料变化。单一数据源不能证明过程状态。

## 8. 安全风险如何映射到物理后果

可以用六步链路描述：

1. **入口**：工程站恶意软件、弱口令、远程维护或供应链文件；
2. **能力**：读取工程、写变量、下载逻辑、修改画面或抑制报警；
3. **控制变化**：SP、模式、联锁、输出或状态机被改变；
4. **过程变化**：压力、液位、速度、温度或设备状态偏离；
5. **保护响应**：BPCS、报警、操作员、机械保护或 SIS 是否阻断；
6. **后果与恢复**：停产、质量损失、设备损坏、环境释放或人身风险，以及恢复所需时间和证据。

这条攻击链必须逐环验证：拥有入口不等于获得工程权限，写入变量不等于输出动作，输出动作也不等于保护必然失效。任何未被工程文件、实验或现场证据支持的环节，都应作为假设单独标注。

### 工控场景：储罐液位操纵

攻击者取得 HMI 写权限后提高液位 SP。控制器正常增加进料阀开度，整个网络流量可能看起来合法。如果高液位报警被同时抑制，操作员失去早期提示；若独立高高液位联锁仍有效，SIS 或硬接线保护可能关闭进料。最终后果取决于保护层独立性、阀门失效位置、出料能力和响应时间，而不是“写操作成功”这一事实本身。

## 9. 防护原则

- 基于角色限制工程、操作、维护和审计权限；
- 将工程下载、强制、旁路、模式切换、配方下发列为高风险操作；
- 对关键工程变更实施双人复核、版本比较和可恢复备份；
- 监测命令语义、变量对象和控制器状态，而不只监测端口；
- 保证报警、历史、控制器诊断和时间同步可以形成证据链；
- 验证 BPCS、机械保护与 SIS 之间不存在未识别的共同原因失效；
- 恢复时同时核对数字状态与现场物理状态，避免带病复产。

## 10. 面试表达

### 60 分答案

> 工业自动化系统通过传感器感知过程、控制器执行逻辑、执行器改变现场，再由 HMI 和 SCADA 提供监视操作。PLC 常用于逻辑和设备控制，DCS 常用于连续过程，SCADA 常用于分布式监控，SIS 用于独立安全保护。

### 90 分答案

> 我会把自动化系统理解为物理过程、测量执行、实时控制、监控操作、工程管理和安全保护共同组成的闭环。安全分析不能止于某台主机是否存在漏洞，而要追踪数据流、控制流、状态流和能量流：攻击者改变了哪个工程对象或控制状态，怎样经过算法和执行器影响过程变量，报警、BPCS、人工操作和 SIS 是否能够发现并阻断，以及恢复时如何证明工程版本和现场状态一致。这样才能把技术漏洞转化为可信的工业风险结论。

### 常见追问

**问：自动化系统与普通 IT 系统最大的区别是什么？**

答：自动化系统直接控制持续变化的物理过程，实时性、确定性、生产连续性和安全约束更强。停机、重启、扫描和补丁等 IT 常规动作可能改变控制或生产状态。

**问：为什么不能只根据 Purdue 层级判断风险？**

答：层级描述功能位置，不自动等于真实信任边界。工程站、远程维护、历史数据同步和临时接入可能跨层通信，必须结合实际连接、身份、权限和命令语义判断。

**问：发现 PLC 可写是否等于存在高危物理风险？**

答：还需要确认可写对象、控制器模式、程序逻辑、当前过程状态、执行器能力和独立保护层。写入普通统计量与写入直接输出、联锁旁路或危险设定值的后果完全不同。

## 11. 实操建议与实验验证

### 任务一：画最小闭环

选择水箱或加热器，标出 SP、PV、MV、扰动、报警和联锁。为每条信号注明单位、正常范围、刷新周期和数据来源。

### 任务二：建立资产—功能矩阵

为 PLC、HMI、工程站、历史服务器和 SIS 填写：主要功能、可执行高风险动作、依赖对象、关键证据和恢复要求。

### 任务三：做一次异常假设

假设 PV 固定不变，分别列出过程稳定、传感器卡死、通信中断和显示欺骗的验证步骤。要求至少使用三个相互独立的数据源。

## 12. 总结与检查清单

- [ ] 能画出最小控制闭环并解释 SP、PV、MV 和扰动；
- [ ] 能区分 PLC、DCS、SCADA 与 SIS 的职责和边界；
- [ ] 能沿数据流、控制流、状态流和能量流分析一次操作；
- [ ] 能把数字行为映射到过程变量、保护响应和业务后果；
- [ ] 能说明为什么层级模型、单一日志或一次写入成功都不足以单独证明风险。
