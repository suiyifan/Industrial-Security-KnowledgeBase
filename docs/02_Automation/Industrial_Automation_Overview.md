# 工业自动化概览

## 1. 工业自动化是什么

工业自动化是用控制系统、仪表、软件和网络，让工业过程按照预定目标自动运行。

白话理解：

> 人不可能一直盯着温度、压力、液位、电机、阀门，也不可能每秒手动调节。工业自动化就是让传感器负责感知，让控制器负责判断，让执行器负责动作。

## 2. 工业自动化解决什么问题

主要解决五类问题：

1. 稳定：让温度、压力、液位、速度等维持在目标范围。
2. 效率：减少人工操作，提高生产效率。
3. 安全：在异常情况下报警、停机或联锁保护。
4. 质量：保证产品参数稳定。
5. 可追溯：记录历史数据、报警、操作日志。

## 3. 一个最小控制系统

最小控制系统通常包含：

- 传感器：测量现场状态。
- 控制器：根据逻辑做判断。
- 执行器：执行动作。
- 被控对象：真实物理过程。

示例：水箱液位控制。

```text
液位传感器 -> PLC -> 水泵 / 阀门 -> 水箱液位变化 -> 液位传感器
```

如果液位低于设定值，PLC 控制水泵启动；如果液位高于设定值，PLC 控制水泵停止或关闭阀门。

## 4. 工业控制系统常见组成

| 组件 | 作用 | 安全关注点 |
|---|---|---|
| 传感器 | 测量温度、压力、液位等 | 数据是否被伪造、干扰、篡改 |
| 执行器 | 控制阀门、电机、泵等 | 是否被异常启停或错误控制 |
| PLC | 现场逻辑控制 | 逻辑是否被篡改、是否未授权访问 |
| RTU | 远程测控终端 | 远程通信安全、弱认证 |
| DCS | 分布式控制系统 | 大规模流程控制、权限、组态安全 |
| SCADA | 监控与数据采集 | 远程监控、调度、数据完整性 |
| HMI | 人机界面 | 操作权限、误导显示、命令下发 |
| 工程站 | 编程、配置、下载工程 | 高价值目标，可能改控制逻辑 |
| 操作员站 | 日常监控与操作 | 操作权限、报警确认、误操作 |
| 历史数据库 | 存储趋势、报警、过程数据 | 数据完整性、追溯可信度 |

## 5. 自动化系统和普通 IT 系统的区别

| 对比项 | IT 系统 | OT / 工控系统 |
|---|---|---|
| 主要目标 | 数据、业务、效率 | 安全、稳定、连续生产 |
| 资产重点 | 服务器、数据库、应用 | PLC、DCS、HMI、现场设备 |
| 更新频率 | 相对频繁 | 非常谨慎，停机成本高 |
| 可用性要求 | 高 | 极高，有时不能随意重启 |
| 影响后果 | 数据泄露、业务中断 | 停产、设备损坏、人身安全风险 |
| 协议特点 | HTTP、数据库协议等 | Modbus、S7、OPC UA、IEC 104 等 |

## 6. 工控安全视角下的核心问题

安全研究时不要只问“有没有漏洞”，还要问：

- 攻击者能不能进入控制网络？
- 能不能识别资产和协议？
- 能不能读取过程数据？
- 能不能下发控制命令？
- 能不能修改工程文件和控制逻辑？
- 能不能隐藏报警、篡改趋势、误导操作员？
- 能不能影响安全联锁或紧急停机？

## 7. 本章面试表达

可以这样说：

> 我理解工业自动化系统的核心是传感器、控制器、执行器和物理过程构成的闭环。工控安全分析不能只看主机漏洞，还要判断漏洞处在控制链路的哪个位置，以及它是否可能影响控制逻辑、设备状态、报警显示、历史数据或安全联锁。
