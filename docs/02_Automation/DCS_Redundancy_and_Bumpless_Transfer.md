# DCS 冗余与无扰切换验证表

冗余设计只有经过受控测试才能成为可信能力。本专题提供验证清单；生产切换必须由系统负责人、操作人员和厂商按批准方案执行。

## 1. 验证对象

| 层级 | 静态一致性 | 动态一致性 |
| --- | --- | --- |
| 控制器 | 程序、组态、固件、任务 | PID 积分、模式、步位、定时器 |
| I/O | 模块、通道、故障值 | 当前值、质量、连接状态 |
| 网络 | 节点、VLAN、路由、优先级 | 会话、路径、错误与重连 |
| 服务器 | 配置、权限、接口 | 报警、队列、会话、历史缓存 |
| 时间 | 时间源、时区、层级 | 当前偏差与切换连续性 |

## 2. 测试前条件

- 变更冻结并确认主备版本一致；
- 当前过程稳定，回路与设备状态清楚；
- 旁路、强制、手动和维护状态已记录；
- 备份和回退已验证；
- 定义触发方式、预期行为、停止条件和责任人；
- 操作员监控 SP、PV、MV、模式、阀位和关键报警；
- 明确哪些 SIS 或设备保护不参与测试。

## 3. 切换观察点

- 故障检测与仲裁耗时；
- 主备角色和脑裂防护；
- 输出是否连续、最大突跳是多少；
- PID、顺控、累计量和通信状态是否连续；
- 远程 I/O 与驱动器是否重连；
- HMI 控制权、报警和趋势是否中断；
- 历史是否有缺口或重复；
- 操作和 SOE 时间是否可关联。

## 4. 典型测试场景

- 计划性主备切换；
- 主控制器故障；
- 单条冗余网络中断；
- 服务器主备切换；
- 备机重新加入与状态同步；
- 在线变更后的切换；
- 公共电源或交换机故障桌面推演。

禁止为验证而在生产制造未批准的断电、网络风暴或双主场景。

## 5. 失败判据

- 输出突跳超过工艺允许值；
- 控制器出现双主或无主；
- 关键状态、参数或步位回退；
- 设备意外启停、模式变化或联锁异常；
- 报警和历史缺失无法解释；
- 切换后备机无法重新同步；
- 过程偏差超过预设停止条件。

## 6. 测试后恢复

确认角色稳定、同步健康、所有通信恢复、输出与现场反馈一致、报警旁路和临时措施清除、历史缺口已标注，并把结果与工程版本和工单归档。

## 7. 安全视角

攻击者可能利用同步通道、仲裁、版本差异或重连路径。监测应覆盖角色变化、同步丢失、备机重新加入和切换后的参数差异，而不只看设备在线。

## 8. 面试速答

> 无扰切换要求的不只是程序一致，还包括 PID 积分、模式、顺控步位、I/O 质量和通信状态连续。我会在稳定工况下定义预期和停止条件，观察 SP/PV/MV、真实阀位、报警与 SOE，并检查共同电源、网络和工程错误，避免把两台设备等同于真正独立冗余。
