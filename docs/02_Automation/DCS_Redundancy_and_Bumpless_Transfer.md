# DCS 冗余与无扰切换

## 1. 原理解释：DCS 为什么需要冗余

DCS 常用于连续过程工业，例如炼化、电力、化工、水处理。系统一旦停机，可能造成生产中断、设备损坏或安全风险。

冗余的目标不是“多买一套备用”，而是提高可用性。

核心目标：

- 控制不中断。
- 故障自动切换。
- 切换前后输出连续。
- 操作员感知最小。

## 2. 工程实现：主备控制器

典型 DCS 控制站可能采用主备 CPU：

```text
Primary Controller
  <-> 状态同步链路
Standby Controller
```

主控制器负责实际输出，备控制器同步程序和运行状态。一旦主控制器故障，备控制器接管。

## 3. 无扰切换是什么

无扰切换 `Bumpless Transfer` 指切换前后控制输出不产生明显跳变。

如果切换瞬间阀门开度从 60% 跳到 20%，可能造成工艺扰动。

因此备机必须同步：

- 程序
- 当前模式
- PID 积分
- 上次误差
- 定时器 / 计数器
- 报警状态
- I/O 状态
- 控制输出

## 4. 心跳与脑裂

主备之间通过心跳判断对方是否存活。

风险场景：

```text
主控制器正常
备控制器收不到心跳
备控制器误认为主故障
两个控制器都认为自己是主
```

这叫脑裂 `Split Brain`。

需要仲裁机制确保任何时刻只有一个主控制器拥有输出权。

## 5. 数据流：一次故障切换

```text
主控制器运行
  -> 备控制器持续同步状态
  -> 主控制器故障
  -> 心跳超时
  -> 仲裁确认
  -> 备控制器成为主
  -> 使用同步状态继续输出
```

## 6. 安全视角

冗余机制本身也是攻击面：

- 攻击心跳导致误切换。
- 攻击同步链路导致备机状态落后。
- 攻击仲裁导致脑裂。
- 攻击备机使切换后进入异常状态。
- 攻击时间同步导致日志和切换记录混乱。

## 7. 工业案例：PID 状态不同步

如果主备控制器切换时，备机没有同步 PID 积分状态，切换后输出可能跳变。

后果：

- 阀门突然动作。
- 压力或流量扰动。
- 操作员看到趋势突变。
- 严重时触发联锁。

## 8. 面试表达

### 60 分答案

> DCS 冗余用于提高连续过程控制系统可用性。主备控制器通过状态同步和心跳检测实现故障切换，无扰切换要求切换前后控制输出保持连续。

### 90 分答案

> 我会从可用性和控制连续性理解 DCS 冗余。主备控制器不仅要同步程序，还要同步 PID 积分、上次误差、报警状态、定时器和输出状态，否则切换时可能出现控制输出跳变。安全上，心跳、同步链路和仲裁机制都是攻击面，攻击者可能造成误切换、备机状态异常或脑裂。

### 常见追问

问：冗余是不是一定安全？

答：

> 不是。冗余提高可用性，但也增加了同步、心跳和仲裁链路。如果这些机制设计或配置不当，反而可能成为新的攻击面。

## 9. 实操建议

短期可以先做资料学习：

- 查 DCS 主备冗余、热备、无扰切换概念。
- 画出主备控制器切换流程。
- 用报告模板描述“同步链路异常”的安全影响。

长期可以在仿真或厂商文档中观察：

- 主备切换日志。
- 切换前后输出趋势。
- 报警和事件记录。
