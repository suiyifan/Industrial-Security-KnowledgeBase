# SIS、安全联锁与失效安全

基本过程控制系统 BPCS 负责正常生产，安全仪表系统 SIS 在特定危险条件下把过程带到或保持在安全状态。SIS 不是“备用控制器”，也不是所有联锁的统称。工控安全工程师需要理解安全生命周期、独立保护层、SIL、旁路和验证，才能判断控制系统受损后是否仍有可信保护。

## 1. 原理解释：为什么正常控制之外还需要保护

BPCS 可能因测量故障、执行器失灵、软件错误、人员误操作或攻击而无法维持安全边界。风险降低通常依靠多层保护：

```text
本质安全设计
 → BPCS 正常控制与约束
 → 过程报警 + 人工响应
 → SIS / ESD
 → 机械泄放、围堰等缓解
 → 应急响应
```

保护层只有在足够独立、可测试且对目标危险有效时，才能作为独立保护层计入风险降低。

## 2. 工程实现：SIF 与 SIS

- **SIF，安全仪表功能**：针对一个危险场景的完整检测—逻辑—最终元件功能；
- **SIS**：实现一个或多个 SIF 的传感器、逻辑解算器、最终元件及支持系统；
- **ESD**：紧急停车系统，常实现多项停车功能；
- **BPCS 联锁**：普通控制系统中的保护逻辑，不自动等同独立 SIF。

一项 SIF 示例：反应器压力达到高高限时，独立压力传感器触发安全逻辑，关闭进料切断阀并停止加热。

## 3. 安全生命周期

安全功能需要贯穿：

1. 危害与风险分析；
2. 分配保护层；
3. 安全需求规格 SRS；
4. 设计与实现；
5. 验证与确认；
6. 安装、调试和投运；
7. 运行、维护和定期证明测试；
8. 修改、退役和功能安全评估。

安全逻辑正确只是其中一环。传感器取样、阀门失效、测试周期和管理流程同样决定真实风险降低。

## 4. SIL 与失效概率

SIL 表示安全功能所需的风险降低能力等级，不是设备“安全质量分数”。低需求模式常用要求时失效概率 PFDavg，高需求或连续模式常用每小时危险失效概率 PFH。

SIL 验证需要考虑：

- 传感器、逻辑解算器和最终元件失效率；
- 架构冗余与表决；
- 诊断覆盖率；
- 证明测试间隔与有效性；
- 共因失效和人为错误；
- 维修时间与旁路暴露。

不能仅凭“使用安全 PLC”就宣称达到某个 SIL。

## 5. 表决架构与独立性

常见表决如 1oo1、1oo2、2oo3：

- 1oo1 简单，但单点危险失效直接影响功能；
- 1oo2 任一通道动作即可跳闸，提高安全性但可能增加误停车；
- 2oo3 需三通道中两通道同意，在安全性和可用性间折中。

冗余通道若共享取压点、供电、I/O、网络、工程站或同一错误参数，可能发生共因失效。

## 6. 失效安全与最终元件

安全动作由具体危险决定：关闭燃料阀、打开冷却阀、停止压缩机、切断加热或泄放压力。最终元件常是整个 SIF 失效概率的重要贡献者。

需确认：

- 阀门失气/失电位置；
- 电磁阀、执行机构和空气源；
- 电机接触器、断路器或 STO；
- 动作时间和过程安全时间；
- 卡涩、泄漏、部分行程测试和全行程测试；
- 安全动作后的锁存与复位。

“输出置零”不是普遍安全状态。

## 7. BPCS 与 SIS 边界

独立性需要从多方面验证：

- 传感器和取样是否独立；
- 逻辑解算器、I/O 与电源是否独立；
- 最终元件是否共享及如何评估；
- 网络和工程站是否共享；
- 权限、账号和维护流程是否隔离；
- 同一软件、库或人员错误是否会同时影响；
- BPCS 操作是否能修改 SIS 设定或旁路。

共享并非绝对禁止，但必须在风险分析中证明不会破坏所需独立性。

## 8. 旁路、抑制、强制与复位

检修可能需要旁路某个传感器或最终元件。旁路应具备：

- 明确授权和书面原因；
- 补偿措施和剩余风险评估；
- 明显、持续的操作员提示；
- 自动超时或交接复核；
- 进入、退出和当前状态审计；
- 恢复后功能验证。

复位只应在危险条件消除、设备处于可恢复状态且人员确认后进行。远程连续复位可能掩盖真实故障。

## 9. 证明测试与覆盖率

诊断只能发现部分故障。证明测试用于发现潜在危险失效，例如阀门卡死、传感器漂移和输出回路断开。

测试必须说明：

- 覆盖哪些故障模式；
- 是否测试完整传感器—逻辑—最终元件链；
- 测试期间如何管理旁路；
- 发现故障如何修复和复测；
- 测试后如何确认全部恢复；
- 测试记录是否可追溯。

只模拟 PLC 输入而不让阀门动作，不能证明最终元件功能。

## 10. 数据流、控制流与安全状态流

```text
独立传感器 → 安全输入 → SIF 逻辑
 → 安全输出 → 最终元件 → 过程安全状态

状态：Healthy → Demand → Tripped → Latched
                    ↘ Bypassed / Faulted → Tested → Reset
```

安全调查应关联需求发生、逻辑动作、最终元件反馈、过程响应、旁路和复位。

## 11. 安全视角：SIS 攻击与失效路径

- 修改安全设定、逻辑、表决或延时；
- 强制安全输入或输出；
- 长期旁路关键通道；
- 欺骗诊断和最终元件反馈；
- 污染工程站、程序库或签名密钥；
- 破坏 BPCS 与 SIS 的独立性；
- 造成误停车，影响生产可用性；
- 在过程接近危险时阻止真实需求动作。

高风险测试必须在授权的仿真或专用测试环境完成，不能主动触发生产安全动作。

### 11.1 安全风险的两种方向

SIS 安全风险既包括“真实需求时拒动”，也包括“无需求时误动”。拒动可能让压力、温度、液位或可燃气体越过可接受边界；误动则可能造成装置紧急停车、旋转设备冲击、公用工程失稳，并在错误复位或不完整吹扫后引入二次危险。因此评估不能只追求系统在线，还要验证它在正确条件、正确时间进入设计的安全状态。

数字安全证据必须与安全功能语义对应。一次登录失败的严重度通常较低，而未授权修改高高压设定值、延长投票延时、旁路最终元件或改变复位方式，会直接削弱某条 SIF 的风险降低能力。资产清单应映射 SIF 编号、危险场景、传感器、逻辑、最终元件、SIL 要求、允许旁路时间和证明测试记录。

### 11.2 工程化攻击链

典型攻击链可描述为：攻击者先控制 SIS 工程站或携带受污染项目文件，经维护窗口获得下载条件；随后识别目标 SIF，修改设定、投票或输出映射，利用版本管理薄弱使变更进入控制器；再清除工程审计或保持 HMI 显示不变。当真实过程需求出现时，逻辑拒动或最终元件未到安全位置，危险由 BPCS 偏差发展为保护层失效。

但这条攻击链并非天然成立。攻击者需要理解安全程序、设备钥匙和模式、下载授权、在线与离线版本关系以及过程工况；独立机械保护、现场硬接线、只读校验、双人审批和完整证明测试都可能阻断它。审计时应为每一步列出必要条件与可获得证据，不能仅因工程站存在漏洞就断言 SIS 已被绕过。

### 11.3 旁路状态的完整控制流

旁路不是一个孤立布尔位，而是一段受治理的控制流：提出申请、确认危险与补偿措施、授权、实施、现场提示、定时提醒、班次交接、解除和恢复测试。状态流至少包括 `正常 → 申请中 → 已批准 → 已旁路 → 到期/异常 → 已解除 → 已验证`。若系统只记录“当前已旁路”，却缺少申请人、批准人、对象、原因、期限和补偿措施，就无法判断这是不是授权维护。

监测还应关联过程需求。如果某通道处于旁路且过程接近跳闸设定，风险会动态上升；单独统计旁路数量无法反映这一点。可将旁路状态、PV 距设定的裕度、剩余允许时间和替代保护可用性组合成运营视图。

### 11.4 事件后的安全恢复

SIS 事件恢复不能以控制器重新运行结束。应核对在线程序和安全签名、钥匙与运行模式、强制和旁路、诊断故障、输入真实性、最终元件位置以及过程是否具备复位条件。复位必须遵守 SRS 定义，避免远程批量复位掩盖仍存在的危险。对被怀疑篡改的系统，恢复后的功能测试应覆盖传感器到最终元件，并由安全、工艺、仪表和操作共同确认。

## 12. 工控场景：共享压力测点导致共因失效

BPCS 压力控制和 SIS 高高压跳闸都使用同一变送器及同一取压管。取压管堵塞后两套系统同时看到稳定低压：BPCS 继续加热，SIS 也不触发。逻辑和网络均正常，但保护独立性在物理测量层已失效。

风险分析必须覆盖一次元件、取样、供电和最终元件，而不能只看控制器是否独立。

## 13. 防护与治理

- 按安全生命周期管理 SRS、逻辑、测试和变更；
- SIS 工程、账号、网络和介质实施更严格隔离；
- 程序与参数使用签名、校验和受控下载；
- 旁路、强制和钥匙位置持续监测并交接；
- 定期执行覆盖完整回路的证明测试；
- 分析共因失效，不把冗余数量等同独立性；
- 事件后验证传感器、逻辑、最终元件和真实过程状态；
- 恢复前确认所有临时旁路和测试状态已清除。

## 14. 面试表达

### 60 分答案

> BPCS 负责正常控制，SIS 在危险条件下执行独立安全动作。SIF 包含传感器、逻辑解算器和最终元件，SIL 表示所需风险降低能力。

### 90 分答案

> 我不会把所有 PLC 联锁都叫 SIS。SIS 要从危害分析和安全需求出发，由完整 SIF 实现，并通过架构、诊断、证明测试和生命周期管理达到所需风险降低。评估时我会检查 BPCS 与 SIS 在传感器、取样、电源、I/O、工程站、网络和最终元件上的独立性，审阅旁路强制和复位，并确认测试覆盖真实最终元件。安全 PLC 或冗余本身不能证明 SIL，仍要考虑共因失效和测试间隔。

### 常见追问

**问：报警与 SIS 有什么区别？**

答：报警依赖人员识别和响应，SIS 在达到需求条件时自动执行安全动作；二者的独立性和风险降低信用不同。

**问：为什么 SIS 也可能造成风险？**

答：误动作会导致停车和设备冲击，拒动会在真实危险时失去保护，旁路与错误恢复也可能扩大风险。

## 15. 实操建议

### 实验一：SIF 因果图

选择高液位保护，画出传感器、表决、逻辑、最终元件、反馈和安全状态，并列出共因依赖。

### 实验二：旁路治理桌面演练

设计旁路申请、补偿措施、操作提示、超时、交接和恢复测试流程，不连接真实 SIS。

### 实验三：证明测试覆盖

比较只模拟输入、测试逻辑输出和完整阀门行程三种测试能发现的故障差异。

### 实验四：SIF 变更攻击链

使用纸面 SIF 或安全 PLC 仿真工程，分别设计设定值修改、输入强制、输出映射改变和旁路超期四种情形。为每种情形列出前提、审计证据、可能后果与独立阻断措施，不连接生产安全系统。

### 实验五：动态旁路风险

建立包含旁路状态、PV、跳闸设定、允许时长和替代保护的表格。模拟 PV 接近设定且旁路即将超时，设计比“检测到旁路”更有工程意义的分级告警与升级流程。

## 16. 总结与检查清单

- [ ] 能区分 BPCS 联锁、SIF、SIS 和 ESD；
- [ ] 能解释安全生命周期与 SRS；
- [ ] 能正确理解 SIL、PFD/PFH 和表决架构；
- [ ] 能分析 BPCS/SIS 独立性与共因失效；
- [ ] 能审阅旁路、复位和证明测试；
- [ ] 能验证安全动作到真实最终元件和过程状态。
