# 功能安全、SIS 与安全联锁

## 1. 为什么要学安全联锁

工控系统不只追求生产效率，还必须保证安全。

某些异常状态下，系统必须自动采取保护动作，例如停机、关闭阀门、切断燃料、泄压。

这类保护逻辑与普通控制逻辑不同，它关系到设备和人员安全。

## 2. 基本概念

### 2.1 BPCS

BPCS 是 Basic Process Control System，基本过程控制系统。

它负责日常生产控制，例如温度、压力、流量调节。

### 2.2 SIS

SIS 是 Safety Instrumented System，安全仪表系统。

它负责在危险状态下执行安全保护动作。

### 2.3 Interlock

Interlock 是联锁。

当某些条件满足时，系统自动禁止某些动作或强制执行保护动作。

例子：

```text
压力超过高高限 -> 自动关闭进料阀 -> 启动泄压流程
```

## 3. 报警和联锁的区别

| 项目 | 报警 | 联锁 |
|---|---|---|
| 作用 | 提醒人 | 自动保护 |
| 是否自动动作 | 通常不直接动作 | 会自动执行保护动作 |
| 风险等级 | 可能较低或中等 | 通常更高 |
| 依赖对象 | 操作员响应 | 系统逻辑 |

报警是“告诉你有问题”，联锁是“系统自己采取保护动作”。

## 4. ESD

ESD 是 Emergency Shutdown，紧急停车。

当系统进入危险状态时，ESD 会让设备进入安全状态。

安全分析中，ESD 相关系统通常非常敏感，测试必须极其谨慎，不能随意扫描、发包或修改。

## 5. 安全完整性等级

SIL 是 Safety Integrity Level，安全完整性等级。

初学阶段先理解：

- SIL 表示安全功能降低风险的能力。
- SIL 越高，安全要求越严格。
- 不要把 SIL 当成普通网络安全等级。

## 6. 安全视角

攻击者如果影响安全系统，可能造成：

- 报警被屏蔽。
- 联锁条件被修改。
- 安全动作无法触发。
- 错误触发停机。
- 操作员误判风险。

## 7. 工控安全测试边界

涉及 SIS、ESD、联锁的系统，测试原则：

- 不做未经授权的主动测试。
- 不随意下发写命令。
- 不在生产环境做破坏性验证。
- 优先使用离线工程、仿真环境、镜像流量、厂商资料。
- 报告中明确风险，不追求危险复现。

## 8. 面试表达

可以这样说：

> 我理解工控系统中报警、联锁和 SIS 的安全意义不同。报警主要提醒操作员，联锁和 SIS 会在危险条件下自动执行保护动作。做工控安全测试时，涉及 SIS、ESD 或关键联锁的部分必须非常谨慎，优先用离线分析和仿真验证，避免影响生产安全。
