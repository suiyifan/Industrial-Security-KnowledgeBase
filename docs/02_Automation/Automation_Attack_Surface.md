# 自动化系统攻击面与物理影响

自动化攻击面不仅包括主机漏洞和网络端口，还包括工程项目、控制命令、模式、配方、I/O 组态、现场参数、报警、旁路和恢复流程。成熟的风险分析必须证明攻击能力如何改变控制状态，并经过执行器影响物理过程，而不是从“获得权限”直接跳到“导致爆炸”。

## 1. 原理解释：攻击面由能力和语义构成

同一资产可能提供不同能力：

- **观察**：读取画面、趋势、点表、工程和诊断；
- **操作**：改 SP、模式、配方，执行启停和复位；
- **维护**：强制、旁路、修改仪表或驱动参数；
- **工程**：上传、下载、在线修改和硬件组态；
- **管理**：用户、证书、时间、备份和系统服务；
- **破坏可用性**：停止服务、耗尽资源或切换模式。

风险取决于能力作用的对象、当前过程状态和保护层，而不只取决于登录了哪台主机。

## 2. 工程实现：按资产识别攻击面

| 资产 | 关键能力 | 典型后果路径 |
| --- | --- | --- |
| 工程师站 | 工程、下载、强制、固件 | 大范围逻辑与组态改变 |
| 操作员站/HMI | SP、模式、命令、报警 | 误操作、态势欺骗 |
| PLC/DCS 控制器 | 变量、任务、程序、模式 | 控制与输出改变 |
| SCADA/RTU | 遥测遥控、队列、站点管理 | 分布式站点操纵 |
| 历史/报警服务器 | 趋势、事件、接口 | 证据和态势感知受损 |
| SIS 工程与逻辑 | 安全功能、旁路、复位 | 误停车或保护拒动 |
| VFD/IED/仪表 | 参数、命令、保护定值 | 设备或测量语义改变 |
| 网关/远程 I/O | 协议映射、质量、故障值 | 跨层语义与现场信号改变 |
| 维护终端/介质 | 便携工程工具与文件 | 绕过固定边界、供应链传播 |

## 3. 按生命周期识别攻击面

- 设计：错误信任边界、默认账号、不安全接口；
- 集成：临时旁路、测试账号、工程文件交换；
- 调试：强制、仿真、宽松权限和远程支持；
- 运行：共享账号、长期在线工程站、陈旧服务；
- 检修：移动介质、承包商终端、临时网络和维护模式；
- 升级：库、固件、设备包、迁移和回退；
- 恢复：不可信备份、状态不一致和带病复产；
- 退役：存储介质、工程备份和凭据泄露。

很多高风险入口只在检修窗口出现，静态资产扫描无法覆盖。

## 4. 控制闭环攻击分类

### 4.1 目标操纵

修改 SP、配方、排产或远程调度，使控制系统正常追求错误目标。

### 4.2 反馈操纵

冻结、偏置、缩放或回放 PV 和状态，使控制器或操作员形成错误判断。

### 4.3 控制器操纵

修改算法、任务、模式、步序、限幅、联锁或程序调用。

### 4.4 执行器操纵

直接写输出、强制变量、改变 VFD/阀门参数或控制源。

### 4.5 态势感知操纵

改变画面、报警、质量、趋势、历史和操作日志，隐藏过程变化。

### 4.6 保护层操纵

旁路联锁、修改安全设定、伪造最终元件反馈或破坏独立性。

隐蔽攻击往往组合多类路径，而非单点写入。

## 5. 从数字能力到物理后果

使用统一链路：

```text
入口与身份
 → 获得的技术能力
 → 被改变的工程对象/变量/状态
 → 控制逻辑与操作行为变化
 → 执行器和能量流变化
 → 过程变量偏移
 → 报警、设备保护、BPCS、SIS 和人员响应
 → 安全、环境、质量、设备、生产后果
 → 隔离、回退和恢复验证
```

任何缺失环节都应列为待验证，而不是用最坏想象补齐。

## 6. 攻击前提与约束

需要确认：

- 网络是否可达，协议与会话是否允许目标操作；
- 账号角色、控制权和控制器模式；
- 变量是否外部可写，写入是否被程序覆盖；
- 当前设备与过程状态；
- 命令是否需要许可、确认、序列号或现场条件；
- 执行器能力、变化速率和物理极限；
- 攻击必须持续多久；
- 保护和操作员能否发现并阻断。

“存在未认证写功能”是重要发现，但物理后果仍需上述约束支持。

## 7. 可用性攻击与安全停机

拒绝服务可能导致：

- HMI 失去显示，但 PLC 闭环继续；
- 远程站失联，现场自治；
- 控制器看门狗故障，输出进入配置状态；
- 远程 I/O 进入保持或安全值；
- 历史和报警缺失，操作员失去态势感知；
- 工程站不可用，故障期间无法维护。

停机不一定安全。某些过程需要受控降负荷、冷却、泄压或保持搅拌，突然失电可能加剧危险。

## 8. 隐蔽性与过程模型

简单攻击让 PV 或输出明显越限，容易被发现。更隐蔽的攻击可能：

- 缓慢改变 SP 或偏置；
- 在正常波动范围内操纵；
- 同时修改反馈与画面；
- 利用过程滞后延迟异常；
- 选择检修、切换或启动等复杂状态；
- 控制在保护阈值以下长期造成质量或磨损。

检测需使用命令—反馈—响应、物料能量平衡、冗余测量和状态一致性，而不只靠静态阈值。

## 9. 保护层与攻击路径交集

评估每层保护是否真正独立：

- 报警是否与被欺骗 PV 使用同一测点；
- BPCS 联锁是否与正常逻辑同一程序和 I/O；
- SIS 是否共享工程站、网络、供电或最终元件；
- 机械保护能否限制最坏后果；
- 人员是否仍有可信信息和足够响应时间。

攻击者若只能控制 BPCS，而独立 SIS 有效，最坏结果可能是安全停车；若共因依赖被利用，后果边界会改变。

## 10. 证据与可信度分级

风险结论可分为：

- **理论可能**：依据架构和功能推断，未验证具体实现；
- **工程可信**：有工程文件、点表和逻辑支持；
- **实验验证**：在同类隔离环境复现控制行为；
- **现场证实**：有授权现场证据或历史事件；
- **后果证实**：真实过程响应和保护动作已被记录。

报告应明确分级、假设和验证边界，避免用实验 PLC 结果无条件外推生产装置。

### 10.1 用三类流验证攻击链

完整攻击链需要同时解释数据流、控制流和状态流。数据流回答攻击者改动的值来自哪里、经过哪些网关或缓存、质量与时间戳是否保留、哪些消费者会收到；控制流回答该值怎样经过权限、模式、任务、联锁和仲裁变成输出；状态流回答设备和过程当时处于启动、自动、手动、降级、维护还是停车状态，以及该操作能否触发合法或非法迁移。

例如捕获到一次 PLC 写请求，只能证明网络上出现了写入意图。还需确认控制器响应、变量是否为在线生效对象、程序是否在下一扫描覆盖该值、设备是否具有远程控制权、最终元件是否动作、PV 是否变化、报警与保护是否响应。缺少任一环节时，应把结论限制在已证明的能力层级。

三类流也帮助识别更隐蔽的操纵：攻击者可能不直接改变控制输出，而是污染进入优化器的数据流；利用手自动切换改变控制流的责任主体；或者在维护状态流中植入一个待恢复后才生效的参数。只检查实时协议写入会漏掉这些生命周期攻击面。

### 10.2 攻击图不能替代过程模型

传统攻击图常以主机权限为节点，以漏洞利用为边，但获得管理员权限不等于获得物理操纵能力。工控攻击图还应加入控制对象、运行模式、工艺约束、执行机构能力、保护层与时间窗口。推荐为每条边记录：前置权限、所需工程知识、在线条件、可观测证据、失败方式、最大可信后果和防护控制。

以“工程站管理员 → PLC 下载”为例，中间可能仍要求专用软件版本、项目密码、硬件钥匙、控制器允许下载、冗余协调和停车窗口；下载后还可能导致安全停车而不是隐蔽运行。把这些条件显式化，才能把“技术可达”与“工程可实施”区分开。

### 10.3 安全风险的动态优先级

同一弱点的安全风险会随工况变化。处于年度检修、空罐、冷态启动、满负荷生产或应急降负荷时，可利用窗口与后果边界不同。风险台账应关联生产单元、关键控制功能、当前模式、危险物料、人员暴露和独立保护可用性，而不是给某个 CVE 永久赋一个静态物理后果。

动态评估不意味着频繁修改漏洞评分，而是用运营条件决定处置顺序。例如一台即将进入联锁旁路检修的控制器，即便基础漏洞评分不变，其临时远程访问、保护降级和人员作业会使风险上升；维护结束后必须撤销账号、清除强制、恢复旁路并验证现场状态。

### 10.4 从检测到响应的闭环

检测规则应面向工程语义，例如非窗口下载、控制器模式切换、关键 SP 大幅变化、VFD 命令源改变、SIS 旁路超期以及命令与过程响应不一致。告警后先确认资产、对象、模式和工况，再决定隔离策略。贸然阻断控制通信或断电可能破坏受控停车，因此响应预案要由网络、自动化、工艺、设备和安全专业共同制定。

事件处置的状态流可写为 `发现 → 工况确认 → 风险稳定 → 证据保全 → 数字恢复 → 现场验证 → 受控启动 → 持续观察`。每次迁移都应有责任人、进入条件、退出证据和回退方案，避免只完成恶意文件清除却留下错误设定、残余物料或未复位保护。

## 11. 工控场景：储罐闭环协同操纵

攻击者取得工程权限后：

1. 缓慢提高进料 SP；
2. 给液位 PV 叠加负偏置；
3. 把高液位报警限提高；
4. 保持 HMI 趋势在正常范围；
5. 真实液位持续上升。

若独立高高液位开关和 SIS 仍有效，系统应最终切断进料；若它与 BPCS 共用被操纵变送器，则可能失去保护。检测可依赖进出流量物料平衡、罐重、独立测点和阀位响应。

## 12. 恢复不是简单重装

事件后需要同时恢复：

- 可信工程、固件、组态和账号；
- 在线参数、保持数据、强制和旁路；
- 控制器、服务器和主备一致性；
- 仪表、I/O、驱动器与安全系统参数；
- 现场阀位、设备位置、物料和能量状态；
- 报警、历史、时间同步和证据链；
- 受控启动顺序与生产验收。

未经过程确认直接恢复 RUN 或批量重启，可能触发残留命令或错误状态。

## 13. 防护原则

- 基于过程功能和高风险动作实施最小权限；
- 工程、操作、维护和安全系统分离职责；
- 对下载、强制、旁路、模式、配方和控制参数做语义审计；
- 监测值、质量、时间、命令、反馈和过程响应；
- 保护层保持独立并定期验证最终元件；
- 检修和远程维护纳入临时访问与退出检查；
- 备份通过恢复演练证明可用；
- 主动安全测试只在授权隔离环境实施。

## 14. 面试表达

### 60 分答案

> 自动化攻击面包括工程站、HMI、控制器、远程 I/O、仪表、驱动器、协议、工程文件和远程维护。风险可能影响控制逻辑、设备状态、报警和生产。

### 90 分答案

> 我会按能力而不是只按资产列攻击面，区分观察、操作、维护、工程和管理权限。然后沿入口—对象—控制状态—执行器—能量流—过程变量—保护响应—业务后果走完整链路。对于未认证写入，我会继续确认控制器模式、目标变量、任务覆盖、当前工况、命令握手和 SIS 独立性。结论标明理论可能、工程可信或实验验证等级，恢复则同时检查工程版本、保持状态、强制旁路和现场物料设备状态。

### 常见追问

**问：RCE 在工程站上为什么比普通办公终端严重？**

答：工程站可能拥有程序、点表、控制器信任和下载能力，攻击者可获得控制语义和工程权限，但实际后果仍取决于网络、控制器模式、过程状态和保护层。

**问：如何避免夸大漏洞物理影响？**

答：列出每个必要前提，用工程、实验和现场证据逐项验证，并明确独立保护和未验证假设。

## 15. 实操建议

### 实验一：攻击路径表

选择一个仿真液位回路，分别分析 SP、PV、MV、模式、报警和联锁操纵，记录前提、证据、保护和恢复。

### 实验二：多源检测

注入缓慢 PV 偏置，使用物料平衡、独立液位、阀位和流量检测，比较单阈值与多源方法。

### 实验三：恢复桌面演练

假设工程站和 PLC 被篡改，制定隔离、取证、可信备份验证、现场核对、受控启动和观察计划。

### 实验四：三类流证据矩阵

在仿真液位系统中执行一次合法设定值修改，再模拟未授权修改。分别收集网络请求、控制器响应、变量质量、模式、阀位反馈、液位趋势和联锁事件，将证据归入数据流、控制流和状态流，判断每种证据单独能够支持到什么结论。

### 实验五：动态风险桌面推演

对同一远程工程访问分别设置正常生产、联锁旁路检修和冷态停车三种工况。比较攻击前提、人员暴露、可接受隔离动作、最大可信后果与恢复顺序，形成不依赖单一漏洞分数的处置优先级。

## 16. 总结与检查清单

- [ ] 能按资产、能力和生命周期识别攻击面；
- [ ] 能区分目标、反馈、控制器、执行器、显示和保护操纵；
- [ ] 能证明数字行为到物理后果的每个环节；
- [ ] 能分析可用性攻击与安全停机差异；
- [ ] 能评估保护层独立性和结论可信等级；
- [ ] 能设计包含数字与物理状态的恢复流程。
