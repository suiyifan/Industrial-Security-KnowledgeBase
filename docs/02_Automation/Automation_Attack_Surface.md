# 自动化系统攻击面

## 1. 什么是攻击面

攻击面是攻击者可能进入、影响或滥用系统的入口集合。

在工控自动化系统中，攻击面不仅包括 Web 服务和主机漏洞，还包括工程接口、工业协议、远程运维、组态文件、账号权限、现场接口等。

## 2. 资产视角攻击面

### 2.1 工程站

风险：

- 工程软件漏洞
- 工程文件泄露
- 控制逻辑被篡改
- 下载工程被滥用
- 账号权限过高

### 2.2 操作员站 / HMI

风险：

- 操作界面被控制
- 命令被非法下发
- 画面数据被伪造
- 报警被屏蔽
- 远程桌面暴露

### 2.3 PLC / RTU

风险：

- 未授权读写变量
- 工程服务暴露
- 默认口令
- 固件漏洞
- 协议缺少认证

### 2.4 SCADA Server

风险：

- 通信驱动漏洞
- Web 管理漏洞
- 数据采集异常
- 横向移动枢纽
- 权限配置错误

### 2.5 历史数据库

风险：

- 数据篡改
- 数据删除
- 凭据泄露
- 办公网与控制网边界风险

### 2.6 工业网关

风险：

- 协议转换暴露
- 默认密码
- 远程维护入口
- Web 管理漏洞
- VPN 配置不当

## 3. 功能视角攻击面

| 功能 | 可能风险 |
|---|---|
| 远程运维 | VPN、RDP、堡垒机、账号权限风险 |
| 工程下载 | 控制逻辑被篡改 |
| 变量读写 | 设备状态、设定值被改变 |
| 报警管理 | 报警被屏蔽或误导 |
| 趋势记录 | 历史追溯失真 |
| 配方管理 | 生产参数被篡改 |
| 用户管理 | 越权、共用账号、弱口令 |

## 4. 协议视角攻击面

常见问题：

- 明文传输。
- 缺少认证。
- 缺少完整性保护。
- 指令功能过强。
- 厂商私有协议可逆向。
- 工程协议暴露。

典型风险：

- 资产识别。
- 读取变量。
- 写变量。
- 停止 / 启动控制器。
- 上传 / 下载工程。
- 读取设备信息。

## 5. 管理视角攻击面

技术漏洞之外，还有管理问题：

- 资产不清。
- 网络边界不清。
- 账号共用。
- 补丁长期不更新。
- 备份不可用。
- 缺少变更审批。
- 缺少日志审计。
- 第三方运维权限过大。

## 6. 攻击路径示例

一个常见路径：

```text
办公网钓鱼
-> 获取运维账号
-> 远程桌面进入工程站
-> 获取工程文件和点表
-> 识别 PLC 地址
-> 修改控制逻辑或变量
-> 影响现场设备状态
```

另一个路径：

```text
边界网关 Web 漏洞
-> 获取设备权限
-> 进入控制网络
-> 扫描工业协议
-> 发现 Modbus TCP 502
-> 未授权读写寄存器
```

## 7. 防护思路

- 资产识别。
- 分区分域。
- 最小权限。
- 强认证。
- 工程文件保护。
- 控制网络访问控制。
- 工业协议审计。
- 远程运维管控。
- 日志和告警。
- 备份与恢复。

## 8. 面试表达

可以这样说：

> 工控系统攻击面要按资产、功能和协议三条线来看。资产上关注工程站、HMI、PLC、SCADA、历史库和工业网关；功能上关注工程下载、变量读写、报警、趋势和远程运维；协议上关注明文、缺认证和未授权读写。最终要把攻击面和真实生产影响关联起来。
