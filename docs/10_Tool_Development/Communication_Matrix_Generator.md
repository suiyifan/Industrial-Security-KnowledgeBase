# 通信矩阵生成器

## 1. 通信矩阵是什么

通信矩阵描述 “谁和谁通信、使用什么协议、访问什么端口、频率如何”。在工控安全评估中，它是分区分域、白名单和风险分析的重要依据。

## 2. 输入来源

- PCAP。
- 防火墙日志。
- 交换机镜像。
- Zeek 日志。
- NetFlow。
- 手工资产表。

## 3. 输出字段

建议字段：

- 源 IP。
- 源资产名称。
- 目的 IP。
- 目的资产名称。
- 目的端口。
- 协议。
- 方向。
- 次数。
- 首次出现。
- 最后出现。
- 是否跨区。
- 风险备注。

## 4. 风险规则

可以内置基础规则：

- 办公网主机直接访问 PLC：高风险。
- 非工程站访问 PLC 工程端口：高风险。
- HMI 访问互联网：异常。
- PLC 主动访问办公网：异常。
- 远程维护入口访问范围过大：高风险。
- 未知资产出现在控制网：需调查。

## 5. 面试表达模板

> 我会把 PCAP 或日志转成通信矩阵，明确源、目的、端口、协议、频率和是否跨区。通信矩阵可以支撑白名单策略和分区分域评估，比如发现非工程站访问 PLC、办公网直连控制区或远程维护权限过大。它比单纯端口扫描更贴近工控现场。
