# 报告自动化

## 1. 为什么要做报告自动化

安全分析工作会产生大量结构化信息：

- 资产清单。
- 漏洞列表。
- CVE 信息。
- 通信矩阵。
- 风险评级。
- 整改建议。

用脚本生成 Markdown 或 Excel 摘要，可以提升复盘和交付效率。

## 2. 可自动化内容

- 从 CSV 资产表生成 Markdown 表格。
- 从 PCAP 摘要生成通信矩阵。
- 从 CVE JSON 生成漏洞卡片。
- 从模板生成漏洞分析报告。
- 统计章节学习进度。
- 生成面试项目说明。

## 3. 报告模板字段

漏洞报告模板：

- 标题。
- 产品和版本。
- 漏洞类型。
- 攻击条件。
- 影响范围。
- 技术细节。
- 工控影响。
- 检测方法。
- 修复建议。
- 参考链接。

## 4. 工具边界

报告自动化负责整理，不负责替你判断。风险等级、影响分析和修复建议仍然需要人工确认。

## 5. 面试表达模板

> 我会把重复性的报告整理工作脚本化，例如把 PCAP 解析结果生成通信矩阵，把 CVE 信息生成漏洞卡片，把资产表转换成 Markdown。这样可以减少手工复制错误，也方便形成 GitHub 作品集。但最终风险判断和工控影响分析仍然需要人工复核。
