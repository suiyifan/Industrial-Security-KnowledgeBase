# Python Socket 工具开发

## 1. 为什么先学 Socket

Socket 是理解 TCP/UDP 通信和工业协议工具开发的基础。很多工业协议都是在 TCP 或 UDP 上承载的，例如 Modbus TCP、S7Comm、部分私有协议。

掌握 Socket 后，你可以写：

- TCP 客户端。
- TCP 服务端。
- 简单端口连通性检测。
- 协议请求构造器。
- 仿真服务。
- 最小 PoC 验证器。

## 2. 工具结构

一个合格 Socket 工具应包含：

- 参数解析：目标 IP、端口、超时、输入文件。
- 连接管理：超时、异常、关闭连接。
- 数据编码：十六进制、字符串、二进制。
- 输出：请求、响应、耗时、错误原因。
- 日志：便于复盘。
- 安全限制：速率、只读提示、授权声明。

## 3. TCP 客户端思路

流程：

1. 创建 socket。
2. 设置超时。
3. 连接目标。
4. 发送请求。
5. 接收响应。
6. 解析或打印响应。
7. 关闭连接。

异常要区分：

- 连接超时。
- 连接被拒绝。
- 对端关闭。
- 响应超时。
- 数据格式错误。

面试中能说清异常处理，会比只会写几行 demo 更可靠。

## 4. 工控安全边界

写网络工具时要注意：

- 不默认扫描大网段。
- 不默认并发过高。
- 不默认发送写入、启停、下载类命令。
- 对可能影响设备的功能增加显式确认。
- 输出中标注 “仅授权环境使用”。

## 5. 面试表达模板

> 我做协议工具时会先用 Python Socket 写最小客户端，明确超时、异常处理和输入输出。工控场景里我会默认只读和低速率，避免对 PLC 或网关造成压力。工具输出不仅显示响应，还会显示目标、端口、耗时和错误原因，方便用于评估报告。
