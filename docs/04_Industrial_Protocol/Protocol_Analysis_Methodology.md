# 工业协议分析方法论

## 1. 总体方法

分析工业协议时，不要停留在端口和功能码。完整链路应该是：

```text
报文字段
  -> 协议操作
  -> 变量 / 对象 / 节点
  -> 控制逻辑
  -> 工业动作
  -> 物理影响
```

## 2. 分析步骤

1. 确认协议和端口。
2. 确认通信双方资产角色。
3. 判断通信方向。
4. 区分读、写、诊断、工程操作。
5. 定位地址、节点、对象或 DB。
6. 结合点表或工程资料解释语义。
7. 判断是否影响 SP、PV、MV、模式、报警、联锁。
8. 结合时间、频率、来源判断是否异常。
9. 输出工控影响。

## 3. 读写风险分级

| 操作 | 风险 |
|---|---|
| 读设备信息 | 资产暴露 |
| 读过程值 | 生产状态泄露 |
| 写设定值 | 改变控制目标 |
| 写命令变量 | 影响设备启停 |
| 写报警/联锁参数 | 隐藏或改变安全边界 |
| 工程下载 | 改变控制逻辑 |
| 控制器 Stop/Start | 直接影响可用性 |

## 4. 报告模板

```text
发现【源资产】通过【协议/端口】访问【目的资产】。
协议操作为【读/写/诊断/工程】。
目标对象为【寄存器/DB/节点/对象】。
结合【点表/工程资料】，该对象对应【工业变量】。
若被未授权利用，可能导致【物理过程影响】。
建议【访问控制/协议白名单/写操作审计/工程操作审批】。
```

## 5. 面试表达

> 我分析工业协议时会从报文字段一路映射到工业影响。比如 Wireshark 看到写寄存器只是第一步，后面要结合点表判断寄存器对应 SP、阀门开度还是普通参数，再结合来源主机和操作时间判断是否异常。
