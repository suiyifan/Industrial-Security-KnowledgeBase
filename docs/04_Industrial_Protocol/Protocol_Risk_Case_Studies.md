# 工业协议风险案例

## 案例 1：Modbus 写设定值

```text
未知主机 -> PLC: TCP 502
Function Code: 0x06
Address: 40010
```

如果 40010 对应阀门开度设定值，则可能改变现场流量。

## 案例 2：S7 写 DB

```text
未知主机 -> PLC: TCP 102
S7 Write Var -> DB20.DBW10
```

如果 DB20.DBW10 对应反应器温度 SP，则可能改变控制目标。

## 案例 3：OPC UA 匿名 Browse

攻击者可枚举节点名，推断工艺结构和关键变量。

风险：

- 点表泄露。
- 关键变量定位。
- 后续写操作更精准。

## 案例 4：IEC 104 非法遥控

未知来源出现遥控选择/执行报文，可能影响开关状态。

## 案例 5：工程协议暴露

工程站之外的主机访问 PLC 工程协议，可能尝试上传/下载程序或诊断控制器。

## 面试总结

> 协议风险不能脱离工程语义。读写寄存器、写 DB、Browse 节点、遥控命令、工程下载的风险等级不同，关键在于它们是否会影响控制变量、设备状态、报警联锁或控制逻辑。
