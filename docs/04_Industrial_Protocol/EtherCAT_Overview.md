# EtherCAT：高实时现场总线

## 1. 原理解释

EtherCAT 是面向高实时控制的工业以太网技术，常用于运动控制、伺服系统、机器人、精密制造。

它强调极低延迟和高同步性。

## 2. 工程实现

典型链路：

```text
EtherCAT Master
  -> Slave 1
  -> Slave 2
  -> Slave 3
  -> ...
```

报文在通过每个从站时被“边走边处理”，提高实时性能。

## 3. 数据流

EtherCAT 通常承载：

- 伺服控制数据
- I/O 状态
- 同步信号
- 设备参数
- 诊断信息

运动控制场景中，数据延迟和同步误差都可能影响设备动作。

## 4. 安全风险

- 主站被攻陷，影响多个从站。
- 参数篡改影响伺服行为。
- 通信干扰导致同步异常。
- 拓扑和设备信息泄露。
- 非授权接入现场总线。

## 5. 面试表达

### 60 分答案

> EtherCAT 是高实时工业以太网，常用于运动控制和伺服系统，强调低延迟和同步。

### 90 分答案

> EtherCAT 的安全分析要结合运动控制场景。它不像普通上位机协议，通信延迟、同步异常或参数篡改都可能影响伺服和机器人动作。评估时要关注主站权限、从站参数、拓扑保护和现场接入风险。

## 6. 实操建议

- 先理解 Master / Slave 和实时通信机制。
- 学习运动控制中的位置、速度、扭矩参数。
- 关注参数篡改如何影响机械动作。
