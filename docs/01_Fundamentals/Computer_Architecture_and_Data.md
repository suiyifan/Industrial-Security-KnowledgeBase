# 计算机组成与数据表示：从电信号到协议字段

工控安全工程师面对的 PLC、工程站、工业网关和服务器虽然形态不同，本质上都在执行指令、搬运数据并与外设交互。不了解数据如何表示、CPU 如何执行、内存如何寻址，就很难真正理解协议字节序、PLC 变量映射、固件架构和内存漏洞。

## 学习目标

- 解释 CPU、寄存器、缓存、内存、总线和 I/O 的关系；
- 掌握二进制、十六进制、补码、位运算和字节序；
- 理解整数、浮点数、字符、字符串和结构体的内存表示；
- 解释中断、DMA、内存映射 I/O 和存储层次；
- 将原始字节映射为工业协议字段和过程变量；
- 识别数据解释错误如何演变为控制偏差与安全风险。

## 1. 为什么安全研究必须理解计算机组成

软件看到的是变量和对象，CPU 看到的是指令与比特，网络传输的是字节，现场过程则表现为温度、压力、阀位和电机状态。同一信息跨层时会经历编码、传输、解析、缩放和控制决策。

```text
物理量 -> 传感器电信号 -> ADC 数值 -> PLC 内存变量
      -> 协议字节 -> SCADA 驱动解析 -> 工程值 -> 画面/报警/控制
```

任何一层对位宽、符号、字节序或缩放的理解错误，都可能让正常字节变成错误工程值。安全分析不仅要问“报文有没有被篡改”，还要问“接收方最终如何解释这些字节”。

## 2. 数字系统与进制

计算机用高低电平表示二进制状态。二进制适合硬件可靠区分，十六进制则能用一位表示四个二进制位，便于阅读地址、机器码、寄存器和数据包。

```text
二进制：1010 1100
十六进制：0xAC
十进制：172
```

安全研究中必须能在三种表示之间转换。抓包工具显示 `00 10` 时，它既可能是十六进制数值 16，也可能是两个独立字段、长度、地址或字符，语义由协议定义决定。

### 2.1 位、字节与字长

- bit：最小二进制单位；
- byte：通常为 8 bit；
- word：依处理器或协议定义，不能假定永远是 16 bit；
- CPU 字长：常指通用寄存器和原生地址处理宽度，如 32 位或 64 位。

位宽决定可表达范围。无符号 16 位整数范围为 `0～65535`，有符号二进制补码范围为 `-32768～32767`。协议文档中的 `WORD`、`DWORD`、`INT`、`DINT` 必须结合厂商定义确认。

### 2.2 位运算

位运算常用于解析状态字和构造协议字段：

```text
AND：提取特定位
OR：设置特定位
XOR：比较差异或翻转位
NOT：按位取反
SHIFT：移动位位置
```

若一个 16 位状态字中 bit 0 表示运行、bit 1 表示故障、bit 4 表示远程模式，可以用掩码判断。必须先确认位编号是从最低有效位开始，且不要混淆“字节顺序”和“位编号约定”。

## 3. 有符号整数与补码

现代系统通常用补码表示有符号整数。补码使加减法可复用同一硬件，并让零只有一种表示。最高位为符号相关位，但不能只把最高位删掉再读数。

以 8 位 `1111 1111` 为例：作为无符号数是 255，作为补码有符号数是 -1。同一字节因数据类型不同产生完全不同含义。

整数安全问题包括：

- 溢出：结果超出位宽后回绕或触发语言规定行为；
- 截断：宽类型写入窄类型丢失高位；
- 符号混淆：负数被当作巨大无符号数；
- 长度计算错误：乘加溢出导致缓冲区分配过小。

工业协议解析器若把报文长度、对象数量或寄存器地址处理错误，可能产生越界访问、拒绝服务或错误控制数据。

## 4. 浮点数与定点数

IEEE 754 浮点数通常由符号、指数和尾数组成。32 位单精度能覆盖很大范围，但很多十进制小数无法被二进制精确表示。因此不能把浮点数简单理解为“带小数点的准确数”。

工业控制中还常使用定点数：用整数保存原始值，再按比例换算。

```text
原始寄存器值：2534
缩放系数：0.1
工程值：253.4 °C
```

浮点或定点解释必须确认：数据类型、字节序、字顺序、比例、偏移、量程和异常值。将两个 16 位寄存器组合成 32 位浮点时，不同设备可能采用大端字节、小端字节或交换字顺序。

错误结果不一定来自攻击，也可能来自组态错误。安全工程师应保留“恶意篡改、协议解析错误、点表错误、传感器故障”多个假设，并通过原始报文、PLC 在线值和现场仪表交叉验证。

## 5. 字符、编码与字符串

字符必须通过编码映射为字节。ASCII 覆盖基础英文字符；UTF-8 对 ASCII 兼容，并用变长字节表示 Unicode。Windows 工业软件还可能使用 UTF-16LE 或区域代码页。

编码问题会影响：

- 设备名称和中文报警显示；
- 配置文件与工程项目解析；
- 日志搜索和取证时间线；
- 路径、用户名及输入校验；
- 协议长度字段与字符数量计算。

C 风格字符串以 `0x00` 结束，若输入缺少终止符或程序错误计算长度，读取可能越过缓冲区。长度前缀字符串则依赖长度字段正确性。逆向时应结合字节模式、引用位置和调用函数识别编码。

## 6. 字节序与数据对齐

大端序先存放高有效字节，小端序先存放低有效字节。网络协议传统上使用网络字节序（大端），x86/x64 主机通常使用小端。协议解析时必须显式转换，不能依赖开发主机的本地顺序。

```text
数值 0x12345678
大端字节：12 34 56 78
小端字节：78 56 34 12
```

结构体成员可能为了访问效率插入填充字节。直接把网络字节强制解释为编译器结构体，可能受到对齐、位域顺序、编译器选项和主机字节序影响。可靠解析器应逐字段检查长度并显式解码。

## 7. CPU 与指令执行

CPU 主要包含控制单元、算术逻辑单元、寄存器和缓存。简化的指令周期为取指、译码、执行和提交结果。现代 CPU 会采用流水线、乱序执行和分支预测提高效率，但软件可见行为仍由指令集架构约束。

常见寄存器角色包括：

- 通用寄存器：保存运算数据和参数；
- 程序计数器：指向下一条指令；
- 栈指针：指向当前栈顶；
- 状态寄存器：记录条件码和处理器状态；
- 控制寄存器：控制内存、权限和运行模式。

架构决定机器码与寄存器。工程站常见 x86/x64，工业网关和嵌入式控制设备常见 ARM、MIPS 等。分析固件前必须识别架构、端序、位宽和 ABI，否则反汇编结果会失真。

## 8. 内存层次与缓存

从快到慢通常包括寄存器、L1/L2/L3 缓存、主内存和持久存储。更快层容量更小、成本更高。缓存利用时间局部性与空间局部性减少访问延迟。

```text
寄存器 -> CPU Cache -> RAM -> SSD/Flash -> 远程存储
  快、小                                      慢、大
```

缓存意味着“CPU 刚写的数据何时被其他核心或设备看到”需要一致性机制。控制与实时系统还关注最坏执行时间：缓存未命中、中断和调度抖动会影响确定性。工控安全不能只看平均性能，拒绝服务评估还要关注任务截止时间和看门狗行为。

## 9. 地址、虚拟内存与保护

程序通常使用虚拟地址，MMU 依据页表映射到物理内存。操作系统利用虚拟内存隔离进程、实现按需分配、共享库和权限控制。页面可具有读、写、执行权限。

安全机制包括不可执行内存、地址空间布局随机化和用户态/内核态隔离。这些机制提高利用难度，但不是漏洞修复。越界写、释放后使用和权限配置错误仍可能破坏数据或控制流。

PLC 与 MCU 的内存模型可能不同：有些采用固定地址、专用数据区或实时任务，没有通用桌面操作系统的完整虚拟内存。分析时必须以具体平台为准。

## 10. 总线、外设与 I/O

CPU 通过总线或互连访问内存与外设。常见 I/O 方式包括：

- 轮询：CPU 反复检查设备状态；
- 中断：设备在事件发生时通知 CPU；
- DMA：设备在较少 CPU 参与下直接与内存交换数据；
- 内存映射 I/O：把设备寄存器映射到地址空间。

工业设备中的网络控制器、串口、ADC、数字 I/O 和存储芯片都通过这些机制进入软件。驱动程序负责把硬件操作抽象为操作系统接口。

DMA 和高权限驱动具有强访问能力，因此固件、驱动和扩展设备属于重要信任边界。异常中断风暴、驱动缺陷或设备寄存器误配置可能导致任务延迟、系统崩溃或输出异常。

## 11. 工控场景：从现场信号到内存变量

以 4–20 mA 温度信号为例：输入模块完成采样与模数转换，产生原始整数；PLC 程序按量程缩放为工程值；通信服务把变量编码成协议字段；SCADA 解码并显示。

```text
4–20 mA -> ADC 原始值 -> 输入映像区 -> 缩放函数 -> REAL 温度
        -> 协议寄存器/对象 -> SCADA 驱动 -> 趋势与报警
```

每个箭头都是可能的故障或攻击点：量程配置错误、内存地址错误、字节序错误、比例篡改、协议值修改、HMI 标签绑定错误。真实物理影响取决于该值是只用于显示，还是继续参与反馈控制、联锁和安全功能。

### 数据流、控制流与状态流

数据流描述原始采样值如何经过内存、协议和组态变成工程值；控制流描述 CPU 指令、PLC 任务和通信服务按什么顺序读取、计算并写出；状态流描述输入质量、控制模式、输出命令和现场反馈如何变化。三类流必须同时分析：数值正确不代表执行路径正确，命令写入也不代表设备状态已经改变。

## 12. 工控协议字段解析案例

假设抓到四个字节：

```text
41 C8 00 00
```

它作为大端 IEEE 754 单精度浮点可表示 25.0；作为小端整数、字符或两个寄存器则是完全不同的结果。确认语义需要：

1. 查看协议规范中的字段类型和长度；
2. 确认报文字节序与设备字顺序；
3. 对照点表中的变量类型、量程和单位；
4. 与 PLC 在线值及 HMI 显示交叉验证；
5. 在隔离环境改变输入并观察字节变化。

这就是从“看到十六进制”走向“理解控制数据”的完整证据链。

## 13. 安全风险与攻击链

数据表示层风险包括整数溢出、长度计算错误、越界访问、类型混淆、未对齐访问、编码绕过和浮点特殊值处理错误。攻击链可能是：

```text
畸形协议长度 -> 解析器整数溢出 -> 缓冲区分配过小
-> 越界写入 -> 服务崩溃或控制流破坏 -> 通信中断
-> SCADA 数据失联 -> 操作员态势感知下降
```

物理影响不是自动发生的。需继续评估控制器是否独立闭环、失联策略、输出保持方式、看门狗、冗余和 SIS。报告应分别描述软件后果、系统后果和过程后果。

## 14. 防护与工程原则

- 协议解析前验证总长度、字段长度、数量和算术溢出；
- 明确使用固定位宽类型，避免依赖平台的隐式位宽；
- 显式处理字节序、对齐、符号和缩放；
- 不直接把不可信字节映射为本机结构体；
- 对传感器和控制值执行范围、变化率与状态一致性校验；
- 采用内存安全语言或编译器防护，并配合模糊测试和边界测试；
- 保留原始值、工程值、质量码和转换配置，支持追溯；
- 关键控制和安全功能不能只依赖上位机数据解释。

## 15. 隔离实验

### 实验一：数据表示

使用 Python 的 `int.to_bytes`、`int.from_bytes` 和 `struct`，分别编码有符号/无符号整数、浮点数和字符串，比较大端与小端结果。记录值、格式、字节和解码结果。

### 实验二：结构体与对齐

编写最小 C 程序，输出基础类型和结构体的 `sizeof`、成员偏移及内存字节。改变成员顺序和编译选项，观察填充差异。

### 实验三：过程量映射

在隔离仿真中生成虚拟温度值，将其缩放为整数寄存器或编码为浮点数，通过抓包对照 PLC 值、协议字节和 HMI 显示。禁止连接生产控制器。

## 16. 面试表达

### 60 分回答

计算机最终处理的是比特和字节。协议分析时我会确认字段位宽、符号、字节序、编码和缩放，因为同一组字节按不同类型解释会产生不同结果。CPU 通过寄存器和指令执行程序，内存保存代码与数据，操作系统利用虚拟内存和权限进行隔离。理解这些基础才能分析协议字段、固件架构和内存漏洞。

### 90 分回答

我会把数据表示放进完整工业链路分析：传感器信号经 ADC 形成原始整数，PLC 按量程转换为工程值，再由协议编码传给 SCADA。任何一步的位宽、符号、字节序、字顺序或缩放错误都可能造成显示或控制偏差。分析四字节字段时，我会结合协议规范、点表、PLC 在线值和隔离差分实验确认语义，而不是只用工具自动解析。对于解析器漏洞，还要从长度计算和内存破坏继续评估服务失效、数据失联、控制器失联策略及 SIS 屏障，区分软件、系统和物理后果。

## 本章总结

计算机组成与数据表示连接了硬件、软件、网络和物理过程。二进制是底层形式，数据类型赋予语义，CPU 与内存执行和保存状态，I/O 把数字世界连接到现场。工控安全研究必须沿“原始字节—程序解释—控制变量—现场反馈”建立证据，才能准确分析故障、漏洞与真实风险。
