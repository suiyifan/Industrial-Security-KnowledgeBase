# 01 Fundamentals：计算机与安全基础

本章用于补齐工控安全研究需要的计算机底座。学习重点不是背概念，而是建立“能写工具、能看系统、能读网络流量”的基础能力。

## 本章目标

学完本章后，你应该能做到：

- 用 Git 管理自己的知识库和实验代码。
- 在 Linux 中完成常见文件、权限、进程、网络排查。
- 用 Python 写基础脚本、网络客户端和简单扫描器。
- 理解 TCP/IP、HTTP、DNS、ARP、ICMP 的基本工作过程。
- 用 Wireshark 抓包并解释常见协议流量。
- 把基础知识转化成工控安全岗位能听懂的表达。

## 面向岗位的学习重点

| 方向 | 为什么重要 | 本章对应内容 |
|---|---|---|
| 工具开发 | JD 要求工具开发、编程能力 | Python、Socket、文件处理 |
| 协议分析 | 工控协议研究必须懂网络 | TCP/IP、端口、Wireshark |
| 漏洞研究 | 漏洞复现需要系统和网络基础 | Linux、进程、权限、HTTP |
| 报告输出 | 安全分析报告需要清楚表达 | 每节的面试表达与笔记模板 |
| 作品集 | HR 更容易相信可展示成果 | Git、GitHub、实验记录 |

## 初学者学习方法

不要只看文字。每学一个概念，都要问三件事：

1. 我能不能用一句白话解释它？
2. 我能不能用一个命令或脚本看到它？
3. 它和工控安全岗位有什么关系？

如果三个问题都能回答，这个知识点才算真正进入你的能力体系。
