# 01 篇实践任务与验收清单

本清单用于证明 01 篇知识已经转化为可操作能力。完成标准不是“命令执行过”或“截图保存了”，而是能说明实验目标、环境、输入、关键输出、判断依据、证据局限和工控场景映射。

所有主动测试仅限个人隔离靶场。不得连接生产 PLC、工程站、工业网关或未经授权网络。

## 1. 通用交付标准

每项实验至少保存：

- [ ] 实验目标与前置假设；
- [ ] 虚拟机、操作系统、软件和依赖版本；
- [ ] 网络拓扑、地址、虚拟网络模式和隔离证明；
- [ ] 输入文件、配置和脚本哈希；
- [ ] 实际命令、关键输出和错误信息；
- [ ] 预期结果与实际结果对照；
- [ ] 数据流、控制流和状态流说明；
- [ ] 安全边界、停止条件和清理步骤；
- [ ] 结论、证据局限和下一步验证；
- [ ] Git 提交或受控证据清单。

推荐目录：

```text
labs/01-fundamentals/<lab-name>/
├─ README.md
├─ topology.md
├─ configs/
├─ scripts/
├─ fixtures/
├─ evidence-manifest.txt
└─ report.md
```

真实凭据、客户数据、受限固件和未脱敏 PCAP 不进入公开仓库。

## 2. 阶段 A：计算机、操作系统与编程

### 2.1 计算机组成与数据表示

- [ ] 完成二进制、十六进制、补码和位掩码转换；
- [ ] 用 Python 编解码无符号/有符号整数和 IEEE 754 浮点；
- [ ] 对比大端、小端和 16 位寄存器字顺序；
- [ ] 用 C 输出基础类型、结构体大小和成员偏移；
- [ ] 将虚拟温度原始值、缩放值和协议字节对应；
- [ ] 解释编码、位宽或字节序错误如何形成过程偏差。

交付物：一份数据表示对照表、一组固定测试向量、一份过程量映射报告。

### 2.2 操作系统运行时

- [ ] 观察进程、线程、文件、Socket 和内存映射；
- [ ] 在 Linux 使用 `strace`、Windows 使用 Process Monitor 观察测试程序；
- [ ] 分别制造 CPU、内存、锁等待和磁盘 I/O 状态；
- [ ] 将 TCP 零窗口或应用超时与主机资源证据关联；
- [ ] 解释用户态、内核态、系统调用与权限检查。

交付物：进程运行时对象图、四类资源状态对照报告。

### 2.3 Python 工具工程

- [ ] 编写固定宽度整数和浮点协议编解码函数；
- [ ] 实现带最大长度的增量 TCP 缓冲区；
- [ ] 区分 DNS、超时、拒绝、复位和协议异常；
- [ ] 添加 `argparse`、结构化日志、退出码和类型标注；
- [ ] 使用 `pytest` 覆盖正常、边界、截断和超长输入；
- [ ] 固定虚拟环境与依赖，生成运行说明。

交付物：一个离线协议解析 CLI、测试报告和依赖清单。

### 2.4 C 与内存安全

- [ ] 阅读并解释栈、堆、全局区和共享库映射；
- [ ] 分析指针、数组、长度、所有权和释放路径；
- [ ] 用教学代码复现越界或整数错误，并用 Sanitizer 定位；
- [ ] 修复为逐字段长度检查和安全算术；
- [ ] 比较缓解机制与根因修复的区别；
- [ ] 把服务崩溃继续映射到通信和过程后果。

交付物：教学解析器修复前后差异、调用栈和边界测试集。

## 3. 阶段 B：Linux、Windows 与版本管理

### 3.1 Linux 系统管理与安全分析

- [ ] 用 `systemctl cat/show` 分析测试服务身份、依赖与沙箱；
- [ ] 用 `namei -l`、`stat`、`getfacl` 定位一次权限拒绝；
- [ ] 用 `ps`、`/proc`、`lsof` 和 `ss` 将进程、文件与端口关联；
- [ ] 用 `ip`、`nft`、`tcpdump` 解释一次网络故障；
- [ ] 建立用户、sudo、SUID、Capabilities、timer 与服务基线；
- [ ] 用 journal/audit 建立登录、文件变化和服务重启时间线；
- [ ] 设计一份工业 Linux 主机低影响采集清单。

交付物：Linux 基线清单、一次权限排障和一次事件时间线报告。

### 3.2 Windows 系统管理与安全分析

- [ ] 用 CIM/PowerShell 建立进程、服务、驱动和端口映射；
- [ ] 使用 `whoami /all`、`Get-Acl`、`icacls` 解释令牌与 ACL；
- [ ] 检查测试任务、服务、Run 项和 WMI 订阅；
- [ ] 导出 System/Security/PowerShell 测试事件及原始 XML；
- [ ] 采集本地管理员、共享、RDP/WinRM、软件与文件哈希基线；
- [ ] 模拟工程项目文件变化，证明“文件变化”不等于“PLC 已下装”。

交付物：Windows 基线、事件时间线和工程项目完整性报告。

### 3.3 Git 与研究证据管理

- [ ] 使用 `git cat-file`、`ls-tree` 解释 blob/tree/commit；
- [ ] 创建分支、逻辑小提交、PR 风格评审和附注标签；
- [ ] 用 `git diff`、`--staged`、`diff --check` 完成提交前审查；
- [ ] 仅用虚构令牌演练秘密提交后的撤销与历史处置；
- [ ] 为实验输入和输出建立哈希、证据编号和受控存储引用；
- [ ] 模拟 PLC 工程项目“离线变更—评审—下装—在线校验”状态链。

交付物：一个完整研究 PR、签名/标签说明和工业变更包。

## 4. 阶段 C：网络、服务、数据库与身份

### 4.1 TCP/IP 网络通信

- [ ] 计算多个 IPv4 子网、广播地址和主机范围；
- [ ] 建立两个虚拟网段，观察 ARP、网关和返回路由；
- [ ] 对比 TCP 无响应、RST、应用超时、重传和零窗口；
- [ ] 解释 VLAN、ACL、NAT 与路由的不同安全作用；
- [ ] 将五元组、协议操作、PLC 变量和反馈状态对应。

交付物：虚拟拓扑、路由故障矩阵和 TCP 状态抓包。

### 4.2 网络服务与远程管理

- [ ] 建立本地 DNS/DHCP/NTP 测试服务并观察数据流；
- [ ] 对比名称解析成功、网络可达、TLS 成功和授权成功；
- [ ] 建立测试 CA，验证正确/错误名称及过期证书；
- [ ] 通过跳板访问测试主机，记录 SSH/RDP 会话和文件传输；
- [ ] 设计供应商 VPN—DMZ—堡垒机—工程站的限时授权流程；
- [ ] 证明 RDP 断开与注销、API 200 与设备执行的区别。

交付物：远程维护证据链和服务状态分层矩阵。

### 4.3 数据库与工业数据

- [ ] 建立站点、点表、报警、配方和发布任务测试表；
- [ ] 配置主外键、唯一、范围和对象授权；
- [ ] 比较参数化查询与不安全字符串拼接；
- [ ] 用两个会话观察隔离、锁、回滚和冲突；
- [ ] 使用执行计划比较有无索引的读写成本；
- [ ] 完成测试备份、恢复和 RPO/RTO 记录；
- [ ] 模拟“数据库提交—任务—PLC 模拟器确认—反馈”状态机。

交付物：数据库模型、事务实验、恢复报告与发布状态图。

### 4.4 密码学、PKI 与身份

- [ ] 对虚构配置计算哈希、HMAC 和数字签名并比较语义；
- [ ] 建立本地根 CA、中间 CA 和测试终端证书；
- [ ] 验证链、名称、用途、有效期和撤销；
- [ ] 模拟旧新证书重叠与不停机轮换；
- [ ] 为人员、设备、服务建立唯一身份和对象权限；
- [ ] 分析共享设备证书如何破坏可归责性；
- [ ] 说明 OPC UA 应用证书与用户授权的区别。

交付物：本地 PKI 生命周期记录、授权矩阵和轮换报告。

### 4.5 网络安全工程

- [ ] 为虚拟水箱绘制资产、数据流、信任边界和安全屏障；
- [ ] 建立至少一条完整攻击树和利用条件链；
- [ ] 区分 CVSS 严重性与现场场景风险；
- [ ] 设计预防、检测、响应、恢复和补偿控制；
- [ ] 验证单层失效后纵深控制是否继续生效；
- [ ] 完成一次工程站失陷桌面推演。

交付物：威胁模型、风险登记表和响应行动卡。

## 5. 阶段 D：实验、流量与取证

### 5.1 虚拟化与容器

- [ ] 比较 Bridge、NAT、Host-only 和 Internal 可达性；
- [ ] 证明靶场无法访问生产/办公地址；
- [ ] 比较快照、克隆和独立备份的恢复能力；
- [ ] 以非 root、只读根、最小 Capabilities 运行测试容器；
- [ ] 固定镜像摘要、资源限制和专用内部网络；
- [ ] 建立 OpenPLC/客户端/监测节点三角色靶场。

交付物：隔离验证报告、恢复记录和声明式靶场配置。

### 5.2 Wireshark 工业流量分析

- [ ] 记录镜像/TAP/本地抓包点及其证据边界；
- [ ] 建立 Endpoints、Conversations、协议与周期基线；
- [ ] 分析 TCP 握手、重传、窗口、RST 和流重组；
- [ ] 配对 Modbus 请求响应、事务、功能码、地址与异常；
- [ ] 将协议字段与虚拟点表、PLC 变量和过程反馈关联；
- [ ] 导出原始 PCAP、哈希、关键包号、过滤器和截图。

交付物：一份可复核的工业 PCAP 分析报告。

### 5.3 日志、可观测性与取证

- [ ] 统一 Windows、Linux、Web 和测试 PLC 的时间与资产标识；
- [ ] 区分 Event、Receive、Ingest 和 Analysis Time；
- [ ] 模拟 Agent 停止、队列满、解析失败和时间漂移；
- [ ] 建立身份—进程—协议—PLC—过程反馈的跨层时间线；
- [ ] 创建证据 ID、哈希、分析副本和保管链；
- [ ] 明确每条结论的来源、置信度和待验证项。

交付物：日志源健康面板说明、事件时间线和证据清单。

## 6. 综合项目：虚拟工控安全研究台

### 6.1 项目目标

构建一个无生产网络连通的虚拟水箱/泵站系统，包含：

```text
[工程/分析主机]
        |
[Internal Lab Network] -- [OpenPLC/协议模拟器] -- [过程模型]
        |
[监测节点：PCAP + 日志]
        |
[历史数据库/Web 报表]
```

### 6.2 必做场景

- [ ] 正常周期采集与趋势入库；
- [ ] 一个经过批准的虚拟设定值变化；
- [ ] 一次连接失败或应用资源故障；
- [ ] 一次越权配置尝试被权限/审批阻断；
- [ ] 一次日志源失联被监测发现；
- [ ] 从可信备份恢复数据库或配置；
- [ ] 用 Git/工单/证据清单记录全过程。

### 6.3 最终报告

报告必须包含：

- [ ] 架构图和三类流；
- [ ] 资产、身份、通信矩阵和信任边界；
- [ ] 软件/镜像/配置/样本版本与哈希；
- [ ] 正常基线和异常场景；
- [ ] PCAP、日志、数据库、PLC 变量与过程反馈；
- [ ] 风险、屏障、残余风险和整改；
- [ ] 恢复与回滚结果；
- [ ] 面试 3 分钟项目表达。

## 7. 01 篇最终验收

只有满足以下条件才算完成：

- [ ] 能解释底层原理，而非只背命令和端口；
- [ ] 能根据输出区分网络、主机、应用、数据库和过程问题；
- [ ] 能写带校验、超时、日志、测试和明确边界的工具；
- [ ] 能在 Linux/Windows 完成低影响基线与事件排查；
- [ ] 能建立参数化、最小权限、PKI 和远程访问控制；
- [ ] 能构建隔离靶场并证明其不连接生产网；
- [ ] 能用多源证据建立跨层时间线；
- [ ] 能区分登录、提交、发布、PLC 状态与物理动作；
- [ ] 能说明事实、推断、不确定性和补证方法；
- [ ] 所有成果已脱敏、版本化、可复现和可审阅。
