# 01 篇术语表

本表统一 01 篇使用的计算机与安全术语。定义强调工程语义和边界，不替代具体产品、协议或标准文档。

## 1. 计算机组成与数据

| 术语 | 中文/解释 |
| --- | --- |
| Bit | 位，二进制最小单位，取 0 或 1。 |
| Byte | 字节，通常由 8 bit 组成。 |
| Word | 字，宽度由 CPU、编程环境或协议定义，不能固定理解为 16 位。 |
| Endianness | 字节序，多字节值中字节的排列顺序，常见大端和小端。 |
| Two's Complement | 补码，现代计算机常用的有符号整数表示。 |
| Fixed-point | 定点数，用整数配合固定缩放表示小数。 |
| Floating-point | 浮点数，常按 IEEE 754 表示，存在精度和特殊值问题。 |
| Encoding | 编码，把字符或数据映射为字节；不等于加密。 |
| CPU | 中央处理器，取指、译码、执行指令并管理计算状态。 |
| Register | 寄存器，CPU 内部高速存储位置。 |
| Cache | 缓存，位于 CPU 与内存等层次之间以降低访问延迟。 |
| RAM | 随机存取内存，保存运行中的代码和数据，通常掉电丢失。 |
| MMU | 内存管理单元，负责虚拟地址到物理地址映射和页面权限。 |
| Interrupt | 中断，设备或系统事件通知 CPU 暂停当前执行并处理事件。 |
| DMA | 直接内存访问，设备在较少 CPU 干预下与内存传输数据。 |
| ABI | 应用二进制接口，规定调用约定、类型布局、寄存器使用等。 |

## 2. 操作系统与运行时

| 术语 | 中文/解释 |
| --- | --- |
| Kernel | 内核，管理进程、内存、文件、网络、设备和安全边界。 |
| User Mode | 用户态，普通应用运行的受限特权状态。 |
| Kernel Mode | 内核态，可执行特权操作并访问关键系统资源。 |
| System Call | 系统调用，用户程序请求内核服务的受控接口。 |
| Process | 进程，程序的一次运行实例，拥有地址空间、身份与资源。 |
| Thread | 线程，进程内部可被调度的执行流，共享进程资源。 |
| Scheduler | 调度器，决定可运行线程何时使用 CPU。 |
| Virtual Memory | 虚拟内存，为进程提供隔离地址空间和页面权限。 |
| Stack | 栈，常保存调用帧、局部对象和部分控制状态。 |
| Heap | 堆，用于动态分配、生命周期由程序管理的内存区域。 |
| Handle / File Descriptor | 句柄/文件描述符，进程引用内核对象的标识。 |
| IPC | 进程间通信，包括管道、消息队列、共享内存和 Socket 等。 |
| Race Condition | 竞态条件，结果依赖不可控执行顺序。 |
| Deadlock | 死锁，执行单元相互等待无法释放的资源。 |
| Service / Daemon | 服务/守护进程，由系统管理、长期后台运行的程序。 |

## 3. 编程与软件工程

| 术语 | 中文/解释 |
| --- | --- |
| Object | 对象，Python 等语言中携带类型、值和身份的运行时实体。 |
| Mutable | 可变对象，创建后内部状态可被修改。 |
| Pointer | 指针，保存对象地址并按类型解释目标内存。 |
| Buffer | 缓冲区，用于临时保存字节或数据。 |
| Buffer Overflow | 缓冲区溢出，访问超过对象边界并影响相邻内存。 |
| Use-after-free | 释放后使用，对象生命周期结束后仍通过旧引用访问。 |
| Undefined Behavior | 未定义行为，语言标准不约束程序结果的情况。 |
| Serialization | 序列化，把对象转换为可存储或传输形式。 |
| Deserialization | 反序列化，把外部数据恢复为对象；不可信输入可能带来风险。 |
| Dependency | 依赖，程序构建或运行所需的外部组件。 |
| Unit Test | 单元测试，验证小范围函数或组件行为。 |
| Regression Test | 回归测试，确保已有修复或功能未再次失效。 |
| Fuzzing | 模糊测试，以大量变异输入发现边界和状态缺陷。 |
| CLI | 命令行界面，通过参数、输出和退出码与用户/脚本交互。 |

## 4. Git 与研究工程

| 术语 | 中文/解释 |
| --- | --- |
| Repository | 仓库，Git 对象、引用和工作文件构成的项目。 |
| Working Tree | 工作区，当前检出的可编辑文件。 |
| Index / Staging Area | 暂存区，保存下一次提交的精确快照。 |
| Blob | Git 中保存文件内容的对象。 |
| Tree | Git 中保存目录结构的对象。 |
| Commit | 提交，指向 tree、父提交并包含作者等元数据的对象。 |
| Branch | 分支，指向提交的可移动引用。 |
| HEAD | 当前检出提交或分支的引用。 |
| Merge | 合并，整合不同提交历史。 |
| Rebase | 变基，通过重新应用提交创建新的线性历史。 |
| Tag | 标签，指向特定对象的固定引用，常用于版本发布。 |
| Pull Request | 拉取请求，用于差异评审、检查与合并协作。 |
| SBOM | 软件物料清单，记录制品包含的软件组件和依赖。 |

## 5. 网络通信

| 术语 | 中文/解释 |
| --- | --- |
| Ethernet | 以太网，常见局域网链路层技术。 |
| MAC Address | MAC 地址，链路层接口标识，不应单独作为强身份。 |
| VLAN | 虚拟局域网，划分二层广播域，不等于完整安全隔离。 |
| ARP | 地址解析协议，将同网段 IPv4 地址映射为 MAC 地址。 |
| IP Address | IP 地址，网络层寻址标识。 |
| Subnet | 子网，由地址和前缀/掩码定义的网络范围。 |
| Route | 路由，决定数据包的下一跳和出口接口。 |
| NAT | 网络地址转换，改写地址/端口；不等于身份认证。 |
| ICMP | Internet 控制消息协议，用于错误报告和诊断。 |
| TCP | 可靠、有序、面向连接的字节流传输协议。 |
| UDP | 无连接、保留数据报边界但不保证可靠交付的协议。 |
| Port | 端口，传输层区分服务端点的编号。 |
| Socket | 应用访问网络协议栈的接口/内核对象。 |
| Five-tuple | 五元组：源/目的 IP、源/目的端口和传输协议。 |
| Packet | 包，对网络数据单元的泛称；具体层可称帧、数据包或段。 |
| PCAP/PCAPNG | 保存捕获网络流量及元数据的文件格式。 |

## 6. 网络服务与 Web

| 术语 | 中文/解释 |
| --- | --- |
| DNS | 域名系统，将名称映射到地址或其他资源记录。 |
| DHCP | 动态主机配置协议，分配地址、网关、DNS 等参数。 |
| NTP | 网络时间协议，用于通用系统时间同步。 |
| HTTP | 无状态请求/响应应用协议。 |
| TLS | 传输层安全协议，提供加密、完整性和对端身份机制。 |
| Cookie | 浏览器随请求携带的名称/值状态，可用于会话标识。 |
| Session | 会话，认证后持续交互的服务端或协议状态。 |
| Token | 令牌，代表身份、授权或会话的凭据。 |
| Reverse Proxy | 反向代理，代表后端服务接收客户端请求。 |
| API | 应用程序接口，定义组件间调用与数据交换。 |
| SSH | 安全 Shell 协议，提供加密远程登录和转发。 |
| SMB | Windows 常用文件共享、命名管道和管理协议。 |
| RDP | 远程桌面协议，提供 Windows 图形交互会话。 |
| WinRM | Windows 远程管理服务，PowerShell Remoting 常以其承载。 |
| Bastion Host | 堡垒机/跳板机，集中控制和审计跨边界管理会话。 |

## 7. 数据库与工业数据

| 术语 | 中文/解释 |
| --- | --- |
| Relational Model | 关系模型，以关系/表及约束组织数据。 |
| Primary Key | 主键，唯一标识表中记录。 |
| Foreign Key | 外键，约束记录引用另一表的有效对象。 |
| SQL | 结构化查询语言，用于定义、查询和修改关系数据。 |
| Transaction | 事务，作为一个逻辑工作单元执行的一组数据库操作。 |
| ACID | 原子性、一致性、隔离性和持久性。 |
| Isolation Level | 隔离级别，规定并发事务相互可见程度。 |
| Index | 索引，以额外存储/写成本提升特定查询。 |
| Query Plan | 执行计划，数据库选择的物理查询路径。 |
| Connection Pool | 连接池，复用数据库连接的应用组件。 |
| RPO | 恢复点目标，可接受的数据丢失范围。 |
| RTO | 恢复时间目标，可接受的恢复耗时。 |
| Historian | 工业历史数据库/历史服务，归档带时间和质量的过程数据。 |
| Quality Code | 质量码，表示过程值可信、坏质量、替代或不确定等状态。 |

## 8. 密码学与身份

| 术语 | 中文/解释 |
| --- | --- |
| Hash | 密码学哈希，将任意输入映射为固定长度摘要。 |
| MAC/HMAC | 使用共享秘密验证消息完整性与来源的认证码。 |
| Symmetric Encryption | 对称加密，双方使用同一秘密密钥。 |
| Asymmetric Cryptography | 非对称密码，使用公钥/私钥对。 |
| AEAD | 认证加密，同时保护机密性和完整性。 |
| Nonce | 一次性数值，按算法要求保持唯一或不可预测。 |
| Salt | 口令派生中的独立随机值，抵抗预计算。 |
| KDF | 密钥派生函数，从秘密材料派生用途分离的密钥。 |
| Digital Signature | 数字签名，用私钥签名、公钥验证内容与来源。 |
| PKI | 公钥基础设施，管理证书、CA、信任、撤销和生命周期。 |
| CA | 证书颁发机构，验证并签发数字证书。 |
| Certificate | 数字证书，将公钥与主体身份/用途绑定。 |
| Trust Anchor | 信任锚，本地直接信任的根公钥或证书。 |
| MFA | 多因素认证，组合不同类别的认证因素。 |
| Authentication | 认证，证明主体身份。 |
| Authorization | 授权，决定主体对对象可执行的操作。 |
| Kerberos | 基于 KDC 和票据的网络认证协议。 |
| HSM / TPM | 硬件安全模块/可信平台模块，用于密钥和平台信任。 |

## 9. 安全工程

| 术语 | 中文/解释 |
| --- | --- |
| Asset | 资产，对组织和工业过程有价值、需要保护的对象。 |
| Threat | 威胁，可能造成损害的潜在原因。 |
| Vulnerability | 漏洞/弱点，可被利用或触发并造成不利影响。 |
| Exposure | 暴露，资产在特定路径上可被接触的程度。 |
| Attack Surface | 攻击面，系统可被交互或输入影响的入口集合。 |
| Trust Boundary | 信任边界，身份、权限、责任或安全假设变化的位置。 |
| Risk | 风险，威胁利用弱点造成后果的不确定性。 |
| CIA | 机密性、完整性、可用性。 |
| Safety | 功能/过程安全，防止人员、环境和设备遭受不可接受伤害。 |
| Least Privilege | 最小权限，只授予必要对象、操作、范围和时间。 |
| Separation of Duties | 职责分离，避免单一主体控制完整高风险流程。 |
| Defense in Depth | 纵深防御，以相互独立的多层控制降低风险。 |
| Compensating Control | 补偿控制，无法实施首选措施时降低风险的替代控制。 |
| Residual Risk | 残余风险，控制实施后仍存在的风险。 |
| CVSS | 通用漏洞严重性评分体系；不等同于具体现场风险。 |

## 10. 虚拟化、容器与实验

| 术语 | 中文/解释 |
| --- | --- |
| Hypervisor | 虚拟机监控器，管理虚拟 CPU、内存、设备和 Guest。 |
| Guest | 客体操作系统，运行于虚拟机内。 |
| Snapshot | 快照，保存特定时点差异状态；不等于独立备份。 |
| Bridge Network | 桥接网络，使虚拟机接入宿主所在二层网络。 |
| Host-only Network | 仅主机网络，通常只允许宿主与虚拟机通信。 |
| Internal Network | 内部虚拟网络，仅同一虚拟网内实例互通。 |
| Container | 容器，共享宿主内核的隔离进程环境。 |
| Namespace | Linux 命名空间，隔离 PID、网络、挂载等视图。 |
| cgroup | 控制组，统计和限制 CPU、内存、I/O、进程等资源。 |
| Image | 容器镜像，包含只读文件系统层和运行元数据。 |
| Volume | 容器持久数据存储，生命周期可独立于容器。 |
| Digest | 内容摘要标识，常用于固定镜像的具体内容。 |

## 11. 可观测性与取证

| 术语 | 中文/解释 |
| --- | --- |
| Log | 日志，描述离散事件及上下文。 |
| Metric | 指标，随时间变化的数值。 |
| Trace | 追踪，一次请求跨组件的因果路径记录。 |
| Event Time | 事件时间，事件实际发生的源时间。 |
| Ingest Time | 入库时间，平台解析/保存事件的时间。 |
| Correlation ID | 关联标识，用于连接跨组件的同一事务或请求。 |
| SIEM | 安全信息与事件管理平台，集中存储、关联与告警。 |
| Chain of Custody | 证据保管链，记录证据采集、转移、访问和分析。 |
| Volatile Data | 易失数据，如内存、连接和运行进程，系统状态变化后易丢失。 |
| Timeline | 时间线，将多源事件按校正时间和上下文组织。 |
| Confidence | 置信度，对结论证据强度和不确定性的表达。 |
| SOE | 事件顺序记录，工业系统中常用于高精度状态变化分析。 |

## 12. 工业协议提示

| 术语 | 中文/解释 |
| --- | --- |
| Modbus TCP | 基于 TCP 的 Modbus，常用 502 端口；端口只作识别线索。 |
| S7comm | Siemens PLC 常见通信协议，常运行于 TCP 102。 |
| OPC UA | 具备信息模型、会话与安全机制的工业互操作协议。 |
| IEC 104 | IEC 60870-5-104，电力遥信、遥测、遥控协议。 |
| Function Code | 功能码，表示协议请求的操作类型。 |
| Point Table | 点表，记录变量名称、地址、类型、量程、单位等语义。 |
| Interlock | 联锁，根据状态阻止不安全操作或触发保护动作。 |
| SIS | 安全仪表系统，独立降低过程风险的安全保护系统。 |
