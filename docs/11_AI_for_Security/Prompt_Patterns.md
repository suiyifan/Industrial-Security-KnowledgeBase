# Prompt 模式

## 1. 漏洞公告提取

```text
请从以下漏洞公告中提取结构化信息：
- CVE 编号
- 产品和版本
- 漏洞类型
- 攻击条件
- 影响
- 修复建议
- 工控场景影响
- 仍需人工确认的问题
```

## 2. PoC 审计

```text
请审计以下 PoC 代码，重点找：
- 是否有外连
- 是否下载或执行第二阶段
- 是否读取本地敏感信息
- 是否有删除/加密/破坏操作
- 真正触发漏洞的最小请求是什么
- 运行前需要隔离哪些风险
```

## 3. 协议字段分析

```text
下面是多组请求和响应的十六进制数据。
请对比字段变化，推测：
- 固定头
- 长度字段
- 序列号
- 命令码
- payload
- 校验字段
- 还需要哪些样本确认
不要把推测写成事实。
```

## 4. 代码审计

```text
请审计以下代码片段，按数据流分析：
- 外部输入在哪里进入
- 是否经过校验
- 是否进入危险函数
- 是否存在认证/授权检查
- 可能的漏洞类型
- 需要怎样验证
```

## 5. 面试表达生成

```text
请把下面的技术笔记整理成 2 分钟面试表达：
- 按背景、方法、证据、结论、反思组织
- 不夸大
- 保留限制条件
- 听起来像亲自做过
```

## 6. Prompt 使用原则

- 明确目标。
- 提供版本和上下文。
- 要求区分事实和推测。
- 要求列出需验证的问题。
- 要求输出结构化表格或清单。
- 不让 AI 编造未提供证据。
