# AI 辅助代码审计

## 1. AI 在代码审计中的价值

AI 适合快速帮助你：

- 找外部输入。
- 标记危险函数。
- 生成数据流问题。
- 总结认证和授权逻辑。
- 生成测试用例。
- 解释不熟悉的代码框架。

但 AI 不能替代运行、调试、单元测试和人工判断。

## 2. 审计关注点

工控安全相关代码常看：

- Web API 鉴权。
- 文件上传和固件升级。
- 命令执行。
- 协议解析。
- 长度字段和缓冲区。
- 会话和权限。
- 工程文件导入导出。
- 配置和密钥管理。

## 3. 输入给 AI 的代码方式

不要只贴单个函数。最好包含：

- 路由或入口。
- 参数定义。
- 调用函数。
- 关键数据结构。
- 权限中间件。
- 错误处理。

这样 AI 才能做数据流分析。

## 4. 输出要转化为验证任务

AI 输出 “可能存在命令注入” 后，你要继续验证：

- 参数是否用户可控。
- 是否经过白名单。
- 是否进入 shell。
- 是否需要登录。
- 运行权限是什么。
- 最小请求是什么。

## 5. 面试表达模板

> 我会用 AI 辅助代码审计，让它帮我标出外部输入、危险函数和可能的数据流问题。但每个风险点我都会转化成验证任务，比如确认参数是否可控、是否有鉴权、是否进入危险函数、是否能构造最小请求。最终结论只基于代码证据和实际验证。
