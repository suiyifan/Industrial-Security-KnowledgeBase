# 项目经历故事库

## 故事 1：OpenPLC + Modbus 实验

### 背景

为了理解 PLC 控制逻辑和 Modbus 协议之间的关系，我搭建了一个本地仿真实验环境。

### 方法

使用 OpenPLC 配置简单控制逻辑，通过 Modbus TCP 读取寄存器，用 Wireshark 抓包分析功能码、地址和数据值。

### 证据

- PLC 逻辑。
- Modbus 地址表。
- PCAP。
- 报文字段解析。

### 结论

这个项目让我理解了协议字段和控制语义的映射。比如写寄存器不是单纯网络数据变化，它可能对应设定值或控制输出变化。

## 故事 2：Modbus 流量分析器

### 背景

工控评估中需要快速从流量中看出资产通信关系和异常写操作。

### 方法

开发 Python 工具解析 PCAP，提取源目的 IP、端口、功能码、异常响应和写操作，生成通信矩阵。

### 结果

工具能把抓包文件转成报告可用的表格，辅助识别非工程站访问 PLC、异常写操作和通信关系。

## 故事 3：工控 CVE 分析报告

### 背景

为了训练漏洞研究闭环，我选择一个工控相关 CVE 做公告整理、PoC 审计、环境复现和影响分析。

### 方法

先提取公告信息和影响版本，再审计 PoC 是否安全，复现后提取最小触发样本，并分析根因和工控影响。

### 结论

这个项目训练了我从 “跑通 PoC” 到 “理解根因和影响边界” 的能力。
