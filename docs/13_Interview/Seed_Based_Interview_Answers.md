# Seed 风格面试回答

## 1. 如何解释“控制”的本质？

### 60 分答案

> 控制的本质是通过传感器、控制器和执行机构形成闭环，让工业过程变量在扰动下保持稳定。

### 90 分答案

> 自动控制不是让设备简单自动运行，而是让温度、压力、液位、流量、转速等过程变量在扰动下保持稳定、安全和连续。工控安全研究员要关注控制闭环是否被破坏，例如设定值是否被篡改、反馈是否真实、控制输出是否被异常改变。

### 常见追问

问：攻击 PV 和攻击 SP 有什么区别？

答：

> 攻击 SP 是改变控制目标，攻击 PV 是伪造控制器看到的现场状态。PV 欺骗更隐蔽，因为系统可能在真实异常时仍认为自己运行正常。

## 2. PLC 扫描周期为什么重要？

### 60 分答案

> PLC 通过输入采样、执行程序、输出刷新形成周期扫描，保证控制逻辑确定性。

### 90 分答案

> PLC 扫描周期的关键是输入映像和输出映像。输入在扫描开始时冻结，避免同一轮程序看到不一致的现场状态；输出在扫描末尾统一刷新，避免设备在一次扫描中抖动。安全分析时，扫描周期、任务调度和输入输出映像都会影响攻击效果。

### 常见追问

问：为什么高级攻击会关注扫描周期？

答：

> 因为攻击者可以利用采样时机、任务周期和通信延迟，让 HMI 看到正常数据，同时让控制逻辑在特定时刻执行异常动作。

## 3. DB 区为什么比普通变量更重要？

### 60 分答案

> DB 区通常保存设定值、配方、PID 状态、模式和报警阈值，业务含义更强。

### 90 分答案

> M 区多用于运行时中间状态，而 DB 区保存控制对象的业务状态，例如 PID 积分、上次误差、SP、配方和模式。HMI 修改设定值本质上常常是在写 DB。如果攻击 Retain DB，恶意参数可能在 PLC 重启后仍然保留，形成持久化影响。

## 4. 如何解释 Modbus 写寄存器风险？

### 60 分答案

> Modbus 写寄存器可能修改 PLC 中的设定值、参数或命令变量，因此比读寄存器风险更高。

### 90 分答案

> 我分析 Modbus 写操作时不会只看功能码，而会结合来源 IP、目标地址、写入时间、点表和工艺语义。比如写 Holding Register 可能只是改普通参数，也可能是改阀门开度、PID 设定值或配方。只有把寄存器地址映射到真实变量，才能判断是否会影响物理过程。

## 5. 如何把工具输出映射到控制行为？

### 60 分答案

> 工具输出只是证据，必须结合点表、HMI 画面和控制逻辑解释它代表什么工业动作。

### 90 分答案

> 例如 Wireshark 看到 `Function Code 0x06` 只能说明发生了写单寄存器。下一步要看源主机是否是合法 HMI 或工程站，目标地址对应哪个变量，写入值是否改变设定值或命令，再结合趋势和操作日志判断是否造成控制影响。这样才能从数据包走到工业风险。
