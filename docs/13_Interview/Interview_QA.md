# 面试问答库

补充材料：

## 1. 为什么选择工控安全？

参考回答：

我本身关注自动化、控制系统和网络安全的交叉方向。传统 IT 安全更多关注数据和业务系统，而工控安全还要考虑生产连续性、设备状态、控制逻辑和现场安全。我希望把编程、网络、协议分析和安全研究结合起来，做更贴近工业场景的漏洞研究和安全评估。

## 2. 你怎么理解这个岗位？

参考回答：

这个岗位不是单纯渗透测试，也不是普通安全运维，而是偏工控系统安全研究。工作重点包括研究工控软件、协议和设备，做协议分析、软件逆向、漏洞挖掘、工具开发，同时要能跟踪新漏洞和攻击手法，输出安全分析报告和调研报告。

## 3. 你目前最匹配的地方是什么？

参考回答：

我会从自动化 / 控制系统理解、Python 工具开发、网络协议分析和安全基础几个方面切入。短期内我会重点补齐工业协议、Wireshark 抓包、Web 漏洞和逆向基础，并通过 GitHub 项目沉淀协议解析、CVE 分析和安全评估报告。

## 4. 你如何学习工控协议？

参考回答：

我会先从 Modbus TCP 这种公开、结构清晰的协议入手，理解 MBAP Header、功能码和寄存器读写，再用 Wireshark 抓包分析真实报文，最后用 Python Socket 或 Scapy 构造请求和解析响应。之后再扩展到 S7、OPC UA、IEC 104 等更贴近实际行业场景的协议。
