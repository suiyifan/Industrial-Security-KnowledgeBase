# 自我介绍

## 1 分钟版本

您好，我目前主要围绕工控安全研究员岗位做系统化准备，学习路径覆盖 Python/Git/Linux/网络基础、PLC/DCS/SCADA、工业协议、Web 安全、逆向分析、漏洞研究、IEC 62443 和安全工具开发。

我比较重视把知识落到可验证项目里，例如用 OpenPLC 和 Modbus 搭建仿真实验环境，抓包分析功能码和寄存器语义；开发 Modbus 流量分析器，把 PCAP 转成通信矩阵和写操作摘要；对工控相关 CVE 做公告整理、PoC 审计、复现和影响分析。

我认为这个岗位需要的不只是单点漏洞能力，还需要理解工控系统、协议、资产和安全影响之间的关系。所以我的准备方式是从控制逻辑、协议数据流、安全风险、报告表达一路打通，希望能在岗位中参与工控软件、协议、设备和漏洞方向的安全研究。

## 2 分钟版本

您好，我应聘这个岗位主要是因为它同时包含工控系统理解、协议分析、漏洞研究、逆向和工具开发，这和我目前的学习路线非常匹配。

我把能力建设拆成几个层次：第一层是基础能力，包括 Python、Git、Linux、网络、Wireshark；第二层是工控基础，包括 PLC、DCS、SCADA、HMI、Historian、PID、I/O 和控制回路；第三层是工控安全核心，包括 Modbus、S7、OPC UA、IEC104 等协议分析，工程站、HMI、网关和 PLC 的攻击面理解；第四层是安全研究能力，包括 Web 漏洞、逆向分析、CVE 复现、补丁对比和漏洞报告；最后是工具和作品集，比如 PCAP 解析、通信矩阵生成和工控 CVE 分析报告。

我目前的优势是学习路径比较体系化，并且每个知识点都尽量落到项目或报告里。例如分析 Modbus 时，我不仅看协议字段，还会思考写寄存器在工控场景里可能对应设定值或输出变化；分析漏洞时，我不会只跑 PoC，而会继续看攻击条件、影响版本、根因和现场缓解措施。

如果加入团队，我希望先从漏洞情报跟踪、协议分析、PoC 复现、报告整理和工具脚本这些任务切入，逐步深入到工控软件逆向、设备固件分析和工业协议安全研究。

## 30 秒压缩版

我目前围绕工控安全研究员岗位做系统学习，覆盖 Python、网络、PLC/SCADA、工业协议、Web 安全、逆向、漏洞研究和 IEC 62443。我的准备方式不是只学理论，而是把内容沉淀成 OpenPLC/Modbus 实验、流量分析工具、CVE 复现报告和面试作品集。我希望在岗位中参与工控软件、协议、设备和漏洞方向的安全研究。
