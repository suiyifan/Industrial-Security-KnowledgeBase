# 工控威胁建模

## 1. 威胁建模是什么

威胁建模是用结构化方法回答：谁可能攻击、从哪里进来、沿什么路径移动、影响哪些资产、造成什么后果。

工控威胁建模必须把网络资产和物理过程联系起来。只说 “攻击者获得服务器权限” 不够，还要继续问：这台服务器是否能下发控制命令、修改报警、影响历史数据或成为进入 PLC 网段的跳板。

## 2. 工控威胁建模对象

| 对象 | 关注点 |
| --- | --- |
| 工程站 | 下载程序、修改配置、保存凭据 |
| 操作站/HMI | 操作指令、报警显示、画面脚本 |
| SCADA 服务器 | 数据采集、命令下发、用户权限 |
| Historian | 历史数据、报表、追溯 |
| PLC/RTU | 控制逻辑、I/O、运行模式 |
| 工业网关 | 协议转换、远程维护、边界连接 |
| 安全设备 | 策略、审计、隔离、检测 |
| 远程接入 | VPN、堡垒机、厂商维护 |

## 3. 信任边界

常见信任边界：

- 办公网与生产网。
- 生产网与控制网。
- 工程站与 PLC。
- HMI 与操作员。
- 远程维护入口与现场设备。
- 厂商云平台与边缘设备。

每跨越一个边界，都应该有明确的访问控制、身份认证、审计和最小权限。

## 4. 威胁路径

典型路径：

1. 钓鱼邮件或弱口令进入办公网。
2. 横向移动到运维主机或跳板机。
3. 利用远程维护通道进入生产网。
4. 获取工程站账号或项目文件。
5. 连接 PLC 或网关。
6. 修改设定值、下载程序或造成通信中断。

另一类路径：

1. 暴露在公网的工业网关 Web 漏洞。
2. 获得网关 shell。
3. 抓取和篡改协议流量。
4. 影响 PLC 通信或作为跳板进入控制网。

## 5. STRIDE 在工控中的映射

| STRIDE | 工控例子 |
| --- | --- |
| Spoofing | 冒充工程站、冒充 PLC、伪造操作员 |
| Tampering | 篡改设定值、逻辑、配置、历史数据 |
| Repudiation | 操作日志缺失，无法追溯谁下发命令 |
| Information Disclosure | 泄露工艺参数、项目文件、账号 |
| Denial of Service | PLC 通信中断、HMI 无法刷新、网关崩溃 |
| Elevation of Privilege | 普通用户执行工程师或管理员操作 |

## 6. 威胁建模输出

一份合格输出至少包括：

- 资产清单。
- 网络拓扑。
- 数据流和控制流。
- 信任边界。
- 攻击路径。
- 关键风险。
- 现有控制措施。
- 改进建议。

## 7. 面试表达模板

> 我做工控威胁建模时会先从资产和控制流程开始，而不是只看端口。先确认工程站、HMI、SCADA、Historian、PLC、网关之间的数据流和控制流，再标出信任边界，比如办公网到生产网、工程站到 PLC、远程维护到现场设备。之后分析攻击路径和影响，比如攻击者拿到工程站后能否下载程序、修改设定值或影响报警。最终输出资产、边界、路径、风险和改进建议。
