# Stuxnet 与 TRITON 案例

## 1. 为什么学习经典案例

经典工控安全事件能帮助建立真实感：攻击者不是只拿 shell，而是围绕工程软件、控制逻辑、操作员认知和安全系统构建完整路径。

## 2. Stuxnet 学习点

Stuxnet 的关键学习点：

- 利用多阶段传播进入隔离环境。
- 针对特定工程软件和 PLC。
- 修改控制逻辑，同时隐藏异常。
- 攻击目标与物理过程强相关。

安全启发：

- 工程站是高价值目标。
- PLC 逻辑完整性非常重要。
- 操作员看到的画面不一定是真实过程。
- 离线或隔离不是绝对安全。

## 3. TRITON 学习点

TRITON 针对安全仪表系统，学习点：

- 攻击目标是安全保护系统。
- 重点不是偷数据，而是影响安全保护能力。
- SIS 工程站和控制网络隔离极其重要。
- 异常和误停车可能是攻击失败或试探行为。

安全启发：

- SIS 不能被当作普通控制系统管理。
- 工程变更必须审计。
- 安全系统的异常要高度重视。
- 远程维护和工程站访问需要严格控制。

## 4. 从案例提炼通用攻击链

典型链路：

1. 初始访问。
2. 横向移动。
3. 获取工程资产。
4. 理解工艺和控制逻辑。
5. 修改控制或安全逻辑。
6. 隐藏异常或误导操作员。
7. 造成物理影响或潜在安全后果。

## 5. 面试表达模板

> 我学习 Stuxnet 和 TRITON 时，重点不是记故事，而是提炼工控攻击链。Stuxnet 体现了攻击工程站和 PLC 逻辑并隐藏过程异常，TRITON 体现了针对安全仪表系统的高危攻击。它们说明工控安全必须关注工程软件、控制逻辑完整性、操作员认知和安全系统隔离，而不是只做传统主机加固。
