# 攻击控制回路

## 1. 控制回路视角

工业控制系统的核心是控制回路：

`传感器 -> 控制器 -> 执行器 -> 物理过程 -> 传感器反馈`

攻击控制回路不是只攻击计算机，而是通过数据、逻辑或命令影响这个闭环。

## 2. 攻击点

| 位置 | 攻击方式 | 影响 |
| --- | --- | --- |
| 传感器 | 数据伪造、量程篡改、断线 | 控制器误判状态 |
| 控制器 | 修改逻辑、设定值、模式 | 输出错误控制 |
| 执行器 | 强制输出、阀门/电机误动作 | 直接影响过程 |
| HMI | 画面欺骗、报警隐藏 | 操作员误判 |
| Historian | 历史数据篡改 | 追溯和分析失真 |
| 网络 | 延迟、丢包、重放、篡改 | 控制不稳定 |

## 3. 数据攻击

数据攻击关注 “让系统看到错误状态”：

- 伪造传感器值。
- 修改 HMI 显示值。
- 篡改历史数据。
- 重放旧数据。
- 隐藏报警。

这类攻击可能不直接控制 PLC，但会误导操作员或自动控制逻辑。

## 4. 控制攻击

控制攻击关注 “让系统执行错误动作”：

- 修改设定值。
- 写寄存器或线圈。
- 下载恶意逻辑。
- 强制输出。
- 切换手自动模式。
- 触发启停命令。

控制攻击需要更谨慎评估，因为它可能直接影响生产和安全。

## 5. 安全联锁与 SIS

安全仪表系统 SIS 通常用于防止危险状态。攻击者如果能影响 SIS：

- 可能让保护动作失效。
- 可能造成误停车。
- 可能隐藏真实危险。

因此 SIS 应与基本过程控制系统隔离，远程访问、工程修改和旁路操作必须严格控制。

## 6. 面试表达模板

> 我理解工控攻击不能只看主机权限，而要看对控制回路的影响。攻击可能发生在传感器数据、控制器逻辑、执行器命令、HMI 显示或历史数据上。比如篡改 HMI 数据会误导操作员，修改 PLC 设定值会影响控制输出，攻击 SIS 则可能影响安全保护。评估工控漏洞时，我会把技术能力映射到控制回路中的具体影响点。
