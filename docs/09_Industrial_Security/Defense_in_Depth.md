# 纵深防御体系

## 1. 纵深防御是什么

纵深防御是在多个层面部署互补安全措施，即使某一层失效，也能被其他层限制影响。

工控系统不能只依赖边界防火墙，因为攻击可能来自远程维护、U 盘、工程站、供应链、内部误操作和设备漏洞。

## 2. 防御层次

| 层次 | 措施 |
| --- | --- |
| 治理 | 制度、责任、变更流程、培训 |
| 资产 | 资产清单、版本、拓扑、数据流 |
| 网络 | 分区分域、工业 DMZ、防火墙、单向网闸 |
| 身份 | 最小权限、多因素、账号审计 |
| 主机 | 白名单、补丁、基线、日志 |
| 应用 | 工程软件权限、项目文件保护 |
| 协议 | 通信白名单、异常命令检测 |
| 设备 | PLC 运行模式、下载保护、固件签名 |
| 监测 | IDS、日志、流量分析、异常行为 |
| 恢复 | 备份、应急预案、演练 |

## 3. 分区分域

分区分域的目标：

- 让不同安全需求的资产分开。
- 让跨区通信可见、可控、可审计。
- 降低横向移动范围。

典型区域：

- 办公网。
- 工业 DMZ。
- 生产管理区。
- 监控区。
- 控制区。
- 安全系统区。

## 4. 白名单思维

工控通信相对稳定，适合白名单：

- 哪台主机可以访问哪台 PLC。
- 允许哪些端口。
- 允许哪些协议。
- 允许哪些功能码或命令。
- 允许哪些时间窗口。

白名单不是一劳永逸，需要和变更流程结合。

## 5. 检测与响应

工控检测重点：

- 新出现资产。
- 异常扫描。
- 异常写操作。
- PLC 程序下载。
- 工程站异常连接。
- 远程维护异常登录。
- HMI/SCADA 服务异常。

## 6. 面试表达模板

> 我理解工控纵深防御不是堆设备，而是从资产、网络、身份、主机、应用、协议、设备、监测和恢复多个层面控制风险。比如通过分区分域限制横向移动，通过通信白名单限制工程站到 PLC 的访问，通过日志和流量监测发现异常下载，通过备份和应急预案保障恢复能力。
