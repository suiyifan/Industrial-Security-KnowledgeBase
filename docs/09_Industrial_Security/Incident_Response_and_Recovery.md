# 应急响应与恢复

## 1. 工控应急的特点

工控应急的首要目标是安全生产和连续性，不是第一时间关机重装。

特点：

- 生产不能随意中断。
- 设备可能无法安装取证工具。
- 厂商和现场工程师必须参与。
- 恢复可能依赖工程备份和参数。
- 误操作可能扩大影响。

## 2. 应急流程

1. 发现和确认：告警、异常操作、通信中断、设备重启。
2. 分级和通报：判断影响范围和生产风险。
3. 隔离控制：限制攻击路径，避免扩大。
4. 证据保全：日志、流量、配置、工程文件、内存或磁盘镜像。
5. 根因分析：漏洞、账号、远程入口、供应链或误操作。
6. 清除和加固：删除后门、修改账号、修复漏洞、调整策略。
7. 恢复验证：业务、控制逻辑、通信、报警、备份。
8. 复盘改进：流程、技术、人员、制度。

## 3. 工控证据

需要保留：

- 防火墙和 VPN 日志。
- 工程站登录和操作日志。
- HMI/SCADA 操作记录。
- PLC 程序变更记录。
- 网络流量 PCAP。
- 设备配置和固件版本。
- 项目文件和备份。
- 报警和趋势数据。

## 4. 恢复重点

恢复不仅是主机上线，还要确认：

- PLC 程序和备份一致。
- 参数、设定值、报警阈值正确。
- HMI 显示和现场一致。
- 通信链路稳定。
- 安全联锁和报警有效。
- 远程维护入口已收敛。

## 5. 面试表达模板

> 工控应急不能简单照搬 IT 的断网重装。首先要保障人身和生产安全，和现场工程师确认哪些系统能隔离、哪些不能动。取证时要保留网络、主机、工程站、HMI、PLC 变更和趋势数据。恢复时不仅要让系统重新上线，还要验证控制逻辑、设定值、报警和联锁是否正确。
