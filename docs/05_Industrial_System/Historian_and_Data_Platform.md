# 历史数据库与生产数据平台

## 1. 原理解释

历史数据库用于保存过程变量、报警、事件、趋势和报表数据。它是事故追溯、生产分析和安全调查的重要证据源。

## 2. 工程实现

常见数据包括 PV 趋势、SP 变化、设备状态、报警记录、操作记录、事件日志和生产报表。

## 3. 数据流

```text
PLC / SCADA
  -> 数据采集接口
  -> Historian
  -> 报表 / MES / 数据平台 / 办公网查询
```

## 4. 安全风险

- 历史数据被篡改。
- 趋势缺失导致无法追溯。
- 双网卡历史库成为跨域跳板。
- 数据库账号泄露。
- 上层系统通过历史库反向进入控制网。

## 5. 面试表达

> 历史库的安全价值在于数据可信和事故追溯。如果历史数据被篡改，攻击者可能隐藏真实过程变化；如果历史库双归属办公网和控制网，还可能成为横向移动跳板。
