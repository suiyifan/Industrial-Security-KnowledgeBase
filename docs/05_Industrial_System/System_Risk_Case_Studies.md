# 系统风险案例化表达

## 案例 1：工程站 RDP 暴露

```text
办公网账号泄露 -> RDP 工程站 -> 获取工程文件 -> 连接 PLC -> 下载工程
```

建议：堡垒机、MFA、维护窗口、会话审计。

## 案例 2：HMI 共用账号

风险：无法追踪具体操作人，攻击者可利用合法账号启停设备或确认报警。

建议：账号实名、分权、关键操作二次确认。

## 案例 3：历史库双网卡

风险：历史库成为办公网和控制网之间的跳板。

建议：DMZ 发布只读副本，控制网侧最小采集。

## 面试总结

> 系统风险要按资产角色讲。同样是弱口令，发生在普通终端和工程站上的影响不同；同样是数据库暴露，发生在历史库上会影响生产数据可信和追溯。
