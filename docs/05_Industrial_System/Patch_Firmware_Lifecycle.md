# 工控系统补丁、固件与生命周期

## 1. 原理解释

工控系统补丁和固件升级比 IT 系统更困难，因为生产连续性、兼容性、厂商认证和停机窗口都会限制变更。

## 2. 工程现实

常见情况：

- 老旧 Windows 长期运行。
- PLC 固件多年不升级。
- 工程软件依赖特定版本。
- 补丁需要厂商验证。
- 停机窗口很少。
- 升级失败可能影响生产。

## 3. 管理流程

```text
漏洞通告
  -> 影响版本确认
  -> 厂商兼容性确认
  -> 测试环境验证
  -> 变更审批
  -> 备份
  -> 维护窗口升级
  -> 回退方案
  -> 复测
```

## 4. 面试表达

> 工控补丁管理不能照搬 IT。评估时我会关注资产版本、厂商公告、兼容性验证、备份、维护窗口和回退方案。对于无法立即升级的漏洞，要通过隔离、访问控制、监测和补偿措施降低风险。
