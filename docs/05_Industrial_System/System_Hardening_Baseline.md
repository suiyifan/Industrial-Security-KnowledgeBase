# 系统加固与基线核查

## 1. 原理解释

基线核查不是为了机械打勾，而是确认关键资产是否满足最小安全要求。

## 2. 核查内容

| 类别 | 检查项 |
|---|---|
| 账号 | 默认账号、共享账号、弱口令、离职账号 |
| 服务 | RDP、SMB、FTP、Telnet、Web 管理 |
| 权限 | 最小权限、管理员日常使用 |
| 日志 | 登录、操作、远程会话、工程变更 |
| 软件 | 未授权软件、远控软件、杀毒状态 |
| 网络 | 多网卡、默认网关、异常路由 |
| 备份 | 工程文件、配置、数据库、恢复演练 |

## 3. 安全视角

基线问题要映射到工控影响：

- 共用账号 -> 无法追责关键操作。
- RDP 暴露 -> 攻击者可进入工程站。
- SMB 开放 -> 工程文件泄露。
- 日志缺失 -> 事故无法追溯。

## 4. 面试表达

> 工控基线核查要结合资产角色。工程站的 RDP、工程文件、管理员权限比普通办公终端更敏感；HMI 的共用账号可能直接关联设备操作；历史库的日志和备份关系到事故追溯。
