# 工程站与工程软件安全

## 1. 原理解释

工程站是工控系统中最重要的高价值资产之一。它用于编写、配置、上传、下载、调试和维护控制工程。

## 2. 工程实现

工程站通常安装 PLC 编程软件、DCS 工程软件、HMI/SCADA 组态软件、驱动配置工具、通信诊断工具和厂商授权软件。

它可能保存工程文件、点表、硬件组态、通信配置、控制逻辑、账号或连接信息。

## 3. 数据流

```text
工程师修改工程
  -> 编译
  -> 连接控制器
  -> 下载工程 / 参数
  -> PLC/DCS 执行新逻辑
  -> 现场过程变化
```

## 4. 安全风险

- 工程文件泄露。
- 工程站感染恶意软件。
- 远程桌面暴露。
- 工程软件漏洞。
- 上传/下载无保护。
- Force 变量无审计。
- 版本管理混乱。
- 工程站双网卡跨接。

## 5. 防护建议

- 工程站专机专用。
- 最小化联网。
- 禁止绕过堡垒机远程访问。
- 工程文件加密和版本管理。
- 下载工程审批和日志。
- U 盘管控。
- 定期备份和恢复演练。

## 6. 面试表达

> 工程站是高价值资产，因为它不仅能访问控制器，还可能保存完整工程、点表、通信配置和控制逻辑。攻击工程站可能比直接攻击 PLC 更危险，因为攻击者能理解工艺并进行精准修改。
