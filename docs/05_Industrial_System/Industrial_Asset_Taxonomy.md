# 工业资产分类与价值判断

## 1. 原理解释

工控系统资产不能只按 IP 或主机名分类，而要按它在控制链路中的角色分类。

资产价值取决于：

- 是否能修改控制逻辑。
- 是否能下发操作命令。
- 是否能读取或写入关键变量。
- 是否保存工程文件、点表、账号或历史数据。
- 是否跨安全域。

## 2. 工程实现

| 类别 | 典型资产 | 核心价值 |
|---|---|---|
| 工程资产 | 工程站、工程软件、工程文件 | 修改逻辑、下载工程 |
| 操作资产 | HMI、操作员站 | 下发命令、确认报警 |
| 监控资产 | SCADA Server、报警服务器 | 数据采集、监控、调度 |
| 数据资产 | Historian、报表、生产数据库 | 趋势、追溯、分析 |
| 控制资产 | PLC、DCS 控制站、RTU、IED | 直接控制现场 |
| 边界资产 | 工业网关、防火墙、网闸、堡垒机 | 跨域访问和协议转换 |
| 安全资产 | IDS、日志审计、杀毒、补丁服务器 | 监测、防护、恢复 |

## 3. 数据流

```text
现场设备 -> 控制器 -> SCADA/HMI -> 历史库/报表 -> 生产管理系统
                         ^
                         |
                     工程站维护
```

工程站虽然不一定连续参与生产数据流，但它对控制逻辑有最高影响。

## 4. 安全视角

高价值资产优先级：

1. 工程站
2. 控制器
3. HMI / 操作员站
4. SCADA Server
5. 工业网关
6. 历史数据库
7. 远程运维入口

## 5. 面试表达

> 我判断工控资产价值时不只看主机等级，而看它在控制链路中的功能。工程站能改逻辑，HMI 能操作设备，PLC/DCS 直接控制现场，历史库影响追溯，网关可能跨安全域。不同资产被攻陷后的工控影响完全不同。
